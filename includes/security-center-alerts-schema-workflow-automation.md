### <a name="the-data-model-of-the-schema"></a>Модель данных схемы

|Поле|Тип данных|Описание|
|----|----|----|
|**AlertDisplayName**|Строка|Отображаемое имя предупреждения.|
|**AlertType**|Строка|Тип предупреждения. Оповещения одного типа должны иметь одинаковое значение. Это поле представляет собой строку с ключом, представляющую тип предупреждения, а не экземпляр предупреждения. Все экземпляры предупреждений из одной логики обнаружения или аналитики должны иметь одинаковое значение для типа оповещения.|
|**компромиседентити**|Строка|Отображаемое имя ресурса, наиболее связанного с этим оповещением.|
|**Описание**|Строка|Описание оповещения.|
|**ендтимеутк**|Дата/время|Время последнего события или действия, добавленного в предупреждение.  Поле должно быть строкой, которая соответствует формату ISO8601, включая сведения о часовом поясе в формате UTC.|
|**Сущности**|IEnumerable (Иентити)|Список сущностей, связанных с предупреждением. Этот список может содержать смесь сущностей различных типов. Тип сущности может быть любым из типов, определенных в разделе сущностей. Сущности, которых нет в списке ниже, также могут быть отправлены, однако не гарантируется, что они будут обработаны (предупреждение не будет проверяться новыми типами сущностей).|
|**ExtendedProperties**|Dictionary (строка, строка)|Поставщики могут (при необходимости) содержать настраиваемые поля.|
|**Намерение**|Перечисление|Цель, связанная с цепочкой уничтожения, за предупреждением. Список поддерживаемых значений и объяснение поддерживаемых в центре безопасности Azure целей для цепочки аннулирований см. в разделе [намерения](../articles/security-center/alerts-reference.md#intentions).<br/>Это поле может содержать несколько значений (разделенных запятыми).|
|**Инцидент**|Bool|Это поле определяет, является ли предупреждение инцидентом (составное Группирование нескольких предупреждений) или одно оповещение. Значение по умолчанию для поля равно false (это означает, что это одно предупреждение).|
|**ProcessingEndTime**|Дата/время|Время, когда оповещение было доступно конечному пользователю в исходном продукте, на котором находится оповещение.|
|**ProductName**|Строка|Имя продукта, опубликовавшего это оповещение (центр безопасности Azure, Azure ATP, Microsoft Defender ATP, МКАС и т. д.).|
|**ремедиатионстепс**|Числ<String>|Ручные действия, которые необходимо выполнить для исправления оповещения.|
|**ресаурцеидентифиерс**|Список (идентификаторы ресурсов)|Идентификаторы ресурсов для этого предупреждения, которые можно использовать для направления оповещения в подходящую группу доступности продукта (клиент, Рабочая область, подписка и т. д.). Для каждого предупреждения может существовать несколько идентификаторов разных типов.|
|**Уровень серьезности**|Перечисление|Серьезность предупреждения, сообщаемого поставщиком. Возможные значения: информационный, низкий, средний и высокий.|
|**старттимеутк**|Дата/время|Время первого события или действия, добавленного в оповещение. Поле должно быть строкой, которая соответствует формату ISO8601, включая сведения о часовом поясе в формате UTC.|
|**Состояние**|Перечисление|Состояние жизненного цикла оповещения.<br/>Поддерживаются следующие состояния: "новое", "разрешено", "закрыто", "неизвестно".<br/>Предупреждение, указывающее значение, отличное от поддерживаемых, присваивается состояние "неизвестно".<br/>Оповещению, которое не указывает значение, присваивается состояние "новое".|
|**системалертид**|Строка|Идентификатор предупреждения.|
|**TimeGenerated**|Дата/время|Время создания предупреждения поставщиком предупреждений. Если внутренние поставщики оповещений не сообщают о них, продукт может назначить время, которое он получал для обработки продукта.  Поле должно быть строкой, которая соответствует формату ISO8601, включая сведения о часовом поясе в формате UTC.|
|**ИмяПоставщика**|Строка|Имя поставщика, который вызывает оповещение.|
