## <a name="overview-of-ssh-and-keys"></a>Общие сведения о SSH и ключах

[SSH](https://www.ssh.com/ssh/) — это протокол зашифрованного соединения, обеспечивающий безопасные входы по незащищенным соединениям. SSH — это протокол подключения по умолчанию для виртуальных машин Linux, размещенных в Azure. Хотя SSH обеспечивает зашифрованное подключение, использование паролей с SSH-подключениями по-прежнему оставляет виртуальную машину уязвимой для атак методом подбора. Мы рекомендуем подключаться к виртуальной машине по протоколу SSH с помощью пары открытого и закрытого ключей, также называемых *ключами SSH*. 

- *Открытый ключ* размещается на виртуальной машине Linux.

- *Закрытый ключ* остается в локальной системе. Его нужно защищать и нельзя никому предоставлять.

При использовании клиента SSH для подключения к виртуальной машине Linux (с открытым ключом) Удаленная виртуальная машина проверяет клиент, чтобы убедиться, что он имеет правильный закрытый ключ. Если у клиента есть закрытый ключ, он получает доступ к виртуальной машине. 

В зависимости от политик безопасности организации вы можете использовать отдельную пару из открытого и закрытого ключей для доступа к нескольким виртуальным машинам и службам Azure. Не нужно выделять отдельную пару ключей для каждой виртуальной машины или службы, к которой необходим доступ. 

Открытый ключ можно использовать совместно с любым пользователем, но только вы (или ваша локальная инфраструктура безопасности) должны иметь доступ к вашему закрытому ключу.
