Следующие предостережения относятся к данным во время перемещения в Azure.

- Мы советуем избегать ситуации, когда в один и тот же контейнер запись производит более одного устройства.
- Если у вас есть объект Azure (например, большой двоичный объект или файл) в облаке с тем же именем, что и у копируемого объекта, устройство перезапишет файл в облаке.
- Пустая иерархия каталогов (без файлов), созданная в общих папках, не передается в контейнеры больших двоичных объектов.
- Данные можно скопировать с помощью перетаскивания в проводнике или с помощью командной строки. Если размер совокупного размера копируемых файлов превышает 10 ГБ, рекомендуется использовать программу с массовым копированием, например `Robocopy` или `rsync` . Средства групповой копии повторно Попытайтесь скопировать операцию копирования для периодических ошибок и обеспечивают дополнительную устойчивость.
- Если в общую папку, связанную с контейнером хранилища Azure, загружаются BLOB-объекты, которые не соответствуют типу определенному для общей папки в момент создания, то такие BLOB-объекты обновляться не будут. Например, если вы создаете общую папку блочного BLOB на устройстве, свяжите общую папку с существующим облачным контейнером, который содержит страничные BLOB-объекты, обновите эту общую папку для скачивания файлов, а затем измените некоторые обновленные файлы, которые уже хранятся в виде страничных BLOB-объектов в облаке, вы увидите ошибки передачи.
- После создания файла в общих папках его невозможно переименовать.
- Удаление файла из общей папки не приводит к удалению записи в учетной записи хранения.
- При использовании `rsync` для копирования данных параметр `rsync -a` не поддерживается.
