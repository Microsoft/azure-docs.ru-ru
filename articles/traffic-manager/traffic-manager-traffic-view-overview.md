# <a name="traffic-manager-traffic-view"></a>Представление трафика в диспетчере трафика

Диспетчер трафика обеспечивает маршрутизацию на уровне DNS (системы доменных имен). Эта служба позволяет конечным пользователям направлять работоспособные конечные точки на соответствующие методы маршрутизации. Представление трафика в диспетчере трафика демонстрирует сведения о базах пользователей (на уровне детализации сопоставителя DNS) и характерных для них схемах трафика. При включении представления трафика эти сведения обрабатываются для обеспечения ценной практической информации. 

С помощью представления трафика вы сможете ознакомиться:
- с расположением баз пользователей (с учетом уровня детализации локального сопоставителя DNS);
- с объемом трафика (отображается как DNS-запросы, обрабатываемые Azure Traffic Manager) из этих регионов;
- с репрезентативной задержкой, с которой столкнулись эти пользователи;
- с определенными шаблонами трафика, исходящего из каждой базы пользователя в регионы Azure, где находятся конечные точки. 

Например, можно использовать Представление трафика, чтобы понять, какие регионы имеют большой объем трафика, но могут пострадать от более высоких задержек. Затем эти сведения используются для планирования расширения в новые регионы Azure. Таким образом, пользователи будут иметь более низкую задержку.

## <a name="how-traffic-view-works"></a>Как работает представление трафика

Представление трафика работает, взглянув на входящие запросы, полученные за последние семь дней, для профиля. Из сведений о входящих запросах Представление трафика извлекает исходный IP-адрес сопоставителя DNS, используемый для представления расположения пользователей. Эти сведения группируются на уровне распознавателя DNS для создания областей базового пользователя. Диспетчер трафика обслуживает географические данные IP-адресов. Затем диспетчер трафика просматривает регионы Azure, к которым отправляется запрос, и конструирует карту потока трафика для пользователей из этих регионов.
 
На следующем шаге диспетчер трафика сопоставляет регион базового пользователя с сопоставлением региона Azure с таблицами задержки сетевой аналитики. Эта таблица сохраняется для различных сетей конечных пользователей, чтобы понять среднюю задержку пользователей из этих регионов при подключении к регионам Azure. Затем все эти вычисления объединяются на уровне IP-адреса локального сопоставителя DNS, прежде чем он будет представлен. Вы можете использовать эту информацию разными способами.

Частота обновления данных в представлении трафика зависит от нескольких внутренних переменных службы. Однако данные обновляются каждые 24 часа.

>[!NOTE]
>Задержки, описанные в Представлении трафика, отражают задержку между пользователем и регионом Azure, к которому этот пользователь обращается, а не задержку запроса DNS. Функция "Представление трафика" обеспечивает оптимальную оценку задержки между локальным сопоставителем DNS и регионом Azure, в который направлен запрос. Если данных недостаточно, для задержки возвращается значение NULL. 

## <a name="visual-overview"></a>Визуальное представление

При переходе к разделу **представление трафика** на странице диспетчера трафика вы увидите географическую карту с наложением представление трафика Insights. На карте представлены сведения о базе пользователей и конечных точках из профиля диспетчера трафика.

### <a name="user-base-information"></a>Сведения о базе пользователей

Для этих локальных арбитров имен DNS, для которых доступны сведения о расположении, они отображаются на карте. Цвет сопоставителя DNS обозначает среднее время задержки для пользователей, которые используют этот сопоставитель DNS при запросах к диспетчеру трафика.

Наведя указатель мыши на сопоставитель DNS, обозначенный на карте, вы увидите следующие данные:
- IP-адрес сопоставителя DNS;
- объем трафика для запросов DNS, полученных диспетчером трафика от этого сопоставителя;
- конечные точки, к которым направлялся трафик от этого сопоставителя DNS, графически представленные линиями от сопоставителя DNS до конечных точек; 
- среднее время задержки от этого расположения до конечной точки, закодированный в цвете соединительных линий.

### <a name="endpoint-information"></a>Сведения о конечных точках

Регионы Azure, в которых находятся конечные точки, показаны синими точками на карте. Если конечная точка является внешней и на нее не сопоставлен регион Azure, то они отображаются в верхней части схемы. Выберите любую конечную точку, чтобы просмотреть различные расположения (в зависимости от используемого распознавателя DNS), откуда направляется трафик на эту конечную точку. Подключения отображаются в виде линии между конечной точкой и расположением сопоставителя DNS. Они окрашены в соответствии с репрезентативной задержкой между этой парой. Вы видите имя конечной точки и регион Azure, в котором она выполняется. Также отображаются общие объемы запросов, направляемых в него этим профилем диспетчера трафика.


## <a name="tabular-listing-and-raw-data-download"></a>Табличные списки и скачивание необработанных данных

Данные Представления трафика можно просмотреть на портале Azure в табличном формате. Существует запись для каждой пары IP-адресов и конечных точек сопоставителя DNS, которая показывает:

* IP-адрес сопоставителя DNS.
* Имя.
* Географическое расположение региона Azure, в котором находится конечная точка (если она доступна).
* Количество запросов, связанных с этим сопоставительом DNS с этой конечной точкой.
* Характерная задержка, связанная с конечными пользователями, которые используют этот DNS (если они доступны). 

Также вы можете скачать данные Представления трафика в CSV-файле и применить их в любом рабочем процессе аналитики на ваш выбор.

## <a name="billing"></a>Выставление счетов

При использовании Представление трафика счет выставляется в зависимости от количества точек данных, используемых для создания ценных сведений. Сейчас используется только один тип точек данных — запросы, полученные в профиле диспетчера трафика. Дополнительные сведения о ценах см. на [странице цен на диспетчер трафика](https://azure.microsoft.com/pricing/details/traffic-manager/).
