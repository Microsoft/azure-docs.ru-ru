# <a name="customize-an-assessment"></a>Настройка оценки

В этой статье описывается, как настроить оценки, созданные средством "миграция и оценка" службы миграции Azure.

Служба [Миграция Azure](migrate-services-overview.md) объединяет в себе средства наблюдения за обнаружением, оценкой и миграцией локальных приложений и рабочих нагрузок, а также виртуальных машин частного или общедоступного облака в Azure. Эта служба предоставляет инструменты для оценки и миграции, а также предложения сторонних независимых поставщиков программного обеспечения (ISV).

Вы можете использовать средство "миграция и оценка Azure" для создания оценок для локальных виртуальных машин VMware и виртуальных машин Hyper-V в процессе подготовки к миграции в Azure. Средство обнаружения и оценки оценивает локальные серверы для миграции на виртуальные машины Azure IaaS и решение Azure VMware (AVS).

**Решение Azure VMware (AVS)** | Оценки для переноса локальных серверов в [Решение Azure VMware (AVS)](../azure-vmware/introduction.md). <br/><br/> С помощью этого типа оценки вы можете оценить возможность миграции на Решение Azure VMware (AVS) для локальных [виртуальных машин VMware](how-to-set-up-appliance-vmware.md).[Подробнее](concepts-azure-vmware-solution-assessment-calculation.md)

## <a name="how-is-an-assessment-done"></a>Как выполняется оценка?

Оценка, выполненная в ходе обнаружения и оценки в службе "миграция Azure", включает три этапа. Оценка начинается с анализа пригодности, затем следует оценка размера и, наконец, оценка ежемесячной стоимости. Компьютер переходит на следующий этап, только если проходит предыдущий. Например, если компьютер не прошел проверку пригодности для Azure, то он помечается как непригодный для Azure, и размер и стоимость для него вычисляться не будут. [Подробнее.](./concepts-assessment-calculation.md)

**Свойство** | **Сведения**

**Целевое расположение** | Расположение Azure для миграции.<br/> Сейчас Оценка виртуальных машин Azure поддерживает следующие целевые регионы: Восточная Австралия, Юго-Восточная Австралия, Южная Бразилия, Центральная Канада, Восточная Канада, Центральная Индия, Центральная часть США, Восточный Китай, Северный Китай, Восточная Азия, Восточная часть США, Восточная США 2, Центральная Германия, северо-восток, Восточная Япония, Западная Япония, Юго-Запад, Юго-Центральный регион США, Северная Европа , US Gov (Техас), US Gov (Вирджиния), Западная Центральная часть США, Западная Европа, Западная Индия, Западная часть США и Западная США 2.
**Тип хранилища** | С помощью этого свойства можно указать тип дисков, которые необходимо переместить в Azure.<br/><br/> Для изменения размера в локальной среде можно указать целевой тип хранилища в качестве дисков, управляемых с помощью класса Premium, SSD (цен. категория "Стандартный") управляемых дисков или дисков, управляемых HDD (цен. категория "Стандартный"). Для определения размера на основе производительности можно указать тип целевого диска: "автоматические", "управляемые диски", "HDD (цен. категория"Стандартный")" или "управляемые SSD (цен. категория"Стандартный") диски".<br/><br/> При указании типа хранения в качестве автоматического рекомендация диска выполняется на основе данных производительности дисков (операции ввода-вывода в секунду и пропускная способность). Если вы укажете тип хранилища "Премиум" или "Стандартный", оценка будет рекомендовать номер SKU диска в выбранном типе хранилища. Если вы хотите получить соглашение об уровне обслуживания виртуальной машины с одним экземпляром 99,9%, можно указать тип хранилища в качестве дисков, управляемых с помощью класса Premium. Это гарантирует, что для всех дисков в результате оценки рекомендуется использовать управляемые диски категории "Премиум". Azure
**Зарезервированные экземпляры (RI)** | Это свойство помогает указать, есть ли в Azure [зарезервированные экземпляры](https://azure.microsoft.com/pricing/reserved-vm-instances/) . После этого оценки затрат в оценке будут выполняться в скидки на использование RI. Зарезервированные экземпляры в настоящее время поддерживаются только для предложения с оплатой по мере использования в службе "Миграция Azure".
**Условия определения размера** | Критерий для использования в Azure подходящего размера для виртуальных машин. Вы можете либо сделать размер на *основе производительности* , либо как виртуальные машины *как локальные*, без учета журнала производительности.
**Журнал производительности** | Время, которое следует учитывать при оценке данных о производительности компьютеров. Это свойство применимо, только если критерий изменения размера *основан на производительности*.
**Использование процентиля** | Значение процентиля примера производительности, которое учитывается для выбора правильного размера. Это свойство применимо только в том случае, если размер *основан на производительности*.
**Серия виртуальных машин** |     Вы можете указать серии VM, которые вы хотели бы рассмотреть для правильного размера. Например, если у вас есть рабочая среда, которую вы не планируете переносить на виртуальные машины Azure серии А, эту серию можно исключить из списка, и выбор правильного размера будет выполняться только для указанных серий.
**Фактор комфорта** | При оценке виртуальной машины Azure в процессе оценки учитывается буфер (в качестве комфортного). Этот буфер применяется на основе данных об использовании компьютера для виртуальных машин (показателей ЦП, памяти, диска и сети). Фактор комфорта учитывается, например, для сезонного использования и малого количества записей в журнале с потенциальным повышением в будущем.<br/><br/> Например, если виртуальная машина с 10 ядрами загружена на 20 %, обычно в результате оценки определяется виртуальная машина с двумя ядрами. Но с фактором комфорта 2.0x в результате определяется машина с 4 ядрами.
**Предложение** | [Предложение Azure](https://azure.microsoft.com/support/legal/offer-details/), в котором вы зарегистрированы. Служба "Миграция Azure" оценивает стоимость соответствующим образом.
**Денежный формат** | Валюта для выставления счетов.
**Скидка (%)** | Любая относящаяся к подписке скидка, предоставляемая в рамках предложения Azure.<br/> Значение по умолчанию — 0 %.
**Время доступности виртуальной машины** | Если виртуальные машины не должны работать в Azure круглосуточно, вы можете указать период (число дней в месяц и число часов в день), на протяжении которого они будут работать в Azure. При этом будет выполнена соответствующая оценка расходов.<br/> Значение по умолчанию — 31 день в месяц и 24 часа в день.
**Преимущество гибридного использования Azure** | Укажите, есть ли у вас программное обеспечение Software Assurance и что они подходят для [преимущество гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-use-benefit/). Если установлено значение "Да", то для виртуальных машин Windows действуют цены, отличные от цен Microsoft Azure. Значение по умолчанию — "Да".

Фактор комфорта учитывается, например, для сезонного использования и малого количества записей в журнале с потенциальным повышением в будущем.<br/><br/> Например, если виртуальная машина с 10 ядрами загружена на 20 %, обычно в результате оценки определяется виртуальная машина с двумя ядрами. **Целевой тип развертывания** | Целевой тип развертывания, на котором должна выполняться оценка: <br/><br/> Выберите **рекомендуется**, если вы хотите, чтобы служба миграции Azure оценила готовность серверов SQL Server для миграции в Azure SQL MI и базу данных SQL Azure и рекомендовала наиболее подходящий вариант целевого развертывания, целевой уровень, конфигурацию SQL Azure и ежемесячные оценки.<br/><br/>Выберите **базу данных SQL Azure**, если вы хотите оценить серверы SQL Server для миграции на только базы данных SQL Azure и просматривать целевой уровень, конфигурацию базы данных SQL Azure и ежемесячные оценки.<br/><br/>Выберите **Azure SQL MI**, если вы хотите оценить серверы SQL Server для миграции на только базы данных SQL Azure и просматривать целевой уровень, конфигурацию Azure SQL MI и ежемесячные оценки.

**Журнал производительности** | Журнал производительности указывает продолжительность, используемую при оценке данных производительности.
**Использование процентиля** | Использование процентиля указывает значение процентиля образца производительности, используемого для оптимизации дается задание.
**Фактор комфорта** | Буфер, используемый во время оценки. Они учетируют такие проблемы, как сезонное использование, короткий журнал производительности и, вероятнее всего, увеличение использования в будущем.<br/><br/> Например, 10-ядерный экземпляр с использованием 20% обычно приводит к подвум основным экземплярам. В настоящее время вы можете выбрать только оплату по мере использования или разработку и тестирование с оплатой по мере использования. Обратите внимание, что вы можете получить дополнительную скидку, применив зарезервированную емкость и Преимущество гибридного использования Azure поверх предложения с оплатой по мере использования.
**Уровень служб** | Наиболее подходящий вариант уровня служб для удовлетворения бизнес-потребностей при миграции в базу данных SQL Azure и (или) Управляемый экземпляр Azure SQL:<br/><br/>**Рекомендуется** , если вы хотите, чтобы служба миграции Azure рекомендовала оптимальный уровень служб для серверов. Это может быть "Общего назначения" или "Критически важный для бизнеса". <br/><br/> **Общего назначения** Если вам нужна конфигурация SQL Azure, предназначенная для бюджетных рабочих нагрузок. [Подробнее](https://docs.microsoft.com/azure/azure-sql/database/service-tier-general-purpose) <br/><br/> **Критически важный для бизнеса** Если требуется настроить SQL Azure для рабочих нагрузок с низкой задержкой с высокой устойчивостью к сбоям и быстрой отработке отказа. [Подробнее](https://docs.microsoft.com/azure/azure-sql/database/service-tier-business-critical)
**Денежный формат** | Валюта выставления счетов для вашей учетной записи.
**Скидка (%)** | Все скидки, относящиеся к подписке, полученные на основе предложения Azure. Значение по умолчанию — 0 %.
**Преимущество гибридного использования Azure** | Указывает, имеется ли у вас SQL Server лицензия. <br/><br/> Если вы это сделаете и на них будет распространяться действие Software Assurance для подписок на SQL Server, вы сможете применить Преимущество гибридного использования Azure при миграции лицензий в Azure.

## <a name="edit-assessment-properties"></a>Изменение свойств оценки

Чтобы изменить свойства оценки после создания оценки, выполните следующие действия.

1. В проекте службы "миграция Azure" щелкните **серверы**.
2. В поле " **Миграция Azure": обнаружение и оценка** щелкните счетчик оценок.
3. В поле **Оценка** выберите нужную оценку > **изменить свойства**.
5. Настройте свойства оценки в соответствии с приведенными выше таблицами.
6. Нажмите кнопку **Сохранить**, чтобы обновить оценку.


Вы также можете изменить свойства оценки при создании оценки.
