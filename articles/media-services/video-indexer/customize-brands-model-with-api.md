# <a name="customize-a-brands-model-with-the-video-indexer-api"></a>Настройка модели торговых марок с помощью API Индексатора видео

Индексатор видео поддерживает распознавание торговых марок в речи и визуальном тексте во время индексирования и повторного индексирования видео- и аудиосодержимого. Функция распознавания торговых марок идентифицирует упоминания о продуктах, службах и компаниях, предлагаемых базой данных торговых марок Bing. Например, если корпорация Майкрософт упоминается в видео или звуковом содержимом или отображается в видеоролике, индексатор видео определит его как торговую марку в содержимом. Специальная модель торговых марок позволяет исключить обнаружение некоторых торговых марок и включить в состав торговые марки, которые должны быть частью вашей модели, но могут не находиться в базе данных торговых марок Bing. Дополнительные сведения см. в этом [обзоре](customize-brands-model-overview.md).

> [!NOTE]
> Если ваше видео было проиндексировано до добавления торговой марки, его необходимо переиндексировать.

Вы можете использовать API Индексатора видео для создания, использования и изменения моделей торговых марок, обнаруженных в видео, как описано в этом разделе. Вы также можете использовать веб-сайт Индексатора видео, как описано в статье [Customize a Brands model with the Video Indexer API](customize-brands-model-with-api.md) (Настройка модели торговых марок с помощью API Индексатора видео).

## <a name="create-a-brand"></a>Создание торговой марки

API [создания торговой марки](https://api-portal.videoindexer.ai/docs/services/operations/operations/Create-Brand) создает новый пользовательский бренд и добавляет его в настраиваемую модель торговых марок для указанной учетной записи.

> [!NOTE]
> Параметр `enabled` (в тексте) на true помещает торговую марку в список *включения* для индексатора видео, который нужно обнаружить. Если задано `enabled` значение false, то в список *исключений* помещается торговая марка, поэтому индексатор видео не обнаружит его.

Некоторые другие параметры, которые можно задать в тексте:

* `referenceUrl`Значением может быть любой эталонный веб-сайт для торговой марки, например ссылка на страницу Википедии.
* `tags`Значение представляет собой список тегов для торговой марки. Этот тег отображается в поле *Категория* торговой марки на веб-сайте индексатора видео. Например, торговая марка Azure может быть отмечена тегом или отнесена к категории "Облако".

### <a name="response"></a>Ответ

В ответе содержится информация о торговой марке, которую вы только что создали в соответствии с форматом примера ниже.

```json
{
  "referenceUrl": "https://en.wikipedia.org/wiki/Example",
  "id": 97974,
  "name": "Example",
  "accountId": "SampleAccountId",
  "lastModifierUserName": "SampleUserName",
  "created": "2018-04-25T14:59:52.7433333",
  "lastModified": "2018-04-25T14:59:52.7433333",
  "enabled": true,
  "description": "This is an example",
  "tags": [
    "Tag1",
    "Tag2"
  ]
}
```

## <a name="delete-a-brand"></a>Удаление торговой марки

API [удаления торговой марки](https://api-portal.videoindexer.ai/docs/services/operations/operations/Delete-Brand?) удаляет торговую марку из настраиваемой модели торговых марок для указанной учетной записи. Учетная запись указывается в `accountId` параметре. После успешного вызова торговая марка больше не будет находиться в таких списках торговых марок, как *Включить* или *Исключить*.

### <a name="response"></a>Ответ

При успешном удалении фирменного стиля нет возвращенного содержимого.

## <a name="get-a-specific-brand"></a>Получение определенной торговой марки

API [получения торговой марки](https://api-portal.videoindexer.ai/docs/services/operations/operations/Get-Brand?) позволяет искать сведения о торговой марке в настраиваемой модели торговых марок для указанной учетной записи с помощью фирменного идентификатора.

### <a name="response"></a>Ответ

В ответе содержится информация о торговой марке, которую вы искали (с помощью идентификатора торговой марки), в соответствии с форматом примера ниже.

```json
{
  "referenceUrl": "https://en.wikipedia.org/wiki/Example",
  "id": 128846,
  "name": "Example",
  "accountId": "SampleAccountId",
  "lastModifierUserName": "SampleUserName",
  "created": "2018-01-06T13:51:38.3666667",
  "lastModified": "2018-01-11T13:51:38.3666667",
  "enabled": true,
  "description": "This is an example",
  "tags": [
    "Tag1",
    "Tag2"
  ]
}
```

> [!NOTE]
> `enabled` Установка значения `true` означает, что торговая марка находится в списке *включений* для индексатора видео, а `enabled` значение false означает, что торговая марка находится в списке *исключений* , поэтому индексатор видео не определит его.

## <a name="update-a-specific-brand"></a>Обновление определенной торговой марки

Обновление API для [фирменной символики](https://api-portal.videoindexer.ai/docs/services/operations/operations/Update-Brand?) позволяет искать сведения о торговой марке в настраиваемой модели торговых марок для указанной учетной записи с помощью фирменного идентификатора.

### <a name="response"></a>Ответ

В ответе содержится обновленная информация о торговой марке, которую вы только что обновили в соответствии с форматом примера ниже.

```json
{
  "referenceUrl": null,
  "id": 97974,
  "name": "Example",
  "accountId": "SampleAccountId",
  "lastModifierUserName": "SampleUserName",
  "Created": "2018-04-25T14:59:52.7433333",
  "lastModified": "2018-04-25T15:37:50.67",
  "enabled": false,
  "description": "This is an update example",
  "tags": [
    "Tag1",
    "NewTag2"
  ]
}
```

## <a name="get-all-of-the-brands"></a>Получение всех торговых марок

API [получения всех торговых марок](https://api-portal.videoindexer.ai/docs/services/operations/operations/Get-Brands?) возвращает все торговые марки в настраиваемой модели торговых марок для указанной учетной записи независимо от того, должна ли эта марка быть в списке *включить* или *исключить* торговые марки.

### <a name="response"></a>Ответ

В ответе содержится список всех торговых марок вашей учетной записи и все сведения о них в соответствии с форматом примера ниже.

```json
[
    {
        "ReferenceUrl": null,
        "id": 97974,
        "name": "Example",
        "accountId": "AccountId",
        "lastModifierUserName": "UserName",
        "Created": "2018-04-25T14:59:52.7433333",
        "LastModified": "2018-04-25T14:59:52.7433333",
        "enabled": true,
        "description": "This is an example",
        "tags": ["Tag1", "Tag2"]
    },
    {
        "ReferenceUrl": null,
        "id": 97975,
        "name": "Example2",
        "accountId": "AccountId",
        "lastModifierUserName": "UserName",
        "Created": "2018-04-26T14:59:52.7433333",
        "LastModified": "2018-04-26T14:59:52.7433333",
        "enabled": false,
        "description": "This is another example",
        "tags": ["Tag1", "Tag2"]
    },
]
```

> [!NOTE]
> Именованный *Пример* торговой марки находится в списке *включения* для индексатора видео, а торговая марка с именем *example2* находится в списке *исключений* , поэтому индексатор видео не определит его.

## <a name="get-brands-model-settings"></a>Параметры модели торговых марок

API [параметров получения торговых марок](https://api-portal.videoindexer.ai/docs/services/operations/operations/Get-Brands) Возвращает параметры модели торговых марок в указанной учетной записи. Параметры торговой марки отображают, включена или нет база данных торговой марки Bing. Если торговые марки Bing не включены, индексатор видео будет определять только торговые марки из настраиваемой модели торговых марок указанной учетной записи.

### <a name="response"></a>Ответ

Ответ показывает включены ли торговые марки Bing в соответствии с форматом примера ниже.

```json
{
  "state": true,
  "useBuiltIn": true
}
```

> [!NOTE]
> `useBuiltIn` значение true означает, что торговые марки Bing включены. Если `useBuiltin` имеет значение false, торговые марки Bing отключаются. `state`Значение можно игнорировать, так как оно является устаревшим.

## <a name="update-brands-model-settings"></a>Обновление параметров модели торговых марок

API [обновления торговых марок](https://api-portal.videoindexer.ai/docs/services/operations/operations/Update-Brands-Model-Settings?) обновляет параметры модели торговых марок в указанной учетной записи. Параметры торговой марки отображают, включена или нет база данных торговой марки Bing. Если торговые марки Bing не включены, индексатор видео будет определять только торговые марки из настраиваемой модели торговых марок указанной учетной записи.

`useBuiltIn`Флаг, установленный в значение true, означает, что торговые марки Bing включены. Если `useBuiltin` имеет значение false, торговые марки Bing отключаются.

### <a name="response"></a>Ответ

При успешном обновлении параметра модели торговых марок возвращенное содержимое отсутствует.
