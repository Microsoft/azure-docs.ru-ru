# <a name="azure-cache-for-redis-event-grid-overview"></a>Общие сведения о сетке событий кэша Azure для Redis 

События кэша Azure для событий Redis, такие как исправление, масштабирование, импорт и экспорт (RDB), помещаются с помощью службы " [Сетка событий Azure](https://azure.microsoft.com/services/event-grid/) " на подписчики, такие как функции azure, Azure Logic Apps или даже в собственный прослушиватель HTTP. Сетка событий обеспечивает надежную доставку событий в приложения с помощью многофункциональных политик повтора и недоставленных сообщений.

Полный список событий, поддерживаемых кэшем Azure для Redis, см. в статье [о схеме событий кэша Azure для Redis](../event-grid/event-schema-azure-cache.md) .

|--|-|

|PowerShell    |[Краткое руководство. Маршрутизация кэша Azure для событий Redis в веб-конечную точку с помощью PowerShell](cache-event-grid-quickstart-powershell.md)|
|Azure CLI    |[Краткое руководство. Маршрутизация кэша Azure для событий Redis в веб-конечную точку с Azure CLI](cache-event-grid-quickstart-cli.md)|

## <a name="best-practices-for-consuming-events"></a>Рекомендации по использованию событий

Приложения, обрабатывающие события кэша Azure для Redis, должны следовать следующим рекомендациям:
> [!div class="checklist"]
> * Так как несколько подписок можно настроить для маршрутизации событий в один и тот же обработчик событий, важно, чтобы события из определенного источника не предполагались, а проверить раздел сообщения, чтобы убедиться, что он поступает из кэша Azure для ожидаемого экземпляра Redis.
> * Подобным образом убедитесь, что вы готовы к обработке этого типа событий (eventType), а также не следует предполагать, что все получаемые события будут иметь ожидаемый тип.
> * Кэш Azure для событий Redis гарантирует доставку подписчикам по крайней мере, что гарантирует вывод всех сообщений. Однако из-за повторных попыток или доступности подписок могут возникнуть дублирующиеся сообщения. Дополнительные сведения о доставке сообщений и повторных попытках см. в разделе [Доставка сообщений в сетке событий и повторная](../event-grid/delivery-and-retry.md)попытка.
