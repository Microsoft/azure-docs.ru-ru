# <a name="how-to-create-alerts-for-update-management"></a>Создание оповещений для Управление обновлениями

Оповещения в Azure заблаговременно уведомляют Вас о результатах выполнения заданий Runbook, проблем с работоспособностью служб или других сценариев, связанных с вашей учетной записью службы автоматизации. Служба автоматизации Azure не включает предварительно настроенные правила генерации оповещений, но ее можно создать на основе данных, которые он создает. В этой статье приводятся рекомендации по созданию правил генерации оповещений с использованием метрик, включенных в Управление обновлениями.

## <a name="available-metrics"></a>Доступные метрики

Служба автоматизации Azure создает две отдельные метрики платформы, связанные с Управление обновлениями, которые собираются и пересылаются в Azure Monitor. Эти метрики доступны для анализа с помощью [Обозреватель метрик](../../azure-monitor/essentials/metrics-charts.md) и для оповещений с помощью [правила оповещения о метриках](../../azure-monitor/alerts/alerts-metric.md).

Выдаются две метрики:

* Общее количество запусков развертывания обновлений для компьютера
* Общее количество запусков развертывания обновлений

При использовании для оповещений обе метрики поддерживают измерения, которые содержат дополнительные сведения, позволяющие ограничить область предупреждения конкретными сведениями о развертывании обновлений. В следующей таблице приведены сведения о метрике и измерениях, доступных при настройке оповещения.

|Название сигнала|Измерения|Описание
|---|---|---|
|`Total Update Deployment Runs`|— Имя развертывания обновления<br>— Состояние | Оповещает об общем состоянии развертывания обновления.|
|`Total Update Deployment Machine Runs`|— Имя развертывания обновления</br>— Состояние</br>— Целевой компьютер</br>— Идентификатор запуска развертывания обновления    |Оповещает о состоянии развертывания обновления, нацеленного на определенные компьютеры.|

## <a name="create-alert"></a>Создание оповещения

Выполните следующие действия, чтобы настроить оповещения, чтобы сообщить о состоянии развертывания обновлений. Если вы не знакомы с оповещениями Azure, см. статью [Обзор оповещений Azure](../../azure-monitor/alerts/alerts-overview.md).

1. В учетной записи службы автоматизации выберите **оповещения** в разделе **мониторинг**, а затем выберите **новое правило генерации оповещений**.

1. На странице **Создание правила оповещения** ваша учетная запись службы автоматизации уже выбрана в качестве ресурса. Если вы хотите изменить его, выберите **изменить ресурс**.

1. На странице Выбор ресурса в раскрывающемся списке **Фильтровать по типу ресурса** выберите пункт **учетные записи службы автоматизации** .

1. Выберите учетную запись службы автоматизации, которую вы хотите использовать, и нажмите кнопку **Готово**.

1. Выберите **Добавить условие** , чтобы выбрать сигнал, соответствующий вашему требованию.

1. Выберите допустимое значение измерения из списка. Если нужного значения нет в списке, выберите **\+** рядом с измерением и введите пользовательское имя. Затем выберите нужное значение. Если необходимо выбрать все значения для измерения, нажмите кнопку **выбрать \*** . Если не задать значение измерения, Управление обновлениями будет игнорировать это измерение.

    ![Настройка логики сигналов](./media/manage-updates-for-vm/signal-logic.png)

1. В разделе **логика оповещения** введите значения в полях **время агрегирования** и **порога** , а затем нажмите кнопку **Готово**.

1. На следующей странице введите имя и описание оповещения.

1. Установите для параметра **Серьезность** значение **Информационный (уровень серьезности 2)** для успешного запуска или **Информационный (уровень серьезности 1)** для неудачного запуска.

    ![На снимке экрана показан раздел Определение сведений о предупреждении с выделенными полями имя, описание и серьезность правила оповещения.](./media/manage-updates-for-vm/define-alert-details.png)

1. Выберите **Да** , чтобы включить правило генерации оповещений.

## <a name="configure-action-groups-for-your-alerts"></a>Настройка групп действий для оповещений

После настройки оповещений можно создать для них группу действий, которые будут применяться сразу к нескольким оповещениям. Действия могут включать уведомления по электронной почте, модули Runbook, веб-перехватчики и многое другое. Дополнительные сведения о группах действий см. в разделе [Создание групп действий и управление ими на портале Azure](../../azure-monitor/alerts/action-groups.md).

1. Выберите предупреждение, а затем выберите **Добавить группы действий** в разделе **действия**. Отобразится область **Выбор группы действий для присоединения к этой панели правила оповещения** .

   :::image type="content" source="./media/manage-updates-for-vm/select-an-action-group.png" alt-text="Использование и предполагаемые затраты.":::

1. Установите флажок для присоединенной группы действий и нажмите кнопку Выбрать.
