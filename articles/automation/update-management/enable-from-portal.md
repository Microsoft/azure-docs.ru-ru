# <a name="enable-update-management-from-the-azure-portal"></a>Включение Управления обновлениями на портале Azure

Эта статья содержит сведения о включении функции [Управление обновлениями](overview.md) для виртуальных машин с помощью портала Azure. Чтобы включить масштабирование виртуальных машин Azure, необходимо включить существующую виртуальную машину Azure с помощью Управление обновлениями.

Для количества групп ресурсов, используемых для управления виртуальными машинами, установлены [ограничения развертывания Resource Manager](../../azure-resource-manager/templates/deploy-to-resource-group.md). Для каждого развертывания Resource Manager (не следует их путать с развертываниями обновлений) можно использовать не более пяти групп ресурсов. Две из этих групп зарезервированы для настройки рабочей области Log Analytics, учетной записи службы автоматизации и связанных с ними ресурсов. Это означает, что для управления с помощью функции "Управление обновлениями" вы можете выбрать еще три группы ресурсов. Это ограничение применяется только к одновременным установкам, но не к общему числу групп ресурсов под управлением службы автоматизации.

> [!NOTE]
> При включении функции "Управление обновлениями" только определенные регионы поддерживают связывание рабочей области Log Analytics и учетной записи службы автоматизации. Список поддерживаемых пар сопоставлений см. в статье о [сопоставлении региона для учетной записи службы автоматизации и рабочей области Log Analytics](../how-to/region-mappings.md).

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас ее нет, [активируйте преимущества для подписчиков MSDN](https://azure.microsoft.com/pricing/member-offers/msdn-benefits-details/) или [зарегистрируйте бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F).
* [Учетная запись службы автоматизации](../automation-security-overview.md) для управления компьютерами.
* [Виртуальная машина](../../virtual-machines/windows/quick-create-portal.md).

## <a name="sign-in-to-azure"></a>Вход в Azure

Войдите в Azure по адресу https://portal.azure.com.

## <a name="enable-update-management"></a>Включение управления обновлениями

1. На портале Azure перейдите в раздел **Виртуальные машины**.

2. На странице **виртуальные машины** с помощью флажков выберите виртуальные машины, которые нужно добавить в Управление обновлениями. За один раз виртуальные машины можно добавить в одну, две или три группы ресурсов. Виртуальные машины Azure могут находиться в любом регионе вне зависимости от расположения учетной записи службы автоматизации.

    ![Список виртуальных машин](media/enable-from-portal/vmlist.png)

    > [!TIP]
    > С помощью фильтров выберите виртуальные машины из разных подписок, расположений и групп ресурсов. Вы можете выбрать все виртуальные машины в списке, установив соответствующий флажок вверху.

    [![Включить управление обновлениями](./media/enable-from-portal/onboard-feature.png)](./media/enable-from-portal/onboard-feature-expanded.png#lightbox)

3. Выберите **службы** и выберите **Управление обновлениями** для функции Управление обновлениями.

4. В списке виртуальных машин отображаются только те виртуальные машины, которые находятся в одной и той же подписке и в том же расположении. Если виртуальные машины находятся в более чем трех группах ресурсов, будут выбраны первые три группы.

5. По умолчанию выбрана существующая рабочая область Log Analytics и учетная запись службы автоматизации. Если вы хотите использовать другую рабочую область Log Analytics и учетную запись службы автоматизации, выберите **Настраиваемый** , чтобы выбрать их на странице настраиваемая конфигурация. После выбора рабочей области Log Analytics выполняется проверка того, связана ли она с учетной записью службы автоматизации. Если связанная учетная запись найдена, отобразится сообщение, как на рисунке ниже. Когда все будет готово, нажмите кнопку **ОК**.

    [![Выбор рабочей области и учетной записи](./media/enable-from-portal/select-workspace-and-account.png)](./media/enable-from-portal/select-workspace-and-account-expanded.png#lightbox)

6. Если выбранная рабочая область не связана с учетной записью службы автоматизации, отобразится сообщение, как на рисунке ниже. Выберите учетную запись службы автоматизации и нажмите кнопку **ОК** по завершении.

    ![Рабочая область отсутствует](media/enable-from-portal/no-workspace.png)

7. Отмените выбор любой виртуальной машины, которую вы не хотите включать. Если виртуальную машину подключить невозможно, рядом с ее именем не будет флажка.

8. Выберите **включить** , чтобы включить эту функцию. После включения Управление обновлениями может потребоваться около 15 минут, прежде чем вы сможете просмотреть оценку обновлений.
