# <a name="use-the-azure-portal-to-administer-your-azure-data-box-and-azure-data-box-heavy"></a>Используйте портал Azure для администрирования Azure Data Box и Azure Data Box Heavy

Эта статья относится как к Azure Data Box, так и к Azure Data Box Heavy. В этой статье описываются некоторые сложные рабочие процессы и задачи управления, которые можно выполнять на Azure Data Box устройстве. Вы можете управлять Data Box устройством через портал Azure или через локальный веб-интерфейс.

В этой статье речь идет о задачах, выполняемых с использованием портала Azure. Используйте портал Azure для управления заказами, управления Data Box устройством и наблюдения за состоянием заказа по мере его завершения.

## <a name="cancel-an-order"></a>Отмена заказа

После этого может потребоваться отменить заказ по разным причинам.

Для импорта и экспорта заказов можно отменить только порядок, прежде чем он будет обработан. После обработки заказа и подготовки устройства Data Box вы не сможете отменить заказ.

Для отмены заказа выполните следующие действия.

1.  Перейдите на вкладку **Обзор > Отмена**.

    ![Команда "Отмена" на вкладке "Обзор" для заказа](media/data-box-portal-admin/portal-admin-cancel-command.png)

2.  Введите причину для отмены заказа.  

    ![Диалоговое окно «Отмена порядка»](media/data-box-portal-admin/portal-admin-cancel-order-dbox.png)

3.  После отмены заказа на портале обновляется состояние заказа и отображается значение **Отменено**.

## <a name="clone-an-order"></a>Клонирование заказа

Клонирование полезно в определенных ситуациях. Например, вы использовали Data Box для перемещения данных. По мере создания данных необходимо использовать другое устройство Data Box для перемещения этих данных в Azure. В этом случае можно просто клонировать тот же порядок.

Выполните следующие действия, чтобы клонировать заказ на импорт.

1.  Перейдите на вкладку **Обзор > Клонировать**. 

    ![Команда clone на вкладке "Обзор" для заказа](media/data-box-portal-admin/portal-admin-clone-command.png)

2.  Все сведения о заказе остаются теми же. Имя заказа — это первоначальное имя с добавлением *-Clone*. Установите флажок, чтобы подтвердить, что вы просмотрели сведения о конфиденциальности. Нажмите кнопку **Создать**.

Клон создается в течение нескольких минут, после чего портал обновляется и отображается новый заказ.


## <a name="delete-order"></a>Удаление заказа

Вы можете удалить заказ после его обработки. Заказ содержит ваши личные сведения, такие как имя, адрес и контактные данные. При удалении заказа удаляются и личные данные.

Можно удалить только завершенные или отмененные заказы. Чтобы удалить заказ, выполните указанные ниже действия.

1. Щелкните **Все ресурсы**. Найдите свой заказ.

2. Щелкните заказ, который необходимо удалить, и перейдите на вкладку **Обзор**. На панели команд нажмите кнопку **Удалить**.

    ![Команда "Удалить" на вкладке "Обзор" для заказа](media/data-box-portal-admin/portal-admin-delete-command.png)

3. Когда будет предложено подтвердить удаление, введите имя заказа. Щелкните **Удалить**.

## <a name="download-shipping-label"></a>Скачивание транспортной этикетки

Может потребоваться загрузить метку доставки, если отображение Data Box не работает, и метка отгрузки возврата не отображается. На Data Box Heavy не отображаются электронные рукописные данные, поэтому этот рабочий процесс не применяется к Data Box Heavy.

Выполните следующие действия, чтобы скачать транспортную этикетку.

1.  Выберите **Обзор > Скачать транспортную этикетку**. Этот параметр доступен только после отправки устройства. 

    ![Скачивание транспортной этикетки](media/data-box-portal-admin/portal-admin-download-shipping-label.png)

2.  Загрузится новая транспортная этикетка для возврата, которая выглядит следующим образом. Сохраните метку и распечатайте ее. Отогните и вставьте метку в очистку конверта на устройстве. Убедитесь, что этикетка является видимой. Удалите все наклейки, которые находятся на устройстве от предыдущей доставки.

    ![Пример транспортной этикетки](media/data-box-portal-admin/portal-admin-example-shipping-label.png)

## <a name="edit-shipping-address"></a>Изменение адреса поставки

Вам может понадобиться изменить адрес поставки после размещения заказа. Эта возможность доступна только до отправки устройства. После отправки устройства этот параметр больше недоступен.

Выполните следующие действия для изменения адреса доставки.

1. Выберите **Сведения о заказе > Изменить адрес поставки**.

    ![Команда "изменить адрес доставки" в сведениях о заказе](media/data-box-portal-admin/portal-admin-edit-shipping-address-command.png)

2. Отредактируйте и подтвердите адрес доставки, а затем сохраните изменения.

    ![Диалоговое окно "изменение адреса доставки"](media/data-box-portal-admin/portal-admin-edit-shipping-address-dbox.png)

## <a name="edit-notification-details"></a>Изменение сведений об уведомлении

Может потребоваться изменить пользователей, которые получают сообщения о состоянии заказа. Например, пользователя нужно уведомить о доставке или получении устройства. Другого пользователя, возможно, понадобиться уведомить о завершении копирования данных, чтобы он проверил, что данные находятся в учетной записи хранения Azure, перед удалением их из источника. В таких случаях можно изменить сведения об уведомлении.

Выполните следующие действия, чтобы изменить сведения об уведомлении.

1. Перейдите к **Сведения о заказе > Изменить сведения об уведомлении**.

    ![Команда "изменить сведения об уведомлении" в сведениях о заказе](media/data-box-portal-admin/portal-admin-edit-notification-details-command.png)

2. Теперь можно изменить сведения об уведомлении, а затем сохранить изменения.
 
    ![Диалоговое окно «Изменение сведений об уведомлении»](media/data-box-portal-admin/portal-admin-edit-notification-details-dbox.png)


## <a name="download-order-history"></a>Скачивание журнала заказов

Как только заказ Data Box будет завершен, данные на дисках устройства будут удалены. После очистки устройства вы сможете скачать журнал заказов на портале Azure.

Чтобы скачать журнал заказов, сделайте следующее.

1. В заказе Data Box выберите **Обзор**. Убедитесь, что заказ выполнен. Если заказ и очистка устройства выполнены, выберите **Сведения о заказе**. Доступен параметр **Скачать журнал заказов**.

    ![Скачивание журнала заказов](media/data-box-portal-admin/portal-admin-download-order-history.png)

2. Щелкните **Скачать журнал заказов**. Скачанный журнал содержит запись журналов отслеживания несущей. На Data Box Heavy устройстве будет два набора журналов, соответствующих двум узлам. Если вы прокрутите страницу этого журнала до конца, то увидите ссылки на такие элементы:
    
   - **Копировать журналы** — список файлов, которые были выработаны во время копирования данных из Data Box в учетную запись хранения Azure (порядок импорта) или из учетной записи хранения в Data Box (порядок экспорта).
   - **Журналы аудита** — содержат сведения о включении Data Box и доступе к общим ресурсам, если Data Box находится за пределами центра обработки данных Azure.
   - **Файлы спецификации в порядке импорта** — список файлов (также называемый манифестом файла), который можно скачать во время **Подготовка к отправке** и имена файлов, размеры файлов и контрольные суммы файлов.
   - **Подробные журналы в порядке экспорта** — список файлов с именами файлов, размерами файлов и вычислением контрольных сумм при копировании данных из учетных записей хранения Azure в Data Box.

## <a name="view-order-status"></a>Просмотр состояния заказа

При изменении состояния устройства на портале вы получаете уведомление по электронной почте.

### <a name="statuses-for-import-order"></a>Состояния для порядка импорта

|---------|---------|

Для обработки заказа может потребоваться от нескольких дней до нескольких месяцев. Если заказ не может быть выполнен через 90 дней, заказ отменяется и вы получаете уведомления.         |
|Обработано     | Обработка заказа завершена. В соответствии с вашим заказом устройство готово к отправке в центре обработки данных.         |
|Отправлено     | Заказ отправлен. Используйте идентификатор отслеживания, отображаемый в вашем заказе на портале, чтобы отслеживать отправку.        |
|Доставлено     | Отправка была доставлена ​​по адресу, указанному в заказе.        |

|Копирование данных     | Выполняется копирование данных. Отслеживайте ход копирования вашего заказа на портале Azure. <br> Дождитесь завершения копирования данных. |
|Завершено       |Заказ успешно выполнен.<br> Прежде чем удалять локальные данные с серверов, убедитесь, что данные находятся в Azure.         |

Запишите изменения, внесенные в данные. См. [примеры журналов копирования при завершении передачи с предупреждениями](./data-box-logs.md#upload-completed-with-warnings).   |

|Очистка | Данные на дисках устройства стираются. Очистка устройства считается завершенной, когда журнал заказов доступен для скачивания на портале Azure.|

Ниже приведены состояния для заказа на экспорт.

|состояние заказа; |Описание |
|---------|---------|

Для обработки заказа может потребоваться от нескольких дней до нескольких месяцев. Если заказ не может быть выполнен за 90 дней, он отменяется, и вы получаете уведомление.         |
