# <a name="scalability-and-performance-targets-for-premium-page-blob-storage-accounts"></a>Целевые показатели масштабируемости и производительности для учетных записей хранилища BLOB-объектов класса Premium

[!INCLUDE [storage-scalability-intro-include](../../../includes/storage-scalability-intro-include.md)]

## <a name="scale-targets-for-premium-page-blob-accounts"></a>Целевые показатели масштабирования для учетных записей BLOB-объектов класса Premium

Учетная запись хранения больших двоичных объектов с расширенной производительностью оптимизирована для операций чтения и записи. Этот тип учетной записи хранения создает резервную копию неуправляемого диска для виртуальной машины Azure.

> [!NOTE]
> Корпорация Майкрософт рекомендует использовать управляемые диски с виртуальными машинами Azure, если это возможно. Дополнительные сведения об управляемых дисках см. в разделе [Общие сведения о хранилище дисков Azure виртуальных машин](../../virtual-machines/managed-disks-overview.md).

Учетные записи хранения BLOB-объектов уровня "Премиум" имеют следующие целевые показатели масштабируемости:

| Общая емкость учетной записи                            | Общая пропускная способность для учетной записи локально избыточного хранилища                     |
| ------------------------------------------------- | --------------------------------------------------------------------------- |
| Емкость диска: 4 ТБ (отдельный диск)/35 ТБ (совокупное количество всех дисков) <br>Емкость для моментальных снимков: 10 ТБ | До 50 гигабит в секунду для входящих<sup>1</sup> и исходящих подключений<sup>2</sup> |

<sup>1</sup> Все данные (запросы), отправляемые в учетную запись хранилища.

<sup>2</sup> Все данные (отклики), получаемые из учетной записи хранения.

Учетная запись BLOB-объекта уровня "Премиум" — это учетная запись общего назначения, настроенная для повышения производительности. Рекомендуется использовать учетные записи хранения общего назначения версии 2.

Если вы используете учетные записи хранилища BLOB-объектов уровня "Премиум" для неуправляемых дисков, а приложение превышает целевые показатели масштабируемости для одной учетной записи хранения, корпорация Майкрософт рекомендует перейти на управляемые диски. Дополнительные сведения об управляемых дисках см. в разделе [Общие сведения о хранилище дисков Azure виртуальных машин](../../virtual-machines/managed-disks-overview.md).

Если вы не можете выполнить миграцию на управляемые диски, создайте приложение для использования нескольких учетных записей хранения и разбейте данные на эти учетные записи хранения. Например, если необходимо подключить диски объемом 51 ТБ к нескольким виртуальным машинам, распределите их между двумя учетными записями хранения. Максимальный размер, поддерживаемый одной учетной записью хранилища класса Premium, составляет 35 ТБ. Убедитесь, что в одной учетной записи хранения производительности Premium не более 35 ТБ подготовленных дисков.
