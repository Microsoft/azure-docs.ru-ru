# <a name="working-remotely-using-azure-networking-services"></a>Удаленная работа с использованием сетевых служб Azure

>[!NOTE]
> В этой статье описывается, как использовать сетевые службы Azure, сеть Майкрософт и партнерскую экосистему Azure для удаленной работы и устранения проблем с сетью, вызванных пандемией COVID–19.

В этой статье описываются возможности, которые организации могут использовать для организации удаленного доступа пользователей или в качестве дополнения к существующим решениям для увеличения их пропускной способности в периоды пиковой нагрузки. Архитекторам сетей приходится решать следующие проблемы:

- Справляться с более интенсивным использованием сети.
- Обеспечивать надежное подключение для большего числа сотрудников и клиентов компании.
- Предоставлять подключение к удаленным точкам по всему миру.

Не во всех сетях возникают заторы из-за массового подключения удаленных сотрудников. В частности, это касается частных глобальных сетей и основных корпоративных сетей. Узкие места обычно возникают только в домашних широкополосных сетях и VPN-шлюзах корпоративных локальных сетей.

Планировщики сетей могут справиться с проблемой узких мест и контролировать перегрузку сети за счет интеллектуального перенаправления и распределения нагрузки и если будут помнить, что для различных типов трафика требуются разные приоритеты сетевой обработки. Например, при дистанционной консультации у врача очень важно передавать трафик в реальном времени, а задержки и дрожание недопустимы. Вместе с тем при репликации того же трафика между хранилищами задержки несущественны. Первый трафик необходимо направлять по самому оптимальному сетевому маршруту с более высоким качеством обслуживания; а для второго трафика можно выбрать маршрут с неоптимальными параметрами.



## <a name="sharing-our-best-practices---azure-network-is-designed-for-elasticity-and-high-availability"></a>Наши рекомендации: эластичность и высокий уровень доступности — основные характеристики сети Azure

В Azure предусмотрена устойчивость к внезапным изменениям уровня использования ресурсов, что позволяет успешно справляться с периодами пиковой загрузки. Кроме того, корпорация Майкрософт оперирует одной из самых больших сетей в мире. Сеть Майкрософт рассчитана на высокую доступность и способна выдерживать различного рода сбои: от неполадок с одним сетевым элементом до отказа всего региона.

Сеть Майкрософт разработана в соответствии с требованиями и поддерживает оптимальную производительность при передаче различных типов сетевого трафика, включая чувствительный к задержкам мультимедийный трафик Skype и Teams, CDN, анализа больших данных в реальном времени, службы хранилища Azure, Bing и Xbox. Чтобы обеспечить оптимальную производительность для различных типов трафика, сеть Microsoft притягивает весь трафик, передаваемый через ее ресурсы, как можно ближе к его источнику.

>[!NOTE] 
>Описанные ниже сетевые функции Azure позволяют использовать возможности притягивания трафика в глобальной сети Майкрософт и тем самым повысить эффективность работы клиентов в сети. Притягивание трафика в сети Майкрософт помогает максимально быстро выгружать трафик из первого и последнего участков сети, где могут возникать заторы в периоды пиковой нагрузки.
>

## <a name="enable-employees-to-work-remotely"></a>Организация удаленной работы сотрудников

VPN-шлюз Azure поддерживает VPN-подключения типа "точка — сеть" (P2S) и "сеть — сеть" (S2S). С помощью VPN-шлюза Azure можно масштабировать подключения сотрудников для безопасного доступа к развернутым ресурсам Azure и локальным ресурсам. Подробнее см. в статье [Как наладить удаленную работу пользователей](../vpn-gateway/work-remotely-support.md). 

При использовании протокола SSTP (Secure Sockets Tunneling Protocol) поддерживается не более 128 одновременных подключений. Чтобы вам было доступно больше подключений, мы рекомендуем перейти на OpenVPN или IKEv2. Подробнее см. в статье [Переход с SSTP на протокол OpenVPN или IKEv2](../vpn-gateway/ikev2-openvpn-from-sstp.md
).

Для доступа к ресурсам, развернутым в Azure, удаленные разработчики могут использовать вместо VPN-подключения решение “Бастион Azure”, чтобы получить безопасный доступ к оболочке (RDP или SSH), для которого не нужно иметь общедоступные IP-адреса на виртуальных машинах. Подробнее см. в статье [Удаленная работа с помощью Бастиона Azure](../bastion/work-remotely-support.md).

Для агрегации крупномасштабного VPN-подключения, чтобы поддержать любые подключения между ресурсами в разных локальных расположениях по всему миру и в разных региональных виртуальных сетях со звездообразной топологией, а также чтобы оптимизировать использование нескольких домашних широкополосных сетей, можно применить Виртуальную глобальную сеть Azure. Подробнее см. в статье [Сложно организовать удаленную работу из дома? Вам поможет Виртуальная глобальная сеть Azure](../virtual-wan/work-remotely-support.md).

Чтобы поддержать удаленных сотрудников, можно также развернуть инфраструктуру виртуальных рабочих столов (VDI), размещенную в виртуальной сети Azure и защищенную с помощью Брандмауэра Azure. Виртуальный рабочий стол Windows — это служба виртуализации для компьютеров и приложений на основе Azure. С помощью Виртуального рабочего стола Windows можно организовать масштабируемую и гибкую среду в рамках подписки Azure, не запуская дополнительные серверы шлюза. Вы несете единоличную ответственность за виртуальные машины WVD в своей виртуальной сети. Подробнее см. в статье [Поддержка удаленной работы с помощью Брандмауэра Azure](../firewall/remote-work-support.md). 

Azure располагает большой экосистемой партнеров. Сетевые виртуальные устройства на базе Azure от наших партнеров также помогают масштабировать VPN-подключения. Подробнее см. в статье [Сведения об использовании сетевого виртуального модуля (NVA) для дистанционной работы](../vpn-gateway/nva-work-remotely-support.md).

## <a name="extend-employees-connection-to-access-globally-distributed-resources"></a>Расширенные возможности подключения сотрудников для доступа к глобально распределенным ресурсам

Следующие службы Azure помогут сотрудникам получить доступ к вашим глобально распределенным ресурсам. Ресурсы могут находиться в любом регионе Azure, в локальных сетях или даже в других общедоступных либо частных облаках. 

- **Пиринг между виртуальными сетями Azure.** Если вы развертываете ресурсы в нескольких регионах Azure и (или) агрегируете подключения удаленных сотрудников, использующих несколько виртуальных сетей, вы можете установить подключения между несколькими виртуальными сетями Azure с помощью пиринга. Дополнительные сведения см. в статье [Пиринг между виртуальными сетями][VNet-peer].

- **Решение на основе Azure VPN.** Удаленным сотрудникам, подключенным к Azure через VPN-подключение P2S или S2S, можно предоставить доступ к локальным сетям, настроив VPN-подключение S2S между локальными сетями и VPN-шлюзом Azure. Подробнее см. в статье [Создание подключения "сеть — сеть"][S2S].

- **ExpressRoute.** Частный пиринг ExpressRoute позволяет обеспечить частное подключение между развернутыми службами Azure и локальной инфраструктурой или инфраструктурой центра совместного размещения. ExpressRoute также предоставляет доступ к общедоступным конечным точкам сети Майкрософт из вашей локальной сети через пиринг Майкрософт. Подключения ExpressRoute не проходят через общедоступный Интернет. Они обеспечивают более безопасное, надежное и скоростное подключение, чем обычные подключения через Интернет, при меньшем числе задержек, которые проще прогнозировать. Дополнительные сведения см. в статье [Обзор ExpressRoute][ExR]. Если вы обратитесь к поставщику сети, который входит в нашу [экосистему партнеров ExpressRoute][ExR-eco], вы сможете быстрее обеспечить себе подключения с большой пропускной способностью к сети Майкрософт.  [ExpressRoute Direct][ExR-D] позволяет напрямую подключить вашу локальную сеть к магистрали Microsoft. ExpressRoute Direct предлагает двойные каналы с пропускной способностью 10 Гбит/с или 100 Гбит/с. 

- **Виртуальная глобальная сеть Azure.** Виртуальная глобальная сеть Azure обеспечивает эффективное взаимодействие между вашими VPN-подключениями и каналами ExpressRoute. Как уже упоминалось, Виртуальная глобальная сеть Azure также поддерживает любые подключения между ресурсами в разных локальных расположениях по всему миру и в разных региональных сетях со звездообразной топологией.

## <a name="scale-customer-connectivity-to-frontend-resources"></a>Масштабирование подключений клиентов к внешним ресурсам

В те моменты, когда в сеть заходит больше людей, трафик многих корпоративных веб-сайтов увеличивается. Шлюз приложений Azure помогает управлять этой повышенной внешней рабочей нагрузкой. Подробнее см. в статье [Поддержка больших объемов трафика Шлюза приложений](../application-gateway/high-traffic-support.md).

## <a name="microsoft-support-for-multi-cloud-traffic"></a>Поддержка трафика в нескольких облаках от Майкрософт

Корпорация Майкрософт предоставляет глобальные возможности подключения для служб, развернутых в других общедоступных облаках. Для этого можно использовать Виртуальную глобальную сеть Azure, VPN или ExpressRoute. Чтобы расширить возможности подключения Azure к другим облакам, можно настроить VPN-подключение S2S между двумя облаками. Подключение Azure к другим общедоступным облакам можно также организовать с помощью ExpressRoute. Облако Oracle Cloud принадлежит к экосистеме партнеров ExpressRoute. Вы можете [настроить прямое взаимодействие между Azure и инфраструктурой Oracle Cloud][Az-OCI]. Большинство поставщиков услуг, принадлежащих к экосистеме партнеров ExpressRoute, также предлагают частное подключение к другим общедоступным облакам. С их помощью вы можете установить через ExpressRoute частное подключение между службами, развернутыми в Azure, и другими облаками.
