# <a name="how-to-configure-virtual-hub-routing"></a>Настройка маршрутизации для виртуального концентратора

Виртуальный концентратор может содержать несколько шлюзов, таких как VPN-шлюз типа "сеть — сеть", шлюз ExpressRoute, шлюз типа "точка — сеть" и брандмауэр Azure. Возможности маршрутизации в виртуальном концентраторе предоставляются маршрутизатором, который управляет всей маршрутизацией, включая транзитную маршрутизацию, между шлюзами с помощью протокол BGP (BGP). Этот маршрутизатор также обеспечивает транзитное подключение между виртуальными сетями, которые подключаются к виртуальному концентратору и поддерживает суммарную пропускную способность 50 Гбит/с. Эти возможности маршрутизации применяются к стандартным виртуальным клиентам глобальной сети.

Дополнительные сведения см. в статье [о маршрутизации виртуальных концентраторов](about-virtual-hub-routing.md).

## <a name="create-a-route-table"></a><a name="create-table"></a>Создание таблицы маршрутов

1. В портал Azure перейдите к виртуальному концентратору.
2. В разделе **Подключение** выберите **Маршрутизация**. На странице маршрутизации отображаются таблицы маршрутов **по умолчанию** и **без них** .

   :::image type="content" source="./media/how-to-virtual-hub-routing/routing.png" alt-text="Страница «маршрутизация»":::
3. Выберите **+ создать таблицу маршрутов** , чтобы открыть страницу **Создание таблицы маршрутов** .
4. На вкладке " **Основные сведения** " страницы Создание таблицы маршрутов заполните следующие поля.

   :::image type="content" source="./media/how-to-virtual-hub-routing/basics.png" alt-text="Вкладка &quot;Основные сведения&quot;":::

   * **имя**;
   * **Маршруты**
   * **Имя маршрута**
   * **Конечный тип**
   * **Префикс назначения**: можно объединять префиксы. Например: VNet 1:10.1.0.0/24 и VNet 2:10.1.1.0/24 можно объединить как 10.1.0.0/16. Маршруты **ветвей** применяются ко всем ПОДКЛЮЧЕННЫМ VPN-сайтам, каналам ExpressRoute и VPN-подключениям пользователей.
   * **Следующий прыжок**: список подключений к виртуальной сети или брандмауэр Azure.

     При выборе подключения к виртуальной сети вы увидите **настройку статических маршрутов**. Это необязательный параметр конфигурации. Дополнительные сведения см. в разделе [Настройка статических маршрутов](about-virtual-hub-routing.md#static).

      :::image type="content" source="./media/how-to-virtual-hub-routing/next-hop.png" alt-text="Следующий прыжок":::

5. Перейдите на вкладку **метки** , чтобы настроить имена меток. Метки предоставляют механизм для логической группировки таблиц маршрутов.

    :::image type="content" source="./media/how-to-virtual-hub-routing/labels.png" alt-text="Настройка имен меток":::

6. Перейдите на вкладку **связи** , чтобы связать соединения с таблицей маршрутов.
Вы увидите **ветви**, **виртуальные сети** и **текущие параметры** подключений.

    :::image type="content" source="./media/how-to-virtual-hub-routing/associations.png" alt-text="Сопоставление подключений с таблицей маршрутов":::

7. Перейдите на вкладку **распространение** , чтобы распространить маршруты от соединений к таблице маршрутов.

    :::image type="content" source="./media/how-to-virtual-hub-routing/propagations.png" alt-text="Распространение маршрутов":::

8. Выберите **создать** , чтобы создать таблицу маршрутов.

## <a name="to-edit-a-route-table"></a><a name="edit-table"></a>Изменение таблицы маршрутов

В портал Azure выберите таблицу маршрутов виртуального концентратора. Выберите таблицу маршрутов, чтобы изменить любые сведения.

## <a name="to-delete-a-route-table"></a><a name="delete-table"></a>Удаление таблицы маршрутизации

В портал Azure выберите таблицу маршрутов виртуального концентратора. Нельзя удалить таблицу маршрутов по умолчанию или None. Однако можно удалить все пользовательские таблицы маршрутов. Нажмите кнопку **"..."**, а затем выберите **Удалить**.

## <a name="to-view-effective-routes"></a><a name="view-routes"></a>Просмотр эффективных маршрутов

В портал Azure выберите таблицу маршрутов виртуального концентратора. Нажмите кнопку **"..."** и выберите **действующие маршруты** , чтобы просмотреть маршруты, полученные выбранной таблицей маршрутов. Распространенные маршруты из соединения с таблицей маршрутов автоматически заполняются в **действующих маршрутах** таблицы маршрутов. Дополнительные сведения см. в статье [о действующих маршрутах](effective-routes-virtual-hub.md).

:::image type="content" source="./media/how-to-virtual-hub-routing/effective.png" alt-text="Просмотр эффективных маршрутов" lightbox="./media/how-to-virtual-hub-routing/effective-expand.png":::

## <a name="to-set-up-routing-configuration-for-a-virtual-network-connection"></a><a name="routing-configuration"></a>Настройка конфигурации маршрутизации для подключения к виртуальной сети

1. В портал Azure перейдите к виртуальной глобальной сети и в разделе **Подключение** выберите **виртуальные сети подключения**.
1. Выберите **+ Добавить подключение**.
1. Выберите виртуальную сеть из раскрывающегося списка.
1. Настройте конфигурацию маршрутизации для связи с таблицей маршрутов. В поле **связать таблицу маршрутов** выберите таблицу маршрутов из раскрывающегося списка.
1. Настройте конфигурацию маршрутизации для распространения в одну или несколько таблиц маршрутов. Для **передачи в таблицу маршрутов** выберите из раскрывающегося списка.
1. Для **статических маршрутов** настройте статические маршруты для сетевого виртуального устройства (если применимо). Виртуальная глобальная сеть поддерживает один IP-адрес следующего прыжка для статического маршрута в подключении к виртуальной сети. Например, если у вас есть отдельное виртуальное устройство для входящих и исходящих потоков трафика, то лучше всего использовать виртуальные устройства в отдельных виртуальных сетей и подключить виртуальных сетей к виртуальному концентратору.
