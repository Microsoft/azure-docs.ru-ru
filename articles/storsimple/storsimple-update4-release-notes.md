# <a name="storsimple-8000-series-update-4-release-notes"></a>Заметки о выпуске, посвященные обновлению 4 для устройств серии StorSimple 8000

## <a name="overview"></a>Обзор

В приведенных ниже заметках о выпуске описаны новые функции и критические открытые проблемы в обновлении 4 для устройств серии StorSimple 8000. Здесь также содержится список обновлений программного обеспечения StorSimple для этого выпуска. 

Обновление 4 можно применить к любому устройству StorSimple под управлением ПО общедоступной версии или версии с обновлениями 0.1–3.1. Версия устройства, связанная с обновлением 4, — 6.3.9600.17820.

Перед развертыванием обновления в решении StorSimple, просмотрите сведения, содержащиеся в заметках о выпуске.

> [!IMPORTANT]
> * Обновление 4 включает обновления ПО для устройств, встроенного ПО USM, драйвера и встроенного ПО LSI, встроенного ПО для дисков, Storport и Spaceport, обновления безопасности и другие обновления ОС. Установка этого обновления занимает примерно 4,5–2 часа. Обновление встроенного ПО диска нарушает работу системы и приводит к простою для вашего устройства. Мы рекомендуем применить обновление версии 4.0, чтобы поддерживать актуальное состояние устройства. 
> * Для новых выпусков обновления могут отображаться не сразу, поскольку развертывание обновлений выполняется поэтапно. Подождите несколько дней и затем повторите попытку поиска обновлений, так как они станут доступными в ближайшее время.

## <a name="whats-new-in-update-4"></a>Новые возможности в обновлении 4

В обновление 4 внесены следующие основные улучшения и исправления ошибок.

* **Более интеллектуальные алгоритмы реорганизации пространства**. В обновлении 4 были улучшены алгоритмы реорганизации пространства, которые корректируют циклы автоматической реорганизации пространства с учетом ожидаемого доступного пространства в облаке. 

* **Средства повышения производительности для локально закрепленных томов**. В обновлении 4 повышена производительность локально закрепленных томов в сценариях с высокими объемами принимаемых данных (сравнимых с размерами самого тома).

* **Восстановление на основе тепловых карт**. В предыдущих выпусках после аварийного восстановления данные восстанавливались из облака на основе схем доступа, что приводило к низкой производительности. 

    В обновлении 4 реализована новая функция, которая в период до аварийного восстановления отслеживает часто запрашиваемые данные и создает тепловую карту (более часто используемые блоки данных считаются более "теплыми" по сравнению с редко используемыми блоками данных). После аварийного восстановления StorSimple использует тепловую карту для автоматического восстановления и заполнения данных из облака. 

    Теперь все операции восстановления данных выполняются на основе тепловых карт. Дополнительные сведения о том, как запрашивать и отменять задания восстановления и заполнения данных на основе тепловых карт, см. в [справочнике по командлетам Windows PowerShell для StorSimple](/powershell/module/hcs/?viewFallbackFrom=winserverr2-ps).

* **Средство диагностики StorSimple**. В обновление 4 включено средство диагностики StorSimple, которое позволяет легко диагностировать и устранять проблемы, связанные с работой системы и сети, производительностью и исправностью оборудования. Это средство запускается с помощью Windows PowerShell для StorSimple. Дополнительные сведения см. в статье об [устранении неполадок с помощью средства диагностики StorSimple](storsimple-8000-diagnostics.md).

* **Средство миграции StorSimple с пользовательским интерфейсом**. До этой версии перенос данных с устройств серий 5000–7000 подразумевал, что пользователи должны выполнять часть процесса миграции вручную с помощью интерфейса Azure PowerShell. В этом выпуске предоставляется удобное средство миграции StorSimple с пользовательским интерфейсом, которое упрощает процесс миграции. Также это позволяет консолидировать контейнеры восстановления. 

* **Изменения, связанные с FIPS**. Начиная с этого выпуска система FIPS включена по умолчанию на всех устройствах StorSimple серии 8000 как в Microsoft Azure для государственных организаций, так и в общедоступных облачных учетных записях Azure.

* **Изменение обновлений**. В этом выпуске исправлены ошибки, связанные со сбоями при установке обновлений.

* **Предупреждение об ошибках диска**. В этом выпуске добавлено новое предупреждение, которое информирует пользователя об ошибках диска, которые могут произойти в ближайшее время. В случае появления этого предупреждения обратитесь в службу поддержки Майкрософт, чтобы получить заменяющий диск. Дополнительные сведения см. в разделе об [оповещениях оборудования на устройстве StorSimple](storsimple-8000-manage-alerts.md#hardware-alerts).

* **Изменения в замене контроллера**. В этом выпуске добавлен командлет, который позволяет пользователю запросить состояние процесса замены контроллера. Дополнительные сведения можно найти в [описании командлета для запроса состояния замены контроллера](/powershell/module/hcs/?viewFallbackFrom=winserverr2-ps).
