# <a name="use-azure-active-directory-ad-authentication-for-your-storsimple"></a>Использование проверки подлинности Azure Active Directory (AD) для StorSimple

[!INCLUDE [storsimple-8000-eol-banner](../../includes/storsimple-8000-eol-banner.md)]

## <a name="overview"></a>Обзор

Служба диспетчера устройств StorSimple выполняется в Microsoft Azure и подключается к нескольким устройствам StorSimple. Сейчас в службе диспетчера устройств StorSimple используется служба контроля доступа (ACS) для аутентификации службы на устройстве StorSimple. Механизм ACS скоро устареет. Он будет заменен аутентификацией Azure Active Directory (AAD). Дополнительные сведения см. в объявлениях о прекращении поддержки ACS и использовании проверки подлинности AAD.

- [Будущее Azure ACS — Azure Active Directory](https://cloudblogs.microsoft.com/enterprisemobility/2015/02/12/the-future-of-azure-acs-is-azure-active-directory/)
- [Предстоящие изменения в службе контроля доступа Microsoft](https://azure.microsoft.com/blog/acs-access-control-service-namespace-creation-restriction/)

В этой статье представлены сведения об аутентификации AAD, новом регистрационном ключе службы и изменениях в правилах брандмауэра на устройствах StorSimple. Сведения в этой статье применимы только к устройствам StorSimple серии 8000.

Проверка подлинности AAD выполняется на устройстве StorSimple серии 8000 с обновлением 5 или более поздней версии. Введение аутентификации AAD влияет на следующие аспекты:

- шаблоны URL-адресов для правил брандмауэра;
- регистрационный ключ службы.

Эти изменения подробно описаны в следующих разделах.

## <a name="url-changes-for-aad-authentication"></a>Изменения URL-адреса для аутентификации AAD

Чтобы в службе применялась аутентификация на основе AAD, все пользователи должны включить новые URL-адреса аутентификации в правила брандмауэра.

Если вы используете StorSimple серии 8000, убедитесь, что в правила брандмауэра включен следующий URL-адрес:

| Шаблон URL-адреса                         | Cloud | Компонент или функция         |
|------------------------------------|-------|----------------------------------|
| `https://login.windows.net`        | Azure Public |Служба аутентификации AAD      |
| `https://login.microsoftonline.us` | US (США) |Служба аутентификации AAD      |

Полный список шаблонов URL-адресов для устройств StorSimple серии 8000 см. в разделе [Шаблоны URL-адресов для правил брандмауэра](storsimple-8000-system-requirements.md#url-patterns-for-firewall-rules).

Если после наступления даты устаревания URL-адрес проверки подлинности не включен в правила брандмауэра, а устройство использует обновление 5, пользователи получат предупреждение об URL-адресе. Необходимо будет добавить новый URL-адрес проверки подлинности. Если устройство использует более раннюю версию (а не обновление 5), пользователи получат предупреждение о пульсе. В обоих случаях устройство StorSimple не может пройти проверку подлинности с помощью службы, а служба не может связаться с устройством.

## <a name="aad-based-registration-keys"></a>Регистрационные ключи на основе AAD

Начиная с обновления 5 для устройств StorSimple серии 8000, используются новые регистрационные ключи на основе AAD. С их помощью можно зарегистрировать на устройстве службу диспетчера устройств StorSimple.

Для устройства StorSimple серии 8000, где установлено обновление 4 или более раннее (включая более старое устройство), невозможно использовать новые регистрационные ключи службы AAD.
В этом сценарии вам необходимо повторно создать регистрационный ключ службы. После повторного создания новый ключ применим для регистрации всех последующих устройств. Старый ключ уже недействителен.

- Срок действия регистрационного ключа AAD истекает через 3 дня.
- Регистрационные ключи AAD работают только с устройствами StorSimple серии 8000, использующими обновление 5 или более поздней версии.
- Регистрационные ключи AAD длиннее соответствующих регистрационных ключей ACS.

Чтобы повторно создать регистрационный ключ службы AAD, выполните приведенные ниже инструкции.

#### <a name="to-generate-the-aad-service-registration-key"></a>Создание регистрационного ключа службы AAD

1. В **диспетчере устройств StorSimple** выберите **Управление &gt;** **Ключи**. Вы также можете найти _ключи_ с помощью панели поиска.
    
2. Щелкните **Создать ключ**.

    ![Кнопка повторного создания](./media/storsimple-8000-aad-registration-key/aad-click-generate-registration-key.png)

3. Скопируйте новый ключ. Старый ключ больше не будет работать.

    ![Подтверждение повторного создания](./media/storsimple-8000-aad-registration-key/aad-registration-key2.png)

    > [!NOTE] 
    > Если вы создаете облачное устройство StorSimple в службе, зарегистрированной на устройстве StorSimple серии 8000, во время этого процесса не создавайте регистрационный ключ. Дождитесь завершения создания устройства, а затем создайте регистрационный ключ.
