# <a name="frequently-asked-questions-about-vmware-solution-by-cloudsimple"></a>Часто задаваемые вопросы о решении VMware с помощью Клаудсимпле

## <a name="cloudsimple-service"></a>Служба Клаудсимпле

**Что такое решение Azure VMware с помощью Клаудсимпле?**

Решение VMware для Azure Клаудсимпле преобразует и расширяет рабочие нагрузки VMware на частные, выделенные облака в Azure за считанные минуты. Клаудсимпле отвечает за подготовку, управление инфраструктурой и организацию рабочих нагрузок между локальной средой и Azure. Так как ваши приложения работают в локальной среде и в Azure, вы обладаете преимуществами эластичности и служб облака, не усложняя создание архитектуры приложений. Клаудсимпле снижает общую стоимость владения с помощью модели использования облака, которая обеспечивает подготовку по требованию, оплату по мере роста и оптимизацию емкости.  См. статью [что такое решение VMware в Azure с помощью клаудсимпле](cloudsimple-vmware-solutions-overview.md) для функций, преимуществ и сценариев.

**Что такое частное облако Клаудсимпле?**

Частное облако Клаудсимпле — это закрытое выделенное облако, которое состоит из высокопроизводительных вычислений, хранения и сетевой среды, развернутой на Microsoft Azureной инфраструктуре (пространстве оборудования и центра обработки данных) в расположениях Azure.  Частное облако предоставляет собственную платформу VMware как услугу. В терминах VMware Каждое частное облако содержит ровно один экземпляр vCenter Server. VCenter Server управляет несколькими узлами ESXi, содержащимися в одном или нескольких кластерах vSphere, вместе с соответствующим хранилищем виртуальной сети SAN (vSAN). Служба Клаудсимпле может содержать несколько частных облаков в подписке Azure.  Дополнительные сведения см. в статье [Общие сведения о частном облаке](cloudsimple-private-cloud.md).

**Где доступна служба Клаудсимпле?**

Клаудсимпле доступен в регионах "Восточная часть США", "Западная часть США" и "Западная Европа" с дополнительными регионами в ближайшее время.

**Разделы справки включить подписку для Клаудсимпле?**

[azurevmwaresales@microsoft.com](mailto:azurevmwaresales@microsoft.com)Чтобы включить подписку для службы клаудсимпле, обратитесь к представителям учетная запись Майкрософт. Укажите идентификатор подписки в сообщении электронной почты, для которого требуется включить службу Клаудсимпле.  

**Разделы справки получить доступ к порталу Клаудсимпле?**

Доступ к порталу Клаудсимпле можно получить из портал Azure.  Дополнительные сведения см. в статье [доступ к решению VMware с помощью портала клаудсимпле портал Azure](access-cloudsimple-portal.md).

**Разделы справки увеличить емкость для частного облака?**

Чтобы увеличить емкость, приобретите дополнительные узлы из портал Azure а затем с помощью узлов разверните частное облако на портале Клаудсимпле.  Вы можете добавить дополнительные узлы в существующий кластер vSphere или добавить их в новый кластер vSphere.  Дополнительные сведения см. [в разделе Расширение частного облака клаудсимпле](expand-private-cloud.md).

**Что происходит с моим частным облаком во время обслуживания?**

Клаудсимпле предоставляет уведомление через несколько дней до интервала запланированного обслуживания.  Обслуживание выполняется без нарушения работы, чтобы обеспечить доступность частного облака.  Обслуживание может иметь следующие типы:

* **Инфраструктура клаудсимпле**.  Инфраструктура Клаудсимпле разработана для обеспечения высокой доступности.  Во время этого типа интервала обслуживания избыточные компоненты обновляются по одному за раз, чтобы избежать перерыва в работе служб. Вы сохраняете доступ к Вашему частному облаку vCenter, всем виртуальным машинам, подключению к Интернету из частного облака и подключениям к локальной среде или Azure.
* **Портал клаудсимпле**. В течение этого типа интервала обслуживания некоторые функции на портале Клаудсимпле могут быть отключены или недоступны.  Уведомление до интервала обслуживания включает сведения об ограничениях функций во время обслуживания.

## <a name="connectivity"></a>Соединение

**Какие варианты подключения к сети региона Клаудсимпле?**

Клаудсимпле предоставляет следующие варианты подключения для подключения к сети Клаудсимпле Region. В одно и то же время можно использовать несколько параметров.

* **Подключение ExpressRoute из локального центра обработки данных к сети клаудсимпле Region**. Это высокоскоростное безопасное частное подключение с низкой задержкой, которое использует Global Reach для моста локального канала ExpressRoute в канал ExpressRoute Клаудсимпле. Инструкции по настройке подключения см. в статье [подключение из локальной среды к клаудсимпле с помощью ExpressRoute](on-premises-connection.md).
* **Подключение ExpressRoute из виртуальной сети Azure к сети регионов клаудсимпле**. Это высокоскоростное безопасное частное подключение с низкой задержкой, которое использует шлюзы виртуальной сети для моста виртуальной сети в Azure до канала Клаудсимпле ExpressRoute. Инструкции по настройке подключения см. в статье [Подключение среды частного облака клаудсимпле к виртуальной сети Azure с помощью ExpressRoute](azure-expressroute-connection.md).
* **VPN-подключение типа "сеть — сеть" из локального центра обработки данных к сети региона клаудсимпле**. Это безопасная виртуальная частная сеть из локального VPN-устройства в регион частного облака Клаудсимпле.  Дополнительные сведения см. [в статье Настройка VPN-шлюзов в клаудсимпле Network](vpn-gateway.md).

**Разделы справки подключиться к частному облаку?**

Вы можете просмотреть сведения о частном облаке на портале Клаудсимпле. Чтобы подключиться к vCenter, который соответствует частному облаку, сначала убедитесь, что сетевое подключение установлено с помощью VPN типа "сеть — сеть", VPN-подключение типа "точка — сеть" или ExpressRoute. Затем запустите портал Клаудсимпле из портал Azure и щелкните **запустить клиент vSphere** на домашней странице или на странице сведений о частном облаке.

**Каковы преимущества каналов ExpressRoute?**

Канал ExpressRoute Azure — это высокоскоростное безопасное подключение с низкой задержкой.  Клаудсимпле предоставляет выделенный канал ExpressRoute для каждого клиента.  С помощью этого канала можно установить безопасное подключение из локальной среды или подписки Azure.

**Каковы расходы на сеть для подключения к Клаудсимпле?  Взимается ли плата за исходящий трафик между Клаудсимпле и Azure или между регионами?**

Клаудсимпле плата за исходящий трафик сети не взимается.  Тарифы Azure уровня "Стандартный" применяются ко всем исходящим трафикам из виртуальной сети или из локального канала ExpressRoute.

## <a name="networking"></a>Сеть

**Какие сетевые функции доступны для моего частного облака?**

Вы можете подготавливать виртуальные ЛС (и их подсети) и таблицы брандмауэров, а также назначать общедоступные IP-адреса, которые сопоставляются с виртуальной машиной, работающей в частном облаке. Дополнительные сведения о сетевых функциях см. в разделе [Общие сведения о виртуальных ЛС и подсетях](cloudsimple-vlans-subnets.md), обзор [таблиц брандмауэра](cloudsimple-firewall-tables.md)и [Общие сведения о общедоступном IP-адресе](cloudsimple-public-ip-address.md).

**Разделы справки настроить разные подсети для приложений в частном облаке?**

Виртуальные ЛС создаются в частном облаке на портале Клаудсимпле.  После создания виртуальной ЛС можно создать группу распределенных портов в частном облаке vCenter с помощью виртуальной ЛС и создать виртуальные машины, подключенные к группе распределенных портов.  Вы можете включить таблицы брандмауэра для VLAN или подсети и определить правила брандмауэра для защиты сетевого трафика.

**Какие параметры брандмауэра доступны для моих частных облаков?**

Можно настроить правила для трафика «Север-Юг» и «Восток-Западная».  Правила определяются в таблице брандмауэра.  Таблицу брандмауэра можно подключить к виртуальным ЛС в частном облаке.  Дополнительные сведения см. в разделе [Настройка таблиц брандмауэра и правил для частных облаков](firewall.md).

**Можно ли назначить общедоступные IP-адреса виртуальным машинам в моей частной облачной среде?**

На портале Клаудсимпле можно выделить новый общедоступный IP-адрес и связать его с частным IP-адресом виртуальной машины или устройства.  Можно также создать новые правила брандмауэра или применить существующие правила брандмауэра, чтобы разрешить трафик с конкретных портов и IP-адресов на портале. Дополнительные сведения см. в разделе [выделение общедоступных IP-адресов для среды частного облака](public-ips.md).

## <a name="security"></a>Безопасность

**Что такое мои параметры безопасности на Клаудсимпле?**

Клаудсимпле предоставляет следующие функции безопасности для защиты среды частного облака.

* **Шифрование неактивных данных**. Вы можете зашифровать неактивных данных, расположенных в хранилище vSAN в частном облаке. vSAN поддерживает внешние серверы управления ключами, которые можно развернуть в виртуальной сети Azure или в локальной среде.  Дополнительные сведения см. в статье [Настройка шифрования vSAN для частного облака клаудсимпле](vsan-encryption.md).
* **Сетевая безопасность**. Управление потоком сетевого трафика с помощью правил брандмауэра, которые применяются между частным облаком и Интернетом, частным облаком и локальной средой или в подсетях частного облака.
* **Безопасное, частное подключение**. Между локальной сетью и подпиской Azure устанавливается безопасное, частное подключение.

## <a name="compute"></a>Службы вычислений

**Какие типы узлов доступны?**

Клаудсимпле предлагает следующие типы узлов:

* **CS28 узел:** ЦП: 2x 2,2 ГГц, всего 28 ядер, 48 НТ.  ОЗУ: 256 ГБ.  Хранилище: 1600 ГБ кэша NVMe, 5760 ГБ данных (все Flash). Сеть: 4x25Gbe NIC
* **CS36 узел:** ЦП 2x 2,3 ГГц, всего 36 ядер, 72 HT.  ОЗУ: 512 ГБ.  Хранилище: 3200 ГБ кэша NVMe 11520 ГБ данных (все Flash).  Сеть: 4x25Gbe NIC
* **CS36m узел:** ЦП 2x 2,3 ГГц, всего 36 ядер, 72 HT.  ОЗУ: 576 ГБ.  Хранилище: 3200 ГБ кэша NVMe 13360 ГБ данных (все Flash).  Сеть: 4x25Gbe NIC

**Как обрабатываются сбои оборудования?**

Вся инфраструктура Клаудсимпле постоянно отслеживается платформой Клаудсимпле и группами операций службы.  При обнаружении сбоя оборудования в частное облако добавляется новый узел, и неисправный узел удаляется.

## <a name="storage"></a>Память

**Какой тип хранилища поддерживается в частном облаке?**

Клаудсимпле предлагает все — флэш-накопитель VMware vSAN в каждом частном облаке.  Каждый vSphere создается с собственным хранилищем данных vSAN.  Дополнительные сведения см. в статье [компоненты VMware для частного облака — хранилище vSAN](vmware-components.md#vsan-storage).

**Поддерживается ли шифрование данных?**
Да.  Вы можете настроить хранилище vSAN в частном облаке, чтобы использовать сервер управления ключами (KMS), развернутый локально или в Azure, для шифрования данных, хранящихся в vSAN.

**Как обрабатываются неисправные диски?**

Клаудсимпле постоянно отслеживает все аппаратные компоненты частного облака.  При обнаружении сбоя диска или обнаружении диска как сбой (на основе эвристики) новый узел автоматически добавляется в частное облако.  Узел с неисправным или неисправным диском удаляется из частного облака.

## <a name="vmware"></a>VMware

**Разделы справки выполнить крупномасштабную передачу или перенос приложений и данных из локальной среды?**

Клаудсимпле предоставляет собственное VMware vSphere решение.  С частным облаком можно использовать все средства VMware, предназначенные для переноса данных.  Возможны следующие значения.

* VMware ХККС для выполнения групповой миграции данных.
* Холодный перенос данных при помощи хранилища данных vMotion из локальной среды в Клаудсимпле.

**Можно ли установить любые инструменты VMware?**

Клаудсимпле предоставляет собственное VMware vSphere решение.  Все инструменты VMware, используемые для управления локальной средой vSphere, можно использовать в Клаудсимпле.  Клаудсимпле поддерживает модель с собственной лицензией (BYOL) для установки средств VMware.

**Как осуществляется управление обновлениями и обновлением?**

Клаудсимпле управляет и обновляет все компоненты инфраструктуры частного облака без нарушения работы.  Все обновления и исправления безопасности, выпущенные VMware или поставщиками инфраструктуры, планируются на обновление, как только они будут квалифицированы с помощью Клаудсимпле.

Клаудсимпле не выполняет обновления или обновления приложений, установленных в частном облаке.

## <a name="azure-integration"></a>Интеграция с Azure

**Какие службы Azure поддерживаются?**

Клаудсимпле предоставляет подключение Azure ExpressRoute к вашей подписке в Azure.  Все службы, работающие в вашей подписке, могут подключаться к частному облаку.  Примеры приведены ниже.

* **Azure Active Directory** в качестве источника удостоверений для клаудсимпле vCenter.
* Служба **хранилища Azure** для хранения резервных копий, образов и других данных из частного облака.
* **Гибридные приложения** с архитектурой приложений, охватывающей общедоступные и частные облака.  Например, вы можете создавать веб серверы в Azure, которые обращаются к приложениям и серверам баз данных в частном облаке.
* **Azure Monitor** и **Центр безопасности Azure** для рабочих нагрузок, использующих журналы поддержки VMware, метрики производительности и управление безопасностью.

**Разделы справки сопоставлять мои клиенты VMware с Azure?**

Клаудсимпле обеспечивает уникальную возможность управления виртуальными машинами VMware в частном облаке из портал Azure.  Пул ресурсов vCenter, настроенный с требуемыми ограничениями ресурсов, может быть сопоставлен с вашей подпиской глобальным администратором.  

**Какие преимущества лицензирования можно получить с помощью Azure?**

С помощью Клаудсимпле вы можете воспользоваться преимуществами гибридного использования Azure и сэкономить до 90% лицензий. Это преимущество позволяет сохранить ваши инвестиции в лицензии Майкрософт и снизить совокупную стоимость владения относительно других облачных решений. Вы также получаете расширенные обновления для системы безопасности для Windows Server 2008 и Microsoft SQL Server 2008.  Модель с собственной лицензией (BYOL) позволяет снизить затраты для таких распространенных приложений, как Veeam и Zerto.
