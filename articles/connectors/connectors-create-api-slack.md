# <a name="monitor-and-manage-slack-with-azure-logic-apps"></a>Мониторинг приложения Slack и управление им с помощью Azure Logic Apps

С помощью Azure Logic Apps и соединителя Slack можно создавать автоматизированные задачи и рабочие процессы, которые отслеживают ваши файлы Slack и управляют каналами, сообщениями и группами Slack и т. д., позволяя:

* отслеживать создание новых файлов;
* создавать, перечислять и объединять каналы; 
* публиковать сообщения;
* создавать группы и устанавливать состояние "Не беспокоить".

Вы можете использовать триггеры, которые получают ответы из учетной записи Slack и делают выходные данные доступными для других действий. Вы можете использовать действия, которые выполняют задачи с вашей учетной записью Slack. Кроме того, выходные данные действий Slack могут использоваться другими действиями. Например, когда создается новый файл, можно отправить сообщение электронной почты с помощью соединителя Office 365 Outlook. Если вы не знакомы с приложениями логики, ознакомьтесь со статьей [Что такое Azure Logic Apps](../logic-apps/logic-apps-overview.md).

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, [зарегистрируйтесь для получения бесплатной учетной записи Azure](https://azure.microsoft.com/free/). 

* Ваша учетная запись [Slack](https://slack.com/) и учетные данные пользователя.

  Ваши учетные данные авторизуют приложение логики, чтобы оно могло создать подключение и получить доступ к вашей учетной записи Slack.

* Базовые знания [создания приложений логики](../logic-apps/quickstart-create-first-logic-app-workflow.md).

* Приложение логики, из которого необходимо получить доступ к учетной записи Slack. Чтобы начать работу с триггером Slack, [создайте пустое приложение логики](../logic-apps/quickstart-create-first-logic-app-workflow.md). Чтобы использовать действие Slack, запустите приложение логики с триггером, например триггером Slack или другим, таким как триггер **Периодичность**.

## <a name="connect-to-slack"></a>Подключение к Slack

[!INCLUDE [Create connection general intro](../../includes/connectors-create-connection-general-intro.md)]

1. Войдите на [портал Azure](https://portal.azure.com) и откройте свое приложение логики в конструкторе приложений логики, если оно еще не открыто.

1. Для пустых приложений логики в поле поиска в качестве фильтра введите slack. В списке триггеров выберите нужный триггер. 

   -или-

   В имеющихся приложениях логики на последнем шаге, в котором необходимо добавить действие, выберите **Новый шаг**. 
   В поле поиска введите слово slack в качестве фильтра. 
   В списке действий выберите любое необходимое действие.

   Чтобы добавить действие между шагами, переместите указатель на стрелку между шагами. 
   Выберите появившийся знак "плюс" ( **+** ), а затем щелкните **Добавить действие**.

1. Если будет предложено войти в Slack, войдите в рабочую область Slack. 

   ![Вход в рабочую область Slack](./media/connectors-create-api-slack/slack-sign-in-workspace.png)

1. Разрешите доступ для приложения логики.

   ![Авторизация доступа к Slack](./media/connectors-create-api-slack/slack-authorize-access.png)

1. Укажите необходимые сведения для выбранного триггера или действия. Чтобы продолжать создавать рабочий процесс приложения логики, добавьте дополнительные действия.
