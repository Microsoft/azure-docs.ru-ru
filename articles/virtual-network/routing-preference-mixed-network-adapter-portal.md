# <a name="configure-both-routing-preference-options-for-a-virtual-machine"></a>Настройка двух вариантов параметров маршрутизации для виртуальной машины

В этой статье показано, как реализовать оба варианта настройки [маршрутизации](./routing-preference-overview.md) (с помощью Интернета и глобальной сети Майкрософт) для виртуальной машины. Это достигается благодаря использованию двух виртуальных сетевых интерфейсов: один сетевой интерфейс с общедоступным IP-адресом маршрутизируется через глобальную сеть Майкрософт, а другой с общедоступным IP-адресом маршрутизируется через сеть ISP.

## <a name="prerequisites"></a>Предварительные требования

Создайте на портале Azure виртуальную машину с общедоступным IP-адресом, следуя инструкциям из раздела [Создание виртуальной машины со статическим общедоступным IP-адресом с помощью портала Azure](./virtual-network-deploy-static-pip-arm-portal.md). По умолчанию для общедоступного IP-адреса выбор маршрутизации осуществляется через **глобальную сеть Майкрософт**. После создания виртуальной машины с общедоступным IP-адресом добавьте для нее второй общедоступный IP-адрес, который будет направлять трафик через транзитную сеть ISP благодаря тому, что для нее параметры маршрутизации заданы как для сети **Интернет**.

## <a name="create-a-public-ip-address-with-a-routing-preference-choice-internet"></a>Создание общедоступного IP-адреса с выбором предпочтительного варианта маршрутизации "Интернет"
1. Войдите на [портал Azure](https://portal.azure.com/).
2. Выберите **Создать ресурс**. 
3. В поле поиска введите *общедоступный IP-адрес*.
4. В результатах поиска выберите **Общедоступный IP-адрес**. Затем на странице **Общедоступный IP-адрес** щелкните **Создать**.
5. В диалоговом окне **Предпочтение маршрутизации** выберите **Интернет**.

      ![Создание общедоступного IP-адреса](./media/routing-preference-portal/public-ip-new.png)

    > [!NOTE]
    > Общедоступные IP-адреса создаются с адресами IPv4 или IPv6. Но сейчас при выборе предпочтительного варианта маршрутизации поддерживается только IPV4.

## <a name="create-a-network-interface-and-associate-the-public-ip"></a>Создание сетевого интерфейса и привязка общедоступного IP-адреса

1. Создайте [сетевой интерфейс](./routing-preference-overview.md) на портале Azure с параметрами по умолчанию. 
1. [Привяжите общедоступный IP-адрес к сетевому интерфейсу ](./associate-public-ip-address-vm.md), созданному в предыдущем разделе. Может пройти несколько секунд, прежде чем IP-адрес станет виден. Просмотрите общедоступный IP-адрес, назначенный для IP-конфигурации, как показано ниже:

    ![Привязка общедоступного IP-адреса](./media/routing-preference-mixed-network-adapter-portal/public-ip.png)

## <a name="attach-network-interface-to-the-vm"></a>Подключение сетевого интерфейса к виртуальной машине

1. [Подключите сетевой интерфейс, созданный в предыдущем разделе, к виртуальной машине](./virtual-network-network-interface-vm.md).
2. Теперь можно видеть два виртуальных сетевых интерфейса, привязанных к виртуальной машине, один из них с общедоступным IP-адресом, который маршрутизируется через глобальную сеть Майкрософт, а другой маршрутизируется через сеть ISP, как показано в разделе ![ Подключение сетевого интерфейса к виртуальной машине](./media/routing-preference-mixed-network-adapter-portal/mixed-network-adapter.png)
