# <a name="configure-email-notifications-for-issues-in-azure-active-directory-domain-services"></a>Настройка уведомлений по электронной почте для проблем в Azure Active Directory доменных служб

Работоспособность управляемого домена доменных служб Azure Active Directory (Azure AD DS) отслеживается платформой Azure. На странице состояния работоспособности на портале Azure отображаются все оповещения для управляемого домена. Для своевременного реагирования на проблемы можно настроить уведомления по электронной почте таким образом, чтобы отчеты об оповещениях, касающихся работоспособности, отправлялись сразу после их обнаружения в управляемом домене Azure AD DS.

В этой статье показано, как настроить получателей уведомлений по электронной почте для управляемого домена.

## <a name="email-notification-overview"></a>Общие сведения об уведомлениях по электронной почте

Для оповещения о проблемах с управляемым доменом можно настроить уведомления по электронной почте. В этих уведомлениях по электронной почте указывается управляемый домен, на котором имеется оповещение, а также время обнаружения и ссылка на страницу работоспособности в портал Azure. Затем можно выполнить указанные рекомендации по устранению неполадок, чтобы решить проблемы.

Ниже приведен пример уведомления по электронной почте, которое указывает на то, что в управляемом домене создано критическое предупреждение или оповещение.

![Пример уведомления по электронной почте](./media/active-directory-domain-services-alerts/email-alert.png)

> [!WARNING]
> Прежде чем переходить по ссылкам в сообщении, убедитесь, что сообщение получено от проверенного отправителя Майкрософт. Уведомления по электронной почте всегда приходят с адреса `azure-noreply@microsoft.com`.

### <a name="why-would-i-receive-email-notifications"></a>Зачем мне получать уведомления по электронной почте?

Доменные службы Azure AD отправляют уведомления по электронной почте в случае важных обновлений, касающихся управляемого домена. Эти уведомления создаются только для важных проблем, которые могут существенно повлиять на службу и требуют немедленных действий. Каждое уведомление по электронной почте активируется оповещением об управляемом домене. Оповещения также отображаются на портале Azure. Их можно просмотреть на [странице с данными о работоспособности доменных служб Azure AD][check-health].

Доменные службы Azure AD не отправляют сообщения электронной почты в целях объявлений, обновлений или продаж.

### <a name="when-will-i-receive-email-notifications"></a>Когда я буду получать уведомления по электронной почте?

Уведомление отправляется сразу же после обнаружения [нового оповещения][troubleshoot-alerts] в управляемом домене. Если оповещение не разрешено, каждые четыре дня отправляются дополнительные уведомления в качестве напоминания.

### <a name="who-should-receive-the-email-notifications"></a>Кто должен получать уведомления по электронной почте?

Список получателей сообщений электронной почты для доменных служб Azure Active Directory должен состоять из пользователей, которые могут администрировать управляемый домен и вносить в него изменения. Этот список рассылки следует считать группой экстренного реагирования на возникшие оповещения и проблемы.

Для уведомлений по электронной почте можно добавить до пяти дополнительных получателей электронной почты. Если требуется более пяти получателей уведомлений по электронной почте, создайте список рассылки и добавьте его в список уведомлений.

Кроме того, можно указать, чтобы уведомления по электронной почте получали все *глобальные администраторы* каталога Azure AD, а также все члены группы *администраторов контроллера домена AAD*. Azure AD DS отправляет уведомления только до 100 адресов электронной почты, включая список глобальных администраторов и администраторов AAD контроллеров домена.

## <a name="configure-email-notifications"></a>Настройка уведомлений по электронной почте

Чтобы просмотреть существующих получателей уведомлений по электронной почте или добавить дополнительных получателей, выполните следующие действия.

1. На портале Azure найдите и выберите **Доменные службы Azure AD**.
1. Выберите управляемый домен, например *aaddscontoso.com*.
1. В левой части окна ресурсов доменных служб Azure AD выберите **Параметры уведомлений**. На экране появятся существующие получатели уведомлений по электронной почте.
1. Чтобы добавить получателя сообщений электронной почты, введите адрес электронной почты в таблице дополнительных получателей.
1. По завершении нажмите кнопку **Сохранить** в верхней части окна навигации.

> [!WARNING]
> При изменении параметров уведомлений обновляются параметры уведомлений для всего управляемого домена, а не только для самого себя.

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

### <a name="i-received-an-email-notification-for-an-alert-but-when-i-logged-on-to-the-azure-portal-there-was-no-alert-what-happened"></a>По электронной почте мне пришло уведомление о предупреждении, но у меня не получилось найти это предупреждение на портале Azure. Что произошло?

Если предупреждение было устранено, оно исчезает с портала Azure. Наиболее вероятной причиной является то, что кто-то, кто получит уведомления по электронной почте, разрешил оповещение в управляемом домене или его автоматическое разрешение платформой Azure.

### <a name="why-can-i-not-edit-the-notification-settings"></a>Почему я не могу изменить параметры уведомлений?

Если вы не можете получить доступ к странице параметров уведомлений на портале Azure, у вас нет разрешений на внесение изменений касательно управляемого домена. Обратитесь к глобальному администратору, чтобы получить разрешения на изменение ресурса AD DS Azure, или удалите его из списка получателей.

### <a name="i-dont-seem-to-be-receiving-email-notifications-even-though-i-provided-my-email-address-why"></a>Я не получаю уведомления по электронной почте, хотя мой адрес электронной почты был указан. Почему?

Проверьте нежелательную почту или папку нежелательной почты в электронной почте для уведомления и убедитесь, что разрешен отправитель `azure-noreply@microsoft.com` .
