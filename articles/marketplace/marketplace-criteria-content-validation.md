# <a name="azure-marketplace-listing-guidelines"></a>Рекомендации по публикации в Azure Marketplace

В этой статье описаны требования и рекомендации по публикации новых предложений и служб в Azure Marketplace. Все предложения должны соответствовать [требованиям](#listing-requirements-for-all-offers) , перечисленным ниже. Используйте ссылки справа для перехода к дополнительным требованиям и контрольным спискам для конкретных типов списков.

## <a name="listing-requirements-for-all-offers"></a>Требования к публикациям всех предложений

| 2 | Заголовок предложения | Краткое описание сути предложения. Совпадает с маркетинговым описанием на веб-сайте партнера. | Содержит ключевые слова для поиска. |
| 3 | Логотип | Эмблема отображается правильно.  | Эмблема отображается правильно.  |
| 4 |  Offer description (Описание предложения); | <ul><li> Содержит 2–3 абзаца.</li><li>Суть предложения легко понять с первого взгляда.</li><li>Не содержит орфографические и грамматические ошибки.</li><li>Достаточно подробное и включает информацию о целевой аудитории, типах пользователей, ценности предложения.</li><li>В повествовательной форме и разбито на абзацы с короткими и понятными предложениями.</li></ul> | <ul><li> Указана целевая отрасль (если уместно).</li><li>Соблюдается хороший стиль форматирования, заголовок каждого абзаца содержит одно предложение или фразу, обобщающую последующий текст, используются маркеры списков (где это требуется) для выделения основных преимуществ. | 9 | Цены на решение | Присутствует обязательная вкладка или сведения с ценами на решение в местной валюте той территории, где предоставляется решение партнера. | Должны быть доступны несколько ценовых категорий и вариантов для выставления счетов на выбор клиента. |
| 10 | Дополнительные сведения | Внизу есть ссылки (под описанием, а не ссылки Azure Marketplace в левой части) на страницы с дополнительными сведениями о решении, которые свободно доступны из Интернета и отображаются правильно. |    Эти ссылки ведут на конкретные элементы (например, страницу спецификаций на сайте партнера), а не просто на домашнюю страницу партнера. | 
| 11 | Поддержка и справка для решения | Есть указание хотя бы на один из следующих элементов: <ul><li>номера телефонов;</li><li>поддержка по электронной почте;</li><li>агенты чата;</li><li>форумы сообщества. |<ul><li>Перечислены все способы предоставления поддержки.</li><li>Платная поддержка предоставляется бесплатно в течение *пробного* или *тестового* периода. |
