# <a name="reference-architecture-diagram"></a>Схема эталонной архитектуры

Эталонная схема архитектуры — это модель, которая представляет инфраструктуру, от которой зависит ваше предложение. Для решений Azure IP эта схема должна также показывать, как ваше предложение использует облачные службы Майкрософт в соответствии с техническими требованиями к совместной продаже IP-продаж. Он не предназначен для оценки качества архитектуры. Он предназначен для демонстрации использования службами Майкрософт решения.

Схему эталонной архитектуры можно создать с помощью нескольких средств. Мы рекомендуем использовать Microsoft Visio, так как в нем есть несколько трафаретов, которые описывают модели архитектуры Azure.

Полезной отправной точкой для создания схем эталонной архитектуры является использование [моделей архитектуры Azure](/azure/architecture/browse/).

## <a name="typical-components-of-a-reference-architecture-diagram"></a>Типичные компоненты схемы эталонной архитектуры

- Облачные службы, которые размещают ваше предложение и взаимодействуют с ним, включая те, которые используют ресурсы Azure.
- Подключения к данным, уровни данных и службы данных, используемые вашим предложением
- Облачные службы, используемые для управления безопасностью, проверкой подлинности и пользователями предложения
- Пользовательские интерфейсы и другие службы, предоставляющие предложение пользователям
- Гибридное или локальное подключение и интеграция, а также сочетание

На этом снимке экрана показан пример схемы эталонной архитектуры.

[![Этот образ представляет собой пример диаграммы архитектуры совместной продажи.](./media/co-sell/co-sell-arch-diagram.png)](./media/co-sell/co-sell-arch-diagram.png#lightbox)

В этом примере эталонная схема архитектуры предназначена для вертикальных отраслевых чат-бот, которые можно интегрировать с сайтами интрасети для помощи в сценариях прогнозирования спроса с помощью алгоритма машинного обучения. Это решение использует цепочку поставок и данные графика производства из различных ERP-систем. Программа-робот предназначена для решения вопросов, когда продавец может фиксировать возможные даты доставки заказа.
