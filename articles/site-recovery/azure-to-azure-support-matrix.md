# <a name="support-matrix-for-azure-vm-disaster-recovery-between-azure-regions"></a>Таблица со сведениями о поддержке аварийного восстановления виртуальной машины Azure между регионами Azure

В этой статье приведены общие сведения о поддержке и предварительных требованиях для аварийного восстановления виртуальных машин Azure из одного региона Azure в другой с помощью службы [Azure Site Recovery](site-recovery-overview.md).


## <a name="deployment-method-support"></a>Поддержка метода развертывания

**Deployment** |  **Поддержка**
--- | ---
**Портал Azure** | Поддерживается.
**PowerShell** | Поддерживается. [Дополнительные сведения](azure-to-azure-powershell.md)
**REST API** | Поддерживается.
**CLI** | Сейчас не поддерживается


## <a name="resource-support"></a>Поддержка ресурсов

**Действие с ресурсом** | **Сведения**
--- | ---
**Перемещение хранилищ между группами ресурсов** | Не поддерживается
**Перемещение вычислительных ресурсов, ресурсов хранилищ и сетевых ресурсов между группами ресурсов** | Не поддерживается.<br/><br/> Если вы перемещаете виртуальную машину или связанные компоненты, такие как хранилище или сеть, после ее репликации, вам необходимо отключить репликацию и повторно включить ее для виртуальной машины.
**Репликация виртуальных машин Azure из одной подписки в другую для обеспечения аварийного восстановления** | Поддерживается в том же клиенте Azure Active Directory.
**Миграция виртуальных машин между регионами в рамках поддерживаемых географических кластеров (в пределах подписок и между ними)** | Поддерживается в том же клиенте Azure Active Directory.
**Перемещение виртуальных машин внутри региона** | Не поддерживается.
**Выделенные узлы Azure** | Не поддерживается.

> - Южную Бразилию можно использовать только в качестве исходного региона, из которого виртуальные машины могут реплицироваться с помощью Site Recovery. Ее невозможно использовать в качестве целевого региона. Это обусловлено проблемами задержки из-за географических расстояний. Обратите внимание, что при отработке отказа из Южной Бразилии в качестве исходного региона для целевого объекта поддерживается восстановление в Южную Бразилию из целевого региона.
> - Вы можете работать в регионах, для которых у вас есть соответствующий доступ.
> - Если вы не видите регион, в котором хотите создать хранилище, убедитесь, что ваша подписка имеет доступ для создания ресурсов в этом регионе.
> - Если вы не видите регион в пределах географического кластера при включении репликации, убедитесь, что у вашей подписки есть разрешения на создание виртуальных машин в этом регионе.



## <a name="cache-storage"></a>Хранилище кэша

В следующей таблице приведены сведения о поддержке учетных записей хранения кэша, используемые Azure Site Recovery для репликации.

**Параметр** | **Поддержка** | **Сведения**
--- | --- | ---
Учетные записи хранения общего назначения версии 2 ("горячего" и "холодного" уровней доступа) | Поддерживается | Использование GPv2 не рекомендуется, поскольку цена трансакции для учетных записей хранения версии 2 существенно выше, чем для версии 1.
Хранилище уровня "Премиум" | Не поддерживается | Учетные записи хранения уровня "Стандартный" используются для хранения кэша в целях оптимизации затрат.
Брандмауэры службы хранилища Azure для виртуальных сетей  | Поддерживается | Если в целевой учетной записи хранения или в учетной записи хранения кэша включен брандмауэр, убедитесь, что задан параметр [Разрешить доверенным службам Майкрософт обходить этот брандмауэр](../storage/common/storage-network-security.md#exceptions).<br></br>Кроме того, убедитесь, что разрешен доступ по меньшей мере к одной подсети исходной виртуальной сети.

В следующей таблице перечислены ограничения на количество дисков, которые могут реплицироваться в одну учетную запись хранения.

**Тип учетной записи хранения**    |    **Обновление = 4 Мбит/с на диск**    |    **Обновление = 8 Мбит/с на диск**
---    |    ---    |    ---
Учетная запись хранения v1    |    диски 300    |    диски 150
V2 учетная запись хранения    |    диски 750    |    диски 375

По мере увеличения среднего количества изменений на дисках количество дисков, которые учетная запись хранения может поддерживать, уменьшается. Приведенная выше таблица может использоваться в качестве рекомендации для принятия решений о количестве учетных записей хранения, которые необходимо подготовить.

Обратите внимание, что указанные выше ограничения относятся к сценариям аварийного восстановления из Azure в Azure и зоне.

Группа безопасности сети для подсети | Поддерживается | Свяжите группу безопасности сети с подсетью с помощью сценария автоматизации Azure из плана восстановления.
Зарезервированный (статический) IP-адрес | Поддерживается | Если сетевой интерфейс исходной виртуальной машины имеет статический IP-адрес, а в целевой подсети доступен такой же IP-адрес, он будет назначен виртуальной машине, которая выполняет отработку отказа.<br/><br/> Если в целевой подсети нет такого же IP-адреса, один из доступных IP-адресов в подсети будет зарезервирован для этой виртуальной машины.<br/><br/> Можно также указать фиксированный IP-адрес и подсеть: **Реплицированные элементы** > **Настройки** > **Вычисления и сеть** > **Сетевые интерфейсы**.
Динамический IP-адрес | Поддерживается | Если сетевая карта исходной виртуальной машины имеет динамический IP-адрес, то сетевая карта на восстановленной виртуальной машине также по умолчанию будет динамической.<br/><br/> При необходимости возможно изменить его на фиксированный IP-адрес.
Несколько IP-адресов | Не поддерживается | При отработке отказа виртуальной машины с сетевым адаптером с несколькими IP-адресами сохраняется только основной IP-адрес сетевого адаптера в исходном регионе. Чтобы назначить несколько IP-адресов, можно добавить виртуальные машины в [план восстановления](recovery-plan-overview.md) и присоединить скрипт для назначения дополнительных IP-адресов к плану или внести изменения вручную или с помощью скрипта после отработки отказа.
Диспетчер трафика     | Поддерживается | Диспетчер трафика можно предварительно настроить так, чтобы трафик перенаправлялся в конечную точку в исходном регионе на регулярной основе и в конечную точку в целевом регионе в случае отработки отказа.

Ускорение работы в сети | Поддерживается | На исходной виртуальной машине необходимо включить ускоренную работу в сети. [Подробнее](azure-vm-disaster-recovery-with-accelerated-networking.md).
