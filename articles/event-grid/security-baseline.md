# <a name="azure-security-baseline-for-event-grid"></a>Базовый план безопасности Azure для службы "Сетка событий"

Этот базовый план безопасности применяет рекомендации из [теста безопасности Azure версии 1,0](../security/benchmarks/overview-v1.md) к Microsoft Azure сетке событий. Azure Security Benchmark содержит рекомендации по обеспечению безопасности облачных решений в Azure.
Содержимое группируются по **элементам управления безопасностью** , определенным в тестовом производительности системы безопасности Azure, и связанным рекомендациям, применимым к службе "Сетка событий Azure". **Элементы управления** , неприменимые к службе "Сетка событий Azure", были исключены.

 
Сведения о том, как служба "Сетка событий Azure" полностью сопоставляется с производительностью системы безопасности Azure, см. в статье [полный базовый план безопасности службы "Сетка событий Azure](https://github.com/MicrosoftDocs/SecurityBenchmarks/tree/master/Azure%20Offer%20Security%20Baselines)".

### <a name="11-protect-azure-resources-within-virtual-networks"></a>1,1: защита ресурсов Azure в виртуальных сетях

**Руководство**. частные конечные точки можно использовать, чтобы разрешить прием событий непосредственно из виртуальной сети в разделы и домены сетки событий, защищенные по частной ссылке, без использования общедоступного Интернета. При создании частной конечной точки для раздела или домена службы "Сетка событий" обеспечивается безопасное подключение между клиентами в виртуальной сети и ресурсом сетки событий. Частной конечной точке назначается IP-адрес из диапазона IP-адресов виртуальной сети. Соединение между частной конечной точкой и службой сетки событий использует защищенную закрытую ссылку.

Служба "Сетка событий Azure" также поддерживает общедоступные элементы управления доступом на основе IP-адресов для публикации разделов и доменов. Элементы управления на основе IP-адресов позволяют ограничить издателей раздел или домен только набором утвержденных компьютеров и облачных служб. Эта функция дополняет механизмы проверки подлинности, поддерживаемые службой "Сетка событий".
