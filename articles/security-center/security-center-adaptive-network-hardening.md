## <a name="what-is-adaptive-network-hardening"></a>Что такое Адаптивное усиление защиты сети?
Применение [групп безопасности сети (NSG)](../virtual-network/network-security-groups-overview.md) для фильтрации трафика в ресурсы и из ресурсов повышает уровень безопасности сети. Но в некоторых случаях фактический трафик, проходящий через группу безопасности сети, является подмножеством определенных правил NSG. Чтобы улучшить состояние безопасности в таком случае, можно ужесточить правила NSG с учетом фактического распределения трафика.

Адаптивное усиление защиты сети предоставляет рекомендации для дальнейшей защиты правил NSG. Она использует алгоритм машинного обучения, который учитывает фактический трафик, известную доверенную конфигурацию, аналитику угроз и другие показатели компрометации, а затем предоставляет рекомендации по разрешению трафика только от определенных IP-адресов или кортежей портов.

Например, пусть существующее правило NSG разрешать трафик от 140.20.30.10/24 через порт 22. В зависимости от анализа трафика Адаптивное усиление защиты сети может порекомендовать уменьшить диапазон, чтобы разрешить трафик от 140.23.30.10/29 и запретить весь остальной трафик на этот порт.

>[!Note]
> Адаптивные рекомендации по усилению защиты сети поддерживаются только для следующих портов (для UDP и TCP): 13, 17, 19, 22, 23, 53, 69, 81, 111, 119, 123, 135, 137, 138, 139, 161, 162, 389, 445, 512, 514, 593, 636, 873, 1433, 1434, 1900, 2049, 2301, 2323, 2381, 3268, 3306, 3389, 4333, 5353, 5432, 5555 , 6379, 7000, 7001, 7199, 8081, 8089, 8545, 9042, 9160, 9300, 11211, 16379, 26379, 27017, 37215


## <a name="view-and-manage-hardening-alerts-and-rules"></a>Просмотр предупреждений и правил усиления защиты и управление ими

1. В меню центра безопасности откройте панель мониторинга **Azure Defender** и выберите адаптивную плитку усиления защиты сети (1) или элемент панели Insights, связанный с адаптивной усилением защиты сети (2). 

    :::image type="content" source="./media/security-center-adaptive-network-hardening/traffic-hardening.png" alt-text="Доступ к адаптивным средствам усиления защиты сети" lightbox="./media/security-center-adaptive-network-hardening/traffic-hardening.png":::

    > [!TIP]
    > Панель Insights показывает процент виртуальных машин, защищенных в данный момент с помощью адаптивной защиты сети. 

1. На странице сведений о **адаптивных рекомендациях по усилению защиты сети следует применять рекомендации по использованию виртуальных машин, подключенных к Интернету** , с сетевыми виртуальными машинами, сгруппированными по трем вкладкам:
   * **Неработоспособные ресурсы**: виртуальные машины, которые в настоящее время имеют рекомендации и оповещения, запущенные с помощью адаптивного алгоритма усиления защиты сети. 
   * **Работоспособные ресурсы**: виртуальные машины без оповещений и рекомендаций.
   * **Непроверенные ресурсы**. виртуальные машины, на которых не удается запустить адаптивный алгоритм усиления защиты сети, по одной из следующих причин:
      * **Виртуальные машины являются классическими**: поддерживаются только Azure Resource Manager виртуальные машины.
      * **Недостаточно данных**: для создания точных рекомендаций по усилению защиты трафика центр безопасности требует не менее 30 дней передачи данных трафика.
      * **Виртуальная машина не защищена защитником Azure**: для этой функции подходят только виртуальные машины, защищенные с помощью [защитника Azure для серверов](defender-for-servers-introduction.md) .

    :::image type="content" source="./media/security-center-adaptive-network-hardening/recommendation-details-page.png" alt-text="На виртуальных машинах, подключенных к Интернету, следует использовать страницу сведений рекомендации по адаптивной настройке защиты сети.":::

1. На вкладке **неработоспособные ресурсы** выберите виртуальную машину, чтобы просмотреть ее оповещения и Рекомендуемые правила усиления защиты, которые необходимо применить.

    - На вкладке **правила** перечислены правила, которые Адаптивное усиление защиты сети рекомендует добавить.
    - На вкладке **оповещения** перечислены оповещения, которые были созданы из-за трафика, передаются в ресурс, который не входит в диапазон IP-адресов, допустимый в рекомендуемых правилах.

1. При необходимости измените правила.

    - [Изменение правила](#modify-rule)
    - [Удаление правила](#delete-rule) 
    - [Добавление правила](#add-rule)

3. Выберите правила, которые необходимо применить к NSG, и нажмите кнопку **Применить**.

    > [!TIP]
    > Если допустимые диапазоны исходных IP-адресов отображаются как "нет", это означает, что рекомендуемым правилом является *запрещающее* правило, в противном случае это правило *разрешения* .

    :::image type="content" source="./media/security-center-adaptive-network-hardening/hardening-alerts.png" alt-text="Управление адаптивными правилами усиления защиты сети":::

      > [!NOTE]
      > Принудительные правила добавляются в NSG (ы), защищающие виртуальную машину. (Виртуальная машина может быть защищена NSG, связанным с сетевой картой, или подсетью, в которой находится виртуальная машина, или и то, и другое).

### <a name="modify-a-rule"></a>Изменение правила <a name ="modify-rule"></a>

Может потребоваться изменить параметры правила, которое было рекомендовано. Например, может потребоваться изменить Рекомендуемые диапазоны IP-адресов.

Некоторые важные рекомендации по изменению адаптивного правила усиления защиты сети.

* Вы можете изменить параметры только для правил "разрешить". 
* Нельзя изменить правила "разрешить", чтобы они стали "запрещенными" правилами. 

  > [!NOTE]
  > Создание и изменение правил "deny" выполняется непосредственно в NSG. Дополнительные сведения см. [в статье Создание, изменение или удаление группы безопасности сети](../virtual-network/manage-network-security-group.md).

* Правило " **запретить все трафик** " является единственным типом правила "запретить", которое будет указано здесь и не может быть изменено. Однако его можно удалить (см. раздел [Удаление правила](#delete-rule)).
  > [!NOTE]
  > Правило **запрета всего трафика** рекомендуется, если в результате выполнения алгоритма центр безопасности не определяет трафик, который должен быть разрешен, на основе существующей конфигурации NSG. Поэтому рекомендуемым правилом является запрет всего трафика на указанный порт. Имя этого типа правила отображается как "*система сформирована*". После применения этого правила его фактическое имя в NSG будет представлять собой строку, состоящую из протокола, направления трафика, "DENY" и случайного числа.

*Чтобы изменить Адаптивное правило усиления защиты сети, выполните следующие действия.*

1. Чтобы изменить некоторые параметры правила, на вкладке **правила** щелкните три точки (...) в конце строки правила и нажмите кнопку **изменить**.

   ![Изменение правила](./media/security-center-adaptive-network-hardening/edit-hard-rule.png)

1. В окне **Изменение правила** обновите сведения, которые необходимо изменить, и нажмите кнопку **сохранить**.

   > [!NOTE]
   > После нажатия кнопки " **сохранить**" вы успешно изменили правило. *Однако вы не применили его к NSG.* Чтобы применить его, необходимо выбрать правило в списке и выбрать **Применить** (как описано на следующем шаге).

   ![Выбор команды "Сохранить"](./media/security-center-adaptive-network-hardening/edit-hard-rule3.png)

3. Чтобы применить обновленное правило, выберите обновленное правило в списке и нажмите кнопку **Применить**.

    ![применить правило](./media/security-center-adaptive-network-hardening/enforce-hard-rule.png)

### <a name="add-a-new-rule"></a>Добавить новое правило <a name ="add-rule"></a>

Можно добавить правило "разрешить", которое не рекомендуется центром безопасности.

> [!NOTE]
> Здесь можно добавить только правила "разрешить". Если вы хотите добавить правила "запретить", это можно сделать непосредственно в NSG. Дополнительные сведения см. [в статье Создание, изменение или удаление группы безопасности сети](../virtual-network/manage-network-security-group.md).

*Чтобы добавить Адаптивное правило усиления защиты сети, выполните следующие действия.*

1. Щелкните **Добавить правило** (находится в левом верхнем углу).

   ![Добавление правила](./media/security-center-adaptive-network-hardening/add-hard-rule.png)

1. В окне **новое правило** введите сведения и нажмите кнопку **Добавить**.

   > [!NOTE]
   > После нажатия кнопки " **Добавить**" правило будет добавлено и указано в списке других рекомендуемых правил. Однако вы не применили его к NSG. Чтобы активировать его, необходимо выбрать правило в списке и нажать кнопку **Применить** (как описано на следующем шаге).

3. Чтобы применить новое правило, выберите новое правило из списка и нажмите кнопку **Применить**.

    ![применить правило](./media/security-center-adaptive-network-hardening/enforce-hard-rule.png)


### <a name="delete-a-rule"></a>Удаление правила <a name ="delete-rule"></a>

При необходимости можно удалить рекомендуемое правило для текущего сеанса. Например, можно определить, что применение предлагаемого правила может привести к блокировке легального трафика.

*Чтобы удалить Адаптивное правило усиления защиты сети для текущего сеанса, выполните следующие действия.*

1. На вкладке **правила** щелкните три точки (...) в конце строки правила и нажмите кнопку **Удалить**.  

    ![Удаление правила](./media/security-center-adaptive-network-hardening/delete-hard-rule.png)
