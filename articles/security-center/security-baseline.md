# <a name="azure-security-baseline-for-security-center"></a>Базовый план безопасности Azure для центра безопасности

Этот базовый план безопасности применяет рекомендации из [теста безопасности Azure](../security/benchmarks/overview.md) к центру безопасности Azure. Azure Security Benchmark содержит рекомендации по обеспечению безопасности облачных решений в Azure. Содержимое группируются по **элементам управления безопасностью** , определенным в тестовом производительности системы безопасности Azure, и связанным рекомендациям, применимым к центру безопасности Azure. **Элементы управления** , неприменимые к центру безопасности Azure, были исключены. Сведения о том, как полностью сопоставить центр безопасности Azure с тестовым показателем безопасности Azure, см. в [полном файле сопоставления базовых показателей безопасности центра безопасности Azure](https://github.com/MicrosoftDocs/SecurityBenchmarks/tree/master/Azure%20Offer%20Security%20Baselines).

### <a name="11-protect-azure-resources-within-virtual-networks"></a>1,1: защита ресурсов Azure в виртуальных сетях

**Руководство**. Центр безопасности Azure — это базовое предложение Azure. Виртуальную сеть, подсеть или группу безопасности сети нельзя связать непосредственно с центром безопасности. Если вы включите сбор данных для ресурсов вычислений, центр безопасности сохранит собранные данные с помощью Log Analytics рабочей области. Вы можете настроить эту рабочую область для доступа к данным рабочей области через частную конечную точку в виртуальной сети. Кроме того, если при использовании центра безопасности сбора данных используется агент Log Analytics, который развертывается на серверах для сбора данных безопасности и обеспечения защиты для этих ресурсов вычислений. Агенту Log Analytics требуются определенные порты и протоколы, которые должны быть открыты для правильной работы с центром безопасности. Заблокируйте сети, чтобы разрешить только эти порты и протоколы, и добавьте только дополнительные правила, необходимые приложению для работы через группы безопасности сети.
