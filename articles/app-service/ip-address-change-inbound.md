# <a name="how-to-prepare-for-an-inbound-ip-address-change"></a>Подготовка к изменению входящего IP-адреса

Если вы получили уведомление о том, что входящий IP-адрес приложения Службы приложений Azure будет изменен, следуйте инструкциям в этой статье.

## <a name="determine-if-you-have-to-do-anything"></a>Определение необходимых действий

* Вариант 1. Если у приложения Службы приложений Azure нет личного домена, делать ничего не нужно.

* Вариант 2. Если на портале регистрации домена (для стороннего поставщика DNS или Azure DNS) настроена только запись CNAME (запись DNS, указывающая на URI), делать ничего не нужно.

* Вариант 3. Если на портале регистрации домена (для стороннего поставщика DNS или Azure DNS) настроена запись A (запись DNS, указывающая непосредственно на ваш IP-адрес), измените существующий IP-адрес на новый. Новый IP-адрес можно найти, следуя инструкциям в следующем разделе.

* Вариант 4: Если приложение расположено за подсистемой балансировки нагрузки, IP-фильтром или любым другим механизмом IP-адресации, который требуется для использования IP-адреса вашего приложения, измените существующий IP-адрес на новый. Новый IP-адрес можно найти, следуя инструкциям в следующем разделе.

## <a name="find-the-new-inbound-ip-address-in-the-azure-portal"></a>Поиск нового входящего IP-адреса на портале Azure

Новый входящий IP-адрес, который передается в приложение, можно найти на портале в поле **Виртуальный IP-адрес**. И старый, и новый IP-адреса связаны с вашим приложением. Через какое-то время старый адрес будет деактивирован.

1.  Откройте [портал Azure](https://portal.azure.com).

2.  В меню навигации слева выберите **Службы приложений**.

3.  Выберите в списке приложение Службы приложений Azure.

1.  Если приложение является приложением-функцией, см. раздел [Function app inbound IP address](../azure-functions/ip-addresses.md#function-app-inbound-ip-address) (Входящий IP-адрес приложения-функции).

4.  Под заголовком **Параметры** в области навигации слева щелкните **Свойства** и найдите раздел **Виртуальный IP-адрес**.

5. Скопируйте IP-адрес и измените конфигурацию записи домена или механизм IP-адресации.
