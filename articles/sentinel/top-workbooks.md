# <a name="commonly-used-azure-sentinel-workbooks"></a>Часто используемые книги Azure Sentinel

В следующей таблице перечислены наиболее часто используемые встроенные книги-Sentinel Azure.

Откройте книгу в Azure Sentinel в разделе **книги управления угрозами**  >   слева и найдите книгу, которую хотите использовать. Дополнительные сведения см. в разделе [учебник. Визуализация и мониторинг данных](tutorial-monitor-your-data.md).

|Имя книги  |Описание  |
|---------|---------|
|**Эффективность аналитики**     |  Предоставляет аналитические сведения о эффективностьх правил аналитики, помогающих достичь лучшей производительности SOC. <br><br>Дополнительные сведения см. [в наборе средств для Data-Driven SoC](https://techcommunity.microsoft.com/t5/azure-sentinel/the-toolkit-for-data-driven-socs/ba-p/2143152).|
|**Действия Azure**     |     Предоставляет подробные сведения об активности Azure в Организации путем анализа и сопоставления всех пользовательских операций и событий. <br><br>Дополнительные сведения см. в статье [Аудит с помощью журналов действий Azure](audit-sentinel-data.md#auditing-with-azure-activity-logs).    |
|**Журналы аудита Azure AD**     |  Использует журналы аудита Azure Active Directory для предоставления подробных сведений о сценариях Azure AD. <br><br>Дополнительные сведения см. в разделе Краткое руководство.  [Начало работы с Sentinel](quickstart-get-visibility.md).     |
|**Журналы аудита Azure AD, действия и входа в систему**     |   Предоставляет аналитические сведения о Azure Active Directory аудите, действиях и данных входа с помощью одной книги. Эта книга может использоваться как безопасностью, так и администраторами Azure.      |
|**Журналы входа в Azure AD**     | Использует журналы входа Azure AD для получения ценных сведений о сценариях Azure AD.        |
|**Сертификация модели кибербезопасности дата_вступления_в_силу (КММК)**     |   Предоставляет механизм для просмотра запросов к журналам, согласованных с элементами управления КММК в портфеле Майкрософт, включая предложения безопасности Майкрософт, Office 365, группы, Intune, виртуальные рабочие столы Windows и т. д. <br><br>Дополнительные сведения см. [в разделе Книга кибербезопасности дата_вступления_в_силу Model сертификация (кммк) в общедоступной предварительной версии](https://techcommunity.microsoft.com/t5/azure-sentinel/what-s-new-cybersecurity-maturity-model-certification-cmmc/ba-p/2111184).|
|**Мониторинг работоспособности сбора данных**     |   Предоставляет аналитические сведения о состоянии приема данных в рабочей области. Просмотр мониторов и обнаружение аномалий, которые помогут определить работоспособность сбора данных в рабочих областях.  <br><br>Дополнительные сведения см. [в статье мониторинг работоспособности соединителей данных с помощью этой книги Azure Sentinel](monitor-data-connector-health.md).    |
|**Анализатор событий**     |  Позволяет просматривать, проводить аудит и ускорить анализ журнала событий Windows, включая все сведения о событиях и их атрибуты, такие как безопасность, приложение, система, установка, Служба каталогов, DNS и т. д.       |
|**Exchange Online**     |Предоставляет аналитические сведения о Microsoft Exchange Online путем трассировки и анализа всех операций Exchange и действий пользователя.         |
|**Идентификация и доступ**     |   Предоставляет сведения об операциях идентификации и доступа в использовании продуктов Майкрософт с помощью журналов безопасности, содержащих журналы аудита и входа.     |
|**Обзор инцидентов**     |   Предназначен для помощи в рассмотрении и исследовании, предоставляя подробные сведения об инциденте, включая общие сведения, данные сущности, время рассмотрения, время устранения и комментарии. <br><br>Дополнительные сведения см. [в наборе средств для Data-Driven SoC](https://techcommunity.microsoft.com/t5/azure-sentinel/the-toolkit-for-data-driven-socs/ba-p/2143152).      |
|**Анализ ценных сведений**     | Предоставляет аналитикы с подробными сведениями об инцидентах, закладках и данных сущностей. Общие запросы и подробные визуализации могут помочь аналитикам исследовать подозрительные действия.       |
|**Журналы обнаружения Microsoft Cloud App Security**     |   Сведения о облачных приложениях, используемых в Организации, а также о тенденциях использования и детализации данных для конкретных пользователей и приложений.  <br><br>Дополнительные сведения см. [в разделе Подключение данных из Microsoft Cloud App Security](connect-cloud-app-security.md).|
|**Книга MITRE АТРИ&CK**     |   Содержит сведения о покрытии MITRE АТРИ&CK для Azure Sentinel.      |
|**Office 365.**     |  Предоставляет аналитические сведения о Office 365 путем трассировки и анализа всех операций и действий. Детализируйте данные SharePoint, OneDrive и Exchange.       |
|**Оповещения системы безопасности**     |  Предоставляет панель мониторинга оповещений системы безопасности для оповещений в среде Azure Sentinel. <br><br>Дополнительные сведения см. в разделе [Автоматическое создание инцидентов на основе оповещений системы безопасности Майкрософт](create-incidents-from-alerts.md).      |
|**Эффективность операций безопасности**     |  Предназначен для менеджеров центра управления Operations Center (SOC) для просмотра общих метрик эффективности и мер, касающихся производительности своей команды. <br><br>Дополнительные сведения см. в статье об [управлении приложением SoC с помощью метрик инцидента](manage-soc-with-incident-metrics.md).  |
|**Аналитика угроз**     | Предоставляет аналитические сведения о индикаторах угроз, включая тип и серьезность угроз, действия угроз с течением времени и взаимосвязь с другими источниками данных, включая Office 365 и брандмауэры.  <br><br>Дополнительные сведения см. [в статье Импорт интеллектуального анализа угроз в Azure Sentinel](import-threat-intelligence.md).      |
|**Аудит рабочей области**     |  Предоставляет отчет об аудите рабочей области, который позволяет анализировать выполнение запросов в рабочей области.   <br><br>Дополнительные сведения см. в статье [Аудит запросов и действий Sentinel Azure](audit-sentinel-data.md).  |
