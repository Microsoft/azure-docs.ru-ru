# <a name="connect-your-squid-proxy-to-azure-sentinel"></a>Подключение прокси-сервера Скуид к Azure Sentinel

> [!IMPORTANT]
> Соединитель прокси-сервера Скуид сейчас находится на **этапе предварительной версии**. Ознакомьтесь с дополнительными [условиями использования Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/) предварительных версий для дополнительных юридических условий, которые относятся к функциям Azure, которые доступны в бета-версии, предварительном просмотре или еще не выпущены в общедоступную версию.

В этой статье описывается подключение устройства прокси-сервера Скуид к Azure Sentinel. Соединитель данных прокси-сервера Скуид позволяет легко подключать журналы Скуид с помощью Azure Sentinel, чтобы просматривать данные в книгах, использовать их для создания пользовательских оповещений и внедрять их для улучшения расследования. Интеграция между прокси-сервером Скуид и Azure Sentinel использует локальную обработку файлов агентом Log Analytics.

> [!NOTE]
> Данные будут храниться в географическом расположении рабочей области, на которой вы используете метку Azure.

## <a name="prerequisites"></a>Предварительные условия

- У вас должно быть разрешение на чтение и запись в рабочей области "Sentinel" Azure.

## <a name="forward-squid-proxy-logs-to-the-log-analytics-agent"></a>Пересылка журналов Скуид proxy в агент Log Analytics  

Настройте прокси-сервер Скуид для отправки файлов журнала в рабочую область Azure с помощью агента Log Analytics.

1. В меню навигации меток Azure выберите **соединители данных**.

1. В коллекции **соединителей данных** выберите соединитель **прокси-скуид (Предварительная версия)** и **откройте страницу соединителя**.

1. Следуйте инструкциям на странице соединителя **прокси-сервера скуид** :

    1. Установка и подключение агента для Linux

        - Выберите виртуальную машину Linux в Azure или компьютер, не относящийся к Azure Linux (физический или виртуальный).

    1. Настройка собираемых журналов

        - В дополнительных параметрах рабочей области добавьте пользовательский тип журнала, отправьте пример файла и настройте его как направленный.

## <a name="find-your-data"></a>Поиск данных

После установки успешного подключения данные отображаются в **журналах** в разделе **пользовательские журналы** в `SquidProxy_CL` таблице.

Полезные примеры запросов см. на вкладке **дальнейшие действия** на странице соединителя.

## <a name="validate-connectivity"></a>Проверка подключения

После того, как журналы начнут отображаться в Log Analytics, может пройти до 20 минут.
