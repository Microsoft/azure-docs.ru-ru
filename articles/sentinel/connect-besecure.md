# <a name="connect-your-beyond-security-besecure-to-azure-sentinel"></a>Подключение к Azure Sentinel с защитой безопасности

> [!IMPORTANT]
> В настоящее время доступна **Предварительная версия** соединителя Secure Security. Ознакомьтесь с дополнительными [условиями использования Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/) предварительных версий для дополнительных юридических условий, которые относятся к функциям Azure, которые доступны в бета-версии, предварительном просмотре или еще не выпущены в общедоступную версию.

Помимо ЗАЩИЩЕНного соединителя безопасности, вы можете легко подключать все журналы решений безопасности, используя метку Azure, для просмотра панелей мониторинга, создания пользовательских оповещений и улучшения расследования. Интеграция между безопасностью и Sentinel Azure позволяет использовать REST API.

> [!NOTE]
> Данные будут храниться в географическом расположении рабочей области, на которой вы используете метку Azure.

## <a name="configure-and-connect-besecure"></a>Настройка и подключение в безопасном месте

БЕЗОПАСНОЕ взаимодействие может интегрироваться с и экспортировать журналы непосредственно в Azure Sentinel.

1. В меню навигации меток Azure выберите **соединители данных**.

1. В коллекции **соединителей данных** выберите **больше безопасности безопасность (Предварительная версия)** , а затем **откройте страницу соединителя**.

1. Выполните следующие действия, чтобы настроить безопасное решение для отправки результатов проверки, проверки состояния и журналов аудита в Azure Sentinel.

    **Доступ к меню интеграции:**
    1. Щелкните пункт "Дополнительно".

    1. Выбор сервера

    1. Выбор интеграции

    1. Включение Azure Sentinel 

    **Обеспечьте безопасность с помощью параметров Sentinel Azure:**

      Скопируйте *идентификатор рабочей области* и значения *первичного ключа* со страницы соединителя Azure Sentinel, вставьте их в конфигурацию «незащищенный» и нажмите кнопку **изменить**.

## <a name="find-your-data"></a>Поиск данных

После установки успешного подключения данные отображаются в **журналах** в разделе **CustomLogs** в одной или нескольких следующих таблицах:
  - `beSECURE_ScanResults_CL`
  - `beSECURE_ScanEvents_CL`
  - `beSECURE_Audit_CL`

Чтобы запросить безопасный журнал в правилах аналитики, поиска запросов, исследований или других местах в Azure Sentinel, введите одно из приведенных выше имен таблиц в верхней части окна запроса.
