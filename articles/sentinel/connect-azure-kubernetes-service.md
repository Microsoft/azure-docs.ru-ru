# <a name="connect-azure-kubernetes-service-diagnostics-and-auditing-logs"></a>Подключение журналов диагностики и аудита службы Azure Kubernetes

Azure Kubernetes Service (AKS) — это полностью управляемая служба оркестрации контейнеров с открытым кодом, которая позволяет развертывать и масштабировать контейнеры DOCKER и приложения на основе контейнеров и управлять ими в среде кластера.

Этот соединитель позволяет выполнять потоковую передачу журналов диагностики Azure Kubernetes Service (AKS) в Azure Sentinel, что позволяет постоянно отслеживать действия во всех экземплярах. 

Узнайте больше о [мониторинге службы Kubernetes Azure](../azure-monitor/containers/container-insights-overview.md) и о [телеметрии диагностики AKS](../aks/view-control-plane-logs.md).

## <a name="prerequisites"></a>Предварительные требования

Для приема журналов AKS в Azure Sentinel:

- У вас должны быть разрешения на чтение и запись в рабочей области Sentinel Azure.

- Чтобы использовать политику Azure для применения политики потоковой передачи журналов к ресурсам AKS, необходимо назначить роль владельца для области назначения политики.

## <a name="connect-to-azure-kubernetes-service"></a>Подключение к службе Kubernetes Azure

Этот соединитель использует политику Azure для применения одной конфигурации потоковой передачи журнала службы Azure Kubernetes к коллекции экземпляров, определенной в качестве области. Типы журналов, принимаемые службой Kubernetes Azure, можно просмотреть в левой части страницы соединителя в разделе **типы данных**.

1. В меню навигации меток Azure выберите **соединители данных**.

1. Выберите **Azure Kubernetes Service (AKS)** в коллекции соединителей данных, а затем на панели просмотра выберите **открыть страницу соединителя** .

1. В разделе **Конфигурация** страницы соединитель разверните узел **Включение журналов диагностики в службе KUBERNETES Azure (AKS)**.

1. Нажмите кнопку **запустить мастер назначения политики Azure** .

    Откроется мастер назначения политики, который готов создать новую политику с именем **развертывание. Настройте параметры диагностики для службы Azure Kubernetes в log Analytics рабочей области**.

    1. На вкладке **Основные сведения** нажмите кнопку с тремя точками в **области область** , чтобы выбрать подписку (и, при необходимости, группу ресурсов). Вы также можете добавить описание.

    1. На вкладке **Параметры** выберите рабочую область "Sentinel" Azure из раскрывающегося списка **log Analytics Рабочая область** . Оставшиеся поля раскрывающегося списка представляют доступные типы журналов диагностики. Оставьте помеченным "true" все типы журналов, которые вы хотите принять.

    1. Чтобы применить политику к существующим ресурсам, выберите вкладку " **исправление** " и установите флажок " **создать задачу исправления** ".

    1. На вкладке **Review + create** (Проверить и создать) щелкните **Создать**. Теперь ваша политика назначена выбранной области.

> [!NOTE]
>
> С этим конкретным соединителем данных индикаторы состояния подключения (цветовая полоса в коллекции соединителей данных и значки подключения рядом с именами типов данных) отображаются как *подключенные* (зеленые), только если данные были приняты в течение последних двух недель. По прошествии двух недель без приема данных соединитель будет отображаться как отключенный. В то время, когда поступает больше данных, состояние *Connected* будет возвращено.
