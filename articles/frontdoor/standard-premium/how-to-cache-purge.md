# <a name="cache-purging-in-azure-front-door-standardpremium-preview"></a>Очистка кэша в интерфейсе передней дверцы Azure Standard/Premium (Предварительная версия)

> [!Note]
> Эта документация предназначена для передней дверцы Azure уровня "Стандартный" или "Премиум" (Предварительная версия). Ищете информацию о передней дверце Azure? Просмотреть [здесь](../front-door-overview.md).

Передняя дверца Azure уровня "Стандартный" или "Премиум" кэширует ресурсы, пока не истечет срок жизни (TTL) ресурса. Каждый раз, когда клиент запрашивает ресурс с истекшим сроком жизни, среда передней дверцы Azure получает новую обновленную копию ресурса для обслуживания запроса, а затем сохраняет обновленный кэш.

Рекомендуется убедиться, что пользователи всегда получают последнюю копию ваших ресурсов. Это необходимо для того, чтобы установить версию ресурсов для каждого обновления и опубликовать их как новые URL-адреса. Передняя дверца Azure Standard/Premium будет немедленно получать новые ресурсы для следующих клиентских запросов. Иногда может потребоваться очистить кэшированное содержимое со всех пограничных узлов и принудительно получить новые обновленные ресурсы. Причина, по которой вы хотите очистить кэшированное содержимое, заключается в том, что вы внесли в приложение новые обновления или хотите обновить активы, содержащие неверные данные.

> [!IMPORTANT]
> Передняя дверца Azure Standard/Premium (Предварительная версия) в настоящее время доступна в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены.
> Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

Ознакомьтесь с [кэшированием передней дверцы Azure](concept-caching.md) , чтобы понять, как работает кэширование.

## <a name="configure-cache-purge"></a>Настройка очистки кэша

1. Перейдите на страницу обзора профиля передней дверцы Azure с ресурсами, которые необходимо очистить, а затем выберите **очистить кэш**.

   :::image type="content" source="../media/how-to-cache-purge/front-door-cache-purge-1.png" alt-text="Снимок экрана: Очистка кэша на странице обзора.":::

1. Выберите конечную точку и домен, которые необходимо удалить из граничных узлов. *(Вы можете выбрать несколько доменов)*

   :::image type="content" source="../media/how-to-cache-purge/front-door-cache-purge-2.png" alt-text="Снимок экрана: страница &quot;Очистка кэша&quot;.":::

1. Чтобы очистить все ресурсы, выберите **Очистить все ресурсы для выбранных доменов**. В противном случае в поле **пути** введите путь к каждому ресурсу, который требуется очистить.

Эти форматы поддерживаются в списках путей для очистки:

* **Очистка одного пути**. Очистите отдельные активы, указав полный путь к ресурсу (без протокола и домена) с расширением файла, например/pictures/strasbourg.png.
* **Очистка корневого домена**. Очистите корень конечной точки, указав "/*" в пути.

Очистка кэша в интерфейсе передней дверцы Azure Standard/Преиум не учитывает регистр. Кроме того, они не зависят от строки запроса. Это означает, что очистка URL-адреса приведет к очистке всех его вариантов в строке запроса.
