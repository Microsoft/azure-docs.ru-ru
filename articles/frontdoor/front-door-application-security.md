# <a name="application-layer-security-with-front-door"></a>Защита на прикладном уровне с помощью Front Door
Передняя дверца Azure предоставляет возможность защиты веб-приложений от сетевых атак и общих уязвимостей в Интернете, таких как внедрение кода SQL или межсайтовые сценарии (XSS). Благодаря поддержке внешних интерфейсов HTTP(S) безопасность на прикладном уровне Front Door является глобально распределенной и непрерывной, что позволяет отражать вредоносные атаки на границе сети Azure, далеко от ваших внутренних серверов. Благодаря повышенной безопасности и оптимизации производительности Front Door обеспечивает быструю и безопасную работу веб-приложений пользователей.

## <a name="application-protection"></a>Защита приложений
Система защиты приложений Front Door настраивается в каждой пограничной среде по всему миру в соответствии с приложениями. Она автоматически блокирует трафик, передаваемый в ваши веб-приложения не по протоколу HTTP(S). Наша распределенная мультитенантная архитектура обеспечивает глобальную защиту в нужном масштабе без снижения производительности. Для рабочих нагрузок HTTP(S) служба защиты веб-приложений Front Door предоставляет мощный обработчик правил, позволяющий создавать настраиваемые правила, предварительно настроенный набор правил для защиты от распространенных атак и подробное ведение журнала всех запросов, которые соответствуют правилу. Поддерживаются гибкие действия, в том числе разрешение, блокирование или только ведение журнала.

## <a name="custom-access-control-rules"></a>Настраиваемые правила управления доступом
- Список **разрешений IP-адресов и список блокировок:** Вы можете настроить настраиваемые правила для управления доступом к веб-приложениям на основе списка IP-адресов клиентов. Поддерживаются IPv4- и IPv6-адреса.
- **Управление доступом на основе географического расположения**. Можно задать настраиваемые правила для управления доступом к веб-приложениям на основе кода страны, в которой находится IP-адрес клиента.
- **Фильтрация параметров HTTP**. Можно задать настраиваемые правила доступа на основе сопоставления параметров HTTP(S)-запросов, включая заголовки, URL-адреса и строки запроса.

## <a name="azure-managed-rules"></a>Правила под управлением Azure
- Предварительно настроенный набор правил для защиты от самых распространенных уязвимостей OWASP включен по умолчанию. В предварительной версии этот набор правил включает в себя обнаружение атак путем внедрения кода SQL (SQLI) и выполнения межсайтовых сценариев (XSS). Будут добавлены дополнительные правила. Вы можете начать с ведения журнала, чтобы проверить, работают ли предварительно настроенные правила для ваших приложений ожидаемым образом. 

## <a name="rate-limiting"></a>Ограничение частоты
- Правило управления скоростью позволяет ограничить аномально интенсивный трафик, поступающий с какого-либо IP-адреса клиента.  Можно задать пороговое число веб-запросов, передаваемых с IP-адреса клиента в минуту.

## <a name="centralized-protection-policy"></a>Централизованная политика защиты
- Можно определить несколько правил защиты и добавить их в политику в порядке приоритета. Настраиваемые правила имеют более высокий приоритет, чем управляемый набор правил. Это позволяет разрешить исключения. Отдельная политика связывается с вашим веб-приложением.  Одна политика защиты веб-приложений реплицируется на все пограничные серверы во всех расположениях, что обеспечивает согласованность политик безопасности во всех регионах.

## <a name="configuration"></a>Конфигурация
- На этапе предварительной версии для создания и развертывания правил и политик защиты приложений Front Door можно использовать интерфейсы REST API, PowerShell или интерфейс командной строки. К моменту выпуска общедоступной версии будет добавлена поддержка этих операций на портале. 


## <a name="monitoring"></a>Мониторинг
Front Door предоставляет возможность наблюдения за веб-приложениями для защиты от атак с помощью метрик в реальном времени, интегрированных с Azure Monitor. Это позволяет отслеживать оповещения и легко наблюдать за тенденциями.

## <a name="pricing"></a>Цены
Средства безопасности на прикладном уровне Front Door на этапе предварительной версии предоставляются бесплатно.
