# <a name="get-images-from-your-custom-view"></a>Получение изображений из пользовательского представления

> [!WARNING]
> API Поиска Bing будут перенесены из Cognitive Services в службы Поиска Bing. С **30 октября 2020 г.** подготовку всех новых экземпляров Поиска Bing необходимо будет выполнять в соответствии с процедурой, описанной [здесь](/bing/search-apis/bing-web-search/create-bing-search-service-resource).
> API-интерфейсы Поиска Bing, подготовленные с помощью Cognitive Services, будут поддерживаться в течение следующих трех лет или до завершения срока действия вашего Соглашения Enterprise (в зависимости от того, какой период окончится раньше).
> Инструкции по миграции см. в статье о [службах Поиска Bing](/bing/search-apis/bing-web-search/create-bing-search-service-resource).

Пользовательский поиск изображений Bing расширяет возможности пользовательского поиска, добавляя изображения. Аналогично веб-результатам, пользовательский поиск поддерживает поиск изображений в списке веб-сайтов вашего экземпляра. Чтобы получить изображения, воспользуйтесь API пользовательского поиска изображений Bing или функцией размещенного пользовательского интерфейса. Пользоваться функцией размещенного пользовательского интерфейса очень просто; именно эта функция позволяет быстро начать и осуществлять поиск.  Сведения о настройке размещенного пользовательского интерфейса для включения изображений см. в разделе [Настройка размещенного пользовательского интерфейса](hosted-ui.md).

Если вы хотите более полно контролировать отображение результатов поиска, можно воспользоваться API пользовательского поиска изображений Bing. Поскольку вызов этого API аналогичен вызову API поиска изображений Bing, примеры вызова этого API вы найдете в разделе [Поиск изображений Bing](../Bing-Image-Search/overview.md). Однако перед этим ознакомьтесь с содержимым в [справочнике по API пользовательского поиска изображений](/rest/api/cognitiveservices-bingsearch/bing-custom-images-api-v7-reference). Основные различия заключаются в поддерживаемых параметрах запроса (необходимо включить параметр запроса customConfig) и конечной точке, которой отправляется запрос.
