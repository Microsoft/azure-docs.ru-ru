# <a name="prebuilt-models"></a>Предварительно созданные модели

Предварительно созданные модели предусмотрены для предметных областей, намерений, фраз и сущностей. Вы можете запустить приложение с помощью предварительно созданной модели или добавить соответствующую модель в приложение позже. 

## <a name="types-of-prebuilt-models"></a>Типы предварительно созданных моделей

LUIS предоставляет три типа предварительно созданных моделей. Каждую модель можно добавить в приложение в любое время. 

|Тип модели|Includes|
|--|--|
|[Доменная](luis-reference-prebuilt-domains.md)|Намерения, фразы, сущности|
|Намерения|Намерения, фразы|
|[Сущности](luis-reference-prebuilt-entities.md)|Только сущности| 

## <a name="prebuilt-domains"></a>Предварительно созданные предметные области

Распознавание речи (LUIS) предоставляет предварительно *созданные домены*, которые представляют собой заранее обученные модели [целей](luis-how-to-add-intents.md) и [сущностей](luis-concept-entity-types.md) , которые работают вместе для доменов или общих категорий клиентских приложений. 

Предварительно созданные предметные области обучены и готовы для добавления в приложение LUIS. Цель и сущности предварительно созданного домена можно полностью настроить после добавления в приложение. 

> [!TIP]
> Намерения и сущности в готовой предметной области специально подобраны. Поэтому, по возможности, лучше сочетать намерения и сущности из одной предметной области.
> Предметная область Utilities (Коммунальные службы) содержит намерения, которые можно настроить для использования в любой предметной области. Например, можно добавить `Utilities.Repeat` в приложение и научить распознавать действия пользователя, которые он хочет повторить в приложении. 

### <a name="changing-the-behavior-of-a-prebuilt-domain-intent"></a>Изменение поведения намерения готовой предметной области

Может оказаться, что готовая предметная область содержит намерение, аналогичное намерению, которое вы хотите использовать в приложении LUIS, однако они должны работать по-разному. Например, предварительно  подготовленный домен содержит `MakeReservation` предназначение для создания резервирования ресторана, но вы хотите, чтобы приложение использовало эту цель для резервирования гостиницы. В этом случае можно изменить поведение этого способа, добавив пример фразы продолжительностью, чтобы выполнить резервирование отеля, а затем переучить приложение. 

Полный список готовых предметных областей можно найти в соответствующем [справочнике](./luis-reference-prebuilt-domains.md).

## <a name="prebuilt-intents"></a>Предварительно созданные намерения

LUIS предоставляет готовые намерения и их фразы продолжительностью для каждого из своих предварительно созданных доменов. Намерения можно добавить без добавления всей предметной области. Добавление цели — это процесс добавления цели и ее фразы продолжительностью в приложение. Имя намерения и список фраз можно изменять.  

## <a name="prebuilt-entities"></a>Предварительно созданные сущности

Интеллектуальная служба распознавания речи (LUIS) включает набор предварительно созданных сущностей для распознавания общих типов информации, таких как даты, время, цифры, измерения и валюта. Поддержка предварительно созданной сущности зависит от языка и региональных параметров приложения LUIS. Полный список предварительно созданных сущностей, поддерживаемых LUIS, включая поддержку языка и региональных параметров, см. по ссылке [Entities per culture](./luis-reference-prebuilt-entities.md) (Язык и региональные параметры сущностей).

Если предварительно созданная сущность включена в приложение, ее прогнозы включатся в опубликованное приложение. Реакция на событие предварительно созданных сущностей предварительно подготовлена и **не может** быть изменена. 

> [!NOTE]
> Сущность **builtin.datetime** не рекомендуется. Она заменяется на [**builtin.datetimeV2**](luis-reference-prebuilt-datetimev2.md), которая обеспечивает распознавание диапазонов дат и времени, а также улучшенное распознавание неоднозначных дат и времени.
