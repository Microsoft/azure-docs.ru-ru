# <a name="test-your-luis-app-in-the-luis-portal"></a>Тестирование приложения LUIS на портале LUIS

[Тестирование](luis-concept-test.md) приложения — это итеративный процесс. После обучения приложения LUIS протестируйте его с примерами высказываний на предмет правильного распознавания намерений и сущностей. Если распознавание выполняется неправильно, внесите изменения в приложение, а затем обучите его и протестируйте еще раз.

## <a name="train-before-testing"></a>Обучение перед тестированием

1. Войдите на портал [LUIS](https://www.luis.ai) и выберите **Подписка** и **Ресурс для разработки**, чтобы просмотреть приложения, назначенные этому ресурсу для разработки.
1. Откройте приложение, выбрав его имя на странице " **Мои приложения** ".
1. Чтобы проверить последнюю версию активного приложения, выберите в верхнем меню пункт **обучить** перед тестированием.

## <a name="test-an-utterance"></a>Тестирование высказывания

Тестовый utterance не должен быть точно таким же, как и любой пример фразы продолжительностью в приложении. Тестовый utterance должен включать возможность выбора слов, длины фразы и использования сущностей, которые вы ожидаете использовать для пользователя.

1. Войдите на портал [LUIS](https://www.luis.ai) и выберите **Подписка** и **Ресурс для разработки**, чтобы просмотреть приложения, назначенные этому ресурсу для разработки.
1. Откройте приложение, выбрав его имя на странице " **Мои приложения** ".

1. Чтобы открыть панель **тестового** выхода, выберите **тест** на верхней панели приложения.

    > [!div class="mx-imgBorder"]
    > ![Страница "обучение & тестовое приложение"](./media/luis-how-to-interactive-test/test.png)

1. Введите высказывание в текстовом поле и нажмите клавишу ВВОД. На панели **Test** (Тестирование) можно вводить неограниченное количество тестовых высказываний, но только одно высказывание за раз.

1. Высказывание, его основное намерение и оценка добавляются в список высказываний под текстовым полем.

    > [!div class="mx-imgBorder"]
    > ![Интерактивное тестирование определяет неправильное намерение](./media/luis-how-to-interactive-test/test-weather-1.png)

## <a name="inspect-the-prediction"></a>Проверка прогноза

Сведения о результатах теста можно просмотреть на панели **Проверка** .

1. На открытой выдвигающейся панели **Test** (Тестирование) нажмите кнопку **Inspect** (Проверить) рядом с высказыванием, которое нужно сравнить.

    > [!div class="mx-imgBorder"]
    > ![Нажмите кнопку "Инспектировать" для получения дополнительных сведений о результатах теста.](./media/luis-how-to-interactive-test/inspect.png)

1. Появится панель **проверки** . На панели находится намерение с высокой оценкой, а также любые идентифицированные сущности. На панели показан прогноз выбранного utterance.

    > [!div class="mx-imgBorder"]
    > ![Частичный снимок экрана: панель проверки](./media/luis-how-to-interactive-test/inspect-panel.png)

## <a name="add-to-example-utterances"></a>Добавить к примеру фразы продолжительностью

На панели проверки можно добавить тестовый utterance к намерению, выбрав **Добавить к примеру фразы продолжительностью**.

## <a name="disable-required-features"></a>Отключение необходимых компонентов

Этот параметр позволяет определить, правильно ли обученное приложение прогнозирует ваши сущности на основе требуемых функций. Значение по умолчанию — применить эту функцию, как требуется при прогнозировании. Выберите этот переключатель, чтобы увидеть прогноз, если функция подсущности не требовалась.

### <a name="when-to-disable-required-features"></a>Когда следует отключать необходимые функции

Обученное приложение может неверно предсказать сущность, занятую компьютером, на основе одного из следующих элементов:
* Неверная метка примера фразы продолжительностью.
* Требуемая функция не соответствует тексту.

Примером является сущность, предназначенная для компьютера, с подсущностью имени человека.

:::image type="content" source="media/luis-how-to-interactive-test/disable-required-feature.png" alt-text="Снимок экрана со схемой сущностей, полученной на компьютере с LUIS Portal, с требуемым компонентом":::

Пример utterance для этой сущности, предназначенной для этой машины: `Assign Bob Jones to work on the new security feature` .

Извлечение должно быть `security feature` как описанием билета и `Bob Jones` инженером, двумя подсущностями `Assign ticket` сущности Entity.

Чтобы обеспечить успешное прогнозирование подсущности, добавьте функцию предварительно созданной сущности [персоннаме](luis-reference-prebuilt-person.md) AA в `engineer` подсущность. Если сделать эту функцию обязательной, это означает, что подсущность будет извлечена только в том случае, если для текста прогнозируется предварительно созданная сущность Персоннаме. Это означает, что любое имя в тексте, которое не прогнозируется в подсущность Персоннаме, не будет возвращено как подсущность с меткой, `engineer` .

Если вы используете интерактивную область тестирования и видите подсущность с обязательной функцией, не прогнозирует ее, переключите этот параметр, чтобы определить, будет ли подсущность предсказана без необходимости применения функции. Подсущность может быть правильно предсказана без функции, как это необходимо из-за правильного добавления меток к примеру фразы продолжительностью.

## <a name="view-sentiment-results"></a>Просмотр результатов тональности

Если на странице **[Publish](luis-how-to-publish-app.md#enable-sentiment-analysis)** (Публикация) настроен **анализ тональности**, результаты теста содержат тональность, найденную в высказывании.

## <a name="correct-matched-patterns-intent"></a>Исправление соответствующего намерения шаблона

Если при использовании [шаблонов](luis-concept-patterns.md) высказывание совпало с шаблоном, но было спрогнозировано неправильное намерение, щелкните ссылку **Изменить** в шаблоне, а затем выберите правильное намерение.

## <a name="compare-with-published-version"></a>Сравнение с опубликованной версией

Можно протестировать активную версию приложения с опубликованной версией [конечной точки](luis-glossary.md#endpoint). На панели **Inspect** (Проверка) выберите **Compare with published** (Сравнить с опубликованной). Средства на тестирование на соответствие опубликованной модели вычитаются из суммы квоты на подписку Azure.

> [!div class="mx-imgBorder"]
> ![Сравнение с опубликованной версией](./media/luis-how-to-interactive-test/inspect-panel-compare.png)

Чтобы просмотреть JSON конечной точки, возвращенный для сравнения, выберите **Show JSON view** (Показать представление JSON).

## <a name="additional-settings-in-test-panel"></a>Дополнительные параметры на панели тестирования

### <a name="luis-endpoint"></a>Конечная точка LUIS

При наличии нескольких конечных точек LUIS щелкните ссылку **Дополнительные параметры** на панели "Published" (Опубликовано), чтобы изменить конечную точку, используемую для тестирования. Если вы не уверены, какую конечную точку выбрать, выберите значение по умолчанию **Starter_Key**.
