# <a name="disclosure-design-guidelines"></a>Рекомендации по проектированию для раскрытия информации
Узнайте, как создавать и обслуживать отношения доверия с клиентами, прозрачными относительно искусственного характера вашего голоса.

## <a name="what-is-disclosure"></a>Что такое раскрытие?

Раскрытие — это средство, позволяющее людям понять, что они&#39;перевзаимодействовать с голосовыми данными, которые были искусственно созданы.

## <a name="why-is-disclosure-necessary"></a>Зачем нужно раскрытие?

Необходимость раскрывать искусственные источники созданного компьютером голоса относительно новая. В прошлом, созданные компьютером голоса были, очевидно, были, что никто не будет ошибочно появлял их для реального человека. Однако каждый день, тем не менее, повышаются реальные преимущества искусственных голосов, и они становятся все более различимыми от человеческого голоса.

## <a name="goals"></a>Цели
Ниже приведены принципы, которые следует учитывать при проектировании искусственного голоса.

**Усилить доверие**
<br>Проектирование с намерением завершить тест Turing без ухудшения производительности. Позвольте пользователям в том, что они взаимодействуют с искусственным голосовым процессом, позволяя им беспрепятственно взаимодействовать с ними.

**Адаптация к контексту использования**
<br>Узнайте, когда, где и как пользователи взаимодействуют с искусственным Voice, чтобы предоставить правильный тип раскрытия в нужное время.

**Задать четкие ожидания**
<br>Предоставьте пользователям возможность легко находить и понимать возможности агента. Предлагают возможности для получения дополнительных сведений о технологии искусственного голоса по запросу.

**Принять сбой**
<br>Используйте время сбоя, чтобы усилить возможности агента.

## <a name="how-to-use-this-guide"></a>Как пользоваться руководством

Это поможет вам определить, какие шаблоны раскрытия лучше подходят для искусственного голоса. Затем мы предлагаем примеры того, как и когда их использовать. Каждый из этих шаблонов предназначен для максимально эффективного прозрачности пользователей при использовании искусственного речевого режима, а также для того, чтобы обеспечить их проектирование в центре управления персоналом.

Учитывая обширный текст руководства по проектированию для работы с голосовыми процессами, мы подробно рассмотрим именно следующее:

1. [**Оценка раскрытия**](#disclosure-assessment): процесс определения типа раскрытия, рекомендуемого для использования с искусственным голосовым интерфейсом

2. [**Как раскрывать**](concepts-disclosure-patterns.md): примеры шаблонов раскрытия, которые можно применить к искусственному голосовому интерфейсу

3. [**Когда следует раскрывать**](concepts-disclosure-patterns.md#when-to-disclose): оптимальное время для раскрываемого пути взаимодействия пользователя

## <a name="disclosure-assessment"></a>Оценка раскрытия
Учтите, что ваши пользователи&#39; ожидания взаимодействия и контекста, в котором они будут испытывать речь. Если в контексте было ясно, что искусственный голос &quot; говорит, &quot; раскрытие может быть минимальным, нежелательным или даже ненужным. Основные типы контекста, влияющие на раскрытие, включают тип персонажа, тип сценария и уровень экспозиции. Он также помогает определить, кто может прослушиваться.
