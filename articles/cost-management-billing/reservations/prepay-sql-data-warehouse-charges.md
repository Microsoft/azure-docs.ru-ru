# <a name="save-costs-for-azure-synapse-analytics-charges-with-reserved-capacity"></a>Экономия средств при использовании Azure Synapse Analytics благодаря зарезервированной емкости

Вы можете сократить расходы на Azure Synapse Analytics, предварительно зарезервировав объем потребления cDWU на один или три года. Чтобы приобрести зарезервированную емкость Azure Synapse Analytics, выберите регион Azure и срок. Затем добавьте номер SKU Azure Synapse Analytics в корзину и выберите количество единиц cDWU, которые хотите приобрести.

При приобретении резервирования использование Azure Synapse Analytics, соответствующее атрибутам резервирования, больше не тарифицируется по модели "оплата по мере использования".

Резервирование не покрывает затраты на хранилище и сетевые ресурсы, связанные с использованием Azure Synapse Analytics.

Когда срок действия зарезервированной емкости истекает, экземпляры Azure Synapse Analytics продолжают работать, но плата за них взимается в соответствии с моделью "оплата по мере использования". Резервирования не продлеваются автоматически.

Информацию о ценах см. в разделе о [вариантах зарезервированной емкости Azure Synapse Analytics](https://azure.microsoft.com/pricing/details/synapse-analytics/).

Вы можете приобрести зарезервированную емкость Azure Synapse Analytics на [портале Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade). Платите за резервирование [наперед или ежемесячными платежами](./prepare-buy-reservation.md). Требования для приобретения резервной мощности.

- Необходимо иметь роль "Владелец" по крайней мере для одной подписки с соглашением Enterprise или подписки с оплатой по мере использования.
- Для подписок с Соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com/). Если этот параметр отключен, необходимо быть администратором EA.
- В рамках программы для поставщиков облачных решений (CSP) только агенты администрирования или агенты продаж могут приобрести зарезервированную емкость Azure Synapse Analytics.

Дополнительные сведения о выставлении счетов за приобретение резервирования для корпоративных клиентов и клиентов с оплатой по мере использования см. в статьях [Общие сведения об использовании зарезервированных экземпляров Azure с Соглашением о регистрации Enterprise](understand-reserved-instance-usage-ea.md) и [Общие сведения об использовании резервирования Azure для подписки с оплатой по мере использования](understand-reserved-instance-usage.md).

## <a name="choose-the-right-size-before-purchase"></a>Выберите правильный размер перед покупкой

Объем резервирования Azure Synapse Analytics должен основываться на общем количестве потребляемых вычислительных единиц использования хранилища данных (cDWU). Покупки осуществляются с шагами приращения в 100 cDWU.

Например, предположим, что общий объем потребления Azure Synapse Analytics составляет DW3000c. Вы хотите приобрести резервную мощность для всего. Поэтому следует приобрести 30 единиц резервной мощности cDWU.

## <a name="buy-azure-synapse-analytics-reserved-capacity"></a>Приобретение зарезервированной емкости Azure Synapse Analytics

1. Войдите на [портал Azure](https://portal.azure.com/).
2. Выберите **Все службы** > **Резервирование**.
3. Выберите подписку. Из списка "Подписка" выберите подписку, которая используется для оплаты резервной мощности. Резервная мощность оплачивается с помощью метода оплаты, указанного для подписки. Нужно выбрать подписку с соглашением Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P) или подписку с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR 0023P).
   - для подписки Enterprise плата будет вычитана из остатка на счете предоплаты Azure (прежнее название — денежное обязательство) по Соглашению о регистрации или будет относится к избыточным расходам.
   - Для подписки с оплатой по мере использования плата взимается с помощью метода оплаты (кредитной карты или счета), указанного для подписки.
4. Выберите область. Используйте список "Область" для выбора диапазона подписки.
   - **Одна группа ресурсов** — скидка по резервированию применяется к подходящим ресурсам только в выбранной группе ресурсов.
   - **Одна подписка** — скидка по резервированию применяется к подходящим ресурсам только в выбранной подписке.
   - **Общая область резервирования** — скидка по резервированию применяется к подходящим ресурсам во всех допустимых подписках в контексте выставления счетов. Для клиентов Соглашения Enterprise контекстом выставления счетов считается регистрация. Для отдельных подписок с оплатой по мере использования областью выставления счетов считаются все допустимые подписки, созданные администратором учетной записи.
   - Для клиентов Соглашения Enterprise контекстом выставления счетов считается соглашение о регистрации EA.
   - Для клиентов с подпиской с оплатой по мере использования общая область включает в себя все подписки с оплатой по мере использования, созданные администратором учетной записи.
5. Выберите регион Azure, который охватывает резервная мощность.
6. Выберите количество. Введите количество в 100 единиц Хранилища данных (cDWU), которые вы хотите приобрести.    
   Например, количество 30 будет выдавать 3 000 cDWU резервной мощности каждый час.
7. Просмотрите затраты на резервирование емкости Azure Synapse Analytics в разделе **Затраты**.
8. Щелкните **Приобрести**.
9. Выберите **Просмотреть резервирование**, чтобы просмотреть состояние покупки.

## <a name="cancel-exchange-or-refund-reservations"></a>Отмена, обмен резервирования, возмещение средств за резервирование

Вы можете отменить и обменять резервирования, а также вернуть вложенные в резервирование средства, но при этом применяются определенные ограничения. Дополнительные сведения см. в статье [Self-service exchanges and refunds for Azure Reservations](exchange-and-refund-azure-reservations.md) (Самостоятельное выполнение обмена и возмещения средств для резервирований Azure).

Скидка на резервирование автоматически применяется к количеству экземпляров Azure Synapse Analytics, которые соответствуют области резервирования емкости Azure Synapse Analytics и региону. Область резервирования емкости Azure Synapse Analytics можно обновить с помощью [портала Azure](https://portal.azure.com/), PowerShell, CLI или API.
