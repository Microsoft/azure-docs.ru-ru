# <a name="save-costs-with-azure-app-service-reserved-instances"></a>Экономия за счет зарезервированных экземпляров Службы приложений Azure

В этой статье объясняется, как можно сэкономить за счет зарезервированных экземпляров Службы приложений Azure для экземпляров ценовой категории "Премиум" версии 3 и базовой платы (ценовая категория "Изолированный").

## <a name="save-with-premium-v3-reserved-instances"></a>Экономия за счет использования зарезервированных экземпляров ценовой категории "Премиум" версии 3

Вы можете сэкономить, зарезервировав экземпляр ценовой категории "Премиум" версии 3 в Службе приложений Azure. Скидка на резервирование автоматически применяется к количеству работающих экземпляров, которые соответствуют области и атрибутам резервирования. Чтобы получить скидки, экземпляру не нужно назначать резервирование.

## <a name="determine-the-right-reserved-instance-size-before-you-buy"></a>Определение нужного размера зарезервированного экземпляра перед приобретением

Перед покупкой резервирования нужно определить размер необходимого зарезервированного экземпляра ценовой категории "Премиум" версии 3. Сведения из следующих разделов помогут подобрать подходящий размер зарезервированного экземпляра ценовой категории "Премиум" версии 3.

### <a name="use-reservation-recommendations"></a>Использование рекомендаций по резервированию

Рекомендации по резервированию помогут определить резервирования, которые следует приобрести.

- Рекомендации по приобретению и рекомендуемое количество отображаются на портале Azure при покупке зарезервированного экземпляра ценовой категории "Премиум" версии 3.
- Помощник по Azure предоставляет рекомендации по приобретению для отдельных подписок.
- Вы можете получить рекомендации по приобретению как для общей области, так и для области одной подписки с помощью API-интерфейсов. Дополнительные сведения см. в статье [API-интерфейсы рекомендаций по покупке зарезервированных экземпляров для клиентов с Соглашением Enterprise](/rest/api/billing/enterprise/billing-enterprise-api-reserved-instance-recommendation).
- Для клиентов с Соглашением Enterprise (EA) и Клиентским соглашением Майкрософт (MCA) в составе [пакета содержимого Power BI Azure Consumption Insights](/power-bi/service-connect-to-azure-consumption-insights) доступны рекомендации по приобретению для общей области и области одной подписки.

### <a name="analyze-your-usage-information"></a>Анализ сведений о потреблении

Проанализируйте сведения о потреблении, чтобы определить, какие резервирования следует приобрести. Данные о потреблении доступны в файле потребления и API-интерфейсах. Используйте их вместе, чтобы определить, какое резервирование следует приобрести. Проверьте, есть ли экземпляры ценовой категории "Премиум" версии 3 с высоким уровнем ежедневного потребления, чтобы определить количество резервирований, которые следует приобрести.

В файле потребления отображаются сведения о расходах за период выставления счетов и ежедневном потреблении. Дополнительную информацию см. в статье [Просмотр и скачивание сведений об использовании и расходах Azure](../understand/download-azure-daily-usage.md). Затем с помощью сведений из файла потребления можно [определить, какое резервирование следует приобрести](determine-reservation-purchase.md).


## <a name="buy-a-premium-v3-reserved-instance"></a>Приобретение зарезервированного экземпляра ценовой категории "Премиум" версии 3

Зарезервированные экземпляры ценовой категории "Премиум" версии 3 можно приобрести на [портале Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/documentation/filters/%7B%22reservedResourceType%22%3A%22VirtualMachines%22%7D). Платите за резервирование [наперед или ежемесячными платежами](prepare-buy-reservation.md). К приобретению зарезервированного экземпляра ценовой категории "Премиум" версии 3 применяются следующие требования:

- Необходима роль "Владелец" по крайней мере для одной подписки EA или подписки с оплатой по мере использования.
- Для подписок EA следует включить параметр **Добавить зарезервированные экземпляры** на [портале EA](https://ea.azure.com/). Если этот параметр отключен, необходимо быть администратором подписки EA.
- В рамках программы для поставщиков облачных решений (CSP) только агенты администрирования или агенты продаж могут приобрести резервирования.

Чтобы купить экземпляр, выполните следующие действия.

1. Войдите на [портал Azure](https://portal.azure.com/).
2. Выберите **Все службы** > **Резервирование**.
3. Щелкните элемент **Добавить**, чтобы приобрести новое резервирование, а затем выберите элемент **Экземпляр**.
4. Заполните обязательные поля. Работающие зарезервированные экземпляры ценовой категории "Премиум" версии 3, соответствующие выбранным атрибутам, подходят для получения скидки на резервирование. Фактическое число зарезервированных экземпляров ценовой категории "Премиум" версии 3, получающих скидку, зависит от области и выбранного количества.

Если вы заключили Соглашение EA, для быстрого добавления дополнительных экземпляров можно использовать **параметр "Добавить еще"** . Этот параметр недоступен для подписок других типов.

| **Поле** | **Описание** |
|---|---|
| Подписка | Подписка, используемая для оплаты резервирования. Резервирование оплачивается с помощью метода оплаты, указанного для подписки. Нужно выбрать подписку с соглашением Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P) или Клиентского соглашения Майкрософт либо отдельную подписку с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR 0023P). Плата вычитается из остатка денежных обязательств (при наличии такового) или относится к избыточным расходам. Для подписки с оплатой по мере использования плата взимается с помощью метода оплаты (кредитной карты или счета), указанного для подписки. |
| Область | Область резервирования может охватывать одну или несколько подписок (общая область). Например, если у вас работают 10 зарезервированных экземпляров Standard\_D2 ценовой категории "Премиум" версии 3 в регионе "Восточная часть США", следует указать количество, равное 10, чтобы получить максимальную выгоду для всех работающих экземпляров ценовой категории "Премиум" версии 3. |

## <a name="save-with-isolated-stamp-fees"></a>Экономия за счет базовой платы (ценовая категория "Изолированный")

Вы можете сэкономить расходы на гербовый сбор Службы приложений Azure (цен. категория "Изолированный"), осуществив резервирование для гербового сбора на три года. Чтобы приобрести резервную мощность для оплаты гербового сбора (цен. категория "Изолированный"), необходимо выбрать регион Azure, в котором метка будет развернута, и количество меток для приобретения.

При приобретении резервирования плата за гербовый сбор (цен. категория "Изолированный"), соответствующая атрибутам резервирования, больше не взимается в соответствии с расценками по мере использования. Резервирование автоматически применяется к числу изолированных меток, соответствующих резервной мощности и региону. Не нужно присваивать резервирование изолированной метке. Резервирование не применяется к рабочим ролям, поэтому все прочие ресурсы, связанные с меткой, оплачиваются отдельно.

После истечения срока действия резервной мощности оплата изолированного гербового сбора продолжает выполняться, но в соответствии с расценками по мере использования. Резервирования не продлеваются автоматически.

## <a name="determine-the-right-isolated-stamp-reservation-to-purchase"></a>Определение подходящего резервирования с базовой платой (ценовая категория "Изолированный") для приобретения

Приобретая резервирование, вы фиксируете использование зарезервированных объемов ресурсов в течение следующих трех лет. Проверьте данные об использовании, чтобы определить, сколько меток Службы приложений (цен. категория "Изолированный") вы постоянно используете и можете использовать в будущем.

Кроме того, убедитесь, что вы понимаете, как изолированная метка выдает единицы измерения Linux или Windows.

- Пустая изолированная метка по умолчанию создает единицу измерения меток Windows. Например, без развернутых рабочих ролей. Она продолжает создавать единицу измерения при развертывании рабочих ролей Windows в метке.
- Единица измерения изменится на единицу измерения метки Linux, если вы развернете рабочую роль Linux.
- Метка создает единицу измерения Windows, когда развернуты рабочие роли Linux и Windows.

В результате единица измерения метки может меняться между Windows и Linux в течение срока использования метки.

Приобретите резервирование метки Windows, если в метке есть одна или несколько рабочих ролей Windows. Единственный раз, когда вам необходимо будет приобрести резервирование метки Linux, это если вы планируете иметь _только_ рабочие роли Linux в метке.

## <a name="buy-isolated-stamp-reserved-capacity"></a>Приобретение резервной мощности изолированной метки

Вы можете приобрести резервную мощность изолированной метки на [портале Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/documentation/filters/%7B%22reservedResourceType%22%3A%22AppService%22%7D). Платите за резервирование [наперед или ежемесячными платежами](./prepare-buy-reservation.md). Чтобы приобрести резервную мощность, необходимо иметь роль "Владелец" по крайней мере для одной подписки Enterprise или отдельной подписки с оплатой по мере использования.

- Для подписок с Соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com/). Или, если этот параметр отключен, необходимо быть администратором EA.
- В рамках программы для поставщиков облачных решений (CSP) только агенты администрирования или агенты продаж могут приобрести зарезервированную емкость Azure Synapse Analytics.

**Для покупки:**

1. Перейдите на [портал Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/documentation/filters/%7B%22reservedResourceType%22%3A%22AppService%22%7D).
1. Выберите подписку. Из списка **Подписка** выберите подписку, которая используется для оплаты резервной мощности. Резервная мощность оплачивается с помощью метода оплаты, указанного для подписки. Нужно выбрать подписку с соглашением Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P) или подписку с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR-0023P) или подписка CSP.
    - для подписки Enterprise плата будет вычитана из остатка на счете предоплаты Azure (прежнее название — денежное обязательство) по Соглашению о регистрации или будет относится к избыточным расходам.
    - Для подписки с оплатой по мере использования плата взимается с помощью метода оплаты (кредитной карты или счета), указанного для подписки.
1. Выберите **Область** для выбора диапазона подписки.
    - **Одна группа ресурсов** — скидка по резервированию применяется к подходящим ресурсам только в выбранной группе ресурсов.
    - **Одна подписка** — скидка по резервированию применяется к подходящим ресурсам только в выбранной подписке.
    - **Общая область резервирования** — скидка по резервированию применяется к подходящим ресурсам во всех допустимых подписках в контексте выставления счетов. Для клиентов Соглашения Enterprise контекстом выставления счетов считается регистрация. Для отдельных подписок с оплатой по мере использования областью выставления счетов считаются все допустимые подписки, созданные администратором учетной записи.
1. Выберите **регион** Azure, на который распространяется резервная мощность, и добавьте резервирование в корзину.
1. Выберите тип изолированного плана и щелкните **Выбрать**.  
    ![Пример ](./media/prepay-app-service/app-service-isolated-stamp-select.png)
1. Введите количество меток Службы приложений (цен. категория "Изолированный") для резервирования. Например, количество, равное трем, даст вам три зарезервированные метки региона. Щелкните **Далее: просмотр и покупка**.
1. Просмотрите и нажмите кнопку **Buy now** (Купить сейчас).

После приобретения перейдите в раздел [Резервирования](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/ReservationsBrowseBlade), чтобы просмотреть состояние покупки и отслеживать его в любое время.

## <a name="cancel-exchange-or-refund-reservations"></a>Отмена, обмен резервирования, возмещение средств за резервирование

Вы можете отменить и обменять резервирования, а также вернуть вложенные в резервирование средства, но при этом применяются определенные ограничения. Дополнительные сведения см. в статье [Self-service exchanges and refunds for Azure Reservations](exchange-and-refund-azure-reservations.md) (Самостоятельное выполнение обмена и возмещения средств для резервирований Azure).

## <a name="discount-application-shown-in-usage-data"></a>Приложение для скидок, отображаемое в данных об использовании

В данных об использовании указана нулевая эффективная цена за использование, на которое распространяется скидка на резервирование. В них показана скидка на резервирование для каждого экземпляра метки в каждом резервировании.

Клиенты с Соглашением Enterprise (EA) могут ознакомиться с дополнительными сведениями о том, как отображается скидка на резервирование в данных об использовании, в статье [Get Enterprise Agreement reservation costs and usage](understand-reserved-instance-usage-ea.md) (Получение сведений о затратах на резервирование и его использовании для Соглашения Enterprise). В противном случае см. статью [Understand Azure reservation usage for your individual subscription with pay-as-you-go rates subscription](understand-reserved-instance-usage.md) (Общие сведения об использовании резервирования Azure для отдельной подписки с оплатой по мере использования).
