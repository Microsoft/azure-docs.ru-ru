# <a name="how-reservation-discounts-apply-to-azure-app-service-premium-v3-instances-and-isolated-stamps"></a>Применение скидок на резервирование к меткам экземпляра Службы приложений Azure уровня "Премиум" версии 3 и ценовой категории уровня "Изолированный"

Эта статья поможет понять, как скидки на резервирование применяются к меткам экземпляра Службы приложений Azure уровня "Премиум" версии 3 и ценовой категории уровня "Изолированный".

## <a name="how-reservation-discounts-apply-to-premium-v3-instances"></a>Применение скидок на резервирование к экземплярам уровня "Премиум" версии 3

После покупки зарезервированного экземпляра Службы приложений Azure уровня "Премиум" версии 3 скидка резервирования автоматически будет применяться к экземплярам Службы приложений, атрибуты и количество которых совпадают с резервированием. Резервирование покрывает стоимость экземпляров уровня "Премиум" версии 3. 

### <a name="how-the-discount-is-applied-to-azure-app-service"></a>Как скидка применяется к Службе приложений Azure 

Скидка на резервирование *сгорает, если не используется*. Поэтому, если у вас не существует подходящих ресурсов в течение некоторого часа, вы потеряете количество ресурсов, зарезервированных на этот час. Вы не можете переносить неиспользуемые зарезервированные часы.
При завершении работы ресурса скидка на резервирование автоматически переносится на другой подходящий ресурс в указанной области резервирования. Если в указанной области не найдены подходящие ресурсы, зарезервированные часы теряются.

### <a name="reservation-discount-for-premium-v3-instances"></a>Скидка на резервирование экземпляров уровня "Премиум" версии 3

Скидка на резервирование Azure применяется к запущенным экземплярам уровня "Премиум" версии 3 на почасовой основе. Для применения скидки приобретенное резервирование должно соответствовать потреблению ресурсов работающими экземплярами уровня "Премиум" версии 3. Для экземпляров уровня "Премиум" версии 3, использующихся меньше часа, резервирование будет заполнено из других экземпляров, которые его не используют, включая одновременно работающие экземпляры. Через час применение резервирования для экземпляров блокируется. Если экземпляр работает неполный час или одновременно работающие в течение часа экземпляры не занимают все время резервирования, резервирование становится недостаточным за этот час. Приведенная ниже диаграмма иллюстрирует резервирование экземпляров виртуальной машины с оплачиваемым использованием. На диаграмме представлено одно оплаченное резервирование и два соответствующих экземпляра виртуальных машин.

![Экран "Изображение, показывающее резервирование экземпляров виртуальной машины с оплачиваемым использованием".](./media/reservation-discount-app-service/reserved-premium-v3-instance-application.png)

1.  Ресурсы, потребляемые свыше определенных для резервирования, оплачиваются по мере использования. Если вы используете меньше ресурсов, чем предоставляет резервированние, дополнительная плата не взимается.
2.  В течение первого часа экземпляр 1 работает 0,75 часа, а экземпляр 2 работает 0,5 часа. Общее использование в течение 1 часа — 1,25 часа. За оставшиеся 0,25 часа с вас взимается оплата по мере использования.
3.  В течение второго и третьего часов оба экземпляра работали 1 час. Работа одного экземпляра покрывается резервированием, а за второй взимается оплата по мере использования.
4.  В течение четвертого часа экземпляр 1 работает 0,5 часа, а экземпляр 2 работает 1 час. Использование экземпляра 1 полностью покрываться резервированием. 0,5 часа работы экземпляра 2 также покрываются. За оставшиеся 0,5 часа с вас взимается оплата по мере использования.

Чтобы больше узнать о применении резервирования Azure и просмотреть отчеты о начислении платы за использование, см. статью об [использовании резервирования](understand-reserved-instance-usage-ea.md).

### <a name="discount-can-apply-to-different-sizes"></a>Скидка может распространяться на различные размеры

Если при покупке зарезервированного экземпляра уровня "Премиум" версии 3 вы выбираете **Optimized for instance size flexibility** (Оптимизировано для гибкости размера экземпляра), покрытие скидки применяется к выбранному размеру экземпляра уровня "Премиум" версии 3. Оно также может применяться к другим размерам экземпляров, принадлежащих к группе гибкости размера экземпляров той же серии.

## <a name="how-reservation-discounts-apply-to-isolated-stamps"></a>Как скидки на резервирование применяются к меткам ценовой категории "Изолированный"

После покупки резервной мощности с Ежемесячным гербовым сбором изолированной службы приложений скидка на резервирование автоматически применяется к базовой плате в регионе. Скидка на резервирование применяется к использованию, предусмотренному единицей измерения гербового сбора (изолированный). Рабочие роли, дополнительные роли, связанные с внешним интерфейсом, и любые другие ресурсы, связанные с меткой, продолжают получать счета по обычному тарифу.

### <a name="reservation-discount-application"></a>Применение скидки на резервирование

Скидка на Ежемесячный гербовый сбор изолированной службы приложений применяется к действующим изолированным меткам на почасовой оплате. Если вы не установили метку в течение часа, то резервная мощность будет потрачена впустую на этот час. Она не переносится.

После покупки приобретенное резервирование сопоставляется с изолированной меткой, работающей в определенном регионе. Если вы закроете эту метку, скидки на резервирование будут автоматически применены к любым другим меткам, работающим в регионе. Если метки не существуют, резервирование применяется к следующей метке, созданной в регионе.

Если метки не работают в течение полного часа, резервирование автоматически применяется к другим соответствующим меткам в том же регионе в течение того же часа.

### <a name="choose-a-stamp-type---windows-or-linux"></a>Выбор типа метки — Windows или Linux

Пустая изолированная метка по умолчанию создает единицу измерения меток Windows. Например, когда не развернуты рабочие роли. Она продолжает создавать единицу измерения при развертывании рабочих ролей Windows. Единица измерения изменится на единицу измерения метки Linux, если вы развернете рабочую роль Linux. Метка создает единицу измерения Windows, когда развернуты рабочие роли Linux и Windows.

В результате единица измерения метки может меняться между Windows и Linux в течение срока использования метки. В то же время резервирование зависит от операционной системы. Вам нужно будет купить резервирование, поддерживающее рабочие роли, которые вы планируете развернуть в метке. Для меток только для Windows и смешанных меток используется резервирование Windows. Метки только с рабочими ролями Linux используют резервирование Linux.

Единственный раз, когда вам необходимо будет приобрести резервирование Linux, это если вы планируете иметь _только_ рабочие роли Linux в метке.

### <a name="discount-examples"></a>Примеры скидок

В следующих примерах показано, как применяется скидка на зарезервированный экземпляр изолированного гербового сбора в зависимости от развертываний.

- **Пример 1**: Вы приобретаете один экземпляр мощности изолированных зарезервированных меток в регионе без меток Службы приложений (цен. категория "Изолированный"). Вы развертываете новую метку в регионе и платите по зарезервированным тарифам для этой метки.
- **Пример 2**. Вы приобретаете один экземпляр изолированной зарезервированной метки в регионе, в котором уже развернута метка Службы приложений (цен. категория "Изолированный"). Вы начинаете получать зарезервированный тариф для развернутой метки.
- **Пример 3**: Вы приобретаете один экземпляр мощности изолированной зарезервированной метки в регионе с уже развернутой меткой Службы приложений (цен. категория "Изолированный"). Вы начинаете получать зарезервированный тариф для развернутой метки. Позже вы удалите метку и развернете новую. Вы получаете зарезервированный тариф для новой метки. Скидки не переносятся на время без развернутых меток.
- **Пример 4**. Вы приобретаете один экземпляр мощности изолированной зарезервированной метки Linux в регионе, а затем развертываете новую метку в регионе. Когда метка изначально развертывается без рабочих ролей, она создает единицу измерения меток Windows. Скидка не получена. Когда первая рабочая роль Linux развертывает метку, она создает единицу измерения меток Linux и применяется скидка на резервирование. Если рабочая роль Windows позже развернется в метке, единица измерения меток вернется к Windows. Вы больше не получаете скидку на резервирование изолированной зарезервированной метки Linux.
