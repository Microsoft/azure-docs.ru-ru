# <a name="common-cost-analysis-uses"></a>Использование анализа общих затрат

Пользователи Управления затратами Azure зачастую хотят получать ответы на вопросы, которые запрашивают многие другие пользователи. В этой статье описывается получение результатов для общих задач анализа затрат в службе "Управление затратами".

## <a name="view-forecasted-costs"></a>Просмотр прогнозируемых затрат

Прогнозируемые затраты отображаются в областях анализа затрат в представлениях областей и гистограммах с накоплением. Прогноз основан на данных журнала использования ресурсов. Изменения в использовании ресурсов влияют на прогнозируемые затраты.

На портале Azure перейдите к анализу затрат для вашей области. Пример: Выберите **Cost Management + Billing** (Управление затратами и выставление счетов)  > **Управление затратами** > **Анализ затрат**.

В представлении по умолчанию верхняя диаграмма содержит разделы с фактическими и амортизированными затратами, а также с прогнозируемыми затратами. В одноцветных областях диаграммы отображаются фактические и амортизационные затраты. В затененных областях диаграммы отображаются прогнозируемые затраты.

[![Прогнозируемые затраты](./media/cost-analysis-common-uses/enrollment-forecast.png)](./media/cost-analysis-common-uses/enrollment-forecast.png#lightbox)

## <a name="view-forecasted-costs-grouped-by-service"></a>Просмотр прогнозируемых затрат, сгруппированных по службам

В представлении по умолчанию не отображается прогнозируемая группа затрат по службе, поэтому необходимо добавить группу по выбору.

На портале Azure перейдите к анализу затрат для вашей области. Пример: Выберите **Cost Management + Billing** (Управление затратами и выставление счетов)  > **Управление затратами** > **Анализ затрат**.

Выберите **Группировать по** > **Имя службы**.

В представлении отображаются затраты, сгруппированные для каждой службы. Прогнозируемые затраты не рассчитываются для каждой службы. Они проецируются для **общего** числа служб.

[![Сгруппированные прогнозируемые затраты](./media/cost-analysis-common-uses/forecast-group-by-service.png)](./media/cost-analysis-common-uses/forecast-group-by-service.png#lightbox)

## <a name="view-forecasted-costs-for-a-service"></a>Просмотр прогнозируемых затрат для службы

Вы можете просмотреть прогнозируемые затраты, ограниченные до одной службы. Например, может потребоваться просмотреть прогнозируемые затраты только для виртуальных машин.

1. На портале Azure перейдите к анализу затрат для вашей области. Пример: Выберите **Cost Management + Billing** (Управление затратами и выставление счетов)  > **Управление затратами** > **Анализ затрат**.
1. Выберите **Добавить фильтр**, а затем **Имя службы**.
1. В списке **Выбрать** выберите службу. Например, выберите **Виртуальные шины**.

Проверьте фактические затраты выбранной службы и прогнозируемую стоимость.

В представление можно добавить дополнительные настройки.

1. Добавьте второй фильтр для **единицы измерения** и выберите значение, чтобы отфильтровать отдельный тип единицы измерения для выбранного имени службы.
1. Сгруппируйте по **ресурсу**, чтобы увидеть определенные ресурсы, на которые начисляется плата. Прогнозируемые затраты не рассчитываются для каждой службы. Они проецируются для **общего** числа ресурсов.

[![Прогнозируемые затраты для службы](./media/cost-analysis-common-uses/forecast-by-service.png)](./media/cost-analysis-common-uses/forecast-by-service.png#lightbox)

## <a name="view-your-azure-and-aws-costs-together"></a>Просмотр затрат на Azure и AWS одновременно  

Используйте области групп управления в Azure для одновременного просмотра затрат на Azure и AWS.

1. Создайте группу управления или выберите существующую.
1. Назначьте существующие и надлежащие подписки Azure, необходимые для группы управления.
1. Назначить *ту же* группу управления связанной учетной записи соединителя.
1. Перейдите в раздел "Анализ затрат" и выберите пункт **Накопленные затраты**.
1. Выберите **Группировать по** - **Поставщик**.

Просмотр затрат службой Azure поможет вам лучше разобраться в частях вашей инфраструктуры, которые обходятся дороже всего. Например, затраты на вычислительные ресурсы виртуальных машин могут быть небольшими. Тем не менее, вы можете получить значительные сетевые расходы из-за объема информации, выдаваемой ВМ. Понимание основных факторов, определяющих стоимость ваших услуг Azure, необходимо для того, чтобы вы могли регулировать использование услуг по мере необходимости.

1. На портале Azure перейдите к анализу затрат для вашей области. Пример: Выберите **Cost Management + Billing** (Управление затратами и выставление счетов)  > **Управление затратами** > **Анализ затрат**.
1. Выберите **Стоимость служб раздельно** а затем сгруппируйте по функции **Уровень обслуживания**.
1. Измените представление на **Таблица**.

## <a name="view-your-spot-vm-charges"></a>Просмотр расходов на точечную виртуальную машину

Точечные виртуальные машины могут обеспечить значительную экономию затрат для рабочих нагрузок, которые могут обрабатывать прерывания. Рабочие нагрузки выполняются на неиспользованной емкости Azure. Так как в любое время точечные виртуальные машины могут быть исключены, они обеспечивают значительную экономию. Чтобы просмотреть расходы на точечную виртуальную машину, выполните приведенные ниже шаги.

1. На портале Azure перейдите к анализу затрат для вашей области. Например, выберите **Управление затратами + выставление счетов** > **Управление затратами** > **Анализ затрат**.
2. Добавьте фильтр для поля **Модель ценообразования: точечная**.

![Пример фильтра "Точечная виртуальная машина"](./media/cost-analysis-common-uses/spot-vm-filter.png)

Измерение модели ценообразования также используется для просмотра расходов по требованию, а также расходов на резервирование.

## <a name="view-your-reservation-charges"></a>Просмотр расходов на резервирование

Зарезервированные экземпляры предоставляют возможность сэкономить деньги с помощью Azure. При резервировании вы тратите деньги заранее на определенное количество ресурсов с течением времени. В анализе затрат отображаются расходы по мере их появления в счете. В течение периода резервирования расходы отображаются в качестве фактических затрат или затрат на амортизацию.

1. На портале Azure перейдите к анализу затрат для вашей области. Например, выберите **Управление затратами + выставление счетов** > **Управление затратами** > **Анализ затрат**.
1. Добавьте фильтр для поля **Модель ценообразования: резервирование**.
1. В разделе **Область** и рядом с отображаемыми затратами щелкните символ стрелки вниз, выберите метрику **Фактическая стоимость** или **Амортизированная стоимость**.

![Выбор метрики затрат](./media/cost-analysis-common-uses/metric-cost.png)

Каждая метрика влияет на то, как данные отображаются для расходов на резервирование.

**Фактическая стоимость.** Отображает покупку в том виде, в каком она указана в счете. Например, если вы купили годовое резервирование на 1200 долларов в январе, в анализе затрат отобразится стоимость 1200 долларов для резервирования в январе. Он не отображает стоимость резервирования за другие месяцы в году. Если вы сгруппируете фактическую стоимость по виртуальным машинам, то в виртуальной машине, получившей преимущество резервирования за определенный месяц, будет нулевая стоимость за месяц.

**Амортизированная стоимость.** Отображает разбивку приобретения резервирования как амортизированную стоимость в течение срока резервирования. Используя тот же приведенный выше пример, в анализе затрат мы увидим отображение стоимости в 100 долларов за каждый месяц года, если в январе было приобретено резервирование на весь год за 1200 долларов. Если в этом примере вы сгруппируете расходы по виртуальной машине, отобразится стоимость, связанная с каждой виртуальной машиной, которая получила преимущество резервирования.

## <a name="view-your-reservation-utilization"></a>Просмотр сведений об использования резервирования

После того как вы приобрели резервирование, необходимо отслеживать его использование, чтобы вы получили то, за что заплатили. Например, если вы приобрели 10 виртуальных машин в год и используете только пять из них, фактически половина покупки потрачена впустую. Существует два разных способа оценки использования.

### <a name="view-unused-ri-costs-in-cost-analysis"></a>Просмотр неиспользованных затрат на приобретение зарезервированного экземпляра в анализе затрат

Чтобы определить, какая сумма в настоящее время тратится каждый месяц на резервирования впустую, выполните приведенные ниже действия.

1. На портале Azure перейдите к анализу затрат для области, где применено резервирование. Например, выберите **Управление затратами + выставление счетов** > **Управление затратами** > **Анализ затрат**.
1. Добавьте фильтр для поля **Модель ценообразования: резервирование**.
1. Выберите представление **Амортизированная стоимость**.
1. Задайте для параметра "Детализация" значение **Ежемесячно**.
1. Задайте для периода времени текущий год или срок резервирования.
1. Для типа диаграммы установите значение **Гистограмма (с накоплением)** .
1. Сгруппируйте затраты по **типу расходов**.
1. Просмотрите результаты для значений `unusedreservation`.

[![Пример, показывающий данные об использовании резервирования](./media/cost-analysis-common-uses/view-reservation-cost.png)](./media/cost-analysis-common-uses/view-reservation-cost.png#lightbox)

### <a name="view-utilization-in-reservations"></a>Просмотр данных об использовании в резервированиях

Подробные инструкции см. в разделе [Оптимизация использования резервирования](../reservations/manage-reserved-vm-instance.md#optimize-reservation-use).

## <a name="view-costs-for-a-specific-tag"></a>Просмотр затрат для определенного тега

Поддержка тегов применяется к использованию, отчет о котором поступил *после* того, как к ресурсу был применен тег. Теги не применяются к сверткам затрат задним числом.

1. На портале Azure перейдите к анализу затрат для вашей области. Пример: Выберите **Cost Management + Billing** (Управление затратами и выставление счетов)  > **Управление затратами** > **Анализ затрат**.
1. Выберите **Группировать по** для вашего тега.

## <a name="view-monthly-ea-cost-breakdown"></a>Просмотр ежемесячной детализации затрат EA

Ваша регистрация в EA начисляет затраты для всей организации. Понимание того, как возникают затраты и как они со временем выставляются по счетам, помогает привлечь соответствующие заинтересованные лица, чтобы обеспечить ответственное управление затратами.

Расходы отображаются только для активной регистрации. Если вы перешли с неактивной регистрации на новую (активную), затраты на предыдущую регистрацию не будут отображаться в Управлении затратами.


1. Перейдите к разделу **Cost Management + Billing** (Управление затратами и выставление счетов) > **Обзор** на портале Azure.
1. Щелкните **Разбивка** для текущего месяца и просмотрите запись об израсходованных средствах в рамках предоплаты Azure (прежнее название — денежное обязательство).  
    [![Обзор затрат EA — сводка по разбивке](./media/cost-analysis-common-uses/breakdown1.png)](./media/cost-analysis-common-uses/breakdown1.png#lightbox)
1.  Перейдите на вкладку **Использование и расходы** и просмотрите разбивку за прошлый месяц в выбранном интервале времени.  
    [![Вкладка использования и расходов Azure](./media/cost-analysis-common-uses/breakdown2.png)](./media/cost-analysis-common-uses/breakdown2.png#lightbox)

## <a name="view-enrollment-monthly-cost-by-term"></a>Просмотр месячной стоимости регистрации по срокам

Используйте представление в виде диаграммы ежемесячных затрат на регистрацию, чтобы понять тенденции затрат и суммы счетов за указанный период.

1. На портале Azure перейдите к анализу затрат для вашей области. Пример: Выберите **Cost Management + Billing** (Управление затратами и выставление счетов)  > **Управление затратами** > **Анализ затрат**.
1. Выберите регистрацию и задайте срок регистрации.
1. Задайте для параметра "Детализация" значение "ежемесячно", а затем задайте для представления значение **Столбец (с распределением)** .

Для более подробного анализа данные можно группировать и фильтровать.

[![Ежемесячные затраты на регистрацию по сроку](./media/cost-analysis-common-uses/enrollment-term1.png)](./media/cost-analysis-common-uses/enrollment-term1.png#lightbox)

## <a name="view-ea-enrollment-accumulated-costs"></a>Просмотр накопленных затрат на регистрацию EA

Просматривайте накопленные затраты с течением времени, чтобы понять общие расходы на организацию в течение заданного периода.

1. На портале Azure перейдите к анализу затрат для вашей области. Пример: Выберите **Cost Management + Billing** (Управление затратами и выставление счетов)  > **Управление затратами** > **Анализ затрат**.
1. Выберите регистрацию и просмотрите текущие накопленные затраты.
