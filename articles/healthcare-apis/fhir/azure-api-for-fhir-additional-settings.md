# <a name="additional-settings-for-azure-api-for-fhir"></a>Дополнительные параметры для API Azure для FHIR

В этом пошаговом руководство мы рассмотрим дополнительные параметры, которые вы можете задать в API Azure для FHIR. Существуют дополнительные страницы, на которых подробно рассматривается более подробная информация.

## <a name="configure-database-settings"></a>Настройка параметров базы данных

API Azure для FHIR использует базу данных для хранения своих данных. Производительность базовой базы данных зависит от числа единиц запроса (RU), выбранных во время подготовки службы или в параметрах базы данных после подготовки службы.

Пропускная способность должна быть подготовлена для обеспечения доступности достаточного количества системных ресурсов для базы данных в любое время. Количество запросов, необходимых для вашего приложения, зависит от выполняемых операций. Операции могут варьироваться от простых операций чтения и записи до более сложных запросов.

Дополнительные сведения об изменении параметров по умолчанию см. в разделе [Настройка параметров базы данных](configure-database.md).

## <a name="access-control"></a>Управление доступом

API Azure для FHIR разрешает доступ только полномочным пользователям к API FHIR. Вы можете настроить полномочных пользователей с помощью двух разных механизмов. Основной и рекомендуемый способ настройки контроля доступа — использование [управления доступом на основе ролей Azure (Azure RBAC)](../../role-based-access-control/index.yml), доступ к которому можно получить с помощью колонки **управления доступом (IAM)** . Azure RBAC работает только в том случае, если требуется защитить доступ к плоскости данных с помощью клиента Azure Active Directory, связанного с подпиской. Если вы хотите использовать другой клиент, API Azure для FHIR предлагает локальный механизм управления доступом к плоскости данных FHIR. При использовании локального механизма RBAC Параметры конфигурации имеют более широкие возможности. Для получения дополнительных сведений выберите один из следующих вариантов.

* [Azure RBAC для плоскости данных FHIR](configure-azure-rbac.md). Это предпочтительный вариант при использовании клиента Azure Active Directory, связанного с подпиской.
* [Управление доступом к плоскости данных Local FHIR](configure-local-rbac.md). Используйте этот параметр только в том случае, если необходимо использовать внешний клиент Azure Active Directory для управления доступом к плоскости данных. 

## <a name="enable-diagnostic-logging"></a>Включение ведения журналов диагностики
Вы можете включить ведение журнала диагностики в процессе установки, чтобы иметь возможность отслеживать службу и получать точные отчеты о соответствии требованиям. Дополнительные сведения о настройке ведения журнала диагностики см. в [статье как настроить](enable-diagnostic-logging.md) ведение журнала диагностики, а также несколько примеров запросов. 

## <a name="use-custom-headers-to-add-data-to-audit-logs"></a>Использование пользовательских заголовков для добавления данных в журналы аудита
В API Azure для FHIR может потребоваться включить дополнительные сведения в журналы, которые поступают из вызывающей системы. Чтобы включить эти сведения, можно использовать пользовательские заголовки.

Пользовательские заголовки можно использовать для сбора информации нескольких типов. Пример:

* сведения об авторизации или удостоверении;
* источник вызывающей стороны;
* организация вызывающей стороны;
* сведения о клиентской системе (электронная медицинская запись, портал для пациентов).

Чтобы добавить эти данные в журналы аудита, см. инструкции по [использованию настраиваемых заголовков HTTP для добавления данных в журналы аудита](use-custom-headers.md) .
