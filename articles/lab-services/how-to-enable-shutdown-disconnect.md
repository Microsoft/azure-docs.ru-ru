# <a name="configure-automatic-shutdown-of-vms-for-a-lab"></a>Настройка автоматического завершения работы виртуальных машин для лаборатории

В этой статье показано, как можно настроить автоматическое завершение работы виртуальных машин для лаборатории.

Вы можете включить несколько функций управления затратами на автоматическое завершение работы, чтобы заранее предотвратить дополнительные затраты, если виртуальные машины не используются активно. Сочетание следующих трех функций автоматического завершения работы и отключения перехватывает большую часть случаев, когда пользователи случайно оставляют свои виртуальные машины.
 
* Автоматическое отключение пользователей от виртуальных машин, которые операционная система считает неактивными.
* Автоматически завершать работу виртуальных машин при отключении пользователей.
* Автоматическое завершение работы виртуальных машин, которые запущены, но не подключаются пользователями.

Ознакомьтесь с дополнительными сведениями о функциях автоматического завершения работы в разделе [Параметры автоматического завершения работы](cost-management-guide.md#automatic-shutdown-settings-for-cost-control) .

Администратор учетной записи лаборатории может настроить этот параметр для учетной записи лаборатории, в которой будут создаваться лаборатории. Дополнительные сведения см. [в статье Настройка автоматического завершения работы виртуальных машин для учетной записи лаборатории](how-to-configure-lab-accounts.md). Как владелец лаборатории вы можете переопределить этот параметр при создании лаборатории или после создания лаборатории. 

## <a name="configure-for-the-lab-level"></a>Настройка для уровня лаборатории

Вы можете настроить параметр автоматического завершения работы в [службах лаборатории Azure](https://labs.azure.com/).

* При создании лаборатории (в **политиках лаборатории**) или
* После создания лаборатории (в **параметрах**)

> [!div class="mx-imgBorder"]
> ![Настройка во время создания лаборатории](./media/how-to-enable-shutdown-disconnect/configure-lab-creation.png)

Обязательно ознакомьтесь с подробными сведениями об автоматическом завершении работы в разделе [Параметры автоматического завершения управления затратами](cost-management-guide.md#automatic-shutdown-settings-for-cost-control) .

> [!WARNING]
> При завершении работы операционной системы Linux или Windows (ОС) на виртуальной машине перед отключением сеанса RDP с виртуальной машиной функция автоматического завершения работы не будет работать должным образом.
