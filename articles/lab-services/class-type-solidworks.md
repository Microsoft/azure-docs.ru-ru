# <a name="set-up-a-lab-for-engineering-classes-using-solidworks"></a>Настройка лаборатории для инженерных классов с помощью СОЛИДВОРКС

[Солидворкс](https://www.solidworks.com/) предоставляет среду с трехмерным компьютерным ДИЗАЙНОМ (CAD) для моделирования сплошных объектов и используется в различных видах инженерных полей.  С помощью СОЛИДВОРКС инженеры могут легко создавать, визуализировать, имитировать и документировать свои макеты.

Вариант лицензирования, обычно используемый университетом, — это СОЛИДВОРКС "Сетевое лицензирование".   Этот параметр позволяет пользователям совместно использовать пул лицензий, управляемых сервером лицензирования.  Этот тип лицензии иногда называется "плавающей" лицензией, так как требуется только наличие достаточного количества лицензий для числа одновременных пользователей.  Когда пользователь будет выполнен с помощью СОЛИДВОРКС, его лицензия вернется в централизованно управляемый пул лицензий, чтобы его можно было повторно использовать другим пользователем.

В этой статье мы покажем, как настроить класс, который использует СОЛИДВОРКС 2019 и сетевое лицензирование.

## <a name="license-server"></a>Сервер лицензирования

Для сетевого лицензирования СОЛИДВОРКС требуется, чтобы на сервере лицензирования был установлен и активирован Диспетчер лицензий Солиднетворк.  Этот сервер лицензирования обычно находится в локальной сети или в частной сети в Azure.  Дополнительные сведения о настройке диспетчера лицензий Солиднетворк на сервере см. в разделе [Установка и активация диспетчера лицензий](https://help.solidworks.com/2019/English/Installation/install_guide/t_installing_snl_lic_mgr.htm) руководства по установке солидворкс.  При этом следует запомнить **номер порта** и [**серийный номер**](https://help.solidworks.com/2019/english/installation/install_guide/r_hid_state_serial_number.htm) , которые используются, так как они понадобятся на последующих шагах.

После настройки сервера лицензирования необходимо установить пиринг между [виртуальной сетью (VNet)](./how-to-connect-peer-virtual-network.md) и [учетной записью лаборатории](./tutorial-setup-lab-account.md).  Пиринг сети должен быть создан перед созданием лаборатории, чтобы виртуальные машины лаборатории могли получить доступ к серверу лицензирования и другим способом.

> [!NOTE]
> Убедитесь, что на брандмауэрах открыты соответствующие порты, чтобы разрешить обмен данными между виртуальными машинами лаборатории и сервером лицензирования.  Например, см. инструкции по [изменению портов компьютеров диспетчера лицензий для брандмауэра Windows](http://help.solidworks.com/2019/english/installation/install_guide/t_mod_ports_on_lic_mgr_for_firewall.htm) , в которых показано, как добавить правила входящего и исходящего трафика в брандмауэр сервера лицензирования.  Также может потребоваться открыть порты для виртуальных машин лаборатории.  Выполните действия, описанные в статье о [параметрах брандмауэра для лабораторных занятий](./how-to-configure-firewall-settings.md) , чтобы получить дополнительные сведения об этом, включая получение общедоступного IP-адреса лаборатории.

## <a name="lab-configuration"></a>Настройка лаборатории

Для настройки этой лабораторной работы требуется подписка Azure и учетная запись лаборатории. Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись](https://azure.microsoft.com/free/), прежде чем начинать работу. После получения подписки Azure можно создать новую учетную запись лаборатории в службах лаборатории Azure. Дополнительные сведения о создании новой учетной записи лаборатории см. в руководстве по [настройке учетной записи лаборатории](./tutorial-setup-lab-account.md). Можно также использовать существующую учетную запись лаборатории.

### <a name="lab-account-settings"></a>Параметры учетной записи лаборатории

Включите параметры, описанные в таблице ниже, для учетной записи лаборатории. Дополнительные сведения о том, как включить образы Marketplace, см. в статье [Указание образов Marketplace, доступных авторам лабораторий](./specify-marketplace-images.md).

| Настройки учетной записи лаборатории | Instructions |
| ------------------- | ------------ |
|Образ Marketplace| Включите образ Windows 10 Pro для использования в учетной записи лаборатории.|

> [!NOTE]
> Помимо Windows 10, СОЛИДВОРКС поддерживает другие версии Windows.  Дополнительные сведения см. в разделе [требования к системе солидворкс](https://www.solidworks.com/sw/support/SystemRequirements.html) .

### <a name="lab-settings"></a>Параметры лаборатории

При настройке лаборатории для занятий используйте параметры, приведенные в таблице ниже. Дополнительные сведения о том, как создать лабораторную среду, см. в разделе Настройка лаборатории для занятий.

| Параметры лаборатории | Значение или инструкции |
| ------------ | ------------------ |
|Размер виртуальной машины| **Малый GPU (визуализация)**.  Эта виртуальная машина лучше всего подходит для удаленной визуализации, потоковой передачи, игр и кодирования с помощью таких платформ, как OpenGL и DirectX.|  
|Образ виртуальной машины| Windows 10 Pro|

> [!NOTE]
> Размер виртуальной машины **небольшого графического процессора (визуализация)** настроен для обеспечения высокопроизводительной работы с графикой.  Дополнительные сведения об этом размере виртуальной машины см. в статье [Настройка лаборатории с графическими процессорами](./how-to-setup-lab-gpu.md).

> [!WARNING]
> Не забудьте сделать [виртуальную сеть виртуальной](./how-to-connect-peer-virtual-network.md) сетью для учетной записи лаборатории для сервера лицензирования **перед** созданием лаборатории.

## <a name="template-virtual-machine-configuration"></a>Конфигурация виртуальной машины шаблона

В этом разделе показано, как настроить виртуальную машину шаблона, загрузив файлы установки СОЛИДВОРКС и установив клиентское программное обеспечение.

1. Запустите виртуальную машину шаблона и подключитесь к компьютеру по протоколу RDP.

1. Скачайте файлы установки для клиентского программного обеспечения СОЛИДВОРКС. Есть два варианта загрузки:
   - Скачайте на [портале клиента солидворкс](https://login.solidworks.com/nidp/idff/sso?id=cpenglish&sid=1&option=credential&sid=1&target=https%3A%2F%2Fcustomerportal.solidworks.com%2F).
   - Скачайте из каталога на сервере.  Если вы использовали этот параметр, необходимо убедиться, что сервер доступен с виртуальной машины шаблона.  Например, этот сервер может находиться в той же виртуальной сети, которая связана с вашей учетной записью лаборатории.
  
    Дополнительные сведения см. в статье [Установка на отдельных компьютерах раздела солидворкс](http://help.solidworks.com/2019/english/Installation/install_guide/c_installing_on_individual_computers.htm?id=fc149e8a968a422a89e2a943265758d3#Pg0) в разделе руководства по установке солидворкс.

1. После скачивания файлов установки установите клиентское программное обеспечение с помощью диспетчера установки СОЛИДВОРКС. Дополнительные сведения об [установке клиента лицензий](http://help.solidworks.com/2019/english/installation/install_guide/t_installing_snl_license_client.htm) см. в разделе солидворкс по установке.

    > [!NOTE]
    > В диалоговом окне **Добавление сервера** появится запрос на ввод **номера порта** , используемого для сервера лицензирования, а также имени или IP-адреса сервера лицензирования.

## <a name="cost"></a>Стоимость

Давайте рассмотрим возможную оценку затрат для этого класса. Эта оценка не включает затраты на запуск сервера лицензирования. Мы будем использовать класс из 25 учащихся. Существует 20 часов запланированного времени класса. Кроме того, каждый учащийся получает 10 часов квоты на домашние задания или назначения за пределами запланированного времени класса. Выбранный размер виртуальной машины — это **малый GPU (визуализация)**, то есть 160 лабораторных единиц.

25 учащихся \* (20 запланированных часов + 10 квот) \* 160 единицы лабораторий * 0,01 долларов США в час = 1200,00 долл. США

>[!IMPORTANT]
> Оценка стоимости предназначена только для примера.  Текущие сведения о ценах см. на странице [цен на службы лаборатории Azure](https://azure.microsoft.com/pricing/details/lab-services/).
