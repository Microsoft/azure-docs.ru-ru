# <a name="save-costs-for-resources-with-reserved-capacity---azure-sql-database--sql-managed-instance"></a>Экономия затрат ресурсов с зарезервированной емкостью — база данных SQL Azure & SQL Управляемый экземпляр
[!INCLUDE[appliesto-sqldb-sqlmi](../includes/appliesto-sqldb-sqlmi.md)] 

Экономия денег с помощью базы данных SQL Azure и Управляемый экземпляр SQL путем фиксации в резервировании для вычислений ресурсов по сравнению с ценами оплаты по мере использования. С зарезервированной емкостью вы выполняете обязательства для базы данных SQL и (или) SQL Управляемый экземпляр использовать в течение одного или трех лет для получения значительной скидки на затраты на вычисление. Чтобы приобрести зарезервированную емкость, необходимо указать регион Azure, тип развертывания, уровень производительности и термин.

Не нужно назначать резервирование определенной базе данных или управляемому экземпляру. Возможность сопоставления существующих развертываний, которые уже запущены или развертывались автоматически, дает преимущество. Приобретя резервирование, вы фиксируете затраты на вычисление за период, состоящие из одного или трех лет. После приобретения резервирования стоимость вычислений, которая соответствует атрибутам резервирования, больше не взимается по тарифам с оплатой по мере использования. 

Резервирование применяется как к основной, так и к оплачиваемой вторичной реплике вычислений, но не охватывает программное обеспечение, сетевые подключения или расходы на хранение, связанные со службой. По окончании срока резервирования срок выставления счетов истекает, а база данных или управляемый экземпляр выставляются по цене оплаты по мере использования. Резервирования не обновляются автоматически. Сведения о ценах см. в [предложении зарезервированных мощностей](https://azure.microsoft.com/pricing/details/sql-database/managed/).

Зарезервированную емкость можно приобрести в [портал Azure](https://portal.azure.com). Платите за резервирование [наперед или ежемесячными платежами](../../cost-management-billing/reservations/prepare-buy-reservation.md). Требования для приобретения резервной мощности.

- Необходимо быть в роли владельца по крайней мере для одной корпоративной или отдельной подписки с тарифами с оплатой по мере использования.
- Для подписок с соглашением Enterprise параметр **Добавить зарезервированные экземпляры** следует включить на [портале EA](https://ea.azure.com). Или, если этот параметр отключен, необходимо быть администратором подписки EA. Зарезервированная емкость.

Дополнительные сведения о том, как корпоративные клиенты и клиенты с оплатой по мере использования оплачиваются по покупкам, см. в статье [Использование резервирования Azure для регистрации на предприятии](../../cost-management-billing/reservations/understand-reserved-instance-usage-ea.md) и [сведения об использовании резервирования Azure для подписки с оплатой по мере использования](../../cost-management-billing/reservations/understand-reserved-instance-usage.md).

> [!NOTE]
> Зарезервированная емкость приобретения не позволяет предварительно распределить или зарезервировать определенные ресурсы инфраструктуры (виртуальные машины или узлы).

## <a name="determine-correct-size-before-purchase"></a>Определение правильного размера перед покупкой

Размер резервирования должен основываться на общем объеме вычислений, используемом существующей или ближайшей к развертыванию базой данных или управляемым экземпляром в определенном регионе с использованием одного и того же уровня производительности и поколения оборудования.

Например, предположим, что вы используете одно общее назначение, го поколения – 16 Виртуальное ядро эластичный пул и два критически важных для бизнеса го поколения – 4 Виртуальное ядро отдельных баз данных. Кроме того, предположим, что вы планируете развертывание в течение следующего месяца: дополнительный пул эластичных баз данных общего назначения го поколения – 16 Виртуальное ядро и один критически важный для предприятия го поколения – 32 Виртуальное ядро эластичный пул. При этом вы знаете, что эти ресурсы вам будут необходимы по крайней мере 1 год. В этом случае вы должны приобрести 32 (2x16) виртуальных ядер 1-year резервирования для отдельной базы данных или пула эластичных БД общего назначения — го поколения и 40 (2x4 + 32) Виртуальное ядро 1-летний резервирование для отдельной базы данных или эластичного пула, критического для предприятия-го поколения.

## <a name="buy-reserved-capacity"></a>Приобретение зарезервированной емкости

1. Войдите на [портал Azure](https://portal.azure.com).
2. Выберите **Все службы** > **Резервирование**.
3. Нажмите кнопку **Добавить** , а затем в области **Покупка резервирования** выберите **база данных SQL** , чтобы приобрести новое резервирование для базы данных SQL.
4. Заполните обязательные поля. Существующие базы данных SQL и Управляемый экземпляр SQL, соответствующие выбранным атрибутам, будут получать зарезервированную скидку емкости. Фактическое число баз данных или управляемых экземпляров, которые получают скидку, зависит от выбранной области и количества.

## <a name="cancel-exchange-or-refund-reservations"></a>Отмена, обмен резервирования, возмещение средств за резервирование

Вы можете отменить и обменять резервирования, а также вернуть вложенные в резервирование средства, но при этом применяются определенные ограничения. ## <a name="vcore-size-flexibility"></a>Гибкость размеров для виртуального ядра

Гибкость размеров виртуального ядра позволяет увеличивать или уменьшать масштаб в пределах уровня производительности и региона, не теряя преимуществ приобретенной резервной мощности. Зарезервированная емкость также обеспечивает возможность временного перемещения активных баз данных в эластичные пулы и из них (в пределах одного региона и уровня производительности) в рамках обычных операций без потери преимуществ зарезервированной емкости. Сохраняя неприменяемый буфер в резервировании, вы можете эффективно управлять пиковыми нагрузками без превышения бюджета.

## <a name="limitation"></a>Ограничение

В базе данных SQL нельзя резервировать базы данных на основе DTU (Basic, Standard или Premium).
