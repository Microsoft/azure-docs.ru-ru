# <a name="quickstart-set-up-azure-attestation-with-azure-portal"></a>Краткое руководство. Настройка Аттестации Azure с помощью портала Azure

## <a name="prerequisites"></a>Предварительные требования

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

Ниже описаны шаги, которые позволяют управлять поставщиком аттестации с помощью портала Azure.

## <a name="1-attestation-provider"></a>1. Поставщик аттестации

### <a name="11-create-an-attestation-provider"></a>1.1. Создание поставщика аттестации

#### <a name="111-to-configure-the-provider-with-unsigned-policies"></a>1.1.1 Настройка неподписанных политик для поставщика

1.  На домашней странице или в меню портала Azure выберите **Создать ресурс**.
2.  В поле поиска введите **аттестация**.
3.  В списке результатов выберите **Аттестация Microsoft Azure**.
4.  На странице "Аттестация Microsoft Azure" щелкните **Создать**.
5.  На странице "Создание поставщика аттестации" укажите следующее.
    
    **Подписка**: Выберите подписку.
    
    **Группа ресурсов.** Выберите существующую группу ресурсов или щелкните **Создать** и введите имя группы ресурсов.
    
    **Name** (Имя). Укажите уникальное имя.

    **Расположение.** Выберите удобное расположение. 
    
    **Файл сертификата для подписывания политик.** Чтобы настроить неподписанные политики для поставщика, файл сертификата для подписывания политик отправлять не нужно. 
6.  Указав необходимые сведения, щелкните **Проверка и создание**.
7.  Устраните проблемы, если они будут обнаружены при проверке, и щелкните **Создать**.

#### <a name="112-to-configure-the-provider-with-signed-policies"></a>1.1.2 Настройка подписанных политик для поставщика

1.  На домашней странице или в меню портала Azure выберите **Создать ресурс**.
2.  В поле поиска введите **аттестация**.
3.  В списке результатов выберите **Аттестация Microsoft Azure**.
4.  На странице "Аттестация Microsoft Azure" щелкните **Создать**.
5.  На странице "Создание поставщика аттестации" укажите следующее.
    
    а. **Подписка**: Выберите подписку.
    
    b. **Группа ресурсов.** Выберите существующую группу ресурсов или щелкните **Создать** и введите имя группы ресурсов.
    
    c. **Name** (Имя). Укажите уникальное имя.

    d. **Расположение.** Выберите удобное расположение. 
    
    д) **Файл сертификата для подписывания политик.** Чтобы настроить сертификаты подписи политик для поставщика, отправьте файл сертификатов. Примеры вы найдете [здесь](./policy-signer-examples.md). 
6.  Указав необходимые сведения, щелкните **Проверка и создание**.
7.  Устраните проблемы, если они будут обнаружены при проверке, и щелкните **Создать**.

### <a name="12-view-attestation-provider"></a>1.2 Просмотр поставщика аттестации

1.  На домашней странице или в меню портала Azure выберите **Все ресурсы**.
2.  В поле фильтра введите имя поставщика аттестации и выберите его.

### <a name="13-delete-attestation-provider"></a>1.3 Удаление поставщика аттестации

1.  На домашней странице или в меню портала Azure выберите **Все ресурсы**.
2.  В поле фильтра введите имя поставщика аттестации.
3.  Установите флажок и щелкните **Удалить**.
4.  Введите "Да" и щелкните **Удалить**. [ИЛИ]
1.  На домашней странице или в меню портала Azure выберите **Все ресурсы**.
2.  В поле фильтра введите имя поставщика аттестации.
3.  Выберите поставщик аттестации и перейдите на страницу обзорных сведений.
4.  Щелкните **Удалить** в меню вверху и выберите **Да**.


## <a name="2-attestation-policy-signers"></a>2. Подписывание политик аттестации

### <a name="21-view-policy-signer-certificates"></a>2.1 Просмотр сертификатов для подписывания политик

1.  На домашней странице или в меню портала Azure выберите **Все ресурсы**.
2.  В поле фильтра введите имя поставщика аттестации.
3.  Выберите поставщик аттестации и перейдите на страницу обзорных сведений.
4.  Щелкните **Сертификаты для подписывания политик** в меню ресурсов слева или в области внизу.
5.  Щелкните **Скачать сертификаты для подписывания политик** (эта кнопка отключена для тех поставщиков аттестации, которые созданы без требования подписывать политики).
6.  Скачиваемый текстовый файл будет содержать все сертификаты в формате JWS.
а.  Проверьте количество и содержимое скачанных сертификатов.

### <a name="22-add-policy-signer-certificate"></a>2.2 Добавление сертификата для подписывания политик

1.  На домашней странице или в меню портала Azure выберите **Все ресурсы**.
2.  В поле фильтра введите имя поставщика аттестации.
3.  Выберите поставщик аттестации и перейдите на страницу обзорных сведений.
4.  Щелкните **Сертификаты для подписывания политик** в меню ресурсов слева или в области внизу.
5.  Щелкните **Добавить** в меню сверху (эта кнопка отключена для тех поставщиков аттестации, которые созданы без требования подписывать политики).
6.  Отправьте файл сертификата для подписывания политик и щелкните **Добавить**. Примеры вы найдете [здесь](./policy-signer-examples.md).

### <a name="23-delete-policy-signer-certificate"></a>2.3 Удаление сертификата для подписывания политик

1.  На домашней странице или в меню портала Azure выберите **Все ресурсы**.
2.  В поле фильтра введите имя поставщика аттестации.
3.  Выберите поставщик аттестации и перейдите на страницу обзорных сведений.
4.  Щелкните **Сертификаты для подписывания политик** в меню ресурсов слева или в области внизу.
5.  Щелкните **Удалить** в меню сверху (эта кнопка отключена для тех поставщиков аттестации, которые созданы без требования подписывать политики).
6.  Отправьте файл сертификата для подписывания политик и щелкните **Удалить**. Примеры вы найдете [здесь](./policy-signer-examples.md). 

## <a name="3-attestation-policy"></a>3. Политика аттестации

### <a name="31-view-attestation-policy"></a>3.1 Просмотр политики аттестации

1.  На домашней странице или в меню портала Azure выберите **Все ресурсы**.
2.  В поле фильтра введите имя поставщика аттестации.
3.  Выберите поставщик аттестации и перейдите на страницу обзорных сведений.
4.  Щелкните **Политика** в меню ресурсов слева или в области внизу.
5.  Выберите **тип аттестации** и просмотрите поле **Текущая политика**.

### <a name="32-configure-attestation-policy"></a>3.2 Настройка политики аттестации

#### <a name="321-when-attestation-provider-is-created-without-policy-signing-requirement"></a>3.2.1 Создание поставщика аттестации без требования подписывать политики

##### <a name="upload-policy-in-jwt-format"></a>Отправка политики в формате JWT

1.  На домашней странице или в меню портала Azure выберите **Все ресурсы**.
2.  В поле фильтра введите имя поставщика аттестации.
3.  Выберите поставщик аттестации и перейдите на страницу обзорных сведений.
4.  Щелкните **Политика** в меню ресурсов слева или в области внизу.
5.  В меню вверху выберите **Настроить**.
6.  Если поставщик аттестации создается без требования подписывать политики, пользователь может передать политику в форматах **JWT** или **простого текста**.
7.  В поле **Формат политики** выберите **JWT**.
8.  Отправьте файл политики с содержимым политики в формате **неподписанного или подписанного JWT** и щелкните **Сохранить**. Примеры вы найдете [здесь](./policy-examples.md).
    
    Если выбрана отправка файла, содержимое политики отобразится в текстовом формате для предварительного просмотра без возможности редактирования.

7.  Щелкните **Обновить** в меню вверху, чтобы просмотреть настроенную политику.

##### <a name="upload-policy-in-text-format"></a>Отправка политики в текстовом формате

1.  На домашней странице или в меню портала Azure выберите **Все ресурсы**.
2.  В поле фильтра введите имя поставщика аттестации.
3.  Выберите поставщик аттестации и перейдите на страницу обзорных сведений.
4.  Щелкните **Политика** в меню ресурсов слева или в области внизу.
5.  В меню вверху выберите **Настроить**.
6.  Если поставщик аттестации создается без требования подписывать политики, пользователь может передать политику в форматах **JWT** или **простого текста**.
7.  В поле **Формат политики** выберите **Текст**.
8.  Отправьте файл политики с содержимым в **текстовом формате** или вставьте содержимое политики прямо в текстовое поле и щелкните **Сохранить**. Примеры вы найдете [здесь](./policy-examples.md).

    Если выбрана отправка файла, содержимое политики отобразится в текстовом формате для предварительного просмотра без возможности редактирования.

8.  Щелкните **Обновить**, чтобы просмотреть настроенную политику.

#### <a name="322-when-attestation-provider-is-created-with-policy-signing-requirement"></a>3.2.2 Создание поставщика аттестации с требованием подписывать политики

##### <a name="upload-policy-in-jwt-format"></a>Отправка политики в формате JWT

1.  На домашней странице или в меню портала Azure выберите **Все ресурсы**.
2.  В поле фильтра введите имя поставщика аттестации.
3.  Выберите поставщик аттестации и перейдите на страницу обзорных сведений.
4.  Щелкните **Политика** в меню ресурсов слева или в области внизу.
5.  В меню вверху выберите **Настроить**.
6.  Если поставщик аттестации создается с требованием подписывать политики, пользователь может передать политику только в формате **подписанного маркера JWT**.
7.  Отправьте файл политики в **формате подписанного маркера JWT** и щелкните **Сохранить**. Примеры вы найдете [здесь](./policy-examples.md).

    Если выбрана отправка файла, содержимое политики отобразится в текстовом формате для предварительного просмотра без возможности редактирования.
    
8.  Щелкните **Обновить**, чтобы просмотреть настроенную политику.
