# <a name="how-to-use-postman-to-send-requests-to-the-azure-digital-twins-apis"></a>Как использовать POST для отправки запросов в интерфейсы API цифровых двойников Azure

[Postman](https://www.getpostman.com/) — это средство тестирования REST, предоставляющее основные возможности HTTP-запросов в пользовательском интерфейсе для классических программ и на основе подключаемых модулей. Его можно использовать для создания HTTP-запросов и их отправки в [интерфейсы API двойников для Azure Digital](how-to-use-apis-sdks.md).

В этой статье описывается, как настроить [клиент отправки RESTful](https://www.getpostman.com/) для взаимодействия с API-интерфейсами Azure Digital двойников, выполнив следующие действия.

1. Используйте Azure CLI, чтобы [**получить токен носителя**](#get-bearer-token) , который будет использоваться для выполнения запросов API в POST.
1. Настройте [**коллекцию POST**](#about-postman-collections) и настройте клиент Posting Client на использование токена носителя для проверки подлинности. При настройке коллекции можно выбрать один из следующих вариантов.
    1. [**Импортируйте**](#import-collection-of-azure-digital-twins-apis) предварительно созданную коллекцию запросов API цифровых двойников Azure.
    1. [**Создайте**](#create-your-own-collection) собственную коллекцию с нуля.
1. [**Добавьте запросы**](#add-an-individual-request) в настроенную коллекцию и отправьте их в интерфейсы API цифровых двойников Azure.

В Azure Digital двойников есть два набора интерфейсов API, с которыми вы можете работать: **плоскость данных** и **плоскость управления**. Дополнительные сведения о различиях между этими наборами API см. в разделе Практические руководства. [*Использование интерфейсов API и пакетов SDK для цифровых двойников Azure*](how-to-use-apis-sdks.md). Эта статья содержит сведения для обоих наборов API.

## <a name="get-bearer-token"></a>Получение токена носителя

Теперь, когда вы настроили POST и свой экземпляр Digital двойников для Azure, вам потребуется получить маркер носителя, который направляет запросы, которые можно использовать для авторизации в Azure Digital двойников API.

Получить этот маркер можно несколькими способами. В этой статье используется [Azure CLI](/cli/azure/install-azure-cli) для входа в учетную запись Azure и получения маркера.

Если Azure CLI [установлен локально](/cli/azure/install-azure-cli), можно запустить командную строку на компьютере, чтобы выполнить следующие команды.
В противном случае можно открыть [Azure Cloud Shell](https://shell.azure.com) окно в браузере и выполнить в нем команды.

1. Сначала убедитесь, что вы выполнили вход в Azure с соответствующими учетными данными, выполнив следующую команду:

    ```azurecli-interactive
    az login
    ```

2. Затем используйте команду [AZ Account Get-Access-Token](/cli/azure/account#az_account_get_access_token) , чтобы получить токен носителя с доступом к службе Digital двойников. В этой команде вы передадите идентификатор ресурса для конечной точки службы цифровых двойников Azure, чтобы получить маркер доступа, который может получить доступ к ресурсам Digital двойников. 

    Требуемый контекст для маркера зависит от того, какой набор API вы используете, поэтому используйте вкладки ниже для выбора между [плоскостью данных](how-to-use-apis-sdks.md#overview-data-plane-apis) и API-интерфейсами [плоскости управления](how-to-use-apis-sdks.md#overview-control-plane-apis) .

    # <a name="data-plane"></a>[Плоскость данных](#tab/data-plane)
    
    Чтобы получить маркер для использования с API-интерфейсами **плоскости данных** , используйте следующее статическое значение для контекста маркера: `0b07f429-9f4b-4714-9392-cc5e8e80c8b0` . Это идентификатор ресурса для конечной точки службы цифровых двойников Azure.
    
    ```azurecli-interactive
    az account get-access-token --resource 0b07f429-9f4b-4714-9392-cc5e8e80c8b0
    ```
    
    # <a name="control-plane"></a>[Уровень управления](#tab/control-plane)
    
    Чтобы получить маркер для использования с API-интерфейсами **плоскости управления** , используйте следующее значение для контекста маркера: `https://management.azure.com/` .
    
    ```azurecli-interactive
    az account get-access-token --resource https://management.azure.com/
    ```
    ---


3. Скопируйте значение `accessToken` в результат и сохраните его для использования в следующем разделе. Это **значение токена** , которое вы будете предоставлять для авторизации ваших запросов.

    :::image type="content" source="media/how-to-use-postman/console-access-token.png" alt-text="Снимок экрана консоли, показывающей результат выполнения команды AZ Account Get-Access-Token. Поле accessToken и его примерное значение выделены.":::

>[!TIP]
>Этот токен действителен не менее пяти минут и не может превышать 60 минут. Если вы исчерпали время, выделенное для текущего маркера, можно повторить шаги, описанные в этом разделе, чтобы получить новый.

Далее вы настроите POST, чтобы использовать этот токен для выполнения запросов API к Azure Digital двойников.

## <a name="about-postman-collections"></a>О коллекциях POST

Запросы в POST-данных сохраняются в **коллекциях** (группы запросов). При создании коллекции для группирования запросов можно одновременно применить общие параметры к нескольким запросам. Это может значительно упростить авторизацию, если вы планируете создать несколько запросов к API цифровых двойников Azure, так как эти сведения необходимо настроить только один раз для всей коллекции.

При работе с Azure Digital двойников можно начать с импорта [предварительно созданной коллекции всех запросов на цифровой двойников Azure](#import-collection-of-azure-digital-twins-apis). Это может потребоваться, если вы изучаете API и хотите быстро настроить проект с примерами запросов.

Кроме того, можно также начать с нуля, [создав собственную пустую коллекцию](#create-your-own-collection) и заполняя ее отдельными запросами, которые вызывают только необходимые API. 

В следующих разделах описываются оба этих процесса. Остальная часть статьи выполняется в вашем локальном приложении, поэтому вы можете открыть приложение после этого на компьютере.

## <a name="import-collection-of-azure-digital-twins-apis"></a>Импорт коллекции API-интерфейсов цифрового двойников Azure

Чтобы быстро приступить к работе с Azure Digital двойников, выполните импорт предварительно созданной коллекции запросов для интерфейсов API цифровых двойников Azure.

### <a name="download-the-collection-file"></a>Скачивание файла коллекции

Первым шагом при импорте набора API является скачивание коллекции. На вкладке ниже выберите плоскость или плоскость управления данными, чтобы просмотреть предварительно собранные параметры коллекции.

# <a name="data-plane"></a>[Плоскость данных](#tab/data-plane)

Сейчас доступны две коллекции плоскости данных Azure Digital двойников, которые можно выбрать.
* [**Коллекция почтовых служб Azure Digital двойников**](https://github.com/microsoft/azure-digital-twins-postman-samples). Эта коллекция предоставляет простой интерфейс для начала работы с Azure Digital двойников в публикации. В число запросов входят образцы данных, поэтому их можно запускать с минимальными необходимыми изменениями. Выберите эту коллекцию, если требуется удобную набор запросов API, содержащий образец информации.
    - Чтобы найти коллекцию, перейдите к ссылке на репозиторий и откройте файл с именем *postman_collection.jsв*.
* **[Swagger Azure Digital двойников](https://github.com/Azure/azure-rest-api-specs/tree/master/specification/digitaltwins/data-plane/Microsoft.DigitalTwins)**. в этом репозитории содержится полный файл Swagger для набора API Digital двойников для Azure, который можно скачать и импортировать в качестве коллекции публикации. Это обеспечит полный набор всех запросов API, но с пустыми телами данных, а не образцами данных. Выберите эту коллекцию, если требуется получить доступ к каждому вызову API и заполнить все данные самостоятельно.
    - Чтобы найти коллекцию, перейдите к ссылке на репозиторий и выберите папку для последней версии спецификации. Отсюда откройте файл с именем *digitaltwins.jsв*.

# <a name="control-plane"></a>[Уровень управления](#tab/control-plane)

Коллекция, доступная в настоящее время для плоскости управления, — это [**Swagger в плоскости управления цифровыми двойников Azure**](https://github.com/Azure/azure-rest-api-specs/tree/master/specification/digitaltwins/data-plane/Microsoft.DigitalTwins). Этот репозиторий содержит полный файл Swagger для набора API Azure Digital двойников, который можно скачать и импортировать в качестве коллекции. Это обеспечит полный набор всех запросов API.

Чтобы найти коллекцию, перейдите к ссылке на репозиторий и выберите папку для последней версии спецификации. Отсюда откройте файл с именем *digitaltwins.jsв*.

### <a name="additional-configuration"></a>Дополнительная настройка

# <a name="data-plane"></a>[Плоскость данных](#tab/data-plane)

Если вы создаете коллекцию [плоскости данных](how-to-use-apis-sdks.md#overview-data-plane-apis) , помогите коллекции легко подключаться к ресурсам Azure Digital двойников, настроив некоторые **переменные** , предоставляемые коллекциям. Если для большого числа запросов в коллекции требуется одинаковое значение (например, имя узла для своего экземпляра двойников Azure Digital), можно сохранить значение в переменной, которая применяется к каждому запросу в коллекции. 1. Используйте **имя узла** экземпляра из раздела [*предварительных требований*](#prerequisites) , чтобы установить поле текущего значения соответствующей переменной. Щелкните **Сохранить**.

    :::image type="content" source="media/how-to-use-postman/postman-variables-imported.png" alt-text="Снимок экрана: диалоговое окно редактирования импортированной коллекции в POST, показывающее вкладку &quot;переменные&quot;. Поле &quot;текущее значение&quot; выделено." lightbox="media/how-to-use-postman/postman-variables-imported.png":::

1. Если коллекция содержит дополнительные переменные, заполните и сохраните эти значения.

### <a name="explore-requests"></a>Просмотр запросов

Затем изучите запросы в коллекции API цифровых двойников Azure. Можно развернуть коллекцию, чтобы просмотреть предварительно созданные запросы (отсортированные по категории операций). 

Для разных запросов требуются разные сведения об экземпляре и его данных. Чтобы просмотреть все сведения, необходимые для составления конкретного запроса, найдите сведения о запросе в [справочной документации по цифровому двойников REST API](/rest/api/azure-digitaltwins/).

Вы можете изменить сведения о запросе в коллекции POST, выполнив следующие действия:

1. Выберите его из списка, чтобы получить редактируемые сведения. 

1. Заполните значения переменных, перечисленных на вкладке **params** ( **переменные пути**).

    :::image type="content" source="media/how-to-use-postman/postman-request-details-imported.png" alt-text="Снимок экрана процедуры POST. Коллекция развернута для отображения запроса. Раздел &quot;переменные пути&quot; выделен в сведениях о запросе." lightbox="media/how-to-use-postman/postman-request-details-imported.png":::

1. Укажите необходимые **заголовки** или сведения о **тексте** на соответствующих вкладках.

После получения всех необходимых сведений можно выполнить запрос с помощью кнопки **Отправить** .

Вы также можете добавить собственные запросы в коллекцию, используя процесс, описанный в разделе [*Добавление отдельного запроса*](#add-an-individual-request) ниже.

### <a name="create-a-postman-collection"></a>Создание коллекции Postman

    Выберите тип **коллекции**.

    :::image type="content" source="media/how-to-use-postman/postman-new-collection-2.png" alt-text="Снимок экрана: диалоговое окно &quot;Создание нового&quot; в POST. Параметр &quot;Collection&quot; выделен.":::

1. Откроется вкладка для заполнения сведений о новой коллекции. Щелкните значок редактирования рядом с именем по умолчанию для коллекции (**Новая коллекция**), чтобы заменить его на свой выбор. 

    :::image type="content" source="media/how-to-use-postman/postman-new-collection-3.png" alt-text="Снимок экрана: диалоговое окно редактирования новой коллекции в POST. Выделен значок редактирования рядом с именем &quot;Новая коллекция&quot;." lightbox="media/how-to-use-postman/postman-new-collection-3.png":::

Затем перейдите к следующему разделу, чтобы добавить в коллекцию токен носителя для авторизации.

### <a name="configure-authorization"></a>Настройка авторизации

Выполните следующие действия, чтобы добавить в коллекцию токен носителя для авторизации. Здесь вы будете использовать **значение токена** , собранного в разделе " [получение токена носителя](#get-bearer-token) ", чтобы использовать его для всех запросов API в коллекции.

1. В диалоговом окне редактирования для новой коллекции перейдите на вкладку **авторизация** .

    :::image type="content" source="media/how-to-use-postman/postman-authorization-custom.png" alt-text="Снимок экрана: диалоговое окно редактирования новой коллекции в POST, отображающее вкладку &quot;Авторизация&quot;." lightbox="media/how-to-use-postman/postman-authorization-custom.png":::

1. Задайте тип **OAuth 2,0**, вставьте маркер доступа в поле маркер доступа и нажмите кнопку **сохранить**.

    :::image type="content" source="media/how-to-use-postman/postman-paste-token-custom.png" alt-text="Снимок экрана с диалоговым окном изменения после правки для новой коллекции на вкладке &quot;Авторизация&quot;. Введите &quot;OAuth 2,0&quot;, и поле &quot;маркер доступа&quot; выделено." lightbox="media/how-to-use-postman/postman-paste-token-custom.png":::

После выполнения описанных выше действий вы закончите настройку коллекции. При необходимости можно закрыть вкладку Правка для новой коллекции.

Новую коллекцию можно увидеть на вкладке коллекции коллекций в главном представлении публикации.

### <a name="set-request-details"></a>Задание сведений о запросе

Чтобы отправить запрос POST к одному из интерфейсов API цифровых двойников Azure, вам потребуется URL-адрес API и сведения о необходимых сведениях. Эти сведения можно найти в [справочной документации по цифровым двойников REST API Azure](/rest/api/azure-digitaltwins/).

Чтобы продолжить работу с примером запроса, в этой статье будет использоваться API запросов (и его [Справочная документация](/rest/api/digital-twins/dataplane/query/querytwins)) для запроса всех цифровых двойников в экземпляре.

1. Получите URL-адрес запроса и тип из справочной документации. Для API запросов это *выполняется в настоящий `https://digitaltwins-hostname/query?api-version=2020-10-31`* момент.
1. В окне Post настройте тип запроса и введите URL-адрес запроса, заполняя в URL-адресе заполнители в соответствии с требованиями. Здесь вы будете использовать **имя узла** экземпляра из раздела [*предварительных требований*](#prerequisites) .
    
   :::image type="content" source="media/how-to-use-postman/postman-request-url.png" alt-text="Снимок экрана с подробными сведениями о новом запросе в POST. URL-адрес запроса из справочной документации был заполнен в поле URL-адрес запроса." lightbox="media/how-to-use-postman/postman-request-url.png":::
    
1. Убедитесь, что параметры, показанные для запроса на вкладке **params** , соответствуют описанным в справочной документации. Для этого запроса в Post `api-version` параметр был автоматически заполнен при вводе URL-адреса запроса на предыдущем шаге. Для API запросов это единственный обязательный параметр, поэтому этот шаг выполняется.
1. На вкладке **авторизация** задайте тип **наследовать проверку подлинности от родительского**. Это означает, что в этом запросе будет использоваться авторизация, настроенная ранее для всей коллекции.
1. Убедитесь, что заголовки, отображаемые для запроса на вкладке **заголовки** , соответствуют описанным в справочной документации. Для этого запроса несколько заголовков были заполнены автоматически. Для API запроса ни один из параметров заголовка не является обязательным, поэтому этот шаг выполняется.
1. Убедитесь, что текст, отображаемый для запроса на вкладке **Body** , соответствует требованиям, описанным в справочной документации. Для API запроса текст JSON необходим для предоставления текста запроса. Ниже приведен пример тела запроса, который запрашивает все цифровые двойников в экземпляре:

   :::image type="content" source="media/how-to-use-postman/postman-request-body.png" alt-text="Снимок экрана с подробными сведениями о новом запросе на вкладке &quot;тело&quot; статьи POST. Он содержит необработанный текст JSON с запросом SELECT * FROM ДИГИТАЛТВИНС." lightbox="media/how-to-use-postman/postman-request-body.png":::

   Дополнительные сведения о создании запросов Azure Digital двойников см. в разделе [*инструкции. запрос к графу двойника*](how-to-query-graph.md).

1. Просмотрите справочную документацию по любым другим полям, которые могут потребоваться для вашего типа запроса. Для API запросов все требования теперь выполнены в запросе POST, поэтому этот шаг выполняется.
1. Используйте кнопку **Send (отправить** ), чтобы отправить завершенный запрос.
   :::image type="content" source="media/how-to-use-postman/postman-request-send.png" alt-text="Снимок экрана публикации, показывающий сведения о новом запросе. Кнопка Отправить выделена." lightbox="media/how-to-use-postman/postman-request-send.png":::

После отправки запроса сведения об ответе появятся в окне Posting под запросом. Можно просмотреть код состояния ответа и любой основной текст.

:::image type="content" source="media/how-to-use-postman/postman-request-response.png" alt-text="Снимок экрана отправленного запроса в POST. После сведений о запросе отображается ответ. Состояние — 200 OK, а в тексте — результаты запроса." lightbox="media/how-to-use-postman/postman-request-response.png":::

Вы также можете сравнить ответ с ожидаемыми данными ответа, указанными в справочной документации, чтобы проверить результат или получить дополнительные сведения о возникших ошибках.
