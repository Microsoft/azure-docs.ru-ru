# <a name="determine-user-connection-latency-in-windows-virtual-desktop"></a>Определение задержки подключения пользователей в виртуальном рабочем столе Windows

Виртуальный рабочий стол Windows является глобально доступным. Администраторы могут создавать виртуальные машины в любом требуемом регионе Azure. Задержка подключения будет зависеть от расположения пользователей и виртуальных машин. Службы виртуальных рабочих столов Windows будут непрерывно выдвигаться на новые географические регионы для повышения задержки.

[Средство оценки возможностей виртуальных рабочих столов Windows](https://azure.microsoft.com/services/virtual-desktop/assessment/) помогает определить лучшее расположение для оптимизации задержки виртуальных машин. Мы рекомендуем использовать средство каждые два – три месяца, чтобы убедиться, что оптимальное расположение не изменилось, так как виртуальный рабочий стол Windows разворачивается в новые области.

## <a name="interpreting-results-from-the-windows-virtual-desktop-experience-estimator-tool"></a>Анализ результатов средства оценки возможностей виртуальных рабочих столов Windows

В виртуальных рабочих столах Windows задержка до 150 мс не должна влиять на работу пользователя, не требующую отрисовки или видео. Задержка между 150 мс и 200 мс должна быть точной для обработки текста. Задержка выше 200 мс может повлиять на взаимодействие с пользователем. 

Кроме того, подключение к виртуальному рабочему столу Windows зависит от подключения к Интернету компьютера, на котором пользователь использует службу. Пользователи могут потерять входные задержки подключения или взаимодействия в одной из следующих ситуаций.

 - У пользователя нет стабильного локального подключения к Интернету, а задержка составляет более 200 мс.
 - Сеть перегружена или ограничена частотой.

Рекомендуется выбирать расположения виртуальных машин, которые максимально близки к пользователям. Например, если пользователь находится в Индии, но виртуальная машина находится в США, то задержка будет влиять на общий пользовательский интерфейс. 

## <a name="azure-front-door"></a>Azure Front Door

Виртуальный рабочий стол Windows использует [переднюю дверцу Azure](https://azure.microsoft.com/services/frontdoor/) , чтобы перенаправить подключение пользователя к ближайшему шлюзу виртуальных рабочих столов Windows на основе исходного IP-адреса. Виртуальный рабочий стол Windows всегда будет использовать шлюз виртуальных рабочих столов Windows, который выбирает клиент.
