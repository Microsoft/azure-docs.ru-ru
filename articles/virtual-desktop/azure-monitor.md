# <a name="use-azure-monitor-for-windows-virtual-desktop-to-monitor-your-deployment-preview"></a>Использование Azure Monitor для виртуальных рабочих столов Windows для мониторинга развертывания (Предварительная версия)

>[!IMPORTANT]
>Azure Monitor для виртуальных рабочих столов Windows в настоящее время находится в общедоступной предварительной версии. без соглашений об уровне обслуживания. Мы не рекомендуем использовать ее для выполнения производственных рабочих нагрузок. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Azure Monitor для виртуальных рабочих столов Windows (Предварительная версия) — это панель мониторинга, основанная на Azure Monitor книгах, которые помогают ИТ-специалистам понять свои среды виртуальных рабочих столов Windows. В этом разделе описывается настройка Azure Monitor для виртуальных рабочих столов Windows для мониторинга сред виртуальных рабочих столов Windows.

## <a name="requirements"></a>Требования

Прежде чем приступить к использованию Azure Monitor для виртуальных рабочих столов Windows, необходимо настроить следующие компоненты:

- Все наблюдаемые среды виртуальных рабочих столов Windows должны быть основаны на последнем выпуске виртуального рабочего стола Windows, совместимом с Azure Resource Manager.

- По крайней мере одна настроенная Log Analytics Рабочая область.

- Включите сбор данных для следующих элементов в рабочей области Log Analytics:
    - Все необходимые счетчики производительности
    - Все счетчики производительности или события, используемые в Azure Monitor для виртуальных рабочих столов Windows
    - Данные из средства диагностики для всех объектов в среде, мониторинг которых будет осуществляться.
    - Виртуальные машины (VM) в наблюдаемой среде.

Любой пользователь, отслеживающий Azure Monitor для виртуальных рабочих столов Windows для вашей среды, также должен иметь следующие разрешения на чтение:

- Доступ на чтение к группе ресурсов, в которой находятся ресурсы среды.

- Доступ для чтения к группам ресурсов, в которых находятся узлы сеансов окружения

>[!NOTE]
> Доступ только для чтения позволяет администраторам просматривать данные. Для управления ресурсами на портале виртуальных рабочих столов Windows необходимы разные разрешения.

## <a name="open-azure-monitor-for-windows-virtual-desktop"></a>Открытие Azure Monitor для виртуальных рабочих столов Windows

Вы можете открыть Azure Monitor для виртуальных рабочих столов Windows одним из следующих способов:

- Перейдите по адресу [AKA.MS/azmonwvdi](https://portal.azure.com/#blade/Microsoft_Azure_WVD/WvdManagerMenuBlade/workbooks).

- Найдите и выберите в портал Azure **виртуальный рабочий стол Windows** , а затем щелкните **аналитика**.

- Найдите и выберите **Azure Monitor** из портал Azure. Выберите **центр Insights** в разделе **аналитика** и в разделе **другой** выберите **виртуальный рабочий стол Windows** , чтобы открыть панель мониторинга на странице Azure Monitor.

После того, как вы открыли Azure Monitor для виртуальных рабочих столов Windows, выберите один или несколько флажков с меткой **Подписка**, **Группа ресурсов**, **пул узлов** и **диапазон времени** в зависимости от среды, которую требуется отслеживать.

>[!NOTE]
>В настоящее время виртуальный рабочий стол Windows поддерживает одновременное наблюдение за одной подпиской, группой ресурсов и пулом узлов. Если вы не можете найти среду, которую нужно отслеживать, см. [документацию по устранению неполадок](troubleshoot-azure-monitor.md) для получения известных запросов к функциям и проблем.

## <a name="set-up-with-the-configuration-workbook"></a>Настройка с помощью книги конфигурации

Если вы впервые открываете Azure Monitor для виртуальных рабочих столов Windows, вам потребуется настроить Azure Monitor для ресурсов виртуальных рабочих столов Windows. Чтобы настроить ресурсы, сделайте следующее:

1. Откройте книгу в портал Azure.
2. Выберите **открыть книгу конфигурации**.

Книга конфигурации настраивает среду мониторинга и позволяет проверить конфигурацию после завершения процесса установки. Важно проверить конфигурацию, если элементы на панели мониторинга отображаются неправильно или если группа продуктов публикует обновления, требующие дополнительных точек данных.

## <a name="host-pool-diagnostic-settings"></a>Параметры диагностики для пула узлов

Необходимо включить Azure Monitor параметры диагностики для всех объектов в среде виртуальных рабочих столов Windows, которые поддерживают эту функцию.

1. Откройте Azure Monitor для виртуального рабочего стола Windows по адресу [AKA.MS/azmonwvdi](https://portal.azure.com/#blade/Microsoft_Azure_WVD/WvdManagerMenuBlade/workbooks), а затем выберите **Книга конфигурации**.

2. Выберите среду для мониторинга в разделе **Подписка**, **Группа ресурсов** и **пул узлов**.

3. В разделе **параметры диагностики пула узлов** проверьте, включена ли система диагностики виртуальных рабочих столов Windows для пула узлов. В противном случае появится сообщение об ошибке "не найдена существующая конфигурация диагностики для выбранного пула узлов". 
   
   Должны быть включены следующие таблицы:

    - Контрольная точка
    - Error
    - Управление
    - Подключение
    - HostRegistration

    >[!NOTE]
    > Если сообщение об ошибке не отображается, шаг 4 не требуется.

4. Выберите **Настройка пула узлов**.

5. Выберите **Развернуть**.

6. обновление книги;

Дополнительные сведения о включении диагностики для всех объектов в среде виртуальных рабочих столов Windows или доступе к рабочей области Log Analytics см. в статье [Отправка диагностики виртуальных рабочих столов Windows в log Analytics](diagnostics-log-analytics.md).

## <a name="configure-log-analytics"></a>Настройка Log Analytics

Чтобы приступить к использованию Azure Monitor для виртуальных рабочих столов Windows, вам также потребуется по крайней мере одна Log Analyticsная Рабочая область для сбора данных из среды, которую планируется отслеживать, и предоставления их в книге. Если вы уже установили один из них, перейдите к [настройке счетчиков производительности](#set-up-performance-counters). Чтобы настроить новую рабочую область Log Analytics для подписки Azure, содержащей среду виртуальных рабочих столов Windows, см. статью [создание log Analytics рабочей области на портал Azure](../azure-monitor/logs/quick-create-workspace.md).

>[!NOTE]
>Для Log Analytics будет взиматься плата за использование стандартных хранилищ данных. Для начала рекомендуется выбрать модель с оплатой по мере использования и настроить ее при масштабировании развертывания, а также получить дополнительные данные. Дополнительные сведения см. в разделе [цены на Azure Monitor](https://azure.microsoft.com/pricing/details/monitor/).

### <a name="set-up-performance-counters"></a>Настройка счетчиков производительности

Необходимо включить определенные счетчики производительности для сбора в соответствующем интервале выборки в рабочей области Log Analytics. Эти счетчики производительности являются единственными счетчиками, необходимыми для мониторинга виртуальных рабочих столов Windows. Вы можете отключить все остальные, чтобы сэкономить затраты.

Если счетчики производительности уже включены и вы хотите удалить их, следуйте инструкциям в разделе [Настройка счетчиков](../azure-monitor/agents/data-sources-performance-counters.md) производительности для перенастройки счетчиков производительности. Хотя в статье описывается добавление счетчиков, их также можно удалить в том же расположении.

Если вы еще не настроили счетчики производительности, вот как настроить их для Azure Monitor для виртуальных рабочих столов Windows:

1. Перейдите по адресу [AKA.MS/azmonwvdi](https://portal.azure.com/#blade/Microsoft_Azure_WVD/WvdManagerMenuBlade/workbooks), а затем выберите в нижней части окна **книгу настройки** .

2. В разделе **конфигурация log Analytics** выберите рабочую область, настроенную для подписки.

3. В разделе **счетчики производительности рабочей области** отображается список счетчиков, необходимых для мониторинга. В правой части этого списка проверьте элементы в списке **отсутствующие счетчики** , чтобы включить счетчики, необходимые для начала наблюдения за рабочей областью.

4. Выберите **Настройка счетчиков производительности**.

5. Выберите **применить конфигурацию**.

6. Обновите книгу конфигурации и продолжайте настройку среды.

Можно также добавлять новые счетчики производительности после начальной настройки при каждом обновлении службы и требует новых средств мониторинга. Чтобы убедиться, что все необходимые счетчики включены, выберите их в списке **отсутствующие счетчики** .

>[!NOTE]
>Счетчики производительности входной задержки совместимы только с Windows 10 RS5 и более поздними версиями, Windows Server 2019 и более поздними версиями.

Дополнительные сведения о ручном добавлении счетчиков производительности, которые еще не включены в коллекцию, см. в разделе [Настройка счетчиков производительности](../azure-monitor/agents/data-sources-performance-counters.md).

### <a name="set-up-windows-events"></a>Настройка событий Windows

Затем необходимо включить определенные события Windows для сбора в Log Analytics рабочей области. События, описанные в этом разделе, являются единственными Azure Monitor для потребностей виртуальных рабочих столов Windows. Вы можете отключить все остальные, чтобы сэкономить затраты.

Чтобы настроить события Windows, выполните следующие действия.

1. Если вы уже включили события Windows и хотите удалить их, удалите ненужные события, прежде чем использовать книгу настройки, чтобы включить набор, необходимый для мониторинга.

2. Перейдите в Azure Monitor для виртуального рабочего стола Windows по адресу [AKA.MS/azmonwvdi](https://portal.azure.com/#blade/Microsoft_Azure_WVD/WvdManagerMenuBlade/workbooks), а затем выберите **Книга конфигурации** в нижней части окна.

3. В **конфигурации событий Windows** имеется список событий Windows, необходимых для мониторинга. В правой части этого списка находится список **отсутствующих событий** , где можно найти требуемые имена событий и типы событий, которые в настоящее время не включены для вашей рабочей области. Запишите каждое из этих имен для последующего использования.

4. Выберите пункт **открыть конфигурацию рабочих областей**.

5. Выберите **данные**.

6. Выберите **журналы событий Windows**.

7. Добавьте имена отсутствующих событий из шага 3 и требуемый тип события для каждого из них.

8. Обновите книгу конфигурации и продолжайте настройку среды.

После первоначальной настройки можно добавлять новые события Windows, если для обновления средства мониторинга требуется включить новые события. Чтобы убедиться, что все необходимые события включены, вернитесь к списку **отсутствующих событий** и включите все отсутствующие события, которые вы видите.

## <a name="install-the-log-analytics-agent-on-all-hosts"></a>Установка агента Log Analytics на всех узлах

Наконец, необходимо установить агент Log Analytics на всех узлах пула узлов, чтобы отправить данные с узлов в выбранную рабочую область.

Чтобы установить агент Log Analytics, выполните следующие действия.

1. Перейдите в Azure Monitor для виртуального рабочего стола Windows по адресу [AKA.MS/azmonwvdi](https://portal.azure.com/#blade/Microsoft_Azure_WVD/WvdManagerMenuBlade/workbooks), а затем выберите **Книга конфигурации** в нижней части окна.

2. Если Log Analytics не настроен для всех узлов в пуле узлов, в нижней части раздела конфигурации Log Analytics появится сообщение об ошибке "некоторые узлы в пуле узлов не отправляют данные в выбранную рабочую область Log Analytics". Выберите **Добавить узлы в рабочую область** , чтобы добавить выбранные узлы. Если вы не видите сообщение об ошибке, закройте здесь.

3. Обновите книгу конфигурации.

>[!NOTE]
>Для установки расширения Log Analytics необходимо запустить компьютер узла. Если автоматическое развертывание завершается сбоем на узле, вы всегда можете установить расширение на узле вручную. Сведения об установке расширения вручную см. в разделе [log Analytics расширение виртуальной машины для Windows](../virtual-machines/extensions/oms-windows.md).
