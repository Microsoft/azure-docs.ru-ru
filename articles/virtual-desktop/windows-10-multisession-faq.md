# <a name="windows-10-enterprise-multi-session-faq"></a>Windows 10 Корпоративная с поддержкой нескольких сеансов: часто задаваемые вопросы

В этой статье содержатся ответы на часто задаваемые вопросы и рекомендации по многосеансовой поддержке Windows 10 Enterprise.

## <a name="what-is-windows-10-enterprise-multi-session"></a>Что такое многосеансовая поддержка Windows 10 Корпоративная?

Многосеансовая поддержка Windows 10 Корпоративная, ранее известная как Windows 10 Корпоративная для виртуальных рабочих столов (ЕВД), представляет собой новый узел сеансов удаленный рабочий стол, позволяющий несколько параллельных интерактивных сеансов. Ранее это можно сделать только в Windows Server. Эта возможность дает пользователям привычный опыт работы с Windows 10, а также выгодные преимущества многосеансовой связи и использования существующих лицензий на пользователя Windows, а не клиентских лицензий RDS. Дополнительные сведения о лицензиях и ценах см. на странице [цены на виртуальные рабочие столы Windows](https://azure.microsoft.com/pricing/details/virtual-desktop/).

## <a name="how-many-users-can-simultaneously-have-an-interactive-session-on-windows-10-enterprise-multi-session"></a>Сколько пользователей может одновременно иметь интерактивный сеанс в многосеансовой среде Windows 10?

Количество интерактивных сеансов, которые могут быть активными одновременно, зависит от аппаратных ресурсов системы (виртуальных ЦП, память, диск и виртуальный графический процессор), того, как пользователи используют свои приложения при входе в сеанс и насколько сильно загружена Рабочая нагрузка системы. Мы рекомендуем проверить производительность системы, чтобы понять, сколько пользователей можно использовать в многосеансовой среде Windows 10 Корпоративная. Дополнительные сведения см. в разделе [цены на виртуальные рабочие столы Windows](https://azure.microsoft.com/pricing/details/virtual-desktop/).

## <a name="why-does-my-application-report-windows-10-enterprise-multi-session-as-a-server-operating-system"></a>Почему приложение сообщает о нескольких сеансах Windows 10 Корпоративная как серверная операционная система?

Windows 10 Корпоративная многосеансовая поддержка — это виртуальный выпуск Windows 10 Корпоративная. Одно из отличий заключается в том, что эта операционная система (ОС) сообщает [продукттипе](/windows/win32/cimwin32prov/win32-operatingsystem) как значение 3, то же самое значение, что и Windows Server. Это свойство обеспечивает совместимость операционной системы с существующими средствами управления для сеансов удаленных рабочих столов, приложениями с поддержкой многосеансовой поддержки, а также оптимизацией производительности низкоуровневых систем для сред узлов сеансов удаленных рабочих столов. Некоторые установщики приложений могут блокировать установку в многосеансовой системе Windows 10 в зависимости от того, установлено ли для Продукттипе значение Client. Если приложение не будет установлено, обратитесь к поставщику приложения за обновленной версией.

## <a name="can-i-run-windows-10-enterprise-multi-session-on-premises"></a>Можно ли запускать многосеансовый запуск Windows 10 Корпоративная?

Многосеансовый запуск Windows 10 Enterprise не может выполняться в локальных рабочих средах, поскольку он оптимизирован для службы виртуальных рабочих столов Windows для Azure. Это соглашение о лицензировании для запуска многосеансового подключения Windows 10 Enterprise за пределами Azure в производственных целях. Многосеансовый запуск Windows 10 Enterprise не будет активирован для локальных служб управления ключами (KMS).

## <a name="can-i-upgrade-a-windows-10-vm-to-windows-10-enterprise-multi-session"></a>Можно ли обновить виртуальную машину Windows 10 до Windows 10 Корпоративная с несколькими сеансами?

Нет. Сейчас невозможно обновить существующую виртуальную машину, работающую под управлением Windows 10 профессиональная или Enterprise, с многосеансовой поддержкой Windows 10 Enterprise. Кроме того, если вы развертываете многосеансовую виртуальную машину Windows 10 Корпоративная, а затем обновляете ключ продукта в другом выпуске, вы не сможете снова переключиться на ВИРТУАЛЬную машину в многосеансовую службу Windows 10, и вам потребуется повторно развернуть виртуальную машину.

## <a name="how-do-i-customize-the-windows-10-enterprise-multi-session-image-for-my-organization"></a>Разделы справки настроить многосеансовый образ Windows 10 Enterprise для моей организации?

Вы можете запустить виртуальную машину в Azure с многосеансовой поддержкой Windows 10 Enterprise и настроить ее, установив бизнес-приложения, sysprep/generalize, а затем создав образ с помощью портал Azure.

Чтобы приступить к работе, создайте виртуальную машину в Azure с многосеансовой поддержкой Windows 10 Enterprise. Вместо запуска виртуальной машины в Azure вы можете скачать VHD напрямую. После этого вы сможете использовать скачанный виртуальный жесткий диск для создания новой виртуальной машины поколения 1 на компьютере с Windows 10 с включенным Hyper-V.

Настройте образ в нужных целях, установив бизнес-приложения и Sysprep в образе. Завершив настройку, отправьте образ в Azure с помощью виртуального жесткого диска в. После этого получите виртуальный рабочий стол Windows из Azure Marketplace и используйте его для развертывания нового пула узлов с настроенным образом.

## <a name="how-do-i-manage-windows-10-enterprise-multi-session-after-deployment"></a>Разделы справки управлять многосеансовой поддержкой Windows 10 Enterprise после развертывания?

Можно использовать любое поддерживаемое средство настройки, но рекомендуется Configuration Manager версии 1906, так как она поддерживает многосеансовую поддержку Windows 10 Корпоративная. Сейчас мы работаем над поддержкой Microsoft Intune.

## <a name="can-windows-10-enterprise-multi-session-be-azure-active-directory-ad-joined"></a>Можно ли присоединить многосеансовую поддержку Windows 10 Enterprise Azure Active Directory (AD)?

В настоящее время многосеансовая поддержка Windows 10 Enterprise поддерживается для гибридного подключения к Azure AD. После присоединения нескольких сеансов Windows 10 Enterprise к домену используйте существующий объект групповая политика, чтобы включить регистрацию Azure AD. Дополнительные сведения см. в статье [Планирование реализации гибридного Azure Active Directory Join](../active-directory/devices/hybrid-azuread-join-plan.md).

## <a name="where-can-i-find-the-windows-10-enterprise-multi-session-image"></a>Где можно найти многосеансовый образ Windows 10 Enterprise?

Многосеансовая поддержка Windows 10 Enterprise находится в коллекции Azure. Чтобы найти его, перейдите к портал Azure и выполните поиск выпуска Windows 10 Корпоративная для виртуальных рабочих столов. Чтобы получить образ, интегрированный с Microsoft 365ными приложениями для предприятия, перейдите в портал Azure и выполните поиск **приложений Microsoft Windows 10 + Microsoft 365 для Enterprise**.

## <a name="which-windows-10-enterprise-multi-session-image-should-i-use"></a>Какой многосеансовый образ Windows 10 Enterprise следует использовать?

В галерее Azure имеется несколько выпусков, включая многосеансовую версию Windows 10 Корпоративная, версия 1809 и многосеансовую поддержку Windows 10 Корпоративная, версия 1903. Для повышения производительности и надежности рекомендуется использовать последнюю версию.

## <a name="which-windows-10-enterprise-multi-session-versions-are-supported"></a>Какие версии Windows 10 Enterprise поддерживают многосеансовую поддержку?

Поддерживается многосеансовая поддержка Windows 10 Enterprise, версии 1809 и более поздних версий и доступные в коллекции Azure. Эти выпуски соответствуют той же политике жизненного цикла поддержки, что и Windows 10 Корпоративная, а это означает, что выпуск за март поддерживается в течение 18 месяцев, а сентябрь выпуска составляет 30 месяцев.

## <a name="which-profile-management-solution-should-i-use-for-windows-10-enterprise-multi-session"></a>Какое решение по управлению профилям следует использовать в многосеансовой среде Windows 10?

Рекомендуется использовать контейнеры профилей Фслогикс при настройке Windows 10 Корпоративная в непостоянных средах или в других сценариях, требующих централизованного хранения профиля. Фслогикс гарантирует доступность и актуальность профиля пользователя для каждого сеанса пользователя. Мы также рекомендуем использовать контейнер профиля Фслогикс для хранения профиля пользователя в любой общей папке SMB с соответствующими разрешениями, но при необходимости можно хранить профили пользователей в хранилище BLOB-объектов Azure. Пользователи виртуальных рабочих столов Windows могут использовать Фслогикс без дополнительных затрат.  Фслогикс предварительно установлен во всех многосеансовых образах Windows 10 Enterprise, но ИТ-администратор все еще несет ответственность за настройку контейнера профиля Фслогикс.

Дополнительные сведения о настройке контейнера профиля Фслогикс см. в статье [Настройка контейнера профиля фслогикс](create-host-pools-user-profile.md#configure-the-fslogix-profile-container).

## <a name="why-do-my-apps-disappear-after-i-sign-out"></a>Почему приложения исчезают после выхода из программы?

Это происходит потому, что вы используете многосеансовую поддержку Windows 10 Enterprise с решением для управления профилями, например Фслогикс. Решение "Администратор" или "профиль" настроено для удаления профилей пользователей при выходе пользователей из системы. Такая конфигурация означает, что когда ваша система удаляет профиль пользователя после выхода из системы, она также удаляет все приложения, установленные во время сеанса. Если вы хотите, чтобы установленные приложения были установлены, необходимо попросить администратора предоставить эти приложения всем пользователям в среде виртуальных рабочих столов Windows.

## <a name="how-do-i-make-sure-apps-dont-disappear-when-users-sign-out"></a>Разделы справки убедиться, что приложения не исчезают при выходе пользователей из систему?

Большинство виртуализованных сред настроены по умолчанию, чтобы предотвратить установку пользователями дополнительных приложений в профили. Если вы хотите, чтобы приложение не исчезало после выхода пользователя из виртуальной системы Windows, необходимо предоставить это приложение для всех профилей пользователей в вашей среде. Дополнительные сведения о подготовке приложений см. в следующих ресурсах:

- [Публикация встроенных приложений в виртуальном рабочем столе Windows](publish-apps.md)
- [Параметры командной строки для обслуживания пакета приложения DISM](/windows-hardware/manufacture/desktop/dism-app-package--appx-or-appxbundle--servicing-command-line-options)
- [Add-Аппкспровисионедпаккаже](/powershell/module/dism/add-appxprovisionedpackage)

## <a name="how-do-i-make-sure-users-dont-download-and-install-apps-from-the-microsoft-store"></a>Разделы справки убедиться, что пользователи не загружают и не устанавливают приложения из Microsoft Store?

Вы можете отключить Microsoft Storeное приложение, чтобы пользователи не загружали дополнительные приложения за пределами уже подготовленных для них приложений.

Чтобы отключить приложение магазина, сделайте следующее:

1. Создайте новый групповая политика.
2. Выберите **Конфигурация компьютера**  >  **Административные шаблоны**  >  **компоненты Windows**.
3. Выберите **Магазин**.
4. Выберите **магазин приложение**.
5. Выберите **отключено** и нажмите кнопку **ОК**.
6. Нажмите кнопку **Применить**.
