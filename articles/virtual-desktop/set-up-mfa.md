# <a name="enable-azure-multifactor-authentication-for-windows-virtual-desktop"></a>Включение многофакторной проверки подлинности Azure для виртуальных рабочих столов Windows

>[!IMPORTANT]
> Если вы посещаете эту страницу в документации по виртуальному рабочему столу Windows (классическая модель), не забудьте [вернуться к документации по виртуальному рабочему столу Windows (классическая)](./virtual-desktop-fall-2019/tenant-setup-azure-active-directory.md) после завершения работы.

Клиент Windows для виртуальных рабочих столов для Windows — это отличный вариант для интеграции виртуальных рабочих столов Windows с локальным компьютером. Однако при настройке учетной записи виртуального рабочего стола Windows в клиенте Windows существуют определенные меры, которые необходимо предпринять, чтобы обеспечить безопасность своих пользователей.

При первом входе клиент запрашивает имя пользователя, пароль и многофакторную проверку подлинности Azure. После этого при следующем входе клиент запомнит ваш токен из приложения Azure Active Directory (AD) Enterprise. Когда вы установите флажок **Запомнить меня** в запросе учетных данных для узла сеансов, пользователи смогут войти в систему после перезапуска клиента без необходимости повторного ввода учетных данных.

Хотя запоминание учетных данных удобно, это также может сделать развертывание на корпоративных сценариях или персональных устройствах менее безопасным. Чтобы защитить пользователей, вы можете сделать так, чтобы клиент запросил учетные данные многофакторной проверки подлинности Azure чаще. В этой статье показано, как настроить политику условного доступа для виртуальных рабочих столов Windows, чтобы включить этот параметр.

## <a name="prerequisites"></a>Предварительные условия

Чтобы приступить к работе, необходимо выполнить следующие действия.

- Назначьте пользователям лицензию, включающую Azure Active Directory Premium P1 или P2.
- Группа Azure Active Directory с пользователями, назначенными членами группы.
- Включите многофакторную проверку подлинности Azure для всех пользователей. Дополнительные сведения о том, как это сделать, см. [в разделе как требовать двухфакторную проверку подлинности для пользователя](../active-directory/authentication/howto-mfa-userstates.md#view-the-status-for-a-user).

> [!NOTE]
> Следующий параметр также применяется к [веб-клиенту виртуальных рабочих столов Windows](https://rdweb.wvd.microsoft.com/arm/webclient/index.html).

## <a name="create-a-conditional-access-policy"></a>Создание политики условного доступа

Ниже показано, как создать политику условного доступа, требующую многофакторную проверку подлинности при подключении к виртуальному рабочему столу Windows:

1. Войдите на **портал Azure** с учетными данными глобального администратора, администратора безопасности или администратора условного доступа.
2. Выберите **Azure Active Directory** > **Безопасность** > **Условный доступ**.
3. Выберите **Новая политика**.
4. Присвойте политике имя. Мы рекомендуем организациям присваивать политикам понятные имена.
5. В разделе **Назначения** выберите **Пользователи и группы**.
6. В разделе **включить** выберите **пункт Выбор пользователей и групп**  >  **Пользователи и группы** > выберите группу, созданную на этапе [предварительных требований](#prerequisites) .
7. Нажмите кнопку **Готово**.
8. В разделе **облачные приложения или действия** укажите выберите  >   **приложения**.
9. Выберите одно из следующих приложений в зависимости от используемой версии виртуальных рабочих столов Windows.
   
   - Если вы используете Windows Virtual Desktop (классическая модель), выберите следующие приложения:
       
       - **Виртуальный рабочий стол Windows** (идентификатор приложения 5a0aa725-4958-4b0c-80a9-34562e23f3b7)
       - **Клиент виртуальных рабочих столов Windows** (идентификатор приложения fa4345a4-A730-4230-84a8-7d9651b86739), позволяющий задавать политики для веб-клиента.
       
        После этого перейдите к шагу 11.

   - Если вы используете виртуальный рабочий стол Windows, выберите это приложение:
       
       -  **Виртуальный рабочий стол Windows** (идентификатор приложения 9cdead84-a844-4324-93f2-b2e6bb768d07)
       
        После этого перейдите к шагу 10.

   >[!IMPORTANT]
   > Не выбирайте приложение под названием поставщик Azure Resource Manager Windows Virtual Desktop (50e95039-B200-4007-bc97-8d5790743a63). Это приложение используется только для получения веб-канала пользователя и не должно иметь многофакторную проверку подлинности.
   > 
   > Если вы используете Windows Virtual Desktop (классическая модель), если политика условного доступа блокирует весь доступ и исключает только идентификаторы приложений виртуальных рабочих столов Windows, это можно исправить, добавив идентификатор приложения 9cdead84-a844-4324-93f2-b2e6bb768d07 в политику. Если не добавить этот идентификатор приложения, обнаружение веб-канала будет блокироваться при обнаружении ресурсов виртуальных рабочих столов Windows (классическая модель).

10. Перейдите в раздел **условия**  >  **клиентские приложения**, а затем выберите, к которому нужно применить политику:
    
    - Выберите **браузер** , если хотите применить политику к веб-клиенту.
    - Выберите **мобильные приложения и настольные клиенты** , если вы хотите применить политику к другим клиентам.
    - Установите оба флажка, если хотите применить политику ко всем клиентам.
   
    > [!div class="mx-imgBorder"]
    > ![Снимок экрана страницы клиентских приложений. Пользователь выбрал флажки мобильные приложения и настольные клиенты.](media/select-apply.png)

11. Выбрав приложение, нажмите кнопку **выбрать**, а затем выберите **Готово**.

    > [!div class="mx-imgBorder"]
    > ![Снимок экрана со страницей "облачные приложения" или "действия". Клиентские приложения Windows для виртуальных рабочих столов и виртуальных рабочих столов Windows выделены красным цветом.](media/cloud-apps-enterprise.png)

    >[!NOTE]
    >Чтобы найти идентификатор приложения, который вы хотите выбрать, перейдите в раздел **корпоративные приложения** и в раскрывающемся меню Тип приложения выберите пункт **приложения Майкрософт** .

12. В разделе **Управление доступом**  >  **предоставление** выберите **предоставить доступ**, **требовать многофакторную проверку подлинности**, а затем **выберите**.
13. В разделе сеанс **управления доступом**  >  выберите **периодичность входа**, задайте значение времени между запросами, а затем нажмите **кнопку Выбрать**. Например, если установить значение **1** и единицу измерения в **часах**, будет требоваться многофакторная проверка подлинности, если подключение будет запущено через час после последнего.
14. Подтвердите параметры и задайте для параметра **Включить политику** значение **Включить**.
15. Выберите **создать** , чтобы включить политику.

>[!NOTE]
>При использовании веб-клиента для входа в виртуальный рабочий стол Windows через браузер в журнале будет отображаться идентификатор клиентского приложения как a85cf173-4192-42f8-81fa-777a763e6e2c (клиент виртуальных рабочих столов Windows). Это связано с тем, что клиентское приложение внутренне связано с ИДЕНТИФИКАТОРом серверного приложения, в котором была задана политика условного доступа.
