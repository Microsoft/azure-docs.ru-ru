# <a name="frequently-asked-questions-about-networking-in-azure-functions"></a>Часто задаваемые вопросы о сети в службе "функции Azure"

В этой статье перечислены часто задаваемые вопросы о сетевых подключениях в функциях Azure. Более подробные сведения см. в разделе [функции сетевые подключения](functions-networking-options.md).

## <a name="how-do-i-set-a-static-ip-in-functions"></a>Разделы справки задать статический IP-адрес в функциях?

Развертывание функции в Среда службы приложений является основным способом получения статических входящих и исходящих IP-адресов для ваших функций. Дополнительные сведения об использовании Среда службы приложений см. в статье [Создание и использование внутренней подсистемы балансировки нагрузки с среда службы приложений](../app-service/environment/create-ilb-ase.md).

Вы также можете использовать шлюз NAT виртуальной сети для маршрутизации исходящего трафика через общедоступный IP-адрес, который вы контролируете. Дополнительные сведения см. в разделе [учебник. Управление функциями Azure исходящий IP-адрес с помощью шлюза NAT виртуальной сети Azure](functions-how-to-use-nat-gateway.md). 

## <a name="how-do-i-restrict-internet-access-to-my-function"></a>Разделы справки ограничить доступ к моей функции через Интернет?

Ограничить доступ к Интернету можно несколькими способами.

* [Ограничения IP-адресов](../app-service/app-service-ip-restrictions.md): ограничьте входящий трафик к приложению-функции по диапазону IP.
    * В разделе ограничения IP-адресов можно также настроить [конечные точки службы](../virtual-network/virtual-network-service-endpoints-overview.md), которые ограничивают функцию принимать входящий трафик только от конкретной виртуальной сети.
* Удаление всех триггеров HTTP. Для некоторых приложений достаточно просто избежать триггеров HTTP и использовать любой другой источник событий для активации функции.

Помните, что редактору портал Azure требуется прямой доступ к выполняемой функции. Все изменения кода с помощью портал Azure потребует устройства, которое вы используете для просмотра портала, чтобы его IP-адрес был добавлен в список утвержденных. Но вы все равно можете использовать любые элементы на вкладке функции платформы с ограничениями сети.

## <a name="how-do-i-restrict-my-function-app-to-a-virtual-network"></a>Разделы справки ограничить мое приложение-функцию виртуальной сетью?

Вы можете ограничить **входящий** трафик для приложения-функции в виртуальной сети с помощью [конечных точек службы](./functions-networking-options.md#use-service-endpoints). Эта конфигурация позволяет приложению-функции выполнять исходящие вызовы к Интернету.

Чтобы полностью ограничить функцию таким образом, чтобы весь трафик проходил через виртуальную сеть, можно использовать [частные конечные точки](./functions-networking-options.md#private-endpoint-connections) с интеграцией исходящей виртуальной сети или среда службы приложений. Дополнительные сведения см. в статье [Интеграция функций Azure с виртуальной сетью Azure с помощью частных конечных точек](functions-create-vnet.md).

## <a name="how-can-i-access-resources-in-a-virtual-network-from-a-function-app"></a>Как получить доступ к ресурсам в виртуальной сети из приложения-функции?

Доступ к ресурсам в виртуальной сети можно получить из выполняемой функции с помощью интеграции с виртуальной сетью. Дополнительные сведения см. в статье [Интеграция виртуальной сети](functions-networking-options.md#virtual-network-integration).

## <a name="how-do-i-access-resources-protected-by-service-endpoints"></a>Разделы справки доступ к ресурсам, защищенным конечными точками службы?

С помощью интеграции с виртуальной сетью можно получить доступ к ресурсам, защищенным конечной точкой службы, из выполняемой функции. Дополнительные сведения см. в статье [Интеграция виртуальной сети](functions-networking-options.md#virtual-network-integration).

## <a name="how-can-i-trigger-a-function-from-a-resource-in-a-virtual-network"></a>Как активировать функцию из ресурса в виртуальной сети?

Вы можете разрешить вызывать триггеры HTTP из виртуальной сети с помощью [конечных точек службы](./functions-networking-options.md#use-service-endpoints) или [подключений к частным конечным точкам](./functions-networking-options.md#private-endpoint-connections). 

Вы также можете активировать функцию из всех других ресурсов в виртуальной сети, развернув приложение-функцию в плане Premium, в плане службы приложений или Среда службы приложений. Дополнительные сведения см. в разделе [триггеры виртуальной сети, не относящиеся к HTTP](./functions-networking-options.md#virtual-network-triggers-non-http) .

## <a name="how-can-i-deploy-my-function-app-in-a-virtual-network"></a>Как развернуть мое приложение-функцию в виртуальной сети?

Развертывание в Среда службы приложений является единственным способом создания приложения-функции, которое полностью находится внутри виртуальной сети. Дополнительные сведения об использовании внутренней подсистемы балансировки нагрузки с Среда службы приложений см. в статье [Создание и использование внутренней подсистемы балансировки нагрузки с среда службы приложений](../app-service/environment/create-ilb-ase.md).

Сценарии, в которых необходим только односторонний доступ к ресурсам виртуальной сети или менее полные сетевые изоляции, см. в статье [Общие сведения о функциях сети](functions-networking-options.md).
