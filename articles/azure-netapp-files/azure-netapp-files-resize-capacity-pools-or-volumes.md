# <a name="resize-a-capacity-pool-or-a-volume"></a>Изменение размера пула емкости или тома
При необходимости вы можете изменить размер пула емкости или тома. 

## <a name="resize-the-capacity-pool"></a>Изменение размера пула емкости 

Размер пула емкости можно изменить с шагом в 1 Тиб приращения или уменьшения. Однако размер пула емкости не может быть меньше 4 тиб. При изменении размера пула емкости изменяется емкость приобретенной службы Azure NetApp Files.

1. В колонке "Управление учетной записью NetApp" щелкните пул емкости, размер которого требуется изменить. 
2. Щелкните правой кнопкой мыши имя пула емкости или щелкните значок "…" в конце строки пула емкости, чтобы открыть контекстное меню. 
3. Используя параметры контекстного меню, измените размер пула емкости или удалите пул.

## <a name="resize-a-volume"></a>Изменение размера тома

При необходимости вы можете изменить размер тома. Потребление емкости тома зависит от подготовленной емкости пула.

1. В колонке "Управление учетной записью NetApp" щелкните **Тома**. 
2. Щелкните правой кнопкой мыши имя тома, размер которого требуется изменить, или щелкните значок "…"в конце строки тома, чтобы открыть контекстное меню.
3. Используя параметры контекстного меню, измените размер тома или удалите том.

## <a name="resize-a-cross-region-replication-destination-volume"></a>Изменение размера тома назначения репликации между регионами 

В отношении [репликации между регионами](cross-region-replication-introduction.md) целевой том автоматически изменяется в зависимости от размера исходного тома. Таким образом, не нужно изменять размер целевого тома отдельно. Автоматическое изменение размера применяется, если тома находятся в активном отношении репликации или если пиринг репликации разбивается с помощью операции повторной [синхронизации](cross-region-replication-manage-disaster-recovery.md#resync-replication). 

В следующей таблице описаны действия по изменению размера тома назначения в зависимости от [состояния зеркального отображения](cross-region-replication-display-health-status.md).

|  Состояние зеркального отображения  | Поведение при изменении размера целевого тома |
|-|-|
| *Зеркальный том* | Когда целевой том был инициализирован и готов к получению обновлений зеркального отображения, изменение размера исходного тома автоматически изменяет размер целевых томов. |
| *Рабочие* | При изменении размера исходного тома, если зеркальное состояние *разорвано*, целевой том автоматически изменяется с [операцией](cross-region-replication-manage-disaster-recovery.md#resync-replication)повторной синхронизации.  |
| *Не инициализировано* | При изменении размера исходного тома, когда состояние зеркального отображения по-прежнему не *инициализировано*, изменение размера целевого тома должно выполняться вручную. Поэтому рекомендуется дождаться завершения инициализации (т. е. когда зеркальное состояние становится *зеркальным*), чтобы изменить размер исходного тома. | 

> [!IMPORTANT]
> Убедитесь в наличии достаточного запаса в пулах ресурсов для исходного и целевого томов репликации между регионами. При изменении размера исходного тома целевой том автоматически изменяется. Но если пул емкости, в котором размещен целевой том, не имеет достаточного запаса, изменение размера исходного и целевого томов завершится ошибкой.
