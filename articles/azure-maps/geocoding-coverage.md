# <a name="azure-maps-geocoding-coverage"></a>Покрытие для геокодирования в службе "Карты Azure"

[Служба поиска](/rest/api/maps/search) Azure Maps поддерживает геокодирование, что означает, что ваш запрос API может содержать условия поиска, например адрес или имя места, и возвращает результат в виде координат широты и долготы. Например, [API Azure Maps получения адреса поиска](/rest/api/maps/search/getsearchaddress) получает запросы, содержащие сведения о расположении, и возвращает результаты в виде координат широты и долготы.

Однако [Служба поиска](/rest/api/maps/search) Azure Maps не имеет одинакового уровня информации и точности для всех регионов и стран. Из этой статьи вы узнаете, для каких расположения можно выполнять надежный поиск в каждом регионе.

Возможность геокодирования в стране или регионе зависит от объема данных о поездке и точности геокодирования службы геошифрования. Следующие категории используются для указания уровня поддержки геокодирования в каждой стране или регионе.

* **Адресные точки** — данные адреса можно разрешить в координаты широты и долготы в адресной упаковке (граница свойства). Адресные точки часто называются точными "Руфтоп", что является самым высоким уровнем точности, доступного для адресов.
* **Номера домов.** Адреса интерполируются в координаты широты и долготы на улице.
* **Уровень улицы.** Адреса приводятся к координатам широты и долготы улицы, на которой находится указанный адрес. Номера домов могут не учитываться.
* **Уровень города.** Поддерживаются названия городов.
