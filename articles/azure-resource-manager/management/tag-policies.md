# <a name="assign-policies-for-tag-compliance"></a>Назначение политик для соответствия тегов

Используйте [политику Azure](../../governance/policy/overview.md) для применения правил и соглашений тегов. Создание политики позволяет избежать необходимости развертывания ресурсов в подписке, которые не имеют ожидаемых тегов для вашей организации. Вместо того чтобы вручную применять теги или искать несоответствующие ресурсы, создайте политику, которая автоматически применяет необходимые теги во время развертывания. Теги также можно применить к существующим ресурсам с новым действием [изменения](../../governance/policy/concepts/effects.md#modify) и [задачей исправления](../../governance/policy/how-to/remediate-resources.md). В следующем разделе приведены примеры политик для тегов.
