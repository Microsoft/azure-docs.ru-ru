# <a name="moving-azure-resources-across-regions"></a>Перемещение ресурсов Azure в разных регионах

В этой статье содержатся сведения о перемещении ресурсов Azure в регионах Azure.

Географические области, регионы и зоны доступности Azure формируют основу глобальной инфраструктуры Azure. [Географы](https://azure.microsoft.com/global-infrastructure/geographies/) Azure обычно содержат два или более [региона Azure](https://azure.microsoft.com/global-infrastructure/regions/). Регион — это область внутри географического элемента, содержащая Зоны доступности и несколько центров обработки данных. 

После развертывания ресурсов в определенном регионе Azure существует ряд причин, по которым может потребоваться переместить ресурсы в другой регион.

- **Выровняйте по регионам**: Переместите ресурсы в недавно появившийся регион Azure, который ранее не был доступен.
- **Использовать ресурсы с одинаковыми службами или функциями**. Переместите ресурсы, чтобы воспользоваться преимуществами служб или функций, доступных в определенном регионе.
- **Реагировать на развитие бизнес-модели**. Переместите ресурсы в регион в соответствии с изменением бизнес-модели, например при слиянии или поглощении компаний.
- **Использовать ресурсы с учетом близости расположения**. Переместите ресурсы в регион, который является локальным для вашего бизнеса.
- **Соответствие требованиям к данным**: перемещение ресурсов для соответствия требованиям местонахождение данных или требованиям классификации данных. [Подробнее](https://azure.microsoft.com/mediahandler/files/resourcefiles/achieving-compliant-data-residency-and-security-with-azure/Achieving_Compliant_Data_Residency_and_Security_with_Azure.pdf).
- **Реагировать на требования к развертыванию**. Переместите ресурсы, которые были развернуты по ошибке или в соответствии с потребностями в емкости. 
- **Ответ на списание**: перемещение ресурсов из-за списания регионов.

## <a name="move-resources-with-resource-mover"></a>Перемещение ресурсов с помощью перемещения ресурсов

Ресурсы можно перемещать в другой регион с помощью [перемещения ресурсов Azure](../../resource-mover/overview.md). Resource Mover обеспечивает:

- Один центр для перемещения ресурсов между регионами.
- Сокращение времени и упрощение при перемещении. Все, что вам нужно, находится в одном расположении.
- Простой и согласованный процесс перемещения различных типов ресурсов Azure.
- Простой способ обнаружения зависимостей между ресурсами, которые необходимо переместить. Это помогает перемещать связанные ресурсы вместе, чтобы после перемещения в целевом регионе все правильно работало.
- Автоматическая очистка ресурсов в исходном регионе, если их нужно удалить после перемещения.
- Тестирование. Вы можете попробовать выполнить перемещение, а затем отменить его, если не хотите выполнять полный переход.

Ресурсы можно перемещать в другой регион с помощью нескольких различных методов.

- **Начало перемещения ресурсов из группы ресурсов**. при использовании этого метода вы запускаете перемещение по региону в группе ресурсов. После выбора ресурсов, которые необходимо переместить, процесс продолжится в концентраторе перемещения ресурсов, чтобы проверить зависимости ресурсов и управлять процессом перемещения. [Подробнее](../../resource-mover/move-region-within-resource-group.md).
- **Начните перемещение ресурсов непосредственно из центра перемещения ресурсов**. Этот метод запускает процесс перемещения региона непосредственно в концентраторе. [Подробнее](../../resource-mover/tutorial-move-region-virtual-machines.md).


## <a name="support-for-region-move"></a>Поддержка перемещения по регионам

В настоящее время вы можете использовать перемещение ресурсов для перемещения этих ресурсов в другой регион:

- Виртуальные машины Azure и связанные с ними диски.
- Сетевые карты
- Группы доступности
- Виртуальные сети Azure
- Общедоступные IP-адреса
- Группы безопасности сети (NSG).
- Внутренние и общедоступные подсистемы балансировки нагрузки.
- Базы данных SQL Azure и эластичные пулы.

## <a name="region-move-process"></a>Процесс перемещения региона

Фактический процесс перемещения ресурсов по регионам зависит от перемещаемых ресурсов. Однако существуют некоторые распространенные основные шаги.

1. **Проверка предварительных требований**. Предварительные требования: Убедитесь, что необходимые ресурсы доступны в целевом регионе, проверьте наличие достаточной квоты и убедитесь, что ваша подписка имеет доступ к целевому региону.
2. **Анализ зависимостей**. ваши ресурсы могут иметь зависимости от других ресурсов. Перед перемещением выясните зависимости, чтобы перемещенные ресурсы продолжали функционировать, как и ожидается после перемещения.
3. **Подготовка к перемещению**. это действия, выполняемые в основном регионе перед перемещением. Например, может потребоваться экспортировать шаблон Azure Resource Manager или начать репликацию ресурсов из источника в целевой объект.
4. **Перемещение ресурсов**. Порядок перемещения ресурсов зависит от того, что они представляют. Возможно, потребуется развернуть шаблон в целевом регионе или выполнить неудачные ресурсы для целевого объекта.
5. **Удалить целевые ресурсы**: после перемещения ресурсов может потребоваться просмотреть ресурсы в целевом регионе и решить, есть ли что-либо, что вам не нужно.
6. **Зафиксировать перемещение**: после проверки ресурсов в целевом регионе некоторые ресурсы могут потребовать окончательного действия фиксации. Например, в целевом регионе, который теперь является основным регионом, может потребоваться настроить аварийное восстановление в новом дополнительном регионе. 
7. **Очистка источника**. Наконец, после того, как все готово к работе в новом регионе, можно очистить и списать ресурсы, созданные для перемещения, и ресурсы в основном регионе.
