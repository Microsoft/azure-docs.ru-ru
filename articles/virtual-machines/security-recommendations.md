# <a name="security-recommendations-for-virtual-machines-in-azure"></a>Рекомендации по безопасности для виртуальных машин в Azure


Эта статья содержит рекомендации по безопасности для виртуальных машин Azure. Следуйте этим рекомендациям, чтобы удовлетворить обязательства по обеспечению безопасности, описанные в нашей модели, для совместной ответственности. Рекомендации также помогут повысить общую безопасность решений веб-приложений. Дополнительные сведения о том, что делает корпорация Майкрософт для выполнения обязанностей поставщиков услуг, см. в статье [Общие обязанности для облачных вычислений](https://gallery.technet.microsoft.com/Shared-Responsibilities-81d0ff91).

Некоторые из рекомендаций этой статьи можно автоматически устранить с помощью центра безопасности Azure. Центр безопасности Azure — это первая строка защиты ваших ресурсов в Azure. Он периодически анализирует состояние безопасности для ресурсов Azure, чтобы выявить потенциальные уязвимости. Затем он рекомендует устранить уязвимости. Дополнительные сведения см. [в статье рекомендации по безопасности в центре безопасности Azure](../security-center/security-center-recommendations.md).

Общие сведения о центре безопасности Azure см. в статье [что такое центр безопасности Azure?](../security-center/security-center-introduction.md).

## <a name="general"></a>Общие сведения

| Рекомендация | Комментарии | Центр безопасности |
|-|----|--|
| При создании пользовательских образов виртуальных машин установите последние обновления. | Перед созданием образов установите последние обновления для операционной системы и всех приложений, которые будут входить в образ.  | - |
| Поддерживайте виртуальные машины в актуальном состоянии. | С помощью решения [Управление обновлениями](../automation/update-management/overview.md) в службе автоматизации Azure можно управлять обновлениями операционной системы для компьютеров под управлением Windows и Linux в Azure. | [Да](../security-center/asset-inventory.md) |
| Создайте резервные копии виртуальных машин. | [Azure Backup](../backup/backup-overview.md) помогает защитить данные приложения и обеспечивает минимальные эксплуатационные расходы. Ошибки приложений могут повредить данные, а ошибки пользователей — привести к ошибкам в коде приложений. Azure Backup защищает виртуальные машины под управлением Windows и Linux. | - |
| Используйте несколько виртуальных машин для повышения устойчивости и доступности. | Если на виртуальной машине выполняются приложения, которые должны быть высокодоступными, используйте несколько виртуальных машин или [групп доступности](./availability.md). | - |
| Внедрение стратегии непрерывности бизнес-процессов и аварийного восстановления (BCDR). | Azure Site Recovery позволяет выбирать из различных параметров, предназначенных для обеспечения непрерывности бизнес-процессов. Он поддерживает различные сценарии репликации и отработки отказа. Дополнительные сведения см. в разделе  [About Site Recovery](../site-recovery/site-recovery-overview.md). | - |

## <a name="data-security"></a>Безопасность данных

| Рекомендация | Комментарии | Центр безопасности |
|-|----|--|
| Зашифруйте диски операционной системы. | [Шифрование дисков Azure](../security/fundamentals/azure-disk-encryption-vms-vmss.md) позволяет шифровать диски виртуальных машин IaaS под управлением Windows и Linux. Без необходимых ключей содержимое зашифрованных дисков не читается. Шифрование дисков защищает сохраненные данные от несанкционированного доступа, которые в противном случае могли бы быть доступны при копировании диска.| [Да](../security-center/asset-inventory.md) |
| Шифрование дисков данных. | [Шифрование дисков Azure](../security/fundamentals/azure-disk-encryption-vms-vmss.md) позволяет шифровать диски виртуальных машин IaaS под управлением Windows и Linux. Без необходимых ключей содержимое зашифрованных дисков не читается. Шифрование дисков защищает сохраненные данные от несанкционированного доступа, которые в противном случае могли бы быть доступны при копировании диска.| -  |
| Ограничение установленного программного обеспечения. | Ограничьте установленное программное обеспечение до необходимого для успешного применения решения. Это помогает сократить область атаки вашего решения. | - |
| Используйте антивирусную или антивредоносную программу. | В Azure можно использовать антивредоносное по от поставщиков безопасности, таких как Microsoft, Symantec, Trend Micro и Kaspersky. Это программное обеспечение помогает защитить виртуальные машины от вредоносных файлов, рекламного по и других угроз. Антивредоносное по Майкрософт можно развернуть на основе рабочих нагрузок приложения. Антивредоносное по Майкрософт доступно только для компьютеров под управлением Windows. Используйте базовую, защищенную по умолчанию или расширенную настраиваемую конфигурацию. Дополнительные сведения см. в [статье антивредоносное по Майкрософт для облачных служб и виртуальных машин Azure](../security/fundamentals/antimalware.md). | - |
| Безопасное хранение ключей и секретов. | Упростите управление секретами и ключами, предоставив владельцам приложений защищенный централизованно управляемый параметр. Такое управление снижает риск случайной компрометации или утечки. Azure Key Vault можете безопасно хранить ключи в аппаратных модулях безопасности (HSM), сертифицированных для FIPS 140-2 уровня 2. Если вам нужно использовать FIPs 140,2 уровня 3 для хранения ключей и секретов, можно использовать [выделенный HSM Azure](../dedicated-hsm/overview.md). | - |

## <a name="monitoring"></a>Наблюдение

| Рекомендация | Комментарии | Центр безопасности |
|-|----|--|
| Мониторинг виртуальных машин. | [Azure Monitor для виртуальных машин](../azure-monitor/vm/vminsights-overview.md) можно использовать для мониторинга состояния виртуальных машин Azure и масштабируемых наборов виртуальных машин. Снижение производительности виртуальной машины может привести к прерыванию работы служб, что в свою очередь нарушает доступность, являющуюся одним из основных принципов безопасности. | - |

## <a name="networking"></a>Сеть

| Рекомендация | Комментарии | Центр безопасности |
|-|----|--|
| Ограничьте доступ к портам управления. | Злоумышленники проверяют диапазоны IP-адресов общедоступного облака на наличие открытых портов управления и попытаются выполнить простые атаки, такие как общие пароли и известные неисправленные уязвимости. Вы можете использовать [JIT-доступ к виртуальной машине](../security-center/security-center-just-in-time.md) для блокировки входящего трафика к виртуальным машинам Azure, уменьшая вероятность атак и предоставляя простые подключения к виртуальным машинам, когда они необходимы. | - |
| Ограничьте доступ к сети. | Группы безопасности сети позволяют ограничить доступ к сети и управлять количеством видимых конечных точек. Дополнительные сведения см. [в статье Создание, изменение или удаление группы безопасности сети](../virtual-network/manage-network-security-group.md). | - |
