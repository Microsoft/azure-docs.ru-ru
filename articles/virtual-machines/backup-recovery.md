# <a name="backup-and-restore-options-for-virtual-machines-in-azure"></a>Параметры резервного копирования и восстановления для виртуальных машин в Azure

Для защиты данных можно создавать архивы с регулярным интервалом. Доступно несколько вариантов резервного копирования для виртуальных машин на основе сценариев использования.

## <a name="azure-backup"></a>Azure Backup

Для создания резервных копий виртуальных машин с производственными рабочими нагрузками используйте службу Azure Backup. Azure Backup поддерживает резервные копии с согласованием приложений для виртуальных машин Windows и Linux. Служба архивации Azure создает точки восстановления, которые хранятся в геоизбыточных хранилищах восстановления. Используя точку восстановления, можно восстановить всю виртуальную машину или только определенные файлы. 

Простое и практичное введение в Azure Backup для виртуальных машин Azure см. в разделе [Краткое руководство по Azure Backup](../backup/quick-backup-vm-portal.md).

Дополнительные сведения о работе Azure Backup см. в статье [Планирование инфраструктуры резервного копирования виртуальных машин в Azure](../backup/backup-azure-vms-introduction.md).


## <a name="azure-site-recovery"></a>Azure Site Recovery

Azure Site Recovery позволяет защитить виртуальные машины на случай крупных аварий, при которых из-за масштабного стихийного бедствия или обширного прерывания работы службы весь регион оказывается подвержен сбою. Для виртуальных машин можно настроить службу Azure Site Recovery, которая позволяет быстро восстановить приложение одним щелчком мыши. Репликацию можно выполнять в выбранном регионе Azure, не ограничиваясь парными регионами. 

Вы можете выполнять аварийное восстановление с тестовой отработкой отказа по запросу, не затрагивая производственные рабочие нагрузки или текущую репликацию. Создавайте планы восстановления для оркестрации отработки отказа и восстановления размещения всего приложения, работающего на нескольких виртуальных машинах. Планы восстановления можно интегрировать с модулями Runbook службы автоматизации Azure.

Вы можете начать с [репликации виртуальных машин](../site-recovery/azure-to-azure-quickstart.md). 

## <a name="managed-snapshots"></a>Управляемые моментальные снимки 

В средах разработки и тестирования моментальные снимки обеспечивают быстрое и простое резервное копирование виртуальных машин, использующих управляемые диски. Управляемые моментальный снимок — это полная копия управляемого диска, предназначенная только для чтения. Моментальные снимки существуют независимо от исходного диска и могут использоваться для создания других управляемых дисков, с помощью которых можно воссоздать виртуальную машину. Плата за них взимается в зависимости от используемого объема диска. Например, если создается моментальный снимок управляемого диска с подготовленной емкостью 64 ГБ и фактическим размером используемых данных 10 ГБ, оплачивается только размер используемых данных (10 ГБ).  

Дополнительные сведения о создании моментальных снимков см. в следующих статьях:

* [Создание копии VHD, хранящейся в качестве управляемого диска, с помощью моментальных снимков в Linux](./linux/snapshot-copy-managed-disk.md)
