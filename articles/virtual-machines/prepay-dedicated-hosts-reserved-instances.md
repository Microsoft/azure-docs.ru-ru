# <a name="save-costs-with-azure-dedicated-host-reservations"></a>Экономия расходов с помощью резервирования выделенных узлов Azure

При фиксации в зарезервированном экземпляре выделенных узлов Azure можно сэкономить деньги. Скидка на резервирование автоматически применяется к числу запущенных выделенных узлов, соответствующих области резервирования и атрибутам. Для получения скидок не нужно назначать резервирование выделенному узлу. Покупка зарезервированного экземпляра охватывает только вычислительную часть использования и включает стоимость лицензирования программного обеспечения. См. [Обзор выделенных узлов Azure для виртуальных машин](./dedicated-hosts.md).

## <a name="determine-the-right-dedicated-host-sku-before-you-buy"></a>Определение правильного выделенного номера SKU узла перед покупкой


Прежде чем приобретать резервирование, необходимо определить, какой специальный узел вам нужен. Номер SKU определяется для выделенного узла, представляющего ряд и тип виртуальной машины. 

Начните с перехода на поддерживаемый размер для [виртуальной машины Windows](./sizes.md) или [Linux](./sizes.md) , чтобы найти серию виртуальных машин.

Затем проверьте, поддерживается ли на выделенных узлах Azure. На странице [цен на выделенные узлы Azure](https://aka.ms/ADHPricing) содержится полный список номеров SKU выделенных узлов, сведения о ЦП и различные варианты ценообразования (включая зарезервированные экземпляры).

Вы можете найти несколько номеров SKU, поддерживающих выбранную серию виртуальных машин (с разными типами). Найдите лучший SKU, сравнив емкость узла (число виртуальных ЦП). Обратите внимание, что вы сможете применить резервирование к нескольким выделенным узлам SKU, поддерживающим один и тот же ряд виртуальных машин (например DSv3_Type1 и DSv3_Type2), но не в разных рядах виртуальных машин (таких как DSv3 и ESv3).



## <a name="purchase-restriction-considerations"></a>Сведения об ограничениях на покупку

Зарезервированные экземпляры доступны для большинства выделенных размеров узлов с некоторыми исключениями.

Скидки на резервирование не применяются к следующим параметрам:

- **Облака.** Резервирования недоступны для приобретения в Германии и Китае.

- **Недостаточная квота** . для резервирования в рамках одной подписки должна быть доступна квота виртуальных ЦП в подписке для нового зарезервированного экземпляра. Например, если в целевой подписке превышена квота в 10 виртуальных ЦП для серии DSv3, вы не сможете купить выделенные узлы резервирования, поддерживающие эту серию. Проверка квоты на резервирование включает виртуальные машины и выделенные узлы, уже развернутые в подписке. Чтобы устранить эту проблему, можно [создать запрос на увеличение квоты](../azure-portal/supportability/resource-manager-core-quotas-request.md) .

- **Ограничения емкости** . в редких обстоятельствах Azure ограничивает приобретение новых резервирований подмножеством выделенных номеров SKU узлов из-за нехватки емкости в регионе.

## <a name="buy-a-reservation"></a>Покупка резервирования

Зарезервированный экземпляр выделенного экземпляра узла Azure можно купить в [портал Azure](https://portal.azure.com/#blade/Microsoft_Azure_Reservations/CreateBlade/referrer/documentation/filters/%7B%22reservedResourceType%22%3A%22VirtualMachines%22%7D).

Платите за резервирование [наперед или ежемесячными платежами](../cost-management-billing/reservations/prepare-buy-reservation.md). Эти требования применяются для приобретения зарезервированного выделенного экземпляра узла:

- Необходима роль "Владелец" по крайней мере для одной подписки EA или подписки с оплатой по мере использования.

- Для подписок EA следует включить параметр **Добавить зарезервированные экземпляры** на [портале EA](https://ea.azure.com/). Если этот параметр отключен, необходимо быть администратором подписки EA.

- В рамках программы для поставщиков облачных решений (CSP) только агенты администрирования или агенты продаж могут приобрести резервирования.

Чтобы купить экземпляр, выполните следующие действия.

1. Войдите на [портал Azure](https://portal.azure.com/).

2. Выберите **Все службы** \> **Резервирования**.

3. Выберите **Добавить** , чтобы приобрести новое резервирование, а затем щелкните **выделенные узлы**.

4. Заполните обязательные поля. Запуск выделенных экземпляров узлов, соответствующих выбранным атрибутам, для получения скидки на резервирование. Фактическое число выделенных экземпляров узлов, которые получают скидку, зависит от выбранной области и количества.

Если вы заключили Соглашение EA, для быстрого добавления дополнительных экземпляров можно использовать **параметр "Добавить еще"** . Этот параметр недоступен для подписок других типов.

| **Поле**           | **Описание**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
|---------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Подписка        | Подписка, используемая для оплаты резервирования. Резервирование оплачивается с помощью метода оплаты, указанного для подписки. Нужно выбрать подписку с соглашением Enterprise (номера предложений: MS-AZR-0017P или MS-AZR-0148P) или Клиентского соглашения Майкрософт либо отдельную подписку с оплатой по мере использования (номера предложений: MS-AZR-0003P или MS-AZR 0023P). Плата вычитается из остатка на счете предоплаты Azure (прежнее название — денежное обязательство), при его наличии, или относится к избыточным расходам. Для подписки с оплатой по мере использования плата взимается с помощью метода оплаты (кредитной карты или счета), указанного для подписки. |
| Область               | Область резервирования может охватывать одну или несколько подписок (общая область). | Выделенный размер узла | Размер выделенных экземпляров узлов.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |

- **Одна группа ресурсов** — скидка по резервированию применяется к подходящим ресурсам только в выбранной группе ресурсов.

- **Одна подписка** — скидка по резервированию применяется к подходящим ресурсам только в выбранной подписке.

- **Общая область резервирования** — скидка по резервированию применяется к подходящим ресурсам во всех допустимых подписках в контексте выставления счетов. Для клиентов EA контекстом выставления счетов считается Соглашение о регистрации. Для отдельных подписок с оплатой по мере использования областью выставления счетов считаются все допустимые подписки, созданные администратором учетной записи.

## <a name="usage-data-and-reservation-utilization"></a>Данные об использовании и использование резервирования

В данных об использовании указана нулевая эффективная цена за использование, на которое распространяется скидка на резервирование. Вы можете увидеть, какой экземпляр виртуальной машины получил скидку резервирования для каждого резервирования.

Дополнительные сведения о том, как скидки на резервирование отображаются в данных об использовании, см. в статье [сведения об использовании резервирования Azure для регистрации на предприятии](../cost-management-billing/reservations/understand-reserved-instance-usage-ea.md) , если вы ЯВЛЯЕТЕСЬ клиентом EA. Если у вас есть отдельная подписка, см. статью [сведения об использовании резервирования Azure для подписки с оплатой по мере использования](../cost-management-billing/reservations/understand-reserved-instance-usage.md).

## <a name="change-a-reservation-after-purchase"></a>Изменение резервирования после покупки

После приобретения в резервирование можно внести следующие типы изменений:

- Обновить область резервирования

- Гибкость размера экземпляра (если применимо)

- владельца;

Можно также разделить резервирование на небольшие фрагменты и объединить уже разделенные резервирования. Ни одно из изменений не вызывает новую коммерческую транзакцию или не изменяет дату окончания резервирования.

Следующие типы изменений нельзя внести после приобретения, непосредственно:

- Регион существующего резервирования

- номер SKU

- Количество

- Длительность

Тем не менее можно выполнить *Обмен* резервированиями, если необходимо внести изменения.
