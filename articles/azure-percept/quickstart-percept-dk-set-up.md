# <a name="set-up-your-azure-percept-dk-and-deploy-your-first-ai-model"></a>Настройка Azure Percept DK и развертывание первой модели ИИ

Завершите процесс установки Azure Percept DK, чтобы настроить набор средств разработки и развернуть первую модель искусственного интеллекта. Убедившись, что ваша учетная запись Azure совместима с Azure Percept, вы сможете выполнять перечисленные ниже задачи.

- Подключение набора средств разработки к сети Wi-Fi
- Настройка входа по протоколу SSH для удаленного доступа к набору средств разработки
- Создание нового Центра Интернета вещей для использования с Azure Percept
- Подключение набора средств разработки к Центру Интернета вещей и учетной записи Azure

В случае возникновения проблем во время этого процесса ознакомьтесь с руководством по [устранению неполадок при установке](./how-to-troubleshoot-setup.md) для поиска возможных решений.

## <a name="prerequisites"></a>Предварительные требования

- Azure Percept DK (набор средств разработки).
- Главный компьютер под управлением Windows, Linux или OS X с возможностью подключения по Wi-Fi и веб-браузером.
- Учетная запись Azure с активной подпиской. [Создайте учетную запись бесплатно.](https://azure.microsoft.com/free/?WT.mc_id=A261C142F)
- Учетная запись Azure должна иметь роль **владельца** или **участника** в подписке. Выполните следующие действия, чтобы проверить роль учетной записи Azure. Дополнительные сведения об определениях ролей Azure см. в [документации по управлению доступом на основе ролей в Azure](../role-based-access-control/rbac-and-directory-admin-roles.md#azure-roles).

    > [!CAUTION]
    > Если у вас есть несколько учетных записей Azure, браузер может кэшировать учетные данные из другой учетной записи. Чтобы избежать путаницы, рекомендуется закрыть все неиспользуемые окна браузера и войти на [портал Azure](https://portal.azure.com/) перед началом процесса установки. Дополнительные сведения о том, как войти в систему с использованием правильной учетной записи, см. в руководстве по [устранению неполадок при установке](./how-to-troubleshoot-setup.md).

### <a name="check-your-azure-account-role"></a>Проверка роли учетной записи Azure

Чтобы проверить, является ли ваша учетная запись Azure "владельцем" или "участником" в подписке, выполните указанные ниже действия.

1. Перейдите на [портал Azure](https://portal.azure.com/) и войдите в систему, используя ту же учетную запись Azure, которую вы собираетесь использовать в Azure Percept.

1. Щелкните значок **Подписки** (он выглядит как желтый ключ).

1. Выберите свою подписку в списке. Если вы не видите свою подписку, убедитесь, что вы использовали для входа правильную учетную запись Azure. Если вы хотите создать подписку, выполните [эти действия](../cost-management-billing/manage/create-subscription.md).

1. В меню "Подписка" выберите **Управление доступом (IAM)** .
1. Щелкните **View my access** (Просмотреть мой доступ).
1. Проверьте роль.
    - Если роль указана в списке как **Читатель** или появляется сообщение о том, что у вас нет разрешений для просмотра ролей, потребуется выполнить принятую в организации процедуру, чтобы получить для учетной записи роль с расширенными правами.
    - Если роль указана как **Владелец** или **Участник**, ваша учетная запись будет работать с Azure Percept и вы можете продолжить процесс установки.

## <a name="launch-the-azure-percept-dk-setup-experience"></a>Запуск процедуры настройки Azure Percept DK

1. Подключите главный компьютер непосредственно к точке доступа Wi-Fi набора средств разработки. Как и при подключении к любой другой сети Wi-Fi, откройте параметры сети и Интернета на компьютере, щелкните указанную ниже сеть и введите сетевой пароль при появлении запроса.

    - **Сетевое имя**. В зависимости от версии операционной системы, используемой в наборе средств разработки, именем точки доступа Wi-Fi будет либо **scz-xxxx**, либо **apd-xxxx** (где "xxxx" — последние четыре цифры Mac-адреса набора средств разработки).
    - **Пароль**. Можно найти в карточке с приветствием, поставляемой вместе с набором средств разработки.

    > [!WARNING]
    > При подключении к точке доступа Wi-Fi Azure Percept DK главный компьютер временно теряет подключение к Интернету. Активные вызовы видеоконференций, потоковая передача через Интернет и другие сетевые возможности перестанут работать.

1. После подключения к точке доступа Wi-Fi набора средств разработки компьютер автоматически запустит процедуру установки в новом окне браузера, где в адресной строке будет указано **your.new.device/** . Если вкладка не открывается автоматически, запустите процесс установки, перейдя по адресу [http://10.1.1.1](http://10.1.1.1). Убедитесь, что в браузере выполнен вход с теми же учетными данными Azure, которые вы планируете использовать с Azure Percept.

    :::image type="content" source="./media/quickstart-percept-dk-setup/main-01-welcome.png" alt-text="Страница приветствия":::

    > [!CAUTION]
    > Веб-служба, используемая для процесса установки, завершит работу после 30 минут простоя. В этом случае перезапустите набор средств разработки, чтобы избежать проблем с подключением к его точке доступа Wi-Fi.

## <a name="complete-the-azure-percept-dk-setup-experience"></a>Завершение процедуры настройки Azure Percept DK

1. Нажмите кнопку **Далее** на экране **приветствия**.

1. На странице **Сетевое подключение** щелкните **Connect to a new WiFi network** (Подключиться к новой сети Wi-Fi).

    Если вы уже подключили набор средств разработки к сети Wi-Fi, нажмите кнопку **Пропустить**.

1. Выберите сеть Wi-Fi из списка доступных сетей и нажмите кнопку **подключить**. При появлении запроса введите свой сетевой пароль.

1. После успешного подключения набора средств разработки к выбранной сети на странице отобразится адрес IPv4, назначенный вашему набору средств разработки. **Запишите адрес IPv4, отображаемый на странице.** Этот IP-адрес потребуется при подключении к набору средств разработки по протоколу SSH для устранения неполадок и обновления устройства.

    :::image type="content" source="./media/quickstart-percept-dk-setup/main-04-success-wi-fi.png" alt-text="Скопируйте IP-адрес.":::

    > [!NOTE]
    > Этот IP-адрес может изменяться при каждой загрузке устройства.

1. Прочтите лицензионное соглашение, выберите **I have read and agree to the License Agreement** (Лицензионное соглашение мною прочитано, и я принимаю его условия) (необходимо прокрутить соглашение до конца) и нажмите кнопку **Далее**.

    :::image type="content" source="./media/quickstart-percept-dk-setup/main-05-eula.png" alt-text="Примите условия лицензионного соглашения.":::

1. Введите имя и пароль учетной записи SSH и **запишите данные для входа для дальнейшего использования**.

    > [!NOTE]
    > SSH (Secure Shell) — это сетевой протокол, позволяющий удаленно подключаться к набору средств разработки через главный компьютер.

1. На следующей странице щелкните **Setup as a new device** (Настройка в качестве нового устройства), чтобы создать новое устройство в учетной записи Azure.

1. Щелкните **Копировать**, чтобы скопировать код устройства. Затем щелкните **Вход в Azure**.

    :::image type="content" source="./media/quickstart-percept-dk-setup/main-08-copy-code.png" alt-text="Скопируйте код устройства.":::

1. Откроется новая вкладка браузера с окном, в котором отображается надпись **Введите код**. Вставьте код в окне и нажмите кнопку **Далее**. НЕ закрывайте вкладку **Добро пожаловать**, на которой выполняется процесс установки.

    :::image type="content" source="./media/quickstart-percept-dk-setup/main-09-enter-code.png" alt-text="Введите код устройства.":::

1. Войдите в Azure Percept, используя учетные данные для учетной записи Azure, которая будет использоваться в наборе средств разработки. По завершении оставьте вкладку "Браузер" открытой.

    > [!CAUTION]
    > Браузер может кэшировать другие учетные данные. Дважды проверьте, что вы входите в систему с использованием правильной учетной записи.

    После успешного входа в Azure Percent на устройстве вернитесь на вкладку **Добро пожаловать**, чтобы продолжить процесс установки.

1. Когда на вкладке **Добро пожаловать** появится страница **Assign your device to your Azure IoT Hub** (Назначить устройство Центру Интернета вещей Azure), выполните одно из следующих действий.

    - Если у вас уже есть Центр Интернета вещей, который необходимо использовать с Azure Percept, и он указан на этой странице, выберите его и перейти к шагу 15.
    - Если у вас нет Центра Интернета вещей или вы хотите создать новый, щелкните **Create a new Azure IoT Hub** (Создать Центр Интернета вещей Azure).

    > [!IMPORTANT]
    > Если у вас есть Центр Интернета вещей, но он отсутствует в списке, возможно, вы выполнили вход в Azure Percept с неверными учетными данными. Справочные сведения см. в [руководстве по устранению неполадок при установке](./how-to-troubleshoot-setup.md).

    :::image type="content" source="./media/quickstart-percept-dk-setup/main-13-iot-hub-select.png" alt-text="Выберите Центр Интернета вещей.":::

1. Чтобы создать Центр Интернета вещей, заполните следующие поля:

    - Выберите подписку Azure, которая будет использоваться с Azure Percept.
    - Выберите имеющуюся группу ресурсов. Если она отсутствует, щелкните **Создать** и следуйте инструкциям на экране.
    - Выберите ближайший к вашему расположению регион Azure.
    - Укажите имя нового Центра Интернета вещей.
    - Выберите ценовую категорию S1 (стандарт).

    > [!NOTE]
    > Если для пограничных приложений искусственного интеллекта требуется более высокая [пропускная способность для сообщений](https://docs.microsoft.com/azure/iot-hub/iot-hub-scaling#message-throughput), вы можете в любое время [обновить Центр Интернета вещей до более высокого стандартного уровня](https://docs.microsoft.com/azure/iot-hub/iot-hub-upgrade) на портале Azure. Уровни B и F не поддерживают Azure Percept.

1. Развертывание Центра Интернета вещей может занять несколько минут. Когда развертывание будет завершено, выберите **Регистрация**.

    :::image type="content" source="./media/quickstart-percept-dk-setup/main-16-iot-hub-success.png" alt-text="Центр Интернета вещей успешно развернут.":::

1. Введите имя устройства для своего набора средств разработки и нажмите кнопку **Далее**.

1. Дождитесь скачивания моделей устройства — это займет несколько минут.

    :::image type="content" source="./media/quickstart-percept-dk-setup/main-18-finalize.png" alt-text="Завершение установки.":::

1. Когда появится страница **Настройка устройства завершена**, набор средств разработки будет успешно связан с Центром Интернета вещей и для него будет загружено необходимое программное обеспечение. Ваш набор средств разработки автоматически отключится от точки доступа Wi-Fi, в результате чего отобразятся следующие два уведомления:

    <!---
    > [!NOTE]
    > The onboarding process and connection to the device Wifi access to your host computer shuts down at this point, but your dev kit will stay connected to the internet.   You can restart the onboarding experience with a dev kit reboot, which will allow you to go back through the onboarding and reconnect the device to a different IOT hub associated with the same or a different Azure Subscription..
    --->

    :::image type="content" source="./media/quickstart-percept-dk-setup/main-19-0-warning.png" alt-text="Предупреждение об отключении процедуры настройки":::

1. Подключите главный компьютер к сети той Wi-Fi, к которой был подключен набор средств разработки на шаге 2.

1. Щелкните **продолжить переход на портал Azure**.

    :::image type="content" source="./media/quickstart-percept-dk-setup/main-20-azure-portal-continue.png" alt-text="Перейдите в Azure Percept Studio.":::

## <a name="view-your-dev-kit-video-stream-and-deploy-a-sample-model"></a>Просмотр видеопотока набора средств разработки и развертывание примера модели

1. [Страница обзора Azure Percept Studio](https://go.microsoft.com/fwlink/?linkid=2135819) является отправной точкой для доступа ко многим рабочим процессам для начальной и расширенной разработки пограничных решений для ИИ. Чтобы начать работу, щелкните **Устройства** в меню слева.

    :::image type="content" source="./media/quickstart-percept-dk-setup/portal-01-get-device-list.png" alt-text="Просмотрите свой список устройств.":::

1. Убедитесь, что ваш набор средств разработки указан как **подключенный**, и щелкните его, чтобы просмотреть страницу устройства.

    :::image type="content" source="./media/quickstart-percept-dk-setup/portal-02-select-device.png" alt-text="Выберите свое устройство.":::

1. Щелкните **View your device stream** (Просмотреть поток устройства). Если вы впервые просматриваете видеопоток устройства, в правом верхнем углу отобразится уведомление о том, что выполняется развертывание новой модели. Это может занять несколько минут.

    :::image type="content" source="./media/quickstart-percept-dk-setup/portal-03-1-start-video-stream.png" alt-text="Просмотрите видеопоток.":::

    После развертывания модели отобразится еще одно уведомление со ссылкой **View stream** (Просмотреть поток). Щелкните ссылку, чтобы просмотреть видеопоток с камеры Azure Percept Vision в новом окне браузера. Набор средств разработчика предварительно загружается с моделью искусственного интеллекта, которая автоматически выполняет обнаружение объектов для многих распространенных объектов.

    :::image type="content" source="./media/quickstart-percept-dk-setup/portal-03-2-object-detection.png" alt-text="Просмотрите обнаружение объектов.":::

1. В Azure Percept Studio также имеется ряд примеров моделей ИИ. Чтобы развернуть пример модели в наборе средств разработки, вернитесь на страницу устройства и щелкните **Deploy a sample model** (Развернуть пример модели).

    :::image type="content" source="./media/quickstart-percept-dk-setup/portal-04-explore-prebuilt.png" alt-text="Изучите предварительно созданные модели.":::

1. Выберите пример модели в библиотеке и щелкните **Deploy to Device** (Развернуть на устройстве).

    :::image type="content" source="./media/quickstart-percept-dk-setup/portal-05-2-select-journey.png" alt-text="Изучение обнаружения объектов в действии":::

1. После успешного развертывания модели в правом верхнем углу экрана появится уведомление со ссылкой **View stream**. Чтобы просмотреть вывод модели в действии, щелкните ссылку в уведомлении или вернитесь на страницу устройства и щелкните **View your device stream** (Просмотреть поток устройства). Все модели, запущенные ранее в наборе средств разработки, теперь будут заменены новой моделью.
