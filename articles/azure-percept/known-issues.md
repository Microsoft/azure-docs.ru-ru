# <a name="known-issues"></a>Известные проблемы

Если возникнут какие-либо из этих проблем, нет необходимости открывать ошибку. Если у вас возникли проблемы с любыми обходными способами, откройте проблему.

|Область|Описание проблемы|Обходной путь|
|-------|---------|---------|
| Процедура встроенного взаимодействия | Не удается завершить подключение, если не настроено Wi-Fi устройства (ошибка входа в Azure). | 1. SSH для точки доступа к устройству (10.1.1.1) <br> 2. выявление и копирование IP-адреса устройства Ethernet <br> 3. подключение к интерфейсу встроенного использования с помощью скопированного URL-адреса Ethernet |
| Процедура встроенного взаимодействия | При выборе ссылок в лицензионном соглашении во время работы со встроенной поддержкой иногда не открывается новая веб-страница. | Скопируйте ссылку и откройте ее в отдельном окне браузера. |
| Процедура встроенного взаимодействия | При подключении к хот-споту Mobile Wi-Fi не работает. | Подключите устройство непосредственно к Софтап, сети Wi-Fi или сети через Ethernet. |
| Wi-Fi/Софтап | Иногда Софтап может отключаться или исчезать. | Мы изучаем.  Перезагрузка устройства обычно приводит к его возврату. |
| Wi-Fi | Кнопка "оборудование", переключающаяся на Wi-Fi Софтап, иногда не работает. | Продолжайте использовать нажатие кнопки или перезагрузите устройство. |
| Wi-Fi | Пользователи могут видеть сообщение после подключения к Wi-Fi "Эта Wi-Fiная сеть использует старый стандарт безопасности". | Хот-спот или Софтап DevKit использует алгоритм шифрования WEP. |
| Wi-Fi | Не удалось подключиться к Софтап с компьютера под управлением Windows 10 со следующим сообщением об ошибке: <br> "Не удается подключиться к этой сети" | Перезагрузите DevKit и компьютер. |
| Обновление устройства | Контейнеры не запускаются после обновления OTA. | Подключитесь к устройству по протоколу SSH и перезапустите контейнер IoT Edge с помощью этой команды `systemctl restart iotedge` . Все контейнеры будут перезапущены. |
| Обновление устройства | Пользователи могут получить сообщение об ошибке обновления, даже если оно было выполнено успешно. | Убедитесь, что устройство Обновлено, перейдя к Двойникау устройства в центре Интернета вещей. Это исправлено после первого обновления. |
| Обновление устройства | После первого обновления пользователи могут потерять параметры подключения Wi-Fi. | Чтобы настроить подключение Wi-Fi, выполните процедуру с помощью встроенной процедуры обновления. Это исправлено после первого обновления. |
| Обновление устройства | После выполнения беспроводного обновления пользователи больше не смогут войти через SSH с помощью ранее созданных учетных записей пользователей, а новые пользователи SSH не могут быть созданы с помощью встроенной функции. Эта проблема затрагивает системы, выполняющие обновления OTA из следующих предварительно установленных версий образов: 2020.110.114.105 и 2020.109.101.105. | Чтобы восстановить профили пользователей, выполните следующие действия после обновления OTA: <br> Подключитесь к DevKit по протоколу [SSH](./how-to-ssh-into-percept-dk.md) , используя "root" в качестве имени пользователя. Если вы отключили вход с правами привилегированного пользователя SSH с помощью встроенной функции, необходимо включить ее повторно. Выполните следующую команду после успешного подключения: <br> ```mkdir -p /var/custom-configs/home; chmod 755 /var/custom-configs/home``` <br> Чтобы восстановить домашние данные предыдущего пользователя, выполните следующую команду: <br> ```mkdir -p /tmp/prev-rootfs && mount /dev/mmcblk0p3 /tmp/prev-rootfs && [ ! -L /tmp/prev-rootfs/home ] && cp -a /tmp/prev-rootfs/home/* /var/custom-configs/home/. && echo "User home migrated!"; umount /tmp/prev-rootfs``` |
| Обновление устройства | После обновления OTA группы обновлений теряются. | Обновите тег устройства, выполнив следующие [инструкции](./how-to-update-over-the-air.md#create-a-device-update-group). |
| Установщик пакетов Dev Tools | Необязательная установка Caffe может завершиться ошибкой, если DOCKER не работает должным образом в системе. | Убедитесь, что DOCKER установлен и работает, а затем повторите попытку установки Caffe. |
| Установщик пакетов Dev Tools | Необязательная установка CUDA в несовместимых системах завершается сбоем. | Перед запуском программы установки проверьте совместимость системы с CUDA. |
| DOCKER, сеть, IoT Edge | Если во внутренней сети используется 172. x. x. x, контейнеры DOCKER не смогут подключиться к границе. | Добавьте Специальный раздел Бип в/ЕТК/доккер/daemon.jsв файле, например: `{    "bip": "192.168.168.1/24"}` |
|Azure Percept Studio | Ссылки "Просмотр потока" в Azure Перцепт Studio не открывают новое окно, в котором отображается веб-поток устройства. | 1. Откройте [портал Azure](https://portal.azure.com) и выберите **центр Интернета вещей**. <br> 2. щелкните центр Интернета вещей, к которому подключено устройство. <br> 3. Выберите **IOT Edge** в разделе **Автоматическое управление устройствами** на странице центра Интернета вещей. <br> 4. Выберите устройство из списка. <br> 5. Выберите **задать модули** в верхней части страницы устройства. <br> 6. Щелкните значок корзины рядом с **хостипмодуле** , чтобы удалить модуль. <br> 7. чтобы подтвердить действие, щелкните " **проверить и создать** ", а затем " **создать**". <br> 8. Откройте [Azure Перцепт Studio](https://go.microsoft.com/fwlink/?linkid=2135819) и в левой панели меню щелкните **устройства** . <br> 9. Выберите устройство из списка. <br> 10. на вкладке " **концепция** " щелкните " **просмотреть поток устройства**". Устройство загрузит новую версию Хостипмодуле и откроет вкладку браузера с веб-потоком вашего устройства. |
