# <a name="azure-security-baseline-for-azure-iot-hub"></a>Базовый план безопасности Azure для центра Интернета вещей Azure

Этот базовый план безопасности применяет рекомендации из [теста безопасности Azure версии 1,0](../security/benchmarks/overview-v1.md) к Microsoft Azure центру Интернета вещей. Azure Security Benchmark содержит рекомендации по обеспечению безопасности облачных решений в Azure.
Содержимое группируются по **элементам управления безопасностью** , определенным в производительности системы безопасности Azure, и связанным рекомендациям, применимым к центру Интернета вещей Azure. **Элементы управления** , неприменимые к центру Интернета вещей Azure, были исключены.

 
Чтобы узнать, как центр Интернета вещей Azure полностью сопоставлен с тестовым показателем безопасности Azure, см. [полный базовый файл сопоставления безопасности центра Интернета вещей Azure](https://github.com/MicrosoftDocs/SecurityBenchmarks/tree/master/Azure%20Offer%20Security%20Baselines).

### <a name="11-protect-azure-resources-within-virtual-networks"></a>1,1: защита ресурсов Azure в виртуальных сетях

**Руководство**. центр Интернета вещей — это многоклиентская платформа как услуга (PaaS), разные клиенты используют один и тот же пул ресурсов вычислений, сети и оборудования хранилища. Имена узлов центра Интернета вещей сопоставляются с общедоступной конечной точкой с общедоступным IP-адресом через Интернет. Разные клиенты совместно используют эту общедоступную конечную точку Центра Интернета вещей, и получить к ней доступ могут все устройства Интернета вещей в глобальных и локальных сетях. Корпорация Майкрософт разработала службу для полной изоляции данных каждого клиента и непрерывно работает для обеспечения этого результата.

Для работы функции Центра Интернета вещей, включая маршрутизацию сообщений, отправку файлов и импорт/экспорт устройств, также требуется подключение из Центра Интернета вещей к ресурсу Azure, принадлежащему клиенту, через общедоступную конечную точку. Эти пути подключения формируют исходящий трафик из Центра Интернета вещей на ресурсы клиента.

Рекомендуется ограничить возможности подключения к ресурсам Azure (включая центр Интернета вещей Azure) через виртуальную сеть, которую вы владеете и использовать для снижения вероятности подключения в изолированной сети и подключения к локальной сети непосредственно в магистральной сети Azure. Используйте частную связь Azure и частную конечную точку Azure, где это возможно, чтобы обеспечить частный доступ к службам из других виртуальных сетей. 

После установки частного доступа отключите доступ к общедоступной сети для центра Интернета вещей, чтобы получить дополнительную безопасность. Этот контроль на уровне сети применяется к определенному ресурсу центра Интернета вещей, обеспечивая изоляцию. Чтобы служба оставалась активной для других ресурсов клиента, использующих общедоступный путь, его общедоступная конечная точка остается неразрешимой, можно обнаружить IP-адреса, а порты остаются открытыми. Это не является проблемой, так как корпорация Майкрософт интегрирует несколько уровней безопасности, чтобы обеспечить полную изоляцию между клиентами.

Не закрывайте на устройствах открытые Аппаратные порты, чтобы избежать нежелательного доступа. Кроме того, механизмы сборки для предотвращения или обнаружения физического изменения устройства.

Ответственность за исправление предупреждений в зависимости от степени важности ресурсов и среды Azure, в которых произошел инцидент, лежит на вашем уровне.
