# <a name="scheduled-maintenance-in-azure-database-for-postgresql--hyperscale-citus"></a>Плановое обслуживание в базе данных Azure для PostgreSQL — изменение масштаба (Цитус)

Служба "база данных Azure для PostgreSQL" (Цитус) выполняет периодическое обслуживание, чтобы обеспечить безопасность, стабильность и актуальность управляемой базы данных.  Во время обслуживания все узлы в группе серверов получают новые функции, обновления и исправления.

Основные функции запланированного обслуживания для Цитус:

* Обновления применяются одновременно на всех узлах в группе серверов
* Уведомления о предстоящем обслуживании публикуются в работоспособности службы Azure пять дней заранее.
* Обычно между успешными событиями обслуживания для группы серверов существует по крайней мере 30 дней.

## <a name="notification-about-upcoming-maintenance"></a>Уведомление о предстоящем обслуживании

Уведомления о предстоящем запланированном обслуживании публикуются в службе работоспособности служб Azure и могут быть:

* Отправить по электронной почте по указанному адресу
* Отправлено в Azure Resource Manager роль
* Отправлено в текстовом сообщении (SMS) на мобильные устройства
* уведомления в приложении Azure;
* голосового сообщения.

> [!IMPORTANT]
> Обычно по крайней мере 30 дней между успешным запланированным обслуживанием для группы серверов.
>
> Однако в случае критического аварийного обновления, например при серьезной уязвимости, окно уведомления может появиться раньше чем через пять дней. Критическое обновление может быть применено к серверу, даже если запланированное обслуживание было успешно выполнено в течение последних 30 дней.

Если обслуживание завершается сбоем или отменяется, система создаст уведомление.
Он будет пытаться выполнить обслуживание снова согласно текущим параметрам планирования и уведомить вас за пять дней до следующего события обслуживания.
