# <a name="stopstart-an-azure-database-for-postgresql---flexible-server-preview"></a>Завершение работы и запуск базы данных Azure для PostgreSQL — гибкого сервера (Предварительная версия)

> [!IMPORTANT]
> База данных Azure для PostgreSQL — гибкий сервер в настоящее время находится на этапе предварительной версии.

В этой статье приводятся пошаговые инструкции по запуску гибкого сервера.

## <a name="pre-requisites"></a>Предварительные требования

Вот что вам нужно, чтобы выполнить инструкции, приведенные в этом руководстве:

-   Необходимо иметь гибкий сервер базы данных Azure для PostgreSQL.

## <a name="stop-a-running-server"></a>Завершение работы сервера

1.  В [портал Azure](https://portal.azure.com/)выберите гибкий сервер, который требуется отключить.

2.  На странице **Обзор** нажмите кнопку " **Закрыть** " на панели инструментов.

> [!NOTE]
> После остановки сервера другие операции управления недоступны для гибкого сервера.

Обратите внимание, что остановленные серверы будут автоматически запускаться через семь дней. Все ожидающие обновления обслуживания будут применены при следующем запуске сервера.

## <a name="start-a-stopped-server"></a>Запуск остановленного сервера

1.  В [портал Azure](https://portal.azure.com/)выберите гибкий сервер, который вы хотите запустить.

2.  На странице **Обзор** нажмите кнопку **Запуск** на панели инструментов.

> [!NOTE]
> После запуска сервера все операции управления теперь доступны для гибкого сервера.
