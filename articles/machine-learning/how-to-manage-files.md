# <a name="how-to-create-and-manage-files-in-your-workspace"></a>Создание файлов и управление ими в рабочей области

Узнайте, как создавать файлы и управлять ими в рабочей области Машинное обучение Azure.  Эти файлы хранятся в хранилище рабочей области по умолчанию. Файлы и папки могут предоставляться другим пользователям, у которых есть доступ на чтение рабочей области, и их можно использовать из любого экземпляра вычислений в рабочей области.

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись](https://aka.ms/AMLFree), прежде чем начинать работу.
* Рабочая область машинного обучения. См. сведения о [создании рабочей области машинного обучения Azure](how-to-manage-workspace.md).

## <a name="create-files"></a><a name="create"></a> Создание файлов

Чтобы создать новый файл в папке по умолчанию ( `Users > yourname` ), сделайте следующее:

1. Откройте рабочую область [Студии машинного обучения](https://ml.azure.com).
1. Выберите **Записные книжки** слева.
1. Выберите **+** изображение.
1. Выберите образ  **создать новый файл** .

    :::image type="content" source="media/how-to-run-jupyter-notebooks/create-new-file.png" alt-text="Создание файла":::

1. Присвойте файлу имя.
1. Выберите тип файла.
1. Нажмите кнопку **создания**.

В разделе предварительного просмотра отображаются записные книжки и большинство типов текстовых файлов.  Большинство других типов файлов не имеют предварительной версии.

Чтобы создать новый файл в другой папке, выполните следующие действия.
1. Выберите "..." в конце папки
1. Выберите **Create new file** (Создать файл).

> [!IMPORTANT]
> Содержимое записных книжек и сценариев потенциально может считывать данные из сеансов и получать доступ к данным без Организации в Azure.  Загружать файлы только из надежных источников. Дополнительные сведения см. в разделе рекомендации по [безопасному коду](concept-secure-code-best-practice.md#azure-ml-studio-notebooks).

## <a name="share-files"></a>Общий доступ к файлам

Скопируйте и вставьте URL-адрес для совместного использования файла.  Только другие пользователи рабочей области могут получить доступ к этому URL-адресу.  Узнайте больше о [предоставлении доступа к рабочей области](how-to-assign-roles.md).

## <a name="delete-a-file"></a>Удаление файла

Вы *не можете* удалить **образцы файлов записных книжек** .  Эти файлы являются частью среды Studio и обновляются каждый раз при публикации нового пакета SDK.  

Файлы, найденные в разделе " **файлы** ", *можно* удалить любым из следующих способов.

* В Студии выберите **...** в конце папки или файла.  Убедитесь, что используется поддерживаемый браузер (Microsoft Edge, Chrome или Firefox).
* [Используйте терминал](how-to-access-terminal.md) из любого вычислительного экземпляра в рабочей области. Папка **~/клаудфилес** сопоставлена с хранилищем в учетной записи хранения рабочей области.
* Воспользуйтесь инструментами Jupyter или JupyterLab.
