# <a name="densenet"></a>DenseNet

В этой статье описывается, как использовать модуль **денсенет** в конструкторе машинное обучение Azure для создания модели классификации изображений с помощью алгоритма денсенет.  

Этот алгоритм классификации является защищенным методом обучения и требует наличия каталога с помеченными изображениями. 

> [!NOTE]
> Этот модуль не поддерживает помеченный набор данных, сформированный на основе *меток данных* в студии, но поддерживает только Каталог изображений с меткой, созданный из [преобразования в модуль каталога изображений](convert-to-image-directory.md) . 

Модель можно обучить, предоставив модель и каталог с помеченными изображениями в качестве входных данных для [обучения модели Pytorch](train-pytorch-model.md). Обученную модель затем можно использовать для прогнозирования значений новых входных примеров с помощью [модели изображения оценки](score-image-model.md).

### <a name="more-about-densenet"></a>Дополнительные сведения о Денсенет

Дополнительные сведения о Денсенет см. в справочной документации, в которой [сжаты подключенные в сети свертки](https://arxiv.org/abs/1608.06993).

## <a name="how-to-configure-densenet"></a>Настройка Денсенет

1.  Добавьте модуль **денсенет** в конвейер в конструкторе.  

2.  В качестве **имени модели** укажите имя определенной структуры денсенет и можно выбрать один из поддерживаемых денсенет: ' densenet121 ', ' densenet161 ', ' densenet169 ', ' densenet201 '.

3.  Для предварительного **обучения** укажите, следует ли использовать предварительно обученную модель для ImageNet. Если этот флажок установлен, можно выполнить точную настройку модели на основе выбранной предварительно обученной модели. Если этот флажок не выбран, можно выполнить обучение с нуля.

4.  Для **эффективного использования памяти** укажите, следует ли использовать контрольные точки, что значительно повышает эффективность памяти, но более медленнее. Дополнительные сведения см. в справочной документации по [реализации денсенетс с экономным объемом памяти](https://arxiv.org/pdf/1707.06990.pdf).

5.  Подключите выходные данные модуля **денсенет** модуля, обучения и проверки набора данных изображений к [модели обучения Pytorch](train-pytorch-model.md). 

6. Отправьте конвейер.


## <a name="results"></a>Результаты

После завершения выполнения конвейера, чтобы использовать модель для оценки, подключите модель " [обучение Pytorch модели](train-pytorch-model.md) к [показателям изображений](score-image-model.md)", чтобы спрогнозировать значения для новых входных примеров.

## <a name="technical-notes"></a>Технические примечания  

###  <a name="module-parameters"></a>Параметры модуля  

| Имя             | Диапазон | Тип    | По умолчанию     | Описание                              |
| ---------------- | ----- | ------- | ----------- | ---------------------------------------- |
| Имя модели       | Любой   | Режим    | densenet201 | Имя определенной структуры Денсенет     |
| Предварительно обученные       | Любой   | Логическое значение | True        | Следует ли использовать предварительно обученную модель для ImageNet |
| Эффективность памяти | Любой   | Логическое значение | Неверно       | Следует ли использовать контрольные точки, что намного больше эффективности памяти, но более медленно |
