# <a name="what-tools-are-included-on-the-azure-data-science-virtual-machine"></a>Какие средства включены в виртуальную машину Azure для обработки и анализа данных?

Виртуальная машина для обработки и анализа данных — это простой способ просмотра данных и машинного обучения в облаке. Виртуальные машины для обработки и анализа данных предварительно настроены полной операционной системой, обновлениями для системы безопасности, драйверами, а также популярным программным обеспечением для обработки и анализа данных, а также разработки. Вы можете выбрать среду оборудования, начиная от недорогих ЦП-ориентированных машин, заканчивая очень мощными машинами с несколькими GPU, хранилищами NVMe и большими объемами памяти. На компьютерах с GPU установлены все драйверы, а все платформы машинного обучения соответствуют версии для совместимости с GPU. Кроме того, во все прикладные программы, поддерживающие GPU, включено ускорение.

Виртуальная машина для обработки и анализа данных поставляется с наиболее полезными предварительно установленными средствами обработки и анализа данных.

## <a name="how-to-choosecondaenvironment"></a>Как выбрать среду Conda  

> [!NOTE]
> При переходе на новую версию виртуальной машины для обработки и анализа данных со старой версии вносятся критические изменения. 

**Переход с Windows Server 2016 на Windows Server 2019** </br> 

* Используется среда Сonda python37_default. Мы не поддерживаем python2. Если вы используете TensorFlow или pytorch в среде Python conda, используйте py37_tensorflow или py37_pytorch соответственно. Если вы использовали только AzureML, используйте azureml_py36_automl. </br>

* Если вы используете TensorFlow или pytorch в среде AzureML conda, используйте azureml_py36_tensorflow или azureml_py36_pytorch соответственно. </br>

**Переход с Ubuntu 16.04 на Ubuntu 18.04** </br>

* Если вы используете TensorFlow или pytorch в среде azureml_py36 conda, используйте azureml_py36_tensorflow или azureml_py36_pytorch соответственно. </br>

* Если вы использовали TensorFlow или pytorch в среде PY37 conda, используйте py37_tensorflow или py37_pytorch соответственно. </br>

* В противном случае используйте python37_default или, если вы применяли только azureml_py36, используйте azureml_py36_automl. </br>
