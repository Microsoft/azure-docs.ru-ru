# <a name="security-recommendations"></a>Рекомендации по обеспечению безопасности

Защитник для Интернета вещей сканирует ресурсы Azure и устройства IoT и предоставляет рекомендации по безопасности для сокращения зоны атак.
Рекомендации по безопасности являются практичными и предназначены для помощи клиентам в обеспечении рекомендаций по обеспечению безопасности.

В этой статье вы найдете список рекомендаций, которые можно активировать в центре Интернета вещей и на устройствах IoT.

## <a name="agent-based-recommendations"></a>Рекомендации на основе агентов

Рекомендации по устройствам содержат подробные сведения и рекомендации по улучшению безопасности устройств.

| Статус | Имя | Источник данных | Описание |
|--|--|--|--|
| Средний | Открыть порты на устройстве | Классический модуль безопасности | На устройстве обнаружена конечная точка прослушивания. |
| Средний | В одной из цепочек обнаружена разрешительное политика брандмауэра. | Классический модуль безопасности | Обнаружена разрешенная политика брандмауэра (ВХОДные и выходные данные). Политика брандмауэра должна запрещать весь трафик по умолчанию и определять правила, обеспечивающие необходимое взаимодействие с устройством. |
| Средний | Обнаружено разрешительное правило брандмауэра во входной цепочке | Классический модуль безопасности | Обнаружено правило в брандмауэре, содержащее шаблон разрешительного типа для широкого диапазона IP-адресов или портов. |
| Средний | Обнаружено разрешительное правило брандмауэра в цепочке выходных данных | Классический модуль безопасности | Обнаружено правило в брандмауэре, содержащее шаблон разрешительного типа для широкого диапазона IP-адресов или портов. |
| Средний | Сбой проверки базовых показателей системы для операции | Классический модуль безопасности | Устройство не соответствует [результатам тестирования CIS Linux](https://www.cisecurity.org/cis-benchmarks/). |

### <a name="agent-based-operational-recommendations"></a>Рекомендации по эксплуатации на основе агентов

Рекомендации по эксплуатации предоставляют аналитические сведения и рекомендации по улучшению конфигурации агента безопасности.

| Статус | Имя | Источник данных | Описание |
|--|--|--|--|
| Низкий | Агент отправляет неиспользуемые сообщения | Классический модуль безопасности | 10% или больше сообщений безопасности были меньше 4 КБ за последние 24 часа. |
| Низкий | Конфигурация двойника безопасности не оптимальна | Классический модуль безопасности | Конфигурация двойника безопасности не является оптимальной. |
| Низкий | Конфликт конфигурации двойника безопасности | Классический модуль безопасности | В конфигурации двойника безопасности обнаружены конфликты. |  |


## <a name="built-in-recommendations-in-iot-hub"></a>Встроенные рекомендации в центре Интернета вещей

Оповещения о рекомендациях предоставляют подробные сведения и рекомендации по улучшению безопасности среды.

| Статус | Имя | Источник данных | Описание |
|--|--|--|--|
| Высокий | Идентичные учетные данные для проверки подлинности, используемые несколькими устройствами | Центр Интернета вещей | Учетные данные для проверки подлинности центра Интернета вещей используются несколькими устройствами. Этот процесс может указывать на устройство столкновении, выполняющее олицетворение допустимого устройства. Использование дублирующихся учетных данных увеличивает риск олицетворения устройства вредоносным субъектом. |
| Средний | Политика фильтрации IP-адресов по умолчанию должна быть запрещена | Центр Интернета вещей | Конфигурация IP-фильтра должна иметь правила, определенные для разрешенного трафика, и по умолчанию запретить весь остальной трафик по умолчанию. |
| Средний | Правило фильтрации IP-адресов включает диапазон больших IP-адресов | Центр Интернета вещей | Диапазон IP-адресов источника правила "разрешить IP-фильтры" слишком велик. Более строгие правила могут предоставлять центр Интернета вещей вредоносным субъектам. |
| Низкий | Включение журналов диагностики в центре Интернета вещей | Центр Интернета вещей | Включите журналы и сохраняйте до года. Хранение журналов позволяет воссоздать журнал действий для изучения при возникновении инцидента безопасности или компрометации сети. |
