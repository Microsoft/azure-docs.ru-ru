# <a name="defender-for-iot-glossary"></a>Глоссарий в защитнике IoT

В этом глоссарии содержится краткое описание важных терминов и концепций для платформы "защитник Azure для Интернета вещей". Выберите ссылки **Дополнительные сведения** , чтобы ознакомиться со связанными терминами в глоссарии. Это поможет вам быстрее освоить и использовать средства продукта.

## <a name="a"></a>Объект

| Термин | Описание | Дополнительные сведения |
|--|--|--|
| **Группа доступа** | Поддержка требований к доступу пользователей для больших организаций путем создания правил группы доступа.<br /><br />Правила позволяют управлять доступом к просмотру и конфигурации для локальной консоли управления IoT для конкретных ролей пользователей в соответствующих подразделениях, регионах, сайтах и зонах.<br /><br />Например, разрешите аналитикам безопасности из группы Active Directory доступ к данным «Западная Европа», но не помешает доступу к данным в Африке. | **<br /> <br /> бизнес-единица локальной консоли управления** |

| **Комментарий к предупреждению** | Комментарии, которые аналитики безопасности и администраторы делают в предупредительных сообщениях. Например, комментарий к предупреждению может предоставлять инструкции о действиях по устранению рисков или имена лиц, которые будут обращаться к событию.<br /><br />Пользователи, просматривающие оповещения, могут выбрать комментарий или комментарии, которые лучше соответствуют состоянию события, или шаги, предпринятые для изучения предупреждения. | **получаете** |

| **API** | Позволяет внешним системам получать доступ к данным, обнаруженным защитником для Интернета вещей, и выполнять действия с помощью внешних REST API по SSL-соединениям. | **маркеры доступа** |
| **отчет о векторе атак** | Графическое представление цепочек уязвимостей уязвимых конечных точек в реальном времени.<br /><br />Отчеты позволяют оценить влияние действий по устранению рисков в последовательности атак для определения. Например, можно оценить, нарушает ли обновление системы путь злоумышленника, нарушая цепочку атак или выпустить альтернативный путь атаки. Это определяет приоритеты действий по исправлению и устранению рисков. | **отчет об оценке рисков** |

Эти модули используют аналитику поведения ICS и обработку и анализ данных для непрерывного анализа сетевого трафика для аномалий, вредоносных программ, проблем в работе, нарушений Протокола и отклонения от базовой сетевой активности.<br /><br />Когда подсистема обнаруживает отклонение, запускается оповещение.
