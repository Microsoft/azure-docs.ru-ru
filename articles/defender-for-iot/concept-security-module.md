# <a name="security-module"></a>Модуль безопасности

В этой статье объясняется, как защитник для Интернета вещей использует двойниковы устройств и модули.

## <a name="device-twins"></a>Двойники устройства

Для созданных в Azure решений Интернета вещей двойники устройств играют ключевую роль в управлении устройствами и автоматизации процессов.

Решение "Defender для Интернета вещей" обеспечивает полную интеграцию с платформой управления устройствами Интернета вещей, позволяя управлять состоянием защиты устройства и использовать существующие возможности управления устройствами. Интеграция достигается за счет использования механизма двойника центра Интернета вещей.

Узнайте больше о концепции [двойниковов устройств](../iot-hub/iot-hub-devguide-device-twins.md) в центре Интернета вещей Azure.

## <a name="security-module-twins"></a>Двойников модуля безопасности

Защитник для Интернета вещей поддерживает модуль безопасности двойника для каждого устройства в службе.
Модуль безопасности двойника содержит всю информацию, относящуюся к безопасности устройств для каждого конкретного устройства в решении.
Свойства безопасности устройств хранятся в выделенном модуле безопасности двойника для безопасного обмена данными и для включения обновлений и обслуживания, требующих меньше ресурсов.

Сведения о создании, настройке и настройке двойника см. в статьях [Создание модуля безопасности двойника](quickstart-create-security-twin.md) и [Настройка агентов безопасности](how-to-agent-configuration.md) . Дополнительные сведения о концепции модуля двойников в центре Интернета вещей см. в разделе [Основные сведения о модуле двойников](../iot-hub/iot-hub-devguide-module-twins.md) .
