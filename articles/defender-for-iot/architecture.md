# <a name="azure-defender-for-iot-architecture"></a>Архитектура Azure Defender для Интернета вещей

В этой статье описывается архитектура функциональной системы для безагентного решения Azure Defender для Интернета вещей. Azure Defender для Интернета вещей предлагает два набора возможностей в соответствии с потребностями вашей среды: безагентное решение для организаций и агентное решение для построителей устройств.

## <a name="agentless-solution-for-organizations"></a>Безагентное решение для организаций
### <a name="defender-for-iot-components"></a>Компоненты Defender для Интернета вещей

Azure Defender для Интернета вещей подключается как к облаку Azure, так и к локальным компонентам. Это решение предназначено для обеспечения масштабируемости в крупных и географически распределенных средах с несколькими удаленными расположениями. Это решение позволяет реализовать многоуровневую распределенную архитектуру по странам, регионам, подразделениям или зонам. 

Azure Defender для Интернета вещей содержит следующие компоненты. 

**Среды, подключенные к облаку**

- Виртуальная машина или устройство датчика Azure Defender для Интернета вещей
- Портал Azure для облачного управления и интеграции с Azure Sentinel
- Локальная консоль управления для управления локальным расположением
- Встроенный агент безопасности (необязательно)

**Среды без подключения к Интернету (автономные)**

- Виртуальная машина или устройство датчика Azure Defender для Интернета вещей
- Локальная консоль управления для управления локальным расположением

### <a name="azure-defender-for-iot-sensors"></a>Датчики Azure Defender для Интернета вещей

Датчики Defender для Интернета вещей обнаруживают и постоянно отслеживают сетевые устройства. Датчики собирают сетевой трафик ICS с помощью пассивного мониторинга (без агента) на устройствах IoT и OT. 
 
Специально разработанная для сетей IoT и OT безагентная технология обеспечивает глубокий мониторинг рисков IoT и OT через несколько минут после подключения к сети. Она не оказывает отрицательного влияния на производительность сети и сетевых устройств благодаря отсутствию вмешательства в трафик при анализе сетевого трафика. 
 
Применяя запатентованные средства анализа поведения и технологию Deep Packet Inspection на прикладном уровне 7 в сетях IoT и OT, вы сможете анализировать традиционные решения на основе подписей для немедленного обнаружения сложных угроз IoT и OT (например, вредоносных программ без файлов) из-за аномальных или несанкционированных действий. 
  
Датчики Azure Defender для Интернета вещей подключаются к порту SPAN или к точке доступа к сети и сразу же начинают выполнять анализ DPI в сетях IoT и OT. 
 
Сбор, обработка, анализ данных и оповещения происходят непосредственно на датчике. Этот процесс идеально подходит для расположений со слабой пропускной способностью или подключением с высокой задержкой, поскольку на консоль управления передаются только метаданные.

Датчик обладает пятью аналитическими подсистемами обнаружения. Обработчики запускают оповещения на основе анализа трафика в режиме реального времени или предварительно записанного трафика. Доступны следующие подсистемы. 

#### <a name="protocol-violation-detection-engine"></a>Подсистема обнаружения нарушений протокола
Подсистема обнаружения нарушений протокола определяет использование структур пакетов и значения полей, которые нарушают спецификацию протокола ICS, например: исключение Modbus и предупреждение об инициации кода устаревших функций.

#### <a name="policy-violation-detection-engine"></a>Подсистема обнаружения нарушений политики
С помощью машинного обучения подсистема обнаружения нарушений политики предупреждает пользователей о любых отклонениях от базового поведения, таких как незаконное использование кода определенных функций, доступ к определенным объектам или изменение конфигурации устройства. Например: оповещения об изменении версии ПО DeltaV и несанкционированном программировании PLC. В частности, подсистема нарушений политики моделирует сети ICS как детерминированные последовательности состояний и переходов — с помощью запатентованного метода, именуемого промышленным моделированием конечного состояния. Подсистема обнаружения нарушений политики устанавливает базовые показатели сетей ICS, поэтому для платформы требуется более короткий период обучения для создания базовых показателей сети, чем применение универсальных математических схем или аналитики, изначально разработанных для ИТ, а не для сетей OT.

#### <a name="industrial-malware-detection-engine"></a>Промышленная подсистема обнаружения вредоносных программ
Промышленная подсистема обнаружения вредоносных программ определяет поведение, указывающее на наличие известных вредоносных программ, таких как Conficker, Black Energy, Havex, WannaCry, NotPetya и Triton. 

#### <a name="anomaly-detection-engine"></a>Подсистема обнаружения аномалий
Подсистема обнаружения аномалий обнаруживает необычное взаимодействие и поведение между компьютерами. При моделировании сетей ICS в виде детерминированных последовательностей состояний и переходов платформа требует более короткого периода обучения, чем применение универсальных математических схем или аналитики, изначально разработанных для ИТ, а не для сетей OT. Она также обнаруживает аномалии быстрее, с минимальными ложными срабатываниями. Предупреждения подсистемы обнаружения аномалий активируются в случае чрезмерно частых попыток входа в систему SMB в случае обнаружения сканирования PLC.

#### <a name="operational-incident-detection"></a>Обнаружение рабочих инцидентов
Обнаружение рабочих инцидентов предназначено для обнаружения проблем с работоспособностью, например периодических подключений, которые могут указывать на ранние признаки отказа оборудования. Например, устройство считается отключенным (не отвечает), а команда PLC "Стоп" Siemens S7 отправила оповещения.

### <a name="management-consoles"></a>Консоли управления
Управление Azure Defender для Интернета вещей в гибридных средах осуществляется с помощью двух порталов управления. 
- Консоль датчика
- Локальная консоль управления
- Портал Azure

### <a name="sensor-console"></a>Консоль датчика
Обнаруженные датчиками нарушения отображаются в консоли датчика, где их можно просматривать, исследовать и анализировать на карте сети, в реестре устройств и во множестве подробных отчетов, например отчетов об оценке рисков, запросов на интеллектуальный анализ данных и векторы атак. Кроме того, консоль можно использовать для просмотра и обработки угроз, обнаруженных подсистемами датчиков, переадресации информации в партнерские системы, управления пользователями и т. д.

:::image type="content" source="./media/architecture/sensor-console-v2.png" alt-text="Консоль датчика Azure Defender для Интернета вещей":::

### <a name="on-premises-management-console"></a>Локальная консоль управления
Локальная консоль управления позволяет операторам центра информационной безопасности управлять оповещениями с нескольких датчиков и анализировать их с одной панели мониторинга, а также предоставляет общее представление о работоспособности сетей OT.

Эта архитектура обеспечивает полное унифицированное представление сети на уровне SOC, оптимизированную обработку оповещений и управление сетевой безопасностью, гарантируя тем самым, что управление принятием решений и рисками останется безупречным.

Помимо работы с несколькими клиентами, мониторинга, анализа данных и удаленного управления датчиками, консоль управления предоставляет дополнительные средства обслуживания системы (такие как исключение оповещений) и полностью настраиваемые функции отчетов для каждого из удаленных устройств. Эта архитектура поддерживает локальное управление на уровне сайта, на уровне зоны и глобальное управление в SOC.

Консоль управления можно развернуть для конфигурации высокого уровня доступности, которая предоставляет консоль резервного копирования, периодически получающую резервные копии всех файлов конфигурации, необходимых для восстановления. В случае сбоя основной консоли устройства управления локальным сайтом автоматически переключаются для синхронизации с консолью резервного копирования, чтобы поддерживать доступность без прерывания.

Тесная интеграция с рабочими процессами SOC и объектами Runbook позволяет легко определять приоритеты действий по устранению рисков и учета корреляции угроз.

- Целостность — упрощение благодаря единой унифицированной платформе управления устройствами, рисками и уязвимостью, а также мониторинга угроз с реагированием на инциденты.

- Агрегирование и корреляция — отображение, агрегирование и анализ данных и оповещений, собранных со всех сайтов.

- Управление всеми датчиками — настройка и мониторинг всех датчиков из единого центра.
