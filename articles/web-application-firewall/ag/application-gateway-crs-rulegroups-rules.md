# <a name="web-application-firewall-crs-rule-groups-and-rules"></a>Группы правил и правила CR брандмауэра веб-приложения

Брандмауэр веб-приложения шлюза приложений (WAF) позволяет устранить уязвимости веб-приложений и защитить их от эксплойтов. Это делается с помощью правил, определенных на основе наборов правил OWASP Core, 3,1, 3,0 или 2.2.9. Эти правила можно отключить для каждого правила. Эта статья содержит текущие правила и наборы правил.

## <a name="core-rule-sets"></a>Основные наборы правил

По умолчанию WAF шлюза приложений имеет предварительно настроенный CR 3,0. Но вместо этого можно использовать CR 3,1 или CR 2.2.9. CR 3,1 предлагает новые наборы правил защиты от заражения Java, начальный набор проверок передачи файлов, исправленные ложные срабатывания и многое другое. CR 3,0 предлагает меньше ложных срабатываний по сравнению с CR 2.2.9. Можно также [настроить правила в соответствии со своими потребностями](application-gateway-customize-waf-rules-portal.md).

> [!div class="mx-imgBorder"]
> ![Управление правилами](../media/application-gateway-crs-rulegroups-rules/managed-rules-01.png)

WAF защищает от следующих веб-уязвимостей:

- Атаки путем внедрения кода SQL
- Атаки с использованием межузловых сценариев
- Другие распространенные атаки, такие как внедрение команд, HTTP-запрос несанкционированных, разделение HTTP-ответов и удаленное включение файлов.
- Нарушения протокола HTTP
- Аномалии протокола HTTP, например отсутствие агента пользователя узла и заголовков Accept
- Боты, программы-обходчики и сканеры
- Распространенные нетипичные настройки приложений (например, Apache и IIS)
