# <a name="register-your-verification-method-info-to-reset-your-own-password"></a>Регистрация сведений о способе проверки для сброса собственного пароля

Если вы забыли пароль рабочей или учебной учетной записи, никогда не получали его от службы поддержки своей компании или ваша учетная запись была заблокирована, используйте сведения для защиты и мобильное устройство, чтобы сбросить пароль рабочей или учебной учетной записи.

Администратор должен включить эту возможность, чтобы вы могли зарегистрировать свои сведения и сбросить пароль. Если вы не видите параметр **Я не помню свой пароль**, это означает, что администратор еще не включил эту возможность для вашей организации. Если вы полагаете, что это неправильно, обратитесь в службу технической поддержки за помощью.

>[!Important]
>Эта статья предназначена для пользователей, пытающихся использовать регистрацию для самостоятельного сброса пароля. Это означает, что вы сможете сбросить собственный рабочий или учебный пароль (например alain@contoso.com ,), не требуя помощи администратора. Если вы администратор и ищете сведения о том, как включить самостоятельный сброс пароля для сотрудников или других пользователей, см. [статью о развертывании самостоятельного сброса пароля в AAD и другие статьи](../authentication/howto-sspr-deployment.md).

## <a name="set-up-your-password-reset-verification-method"></a>Настройка способа проверки для сброса пароля

1. Откройте веб-браузер на устройстве и перейдите на [страницу сведений для защиты](https://account.activedirectory.windowsazure.com/PasswordReset/Register.aspx?regref=ssprsetup).

2. В зависимости от того, как администратор настроил вашу организацию, вам будут доступны один или несколько из перечисленных ниже вариантов для настройки способа проверки. Если вариантов несколько, настоятельно рекомендуем использовать более одного способа проверки безопасности на случай, если один из способов станет недоступным.

    - **Приложение для аутентификации**. Выберите в качестве способа проверки безопасности использование Microsoft Authenticator или другого приложения для аутентификации. Дополнительные сведения о настройке приложения см. в статье [Настройка приложения Authenticator в качестве метода проверки](security-info-setup-auth-app.md).

    - **Обмен текстовыми сообщениями**. Выберите этот вариант, чтобы отправить текстовые сообщения на свое мобильное устройство. Дополнительные сведения о настройке текстовых сообщений см. в статье [Настройка текстовых сообщений в качестве метода проверки](security-info-setup-text-msg.md).

    - **Телефонные звонки**. Выберите этот способ проверки, чтобы получить звонок на зарегистрированный номер телефона. Дополнительные сведения о настройке телефонных звонков см. в статье [Настройка номера телефона в качестве метода проверки](security-info-setup-phone-number.md).

    - **Ключ безопасности**. Выберите этот вариант, чтобы использовать совместимый с продуктами Майкрософт ключ безопасности. Дополнительные сведения см. в статье [Настройка ключа безопасности в качестве метода проверки](security-info-setup-security-key.md).

    - **Адрес электронной почты**. Выберите использование альтернативного адреса электронной почты, для которого не нужен забытый или отсутствующий пароль. Этот вариант работает только для сброса пароля, но не в качестве способа проверки безопасности. Дополнительные сведения о настройке адреса электронной почты см. в статье [Настройка адреса электронной почты в качестве метода проверки](security-info-setup-email.md).

    - **Контрольные вопросы**. Выберите этот способ, чтобы указать ответы на заранее определенные контрольные вопросы, которые настроил администратор. Этот вариант работает только для сброса пароля, но не в качестве способа проверки безопасности. Дополнительные сведения о контрольных вопросах см. в статье [Настройка контрольных вопросов в качестве метода проверки](security-info-setup-questions.md).

3. После выбора и настройки способов проверки нажмите **Готово**.

    > [!Note]
    > Если вы добавите такие сведения, как номер телефона или адрес электронной почты, они не попадут в глобальный каталог вашей организации. Эти данные видны только вам и администратору. Ответы на ваши секретные вопросы видите только вы.

 Ниже описаны ошибки, которые встречаются чаще всего, и подсказки по их устранению.

Чтобы сбросить пароль в этом случае, обратитесь за помощью в службу технической поддержки или к администратору организации. |
| Для вашей учетной записи запрещен сброс пароля.<br>К сожалению, ИТ-специалисты не установили для вашей учетной записи возможность использования этой службы.<br>Если хотите, мы можем обратиться к администратору вашей организации для сброса вашего пароля. | Если при вводе ИД пользователя появляется это сообщение об ошибке, это означает, что в вашей организации не включена возможность сброса пароля или вам не разрешено использовать ее. Чтобы сбросить пароль в такой ситуации, щелкните ссылку **Обратитесь к администратору**. При этом в службу технической поддержки или администратору вашей организации будет отправлено сообщение электронной почты, в котором будет указано, что вы хотите сбросить пароль. |
| Не удалось проверить учетную запись.<br>Если хотите, мы можем обратиться к администратору вашей организации для сброса вашего пароля. | Если после ввода ИД пользователя появляется это сообщение об ошибке, это означает, что в вашей организации включена возможность сброса пароля и вы можете использовать ее, но не зарегистрированы для этого. В этом случае вам нужно обратиться за помощью в службу технической поддержки или к администратору организации. Сведения о том, как зарегистрироваться для сброса пароля после возвращения доступа к устройству, см. в описании процесса выше. |
