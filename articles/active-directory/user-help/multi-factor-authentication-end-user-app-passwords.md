# <a name="manage-app-passwords-for-two-step-verification"></a>Управление паролями приложения для двухфакторной проверки подлинности

> [!Important]
>Администратор может запретить использование паролей приложений. Если вы не видите параметр **Пароли приложений**, значит он недоступен для вашей организации.

При использовании паролей приложений необходимо помнить следующее:

- Пароли приложений создаются автоматически и должны быть созданы и введены один раз для каждого приложения.

- Максимальное количество паролей на пользователя — 40. Если вы попытаетесь создать пароль после достижения этого предела, вам будет предложено удалить один из существующих паролей, чтобы создать новый.

    >[!Note]
    >Все клиенты Office 2013 (включая Outlook) поддерживают новые протоколы идентификации и двухфакторную проверку подлинности. Это означает, что после включения двухфакторной проверки подлинности отпадает потребность в использовании паролей приложений для клиентов Office 2013. Дополнительные сведения см. в статье [Современная проверка подлинности в клиентских приложениях Office 2013 и Office 2016](https://support.office.com/article/how-modern-authentication-works-for-office-2013-and-office-2016-client-apps-e4c45989-4b1a-462e-a81b-2a13191cf517).

## <a name="create-new-app-passwords"></a>Создание паролей приложений

Во время начальной регистрации двухфакторной проверки вам предоставляется один пароль приложения. Если требуется несколько паролей, необходимо создать их самостоятельно. Вы можете создавать пароли приложений из нескольких областей в зависимости от того, как в организации настроена двухфакторная проверка подлинности. Дополнительные сведения о регистрации для использования двухфакторной проверки подлинности с рабочей или учебной учетной записью см. в разделе [Обзор двухфакторной проверки и рабочей или учебной учетной записи](multi-factor-authentication-end-user-first-time.md) и связанных с ним статей.

### <a name="where-to-create-and-delete-your-app-passwords"></a>Создание и удаление паролей приложений

Вы можете создавать и удалять пароли приложений в зависимости от того, как используется двухфакторная проверка подлинности:

- **Ваша организация использует двухфакторную проверку подлинности и страницу "Дополнительная проверка безопасности".** Если вы используете рабочую или учебную учетную запись (например, alain@contoso.com) с двухфакторной проверкой подлинности в организации, вы можете управлять паролями приложений на странице [Дополнительная проверка безопасности](https://account.activedirectory.windowsazure.com/Proofup.aspx). Подробные инструкции см. в разделе [Создание и удаление паролей приложений с помощью страницы "Дополнительная проверка безопасности"](#create-and-delete-app-passwords-from-the-additional-security-verification-page) в этой статье.

- **Ваша организация использует двухфакторную проверку подлинности и портал Office 365.** Если вы используете рабочую или учебную учетную запись (например, alain@contoso.com ), двухфакторную проверку подлинности и Microsoft 365 приложения в Организации, вы можете управлять паролями приложения на [странице портала Office 365](https://www.office.com). Подробные инструкции см. в разделе [Создание и удаление паролей приложений на портале Office 365](#create-and-delete-app-passwords-using-the-office-365-portal) в этой статье.

- **Вы используете двухфакторную проверку подлинности с личной учетной записью Майкрософт.** Если вы используете личную учетную запись Майкрософт (например, alain@outlook.com) с двухфакторной проверкой подлинности, вы можете управлять паролями приложений на [странице базовых параметров безопасности](https://account.microsoft.com/security/). Подробные инструкции см. в статье [Использование паролей приложений с приложениями, которые не поддерживают двухфакторную проверку подлинности](https://support.microsoft.com/help/12409/microsoft-account-app-passwords-and-two-step-verification).

## <a name="create-and-delete-app-passwords-from-the-additional-security-verification-page"></a>Создание и удаление паролей приложений на странице дополнительной проверки безопасности

Вы можете создавать и удалять пароли приложений на странице **Дополнительная проверка безопасности** рабочей или учебной учетной записи.

1. Войдите на [страницу "Дополнительная проверка безопасности"](https://account.activedirectory.windowsazure.com/Proofup.aspx) и выберите **Пароли приложений**.

    ![Страница "Пароли приложений" с выделенной вкладкой "Пароли приложений"](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page.png)

2. Нажмите **Создать**, введите имя приложения, которому требуется пароль приложения, а затем выберите **Далее**.

    ![Страница создания паролей приложений с именем приложения, которому требуется пароль](media/multi-factor-authentication-end-user-app-passwords/mfa-create-app-password-page.png)

3. Скопируйте пароль на странице **Ваш пароль приложения** и нажмите **Закрыть**.

    ![Страница пароля приложения с паролем для указанного приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-your-app-password-page.png)

4. На странице **пароли приложений** убедитесь, что приложение указано в списке.

    ![Страница "Пароли приложений" с новым приложением в списке](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page-with-new-password.png)  

5. Откройте приложение, для которого вы создали пароль (например, Outlook 2010), а затем вставьте пароль приложения при появлении запроса. Это необходимо сделать только один раз для каждого приложения.

### <a name="to-delete-an-app-password-using-the-app-passwords-page"></a>Удаление пароля приложения на странице "Пароли приложений"

1. На странице **пароли приложений** выберите **Удалить** рядом с паролем приложения, которое нужно удалить.

   ![Снимок экрана, на котором показано удаление пароля приложения на странице "пароли приложений"](media/multi-factor-authentication-end-user-app-passwords/mfa-app-passwords-page-delete.png)

2. Щелкните **Да**, чтобы подтвердить удаление пароля, а затем выберите **Закрыть**.

    Пароль приложения успешно удален.

## <a name="create-and-delete-app-passwords-using-the-office-365-portal"></a>Создание и удаление паролей приложений с помощью портала Office 365

Если вы используете двухфакторную проверку подлинности с рабочей или учебной учетной записью и Microsoft 365 приложениями, вы можете создавать и удалять пароли приложений с помощью портала Office 365.

### <a name="to-create-app-passwords-using-the-office-365-portal"></a>Создание паролей приложений с помощью портала Office 365

1. Войдите в рабочую или учебную учетную запись, перейдите на [страницу Моя учетная запись](https://myaccount.microsoft.com)и выберите **сведения о безопасности**.

    ![Портал Office, на котором отображается вкладка "сведения о безопасности"](media/multi-factor-authentication-end-user-app-passwords/mfa-security-info.png)

2. Выберите **Добавить метод**, выберите **пароль приложения** в раскрывающемся списке и нажмите кнопку **добавить**.

    ![Страница «сведения о безопасности» с добавлением метода дровпдовн List](media/multi-factor-authentication-end-user-app-passwords/mfa-add-method.png)

3. Введите имя для пароля приложения, а затем нажмите кнопку **Далее**.

    ![Страница создания паролей приложений с именем пароля приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-enter-app-password-name.png)

4. Скопируйте пароль на странице **пароль приложения** и нажмите кнопку **Готово**.

    ![Страница пароля приложения с новым созданным паролем приложения](media/multi-factor-authentication-end-user-app-passwords/mfa-copy-app-password.png)

5. На странице **сведения о безопасности** убедитесь, что указан пароль приложения.

    ![Страница "сведения о безопасности" с новым паролем приложения, показанным в списке](media/multi-factor-authentication-end-user-app-passwords/mfa-verify-app-password.png)  

6. Откройте приложение, для которого вы создали пароль приложения (например, Outlook 2016), а затем вставьте пароль приложения при появлении запроса. Это необходимо сделать только один раз для каждого приложения.

### <a name="to-delete-app-passwords-using-the-security-info-page"></a>Удаление паролей приложений с помощью страницы «сведения о безопасности»

1. На странице **сведения о безопасности** выберите **Удалить** рядом с паролем приложения, которое нужно удалить.

   ![Снимок экрана, на котором показано удаление пароля приложения на странице "сведения о безопасности"](media/multi-factor-authentication-end-user-app-passwords/mfa-delete-app-password.png)

2. В поле подтверждения нажмите кнопку **ОК** .

    Пароль приложения успешно удален.

## <a name="if-your-app-passwords-arent-working-properly"></a>Действия в случае неправильного функционирования паролей приложений

Убедитесь, что пароль был правильно введен. Если вы уверены, что правильно ввели свой пароль, попробуйте снова войти в систему и создать новый пароль приложения. Если ни один из этих вариантов не устранит вашу проблему, обратитесь в службу технической поддержки вашей организации, чтобы удалить существующие пароли приложений и создать новые.
