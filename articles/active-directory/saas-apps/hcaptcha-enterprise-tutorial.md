# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-hcaptcha-enterprise"></a>Руководство по интеграции единого входа Azure Active Directory с hCaptcha Enterprise

В этом руководстве описано, как интегрировать hCaptcha Enterprise с Azure Active Directory (AAD). Интеграция hCaptcha Enterprise с Azure AD обеспечивает следующие возможности.

* Управление доступом к hCaptcha Enterprise с помощью AAD.
* Автоматический вход пользователей в hCaptcha Enterprise с учетными записями Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка hCaptcha Enterprise с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* hCaptcha Enterprise поддерживает единый вход, инициируемый **поставщиком услуг и поставщиком удостоверений**.
* hCaptcha Enterprise поддерживает **JIT**-подготовку пользователей.

## <a name="adding-hcaptcha-enterprise-from-the-gallery"></a>Добавление hCaptcha Enterprise из коллекции

Чтобы настроить интеграцию hCaptcha Enterprise с Azure AD, необходимо добавить hCaptcha Enterprise из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **hCaptcha Enterprise**.
1. Выберите **hCaptcha Enterprise** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-hcaptcha-enterprise"></a>Настройка и проверка единого входа AAD для hCaptcha Enterprise

Настройте и проверьте единый вход AAD в hCaptcha Enterprise с помощью тестового пользователя **B.Simon**. Чтобы обеспечить работу единого входа, нужно установить связь между пользователем AAD и соответствующим пользователем в hCaptcha Enterprise.

Чтобы настроить и проверить единый вход Azure AD в hCaptcha Enterprise, выполните следующие действия:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в hCaptcha Enterprise](#configure-hcaptcha-enterprise-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя hCaptcha Enterprise](#create-hcaptcha-enterprise-test-user)** требуется для того, чтобы в hCaptcha Enterprise существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **hCaptcha Enterprise** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** для изменения параметров.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. Если вы хотите настроить приложение в режиме, инициируемом **поставщиком удостоверений**, в разделе **Базовая конфигурация SAML** введите значения следующих полей.

    а. В текстовом поле **Идентификатор** введите URL-адрес в формате `https://accounts.hcaptcha.com/org/<YOUR_SLUG>/saml/callback`.

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://accounts.hcaptcha.com/org/<YOUR_SLUG>/saml/callback`.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг**, щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес в формате `https://dashboard.hcaptcha.com/org/<YOUR_SLUG>/login`.

    > [!NOTE]
    > Эти значения приведены для примера. Замените их фактическими значениями идентификатора, URL-адреса ответа и URL-адреса входа. Чтобы получить эти значения, обратитесь в [службу поддержки клиентов hCaptcha Enterprise](mailto:support@hcaptcha.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. Приложение hCaptcha Enterprise ожидает проверочные утверждения SAML в определенном формате, поэтому нужно добавить настраиваемые сопоставления атрибутов в конфигурацию атрибутов токена SAML. На следующем снимке экрана показан список атрибутов по умолчанию.

    ![Изображение](common/default-attributes.png)

1. В дополнение к описанному выше приложение hCaptcha Enterprise ожидает несколько дополнительных атрибутов в ответе SAML, как показано ниже. Эти атрибуты также заранее заполнены, но вы можете изменить их в соответствии со своими требованиями.
    
    | Имя |  Исходный атрибут|
    | ----------------- | --------- |
    | groups | user.groups |

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе вы включите единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к hCaptcha Enterprise.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **hCaptcha Enterprise**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-hcaptcha-enterprise-sso"></a>Настройка единого входа в hCaptcha Enterprise

Чтобы настроить единый вход на стороне **hCaptcha Enterprise**, нужно отправить **URL-адрес метаданных федерации приложений** в [службу поддержки hCaptcha Enterprise](mailto:support@hcaptcha.com). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

### <a name="create-hcaptcha-enterprise-test-user"></a>Создание тестового пользователя hCaptcha Enterprise

В этом разделе вы создадите в hCaptcha Enterprise пользователя B.Simon. Приложение hCaptcha Enterprise поддерживает JIT-подготовку пользователей, которая включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. Если пользователь еще не существует в hCaptcha Enterprise, он создается после проверки подлинности.

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов. 

#### <a name="sp-initiated"></a>Инициация поставщиком услуг:

* Выберите **Тестировать приложение** на портале Azure. Вы будете перенаправлены по URL-адресу для входа в hCaptcha Enterprise, где можно инициировать поток входа.  

* Перейдите по URL-адресу для входа в hCaptcha Enterprise и инициируйте поток входа.

#### <a name="idp-initiated"></a>Вход, инициированный поставщиком удостоверений

* Выберите элемент **Тестировать приложение** на портале Azure, и вы автоматически войдете в приложение hCaptcha Enterprise, для которого настроили единый вход. 

Вы можете также использовать портал "Мои приложения" корпорации Майкрософт для тестирования приложения в любом режиме. Щелкнув плитку hCaptcha Enterprise в разделе "Мои приложения", вы автоматически перейдете на страницу входа приложения для инициирования потока входа (при настройке в режиме поставщика службы) или в приложение hCaptcha Enterprise, для которого настроили единый вход (при настройке в режиме поставщика удостоверений). Дополнительные сведения о портале "Мои приложения" см. в [этой статье](../user-help/my-apps-portal-end-user-access.md).
