# <a name="tutorial-azure-active-directory-single-sign-on-integration-with-awarego"></a>Руководство по интеграции единого входа Azure Active Directory в AwareGo

В этом руководстве описано, как интегрировать AwareGo с Azure Active Directory (Azure AD). Интеграция AwareGo с Azure AD обеспечивает следующие возможности:

* С помощью Azure AD вы можете контролировать доступ к AwareGo.
* Автоматический вход пользователей в AwareGo с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка AwareGo с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде. AwareGo поддерживает единый вход, инициированный поставщиком услуг.


## <a name="adding-awarego-from-the-gallery"></a>Добавление AwareGo из коллекции

Чтобы настроить интеграцию AwareGo с Azure AD, необходимо добавить AwareGo из коллекции в список управляемых приложений SaaS (программное обеспечение как услуга).

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области слева выберите службу **Azure Active Directory**.
1. Выберите **Корпоративные приложения** > **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **AwareGo**.
1. В области результатов выберите **AwareGo** и добавьте это приложение. Через несколько секунд приложение будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-awarego"></a>Настройка и проверка единого входа Azure AD для AwareGo

Настройте и проверьте единый вход Azure AD в AwareGo с помощью тестового пользователя **B. Simon**. Чтобы обеспечить работу единого входа, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в AwareGo.

Чтобы настроить и проверить единый вход Azure AD в AwareGo, сделайте следующее:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.  

    а. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя B. Simon.  
    b. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы разрешить пользователю B. Simon использовать единый вход Azure AD.  

1. **[Настройка единого входа в AwareGo](#configure-awarego-sso)** необходима для настройки параметров единого входа на стороне приложения.

    a. **[Создание тестового пользователя приложения AwareGo](#create-an-awarego-test-user)** требуется для того, чтобы в AwareGo существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.  
    b. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Чтобы включить единый вход Azure AD на портале Azure, выполните следующие действия:

1. На портале Azure на странице интеграции с приложением **AwareGo** в разделе **Управление** выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. Чтобы изменить параметры, на панели **Настройка единого входа с помощью SAML** нажмите кнопку **Изменить**.

   ![Снимок экрана: кнопка "Правка" для параметра "Базовая конфигурация SAML".](common/edit-urls.png)

1. В области редактирования в разделе **Базовая конфигурация SAML** сделайте следующее:

    a. В текстовом поле **URL-адрес для входа** введите один из следующих URL-адресов:

    * `https://lms.awarego.com/auth/signin/` 
    * `https://my.awarego.com/auth/signin/`

    b. В текстовом поле **Идентификатор (сущности)** введите URL-адрес в следующем формате: `https://<SUBDOMAIN>.awarego.com`

    c. В поле **URL-адрес ответа** введите URL-адрес в следующем формате: `https://<SUBDOMAIN>.awarego.com/auth/sso/callback`

    > [!NOTE]
    > Приведенные выше значения используются только для примера. Замените их фактическими значениями идентификатора и URL-адресами ответа. Чтобы получить эти значения, обратитесь в [группу поддержки AwareGo](mailto:support@awarego.com). Вы можете также просмотреть примеры в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** напротив параметра **Сертификат (Base64)** щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Снимок экрана: ссылка "Скачать" для скачивания сертификата в области "Сертификат для подписи SAML".](common/certificatebase64.png)

1. В разделе **Настройка AwareGo** скопируйте один или несколько URL-адресов в зависимости от требований.

    ![Снимок экрана: область "Настройка AwareGo" для копирования URL-адресов конфигурации.](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, а затем **Пользователи** > **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. На панели свойств **Пользователь** сделайте следующее:

   а. В поле **Имя** введите **B.Simon**.  
   b. В поле **Имя пользователя** введите имя пользователя в следующем формате: `<username>@<companydomain>.<extension>` (например, B.Simon@contoso.com).  
   c. Установите флажок **Показать пароль** и сохраните значение, которое отображается в поле **Пароль**, для дальнейшего использования.  
   d. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B. Simon, предоставив этому пользователю доступ к AwareGo.

1. На портале Azure выберите **Корпоративные приложения** > **Все приложения**.
1. Из списка **Приложения** выберите **AwareGo**.
1. В разделе **Управление** на странице сводных данных о приложении выберите **Пользователи и группы**:
1. Выберите **Добавить пользователя**, а затем на панели **Добавление назначения** щелкните **Пользователи и группы**.
1. В области **Пользователи и группы** из списка **Пользователи** выберите **B. Simon**, а затем нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, ее можно выбрать в списке **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль *Доступ по умолчанию*.
1. В области **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-awarego-sso"></a>Настройка единого входа AwareGo

Чтобы настроить единый вход на стороне **AwareGo**, отправьте ранее скачанный **сертификат (Base64)** и URL-адреса, скопированные ранее на портале Azure, в [группу поддержки AwareGo](mailto:support@awarego.com). Специалисты группы поддержки создают этот параметр для правильного подключения единого входа SAML для приложения на обеих сторонах.

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD одним из следующих способов: 

* На портале Azure выберите **Проверить это приложение**. Вы будете перенаправлены на страницу входа AwareGo, на которой можно инициировать поток входа. 

* Перейдите непосредственно на страницу входа в AwareGo и инициируйте поток входа.

* Перейдите на портал "Мои приложения" корпорации Майкрософт. При выборе плитки **AwareGo** на портале "Мои приложения" вы будете перенаправлены на страницу входа AwareGo. Дополнительные сведения см. в статье о [входе на портал "Мои приложения" и запуске приложений на нем](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction).
