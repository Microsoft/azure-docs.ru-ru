# <a name="tutorial-azure-active-directory-integration-with-mind-tools-toolkit"></a>Руководство по интеграции Azure Active Directory с набором средств Mind Tools

В этом руководстве описано, как интегрировать набор средств Mind Tools с Azure Active Directory (Azure AD).

Такая интеграция предоставит следующие возможности:

* Контроль доступа к набору средств Mind Tools с помощью Azure AD.
* Поддержка автоматического входа пользователей в Mind Tools с учетными записями Azure AD.
* Централизованное управление учетными записями через портал Azure.

Дополнительные сведения об интеграции приложений SaaS с Azure AD см. в статье о [едином входе в приложения с помощью Azure Active Directory](../manage-apps/what-is-single-sign-on.md).

## <a name="prerequisites"></a>Предварительные требования

Чтобы настроить интеграцию Azure AD с набором средств Mind Tools, вам потребуется:

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка на набор средств Mind Tools с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Набор средств Mind Tools поддерживает единый вход, инициируемый поставщиком услуг.
* Набор средств Mind Tools поддерживает JIT-подготовку пользователей.
* После настройки набора средств Mind Tools вы сможете включить управление сеансом. Этот механизм защищает от утечки данных и несанкционированного доступа к конфиденциальным данным вашей организации в режиме реального времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="add-mind-tools-toolkit-from-the-gallery"></a>Добавление набора средств Mind Tools из коллекции

Чтобы настроить интеграцию набора средств Mind Tools с Azure AD, необходимо добавить этот набор средств из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Mind Tools Toolkit**.
1. Выберите **набор средств Mind Tools** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on"></a>Настройка и проверка единого входа в Azure AD

В этом разделе описана настройка и проверка единого входа Azure AD в набор средств Mind Tools с использованием данных тестового пользователя **B.Simon**. Для работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Mind Tools Toolkit.

Чтобы настроить и проверить единый вход Azure AD в набор средств Mind Tools, выполните действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Mind Tools Toolkit](#configure-mind-tools-toolkit-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя для Mind Tools Toolkit](#create-a-mind-tools-toolkit-test-user)** требуется для того, чтобы в этом наборе средств существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

### <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

В этом разделе вы настроите единый вход Azure AD в набор средств Mind Tools, выполнив следующие шаги:

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с **набором средств Mind Tools** щелкните **Единый вход**.

    ![Раздел "Управление", где выделен "Единый вход"](common/select-sso.png)

1. В диалоговом окне **Выбор метода единого входа** выберите **SAML/WS-Fed**, чтобы включить единый вход.

    ![Диалоговое окно для выбора метода единого входа с выделенным элементом SAML](common/select-saml-option.png)

1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить эти параметры.

    ![Страница "Настройка единого входа с помощью SAML", где выделен значок карандаша для перехода к базовой конфигурации SAML](common/edit-urls.png)

1. В разделе **Базовая конфигурация SAML** в поле **URL-адрес для входа** введите URL-адрес в следующем формате `https://app.goodpractice.net/#/<subscriptionUrl>/s/<locationId>`:

    > [!NOTE]
    > Значение **URL-адрес входа** приведено здесь только для примера. Вместо него необходимо указать фактический URL-адрес входа. Чтобы получить это значение, обратитесь в [службу поддержки клиентов набора средств Mind Tools](mailto:support@goodpractice.com).

1. На странице **Настройка единого входа с помощью SAML** перейдите в раздел **Сертификат подписи SAML**. Справа от поля **XML метаданных федерации** выберите **Скачать**, чтобы скачать XML-файл и сохранить его на локальном компьютере. Содержимое XML зависит от выбранных параметров.

    ![Раздел "Сертификат подписи SAML", где выделена кнопка "Скачать" рядом с полем "XML метаданных федерации"](common/metadataxml.png)

1. В разделе **Настройка Mind Tools Toolkit** скопируйте любой из следующих URL-адресов.

    * **Login URL** (URL-адрес для входа)

    * **идентификатор Azure AD**;

    * **URL-адрес выхода**.

    ![Раздел "Настройка Mind Tools Toolkit", где выделены URL-адреса конфигурации](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как создать на портале Azure тестового пользователя с именем B.Simon.

1. На панели слева на портале Azure выберите **Azure Active Directory** > **Пользователи** > **Все пользователи**.
1. В верхней части экрана щелкните **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите **B.Simon**.  
   1. В поле **Имя пользователя** введите **B.Simon@** _домен_компании_ **.** _расширение_. Например, B.Simon@contoso.com.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **создания**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B.Simon, предоставив этому пользователю доступ к набору средств Mind Tools.

1. На портале Azure выберите **Корпоративные приложения** > **Все приложения**.
1. В списке приложений выберите **набор средств Mind Tools**.
1. На странице "Обзор" приложения перейдите в раздел **Управление** и выберите **Пользователи и группы**.

   ![Раздел "Управление" с выделенным пунктом "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**. В диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

   ![Раздел "Пользователи и группы" с выделенным элементом "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B. Simon** из списка пользователей. Затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор ролей** нужно выбрать соответствующую роль для пользователя из списка. Затем в нижней части экрана нажмите кнопку **Выбрать**.
1. В диалоговом окне **Добавление назначения** выберите **Назначить**.

## <a name="configure-mind-tools-toolkit-sso"></a>Настройка единого входа для набора средств Mind Tools

Для настройки единого входа на стороне **набора средств Mind Tools** необходимо отправить скачанный файл **XML метаданных федерации** и скопированные ранее URL-адреса [службе поддержки набора средств Mind Tools](mailto:support@goodpractice.com). Специалисты службы поддержки правильно настроят подключение единого входа SAML на обеих сторонах подключения.

### <a name="create-a-mind-tools-toolkit-test-user"></a>Создание тестового пользователя для Mind Tools Toolkit

В этом разделе описано, как в наборе средств Mind Tools создать пользователя с именем B.Simon.

Набор средств Mind Tools поддерживает JIT-подготовку, и эта функция включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. Если пользователь в наборе средства Mind Tools еще не существует, он создается при попытке доступа к набору.

### <a name="test-sso"></a>Проверка единого входа

В этом разделе объясняется, как проверить конфигурацию единого входа Azure AD с помощью портала "Мои приложения".

Щелкнув плитку Mind Tools Toolkit на портале "Мои приложения", вы автоматически войдете в набор средств, для которого настроили единый вход. Дополнительные сведения о портале "Мои приложения" см. в статье [Вход и запуск приложений с портала "Мои приложения"](../user-help/my-apps-portal-end-user-access.md).
