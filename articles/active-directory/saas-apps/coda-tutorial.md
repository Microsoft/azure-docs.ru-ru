# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-coda"></a>Руководство. Интеграция единого входа Azure Active Directory с Coda

В этом учебнике описано, как интегрировать Coda с Azure Active Directory (Azure AD). После интеграции Coda с Azure AD доступны следующие возможности.

* Управление доступом к Coda с помощью Azure AD.
* Автоматический вход пользователей в Coda с использованием учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка (Enterprise) на Coda с поддержкой единого входа и отключенной интеграцией GDrive. Обратитесь в службу поддержки [Coda](mailto:support@coda.io), чтобы отключить интеграцию GDrive для вашей организации, если она включена в данный момент.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Coda поддерживает единый вход, инициируемый **поставщиком удостоверений**.

* Coda поддерживает **JIT**-подготовку пользователей.

* После настройки Coda можно применять элементы управления сеансами, которые защищают от хищения и несанкционированного доступа к конфиденциальным данным вашей организации в режиме реального времени. Элементы управления сеансом являются расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-coda-from-the-gallery"></a>Добавление Coda из коллекции

Чтобы настроить интеграцию Coda с Azure AD, необходимо добавить Coda из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Add from the gallery** (Добавление из коллекции) в поле поиска введите **Coda**.
1. Выберите **Coda** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-single-sign-on-for-coda"></a>Настройка и проверка единого входа Azure AD для Coda

Настройте и проверьте единый вход Azure AD в Coda с использованием данных тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Coda.

Чтобы настроить и проверить единый вход Azure AD в Coda, выполните действия в следующих стандартных блоках.

1. **[Начать настройку единого входа в Coda](#begin-configuration-of-coda-sso)** необходимо, чтобы начать настройку единого входа в Coda.
1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
   * **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
   * **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Coda](#configure-coda-sso)** необходима, чтобы завершить настройку единого входа в Coda.
   * **[Создание тестового пользователя приложения Coda](#create-coda-test-user)** необходимо для того, чтобы в Coda существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="begin-configuration-of-coda-sso"></a>Начало настройки единого входа в Coda

Чтобы начать, выполните следующие действия в Coda.

1. В Coda откройте панель **Параметры организации**.

   ![Открытие параметров организации](media/coda-tutorial/org-settings.png)

1. Убедитесь, что для вашей организации отключена интеграция с GDrive. Если в настоящее время она включена, обратитесь в [службу поддержки Coda](mailto:support@coda.io), чтобы упростить миграцию с GDrive.

   ![GDrive отключен](media/coda-tutorial/gdrive-off.png)

1. В разделе **Authenticate with SSO (SAML)** (Проверка подлинности с помощью единого входа (SAML)) выберите параметр **Configure SAML** (Настройка SAML).

   ![Настройка SAML](media/coda-tutorial/saml-settings-link.png)

1. Обратите внимание на значения для **Идентификатор сущности** и **SAML Response URL** (URL-адрес ответа SAML), который потребуется в последующих шагах.

   ![Идентификатор сущности и URL-адрес ответа SAML для использования в Azure](media/coda-tutorial/azure-settings.png)

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Coda** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Настройка единого входа с помощью SAML** введите значения для следующих полей:

   а. В текстовом поле **Идентификатор** введите идентификатор сущности, указанный выше. Он должен соответствовать такому шаблону: `https://coda.io/samlId/<CUSTOMID>`

   b. В текстовом поле **URL-адрес ответа** введите URL-адрес ответа SAML, указанный выше. Он должен соответствовать такому шаблону: `https://coda.io/login/sso/saml/<CUSTOMID>/consume`

   > [!NOTE]
   > Значения будут отличаться от указанных выше — ваши значения можно найти в консоли "Настройка SAML" в Coda. Измените их на фактические значения идентификатора и URL-адреса ответа.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

   ![Ссылка для скачивания сертификата](common/certificatebase64.png)

1. Требуемые URL-адреса можно скопировать из раздела **Настройка Coda**.

   ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к Coda.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. Из списка приложений выберите **Coda**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

   ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-coda-sso"></a>Настройка единого входа в Coda

Для завершения установки необходимо ввести значения из Azure Active Directory на панели **Configure Saml** (Настройка SAML) в Coda.

1. В Coda откройте панель **Параметры организации**.
1. В разделе **Authenticate with SSO (SAML)** (Проверка подлинности с помощью единого входа (SAML)) выберите параметр **Configure SAML** (Настройка SAML).
1. Задайте для параметра **SAML Provider** (Поставщик SAML) значение **Azure Active Directory**.
1. В поле **Identity Provider Login URL** (URL-адрес входа поставщика удостоверений) вставьте значение **Login URL** (URL-адрес входа) из консоли Azure.
1. В поле **Identity Provider Issuer** (Издатель поставщика удостоверений) вставьте значение **Azure AD Identifier** (Идентификатор Azure AD) из консоли Azure.
1. В меню **Identity Provider Public Certificate** (Общедоступный сертификат поставщика удостоверений) выберите элемент **Upload Certificate** (Передать сертификат) и выберите загруженный ранее файл сертификата.
1. Щелкните **Сохранить**.

Этим завершается настройка подключения единого входа SAML.

### <a name="create-coda-test-user"></a>Создание тестового пользователя Coda

В этом разделе вы создадите в Coda пользователя с именем Britta Simon. Приложение Coda поддерживает JIT-подготовку пользователей, которая включена по умолчанию.
