# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-greenlight-compliant-access-management"></a>Руководство. Интеграция единого входа Azure Active Directory с приложением "Управление доступом в соответствии с требованиями Greenlight"

В этом учебнике описано, как интегрировать приложение "Управление доступом в соответствии с требованиями Greenlight" с Azure Active Directory (Azure AD). Интеграция приложения "Управление доступом в соответствии с требованиями Greenlight" с Azure AD обеспечивает следующие возможности.

* Управление доступом к приложению "Управление доступом в соответствии с требованиями Greenlight" в Azure AD.
* Автоматический вход пользователей в приложение "Управление доступом в соответствии с требованиями Greenlight" с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка на приложение "Управление доступом в соответствии с требованиями Greenlight" с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Приложение "Управление доступом в соответствии с требованиями Greenlight" поддерживает единый вход, инициированный **поставщиком услуг и поставщиком удостоверений.** .
* После настройки приложения "Управление доступом в соответствии с требованиями Greenlight" можно применять элементы управления сеансами, которые защищают от хищения и несанкционированного доступа к конфиденциальным данным вашей организации в режиме реального времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-greenlight-compliant-access-management-from-the-gallery"></a>Добавление приложения "Управление доступом в соответствии с требованиями Greenlight" из коллекции

Чтобы настроить интеграцию приложения "Управление доступом в соответствии с требованиями Greenlight" с Azure AD, добавьте это приложение из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Управление доступом в соответствии с требованиями Greenlight**.
1. Выберите **Управление доступом в соответствии с требованиями Greenlight** на панели результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-greenlight-compliant-access-management"></a>Настройка и проверка единого входа Azure AD для приложения "Управление доступом в соответствии с требованиями Greenlight"

Настройте и проверьте единый вход Azure AD в приложение "Управление доступом в соответствии с требованиями Greenlight" с помощью тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в приложении "Управление доступом в соответствии с требованиями Greenlight".

Чтобы настроить и проверить единый вход Azure AD в приложение "Управление доступом в соответствии с требованиями Greenlight", выполните действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    * **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    * **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в приложение "Управление доступом в соответствии с требованиями Greenlight"](#configure-greenlight-compliant-access-management-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    * **[Создание тестового пользователя приложения "Управление доступом в соответствии с требованиями Greenlight"](#create-greenlight-compliant-access-management-test-user)** требуется для того, чтобы в приложении "Управление доступом в соответствии с требованиями Greenlight" существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Управление доступом в соответствии с требованиями Greenlight** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. Если вы хотите настроить приложение в режиме, инициируемом **поставщиком удостоверений**, в разделе **Базовая конфигурация SAML** введите значения следующих полей.

    а. В текстовом поле **Идентификатор** введите URL-адрес в формате `https://<CUSTOMER>.greenlightcorp.com/ebcpresq/checkLoginSAML.do`.

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://<CUSTOMER>.greenlightcorp.com/ebcpresq/checkLoginSAML.do`.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг**, щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес в формате `https://<CUSTOMER>.greenlightcorp.com/ebcpresq/checkLoginSAML.do`.

    > [!NOTE]
    > Эти значения приведены для примера. Замените их фактическими значениями идентификатора, URL-адреса ответа и URL-адреса входа. Чтобы получить эти значения, обратитесь в [службу технической поддержки приложения "Управление доступом в соответствии с требованиями Greenlight"](mailto:support@greenlightcorp.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите элемент **XML метаданных федерации** и выберите **Скачать**, чтобы скачать сертификат и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/metadataxml.png)

1. Требуемые URL-адреса можно скопировать из раздела **Настройка приложения "Управление доступом в соответствии с требованиями Greenlight"** .

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B.Simon, предоставив этому пользователю доступ к приложению "Управление доступом в соответствии с требованиями Greenlight".

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Управление доступом в соответствии с требованиями Greenlight**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-greenlight-compliant-access-management-sso"></a>Настройка единого входа в приложение "Управление доступом в соответствии с требованиями Greenlight"

Чтобы настроить единый вход на стороне приложения **Управление доступом в соответствии с требованиями Greenlight**, нужно отправить скачанный файл **XML метаданных федерации** и соответствующие URL-адреса, скопированные на портале Azure, в [службу технической поддержки приложения "Управление доступом в соответствии с требованиями Greenlight"](mailto:support@greenlightcorp.com). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

### <a name="create-greenlight-compliant-access-management-test-user"></a>Создание тестового пользователя приложения "Управление доступом в соответствии с требованиями Greenlight"

В этом разделе описано, как создать пользователя B.Simon в приложении "Управление доступом в соответствии с требованиями Greenlight". Чтобы добавить пользователей на платформу "Управление доступом в соответствии с требованиями Greenlight", обратитесь в [службу технической поддержки приложения "Управление доступом в соответствии с требованиями Greenlight"](mailto:support@greenlightcorp.com). Перед использованием единого входа необходимо создать и активировать пользователей.
