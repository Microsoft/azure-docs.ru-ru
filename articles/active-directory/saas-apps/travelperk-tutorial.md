# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-travelperk"></a>Руководство по интеграции единого входа Azure Active Directory с TravelPerk

В этом учебнике описано, как интегрировать приложение TravelPerk с Azure Active Directory (Azure AD). Интеграция TravelPerk с Azure AD обеспечивает следующие возможности:

* Контроль доступа к TravelPerk с помощью Azure AD.
* Автоматический вход пользователей в TravelPerk с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Учетная запись TravelPerk с подпиской уровня Premium.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* TravelPerk поддерживает единый вход, инициированный **поставщиком услуг**.

* TravelPerk поддерживает **JIT**-подготовку пользователей

## <a name="adding-travelperk-from-the-gallery"></a>Добавление TravelPerk из коллекции

Чтобы настроить интеграцию TravelPerk с Azure AD, необходимо добавить TravelPerk из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **TravelPerk**.
1. Выберите **TravelPerk** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-travelperk"></a>Настройка и проверка единого входа Azure AD для TravelPerk

Настройте и проверьте единый вход Azure AD в TravelPerk с помощью тестового пользователя **B. Simon**. Чтобы обеспечить работу единого входа, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в TravelPerk.

Чтобы настроить и проверить единый вход Azure AD в TravelPerk, выполните действия из следующих стандартных блоков:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в TravelPerk](#configure-travelperk-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя приложения TravelPerk](#create-travelperk-test-user)** требуется для того, чтобы в TravelPerk существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **TravelPerk** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    а. В текстовом поле **URL-адрес для входа** введите URL-адрес в следующем формате: `https://<COMPANY>.travelperk.com/`.

    b. В текстовом поле **Идентификатор (сущности)** введите URL-адрес в следующем формате: `https://<COMPANY>.travelperk.com/accounts/saml2/metadata/<APPLICATION_ID>`.

    c. В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://<COMPANY>.travelperk.com/accounts/saml2/callback/<APPLICATION_ID>/?acs`.

    > [!NOTE]
    > Эти значения приведены для примера. Вместо них необходимо указать фактические значения URL-адреса входа, URL-адреса ответа и идентификатора. Эти значения можно найти в учетной записи TravelPerk: выберите **Company Settings (Параметры компании)**  > **Integrations (Интеграции)**  > **Single Sign On (Единый вход)** . За помощью обращайтесь в [центр поддержки TravelPerk](https://support.travelperk.com/hc/en-us/articles/360052450271-How-can-I-setup-SSO-for-Azure-SAML-).

1. Приложение TravelPerk ожидает проверочные утверждения SAML в определенном формате, который требует добавить настраиваемые сопоставления атрибутов в вашу конфигурацию атрибутов токена SAML. На следующем снимке экрана показан список атрибутов по умолчанию. В сопоставлении по умолчанию **emailaddress** сопоставлен с **user.mail**. Но приложение TravelPerk ожидает сопоставления **emailaddress** с **user.userprincipalname**. Для TravelPerk необходимо изменить сопоставление атрибутов: щелкните значок **Правка** и измените сопоставление атрибута. Чтобы изменить атрибут, просто щелкните его для открытия в режиме правки.

    ![Изображение](common/default-attributes.png)

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите элемент **XML метаданных федерации** и выберите **Скачать**, чтобы скачать сертификат и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/metadataxml.png)

1. Требуемые URL-адреса можно скопировать из раздела **Настройка TravelPerk**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)
### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B. Simon, предоставив этому пользователю доступ к TravelPerk.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. Из списка приложений выберите **TravelPerk**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-travelperk-sso"></a>Настройка единого входа для TravelPerk

Чтобы настроить единый вход на стороне **TravelPerk**, нужно отправить скачанный **XML-файл метаданных федерации** и соответствующие URL-адреса, скопированные на портале Azure, в [группу поддержки TravelPerk](mailto:trex@travelperk.com). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

### <a name="create-travelperk-test-user"></a>Создание тестового пользователя TravelPerk

В этом разделе описано, как в приложении TravelPerk создать пользователя с именем B. Simon. Приложение TravelPerk поддерживает JIT-подготовку. Эта функция включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. Если пользователь еще не существует в TravelPerk, он создается при попытке доступа к приложению TravelPerk.

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов. 

1. Выберите **Тестировать приложение** на портале Azure. Вы будете перенаправлены по URL-адресу для входа в TravelPerk, где можно инициировать поток входа. 

2. Перейдите по URL-адресу для входа в TravelPerk и инициируйте поток входа.

3. Вы можете использовать Панель доступа (Майкрософт). Щелкнув плитку TravelPerk на Панели доступа, вы перейдете по URL-адресу для входа в TravelPerk. См. дополнительные сведения о [панели доступа](../user-help/my-apps-portal-end-user-access.md)
