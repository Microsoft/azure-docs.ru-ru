# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-iprova"></a>Руководство. Интеграция единого входа Azure Active Directory с iProva

В этом руководстве вы узнаете, как интегрировать iProva с Azure Active Directory (Azure AD). Интеграция iProva с Azure AD обеспечивает следующие возможности.

* С помощью Azure AD вы можете контролировать доступ к iProva.
* Вы можете включить автоматический вход пользователей в iProva с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка iProva с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* iProva поддерживает единый вход, инициированный **поставщиком услуг**.

* После настройки iProva можно применять элемент управления сеансами, который защищает от хищения и несанкционированного доступа к конфиденциальным данным вашей организации в режиме реального времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-iprova-from-the-gallery"></a>Добавление iProva из коллекции

Чтобы настроить интеграцию iProva с Azure AD, необходимо добавить iProva из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **iProva**.
1. Выберите **iProva** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-iprova"></a>Настройка и проверка единого входа Azure AD для iProva

Настройте и проверьте единый вход Azure AD в iProva с помощью тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в iProva.

Чтобы настроить и проверить единый вход Azure AD в iProva, выполните действия в следующих стандартных блоках:

1. **[Получить сведения о конфигурации из iProva](#retrieve-configuration-information-from-iprova)** для подготовки к следующим шагам.
1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Создание тестового пользователя iProva](#create-iprova-test-user)** , чтобы в iProva существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Настройка единого входа в iProva](#configure-iprova-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="retrieve-configuration-information-from-iprova"></a>Получение сведений о конфигурации из iProva

В этом разделе вы получите сведения из iProva, необходимые для настройки единого входа Azure Active Directory.

1. Откройте в браузера **страницу сведений о SAML2** на сайте iProva, используя URL-адрес следующего вида:

    ```https
    https://SUBDOMAIN.iprova.nl/saml2info
    https://SUBDOMAIN.iprova.be/saml2info
    ```

    ![Просмотр страницы сведений о SAML2 в iProva](media/iprova-tutorial/iprova-saml2-info.png)

1. Не закрывайте пока открытую вкладку браузера, а следующие шаги выполняйте в новой вкладке.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **iProva** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. В разделе **Базовая конфигурация SAML** выполните приведенные ниже действия.

    а. В поле **URL-адрес входа** укажите значение, которое отображается под меткой **URL-адреса входа** на **странице сведений о SAML2 в iProva**. Эта страница, которую вы открывали в другой вкладке браузера.

    b. В поле **Идентификатор** укажите значение, которое отображается под меткой **идентификатора сущности** на **странице сведений о SAML2 в iProva**. Эта страница, которую вы открывали в другой вкладке браузера.

    c. В поле **URL-адрес ответа** укажите значение, которое отображается под меткой **URL-адреса ответа** на **странице сведений о SAML2 в iProva**. Эта страница, которую вы открывали в другой вкладке браузера.

1. Приложение iProva ожидает проверочные утверждения SAML в определенном формате, поэтому следует добавить настраиваемые сопоставления атрибутов в вашу конфигурацию атрибутов токена SAML. На следующем снимке экрана показан список атрибутов по умолчанию.

    ![Изображение](common/default-attributes.png)

1. В дополнение к описанному выше приложение iProva ожидает в ответе SAML несколько дополнительных атрибутов, которые показаны ниже. Эти атрибуты также заранее заполнены, но вы можете изменить их в соответствии со своими требованиями.

    | Имя | Исходный атрибут| Пространство имен  |
    | ---------------| -------- | -----|
    | `samaccountname` | `user.onpremisessamaccountname`| `http://schemas.xmlsoap.org/ws/2005/05/identity/claims`|

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

## <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

## <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к iProva.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **iProva**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="create-iprova-test-user"></a>Создание тестового пользователя iProva

1. Войдите в iProva с учетной записью **администратора**.

2. Откройте меню **Go to** (Перейти).

3. Выберите **Application management** (Управление приложением).

4. Выберите **Users** (Пользователи) на панели **Users and user groups** (Пользователи и группы пользователей).

5. Выберите **Добавить**.

6. В текстовом поле **Username** (Имя пользователя) введите имя пользователя, например `B.Simon@contoso.com`.

7. В поле **Полное имя** введите полное имя пользователя, **B.Simon**.

8. Выберите вариант **No password (use single sign-on)** (Без пароля (использование единого входа)).

9. В текстовом поле **Email address** (Адрес электронной почты) введите адрес электронной почты пользователя, например `B.Simon@contoso.com`.

10. Внизу страницы нажмите кнопку **Finish** (Завершить).

## <a name="configure-iprova-sso"></a>Настройка единого входа iProva

1. Войдите в iProva с учетной записью **администратора**.

2. Откройте меню **Go to** (Перейти).

3. Выберите **Application management** (Управление приложением).

4. Выберите **General** (Общие) на панели **System settings** (Параметры системы).

5. Выберите **Изменить**.

6. Прокрутите вниз до раздела **Access control** (Управление доступом).

    ![Параметры управления доступом в iProva](media/iprova-tutorial/iprova-accesscontrol.png)

7. Найдите параметр **Users are automatically logged on with their network accounts** (Пользователи автоматически входят в систему с сетевыми учетными записями) и выберите для него значение **Yes, authentication via SAML** (Да, с аутентификацией через SAML). Появятся дополнительные параметры.

8. Выберите **Настроить**.

9. Выберите **Далее**.

10. iProva предложит скачать данные федерации по URL-адресу или загрузить их из файла. Выберите вариант **From URL** (По URL-адресу).

    ![Скачивание метаданных AAD](media/iprova-tutorial/iprova-download-metadata.png)

11. Вставьте URL-адрес метаданных, который вы сохранили на последнем шаге раздела Настройка единого входа.

12. Чтобы скачать метаданные из AAD, нажмите кнопку в форме стрелки.

13. Когда скачивание завершится, появится сообщение с подтверждением **Valid Federation Data file downloaded** (Скачан допустимый файл с данными федерации).

14. Выберите **Далее**.

15. Пока пропустите действие **Test login** (Тестовый вход) и выберите **Next** (Далее).

16. В раскрывающемся списке **Claim to use** (Утверждение для использования) выберите **windowsaccountname**.

17. Нажмите кнопку **Готово**.

18. Теперь вы снова попадете на экран **Edit general settings** (Редактирование общих параметров). Внизу страницы нажмите кнопку **ОК**, чтобы сохранить конфигурацию.
