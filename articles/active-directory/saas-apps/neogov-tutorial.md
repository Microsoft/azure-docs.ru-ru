# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-neogov"></a>Руководство по интеграции единого входа Azure Active Directory с NEOGOV

В этом руководстве описано, как интегрировать NEOGOV с Azure Active Directory (Azure AD). Интеграция NEOGOV с Azure AD обеспечивает следующие возможности.

* С помощью Azure AD вы можете контролировать доступ к NEOGOV.
* Вы можете включить автоматический вход пользователей в NEOGOV с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* подписка NEOGOV с поддержкой единого входа (SSO).

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* NEOGOV поддерживает единый вход, инициируемый **поставщиком удостоверений**

## <a name="adding-neogov-from-the-gallery"></a>Добавление NEOGOV из коллекции

Чтобы настроить интеграцию NEOGOV с Azure AD, необходимо добавить NEOGOV из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **NEOGOV**.
1. Выберите **NEOGOV** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-single-sign-on-for-neogov"></a>Настройка и проверка единого входа Azure AD для NEOGOV

Настройте и проверьте единый вход Azure AD в NEOGOV с использованием тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в NEOGOV.

Чтобы настроить и проверить единый вход Azure AD в NEOGOV, выполните действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    * **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    * **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в NEOGOV](#configure-neogov-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    * **[Создание тестового пользователя NEOGOV](#create-neogov-test-user)** требуется для того, чтобы в NEOGOV существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **NEOGOV** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Настройка единого входа с помощью SAML** введите значения для следующих полей:

    а. В текстовом поле **Идентификатор** введите URL-адрес в таком формате:

    | Среда | Шаблон URL-адреса |
    | -- | -- |
    | Производство | `https://www.neogov.com/` |
    | Песочница | `https://www.uat.neogov.net/` |
    | | |

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в таком формате:

    | Среда | Шаблон URL-адреса |
    | -- | -- |
    | Производство | `https://login.neogov.com/authentication/saml/consumer` |
    | Песочница | `https://login.uat.neogov.net/authentication/saml/consumer` |
    | | |

1. Приложение NEOGOV ожидает проверочные утверждения SAML в определенном формате, что требует добавить настраиваемые сопоставления атрибутов в вашу конфигурацию атрибутов токена SAML. На следующем снимке экрана показан список атрибутов по умолчанию, где **nameidentifier** сопоставляется с **user.userprincipalname**. Приложение NEOGOV ожидает, что **nameidentifier** будет сопоставляться с **user.objectid**, поэтому необходимо изменить сопоставление атрибутов, щелкнув значок **Изменить**.

    ![Изображение](common/edit-attribute.png)

1. В дополнение к описанному выше приложение NEOGOV ожидает несколько дополнительных атрибутов в ответе SAML, как показано ниже. Эти атрибуты также заранее заполнены, но вы можете изменить их в соответствии со своими требованиями.

    | Имя |  Исходный атрибут|
    | -------|--------- |
    | mail | user.mail |

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как разрешить пользователю B.Simon использовать единый вход Azure, предоставив этому пользователю доступ к NEOGOV.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. Из списка приложений выберите **NEOGOV**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-neogov-sso"></a>Настройка единого входа NEOGOV

Для настройки единого входа на стороне **NEOGOV** необходимо отправить **URL-адрес метаданных федерации приложений** в [службу поддержки NEOGOV](mailto:itops@neogov.net). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

### <a name="create-neogov-test-user"></a>Создание тестового пользователя NEOGOV

В этом разделе описано, как создать пользователя B.Simon в приложении NEOGOV. Чтобы добавить пользователей на платформу NEOGOV, обратитесь в [службу поддержки NEOGOV](mailto:itops@neogov.net). Перед использованием единого входа необходимо создать и активировать пользователей.
