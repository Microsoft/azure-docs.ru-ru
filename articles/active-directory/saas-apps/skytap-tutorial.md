# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-single-sign-on-for-skytap"></a>Руководство по интеграции единого входа Azure Active Directory с приложением "Единый вход для Skytap"

В этом руководстве описано, как интегрировать Azure Active Directory (Azure AD) с приложением "Единый вход для Skytap". Интеграция приложения "Единый вход для Skytap" с Azure AD обеспечивает следующие возможности.

* С помощью Azure AD вы можете контролировать доступ к приложению "Единый вход для Skytap".
* Автоматический вход пользователей в приложение "Единый вход для Skytap" с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Дополнительные сведения об интеграции приложений SaaS с Azure AD см. в статье о [едином входе в приложения с помощью Azure Active Directory](../manage-apps/what-is-single-sign-on.md).

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка приложения "Единый вход для Skytap" с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Приложение "Единый вход для Skytap" поддерживает единый вход, инициируемый поставщиком услуг и поставщиком удостоверений.
* После настройки приложения "Единый вход для Skytap" можно включить управление сеансом. Это защищает от утечки данных и несанкционированного доступа к конфиденциальным данным вашей организации в режиме реального времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="add-single-sign-on-for-skytap-from-the-gallery"></a>Добавление приложения "Единый вход для Skytap" из коллекции

Чтобы настроить интеграцию Azure AD с приложением "Единый вход для Skytap" для включения единого входа, вам необходимо добавить это приложение из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Единый вход для Skytap**.
1. Выберите **Единый вход для Skytap** в области результатов и добавьте приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-single-sign-on-for-skytap"></a>Настройка и проверка единого входа Azure AD для приложения "Единый вход для Skytap"

Настройте и проверьте единый вход Azure AD в приложение "Единый вход для Skytap" с помощью тестового пользователя **B. Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в приложении "Единый вход для Skytap".

Ниже приведены основные действия по настройке и проверке единого входа Azure AD с помощью приложения "Единый вход для Skytap":

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.

    а. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя B.Simon.

    b. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в приложение "Единый вход для Skytap"](#configure-single-sign-on-for-skytap-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.

    а. **[Создание тестового пользователя в приложении "Единый вход для Skytap"](#create-single-sign-on-for-skytap-test-user)** требуется для того, чтобы в приложении "Единый вход для Skytap" существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** необходима, чтобы убедиться в корректной работе конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Единый вход для Skytap** найдите раздел **Управление**. Выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить эти параметры.

   ![Снимок экрана: настройка единого входа на странице SAML с выделенным значком пера](common/edit-urls.png)

1. Если вы хотите настроить приложение в режиме, инициируемом **поставщиком удостоверений**, в разделе **Базовая конфигурация SAML** введите значения следующих полей.

    а. В текстовом поле **Идентификатор** введите URL-адрес в следующем формате: `http://pingone.com/<custom EntityID>`.

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в следующем формате: `https://sso.connect.pingidentity.com/sso/sp/ACS.saml2`.

1. При необходимости вы можете настроить приложение для работы в режиме, инициируемом **поставщиком услуг**. Для этого щелкните **Задать дополнительные URL-адреса** и выполните следующие действия:

    а. В текстовом поле **URL-адрес для входа** введите URL-адрес в следующем формате: `https://sso.connect.pingidentity.com/sso/sp/initsso?saasid=<saasid>&idpid=<idpid>`.

    
    b. В текстовом поле **Состояние ретранслятора** введите URL-адрес в следующем формате: `https://pingone.com/1.0/<custom ID>`.

    > [!NOTE]
    > Эти значения приведены для примера. Замените эти значения фактическим идентификатором, URL-адресом ответа, URL-адресом для входа и состоянием ретранслятора. Чтобы получить их, обратитесь в [группу поддержки клиентов приложения "Единый вход для Skytap"](mailto:support@skytap.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** выберите **XML метаданных федерации**. Выберите **Скачать**, чтобы скачать файл метаданных и сохранить его на компьютере.

    ![Снимок экрана ссылки для скачивания сертификата](common/metadataxml.png)

1. Требуемый URL-адрес можно скопировать из раздела о **настройке приложения "Единый вход для Skytap"** .

    ![Снимок экрана копирования URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. Слева на портале Azure выберите **Azure Active Directory** > **Пользователи** > **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Выберите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **создания**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B. Simon, предоставив этому пользователю доступ к приложению "Единый вход для Skytap".

1. На портале Azure выберите **Корпоративные приложения** > **Все приложения**.
1. В списке приложений выберите **Единый вход для Skytap**.
1. На странице сводных сведений о приложении откройте раздел **Управление** и выберите **Пользователи и группы**.

   ![Снимок экрана: раздел "Управление" с выделенным пунктом "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**. В диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Снимок экрана: страница "Пользователи и группы" с выделенным элементом "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B. Simon** из списка пользователей. Затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если вы хотите получить значение роли в утверждении SAML, выберите в диалоговом окне **Выбор роли** подходящую роль для пользователя в предложенном списке. Затем в нижней части экрана нажмите кнопку **Выбрать**.
1. В диалоговом окне **Добавление назначения** выберите **Назначить**.

## <a name="configure-single-sign-on-for-skytap-sso"></a>Настройка единого входа для приложения "Единый вход для Skytap"

Чтобы настроить единый вход на стороне приложения "Единый вход для Skytap", отправьте скачанный **XML-файл метаданных федерации** и соответствующие URL-адреса, скопированные на портале Azure, в [группу поддержки приложения "Единый вход для Skytap"](mailto:support@skytap.com). Специалисты службы поддержки правильно настроят подключение единого входа SAML на обеих сторонах подключения.


### <a name="create-single-sign-on-for-skytap-test-user"></a>Создание тестового пользователя для приложения "Единый вход для Skytap"

В этом разделе описано, как создать пользователя B. Simon в приложении "Единый вход для Skytap". Чтобы добавить пользователей на платформу "Единый вход для Skytap", обратитесь в [службу поддержки клиентов этой платформы](mailto:support@skytap.com). Единый вход нельзя использовать до создания и активации пользователей.
