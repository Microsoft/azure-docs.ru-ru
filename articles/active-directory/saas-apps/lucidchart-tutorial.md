# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-lucidchart"></a>Руководство по Интеграция единого входа Azure Active Directory с Lucidchart

В этом руководстве описано, как интегрировать Lucidchart с Azure Active Directory (Azure AD). Интеграция Lucidchart с Azure AD обеспечивает следующие возможности.

* С помощью Azure AD вы можете управлять доступом к Lucidchart.
* Автоматический вход пользователей в Lucidchart с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Lucidchart с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Lucidchart поддерживает единый вход, инициированный **поставщиком услуг**.
* Lucidchart поддерживает **JIT**-подготовку пользователей.

## <a name="add-lucidchart-from-the-gallery"></a>Добавление Lucidchart из коллекции

Чтобы настроить интеграцию Lucidchart с Azure AD, необходимо добавить Lucidchart из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Lucidchart**.
1. Выберите **Lucidchart** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-lucidchart"></a>Настройка и проверка единого входа Azure AD для Lucidchart

Настройте и проверьте единый вход Azure AD в Lucidchart с помощью тестового пользователя **B.Simon**. Чтобы обеспечить работу единого входа, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Lucidchart.

Чтобы настроить и проверить единый вход Azure AD в Lucidchart, выполните следующие действия:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    * **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    * **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Lucidchart](#configure-lucidchart-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    * **[Создание тестового пользователя Lucidchart](#create-lucidchart-test-user)** требуется, чтобы в Lucidchart существовал пользователь B.Simon, связанный с представлением этого пользователя в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **Lucidchart** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** для изменения параметров.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

4. На странице **Базовая конфигурация SAML** введите значения следующих полей.

   В текстовом поле **URL-адрес входа** введите URL-адрес: `https://chart2.office.lucidchart.com/saml/sso/azure`

5. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** щелкните **Скачать**, чтобы скачать нужный вам **XML метаданных федерации**, и сохраните его на компьютере.

    ![Ссылка для скачивания сертификата](common/metadataxml.png)

6. Требуемый URL-адрес можно скопировать из раздела **Настройка Lucidchart**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

    
### <a name="create-an-azure-ad-test-user&quot;></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name=&quot;assign-the-azure-ad-test-user&quot;></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B.Simon, предоставив этому пользователю доступ к Lucidchart.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Lucidchart**.
1. На странице &quot;Обзор&quot; приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль &quot;Доступ по умолчанию&quot;.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name=&quot;configure-lucidchart-sso&quot;></a>Настройка единого входа в Lucidchart

1. В другом окне веб-браузера зайдите на веб-сайт компании Lucidchart в качестве администратора.

2. В меню в верхней части страницы щелкните **Группа**.

    ![Команда](./media/lucidchart-tutorial/ic791190.png &quot;Группа")

3. Выберите **Приложение \> Управление SAML**.

    ![Управление SAML](./media/lucidchart-tutorial/ic791191.png "Управление SAML")

4. На странице диалогового окна **Параметры проверки подлинности SAML** выполните следующее:

    а. Выберите **Включить проверку подлинности SAML**, а затем щелкните **Дополнительно**.

    ![Параметры проверки подлинности SAML](./media/lucidchart-tutorial/ic791192.png "Параметры проверки подлинности SAML")

    b. В текстовом поле **Домен** введите имя домена и щелкните **Изменить сертификат**.

    ![Изменить сертификат](./media/lucidchart-tutorial/ic791193.png "Изменение сертификата")

    c. Откройте скачанный файл метаданных, скопируйте его содержимое и вставьте его в текстовое поле **Отправка метаданных**.

    ![Отправка метаданных](./media/lucidchart-tutorial/ic791194.png "Отправка метаданных")

    d. Установите флажок **Автоматически добавлять новых пользователей в группу**, а затем нажмите кнопку **Сохранить изменения**.

    ![Сохранить изменения](./media/lucidchart-tutorial/ic791195.png "Сохранить изменения")

### <a name="create-lucidchart-test-user"></a>Создание тестового пользователя в Lucidchart

Элемент действия для настройки подготовки пользователей в Lucidchart отсутствует.  Когда назначенный пользователь пытается войти в Lucidchart с помощью панели доступа, Lucidchart проверяет, существует ли данный пользователь.  

Если учетная запись пользователя отсутствует, Lucidchart автоматически создает ее.

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов. 

* Выберите **Тестировать приложение** на портале Azure. Вы будете перенаправлены по URL-адресу для входа в Lucidchart, где можно инициировать поток входа. 

* Перейдите по URL-адресу для входа в Lucidchart и инициируйте поток входа.

* Вы можете использовать портал "Мои приложения" корпорации Майкрософт. Щелкнув плитку Lucidchart на портале "Мои приложения", вы автоматически войдете в приложение Lucidchart, для которого настроили единый вход. Дополнительные сведения о портале "Мои приложения" см. в [этой статье](../user-help/my-apps-portal-end-user-access.md).
