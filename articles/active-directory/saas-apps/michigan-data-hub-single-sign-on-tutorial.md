# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-michigan-data-hub-single-sign-on"></a>Руководство по интеграции единого входа Azure Active Directory с приложением "Единый вход для Michigan Data Hub"

В этом руководстве объясняется, как интегрировать Azure Active Directory (Azure AD) с приложением "Единый вход для Michigan Data Hub". Интеграция приложения "Единый вход для Michigan Data Hub" с Azure AD обеспечивает следующие возможности:

* С помощью Azure AD вы сможете контролировать доступ к приложению "Единый вход для Michigan Data Hub".
* Включить автоматический вход пользователей в приложение "Единый вход для Michigan Data Hub" с помощью их учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка на приложение "Единый вход для Michigan Data Hub" с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Приложение "Единый вход для Michigan Data Hub" поддерживает единый вход, инициированный **поставщиком услуг**.
* После настройки приложения "Единый вход для Michigan Data Hub" вы можете применить управление сеансами, которое защищает от хищения конфиденциальных данных вашей организации и несанкционированного доступа к ним в реальном времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-michigan-data-hub-single-sign-on-from-the-gallery"></a>Добавление приложения "Единый вход для Michigan Data Hub" из коллекции

Чтобы настроить интеграцию Azure AD с приложением "Единый вход для Michigan Data Hub", вам необходимо добавить приложение "Единый вход для Michigan Data Hub" из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Единый вход для Michigan Data Hub**.
1. Выберите вариант **Единый вход для Michigan Data Hub** в области результатов и добавьте приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-michigan-data-hub-single-sign-on"></a>Настройка и проверка единого входа Azure AD для приложения "Единый вход для Michigan Data Hub"

Настройте и проверьте единый вход Azure AD в приложение "Единый вход для Michigan Data Hub" с помощью тестового пользователя **B.Simon**. Чтобы обеспечить единый вход, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в приложении "Единый вход для Michigan Data Hub".

Чтобы настроить и проверить единый вход Azure AD в приложение "Единый вход для Michigan Data Hub", выполните инструкции в следующих базовых блоках:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в приложение "Единый вход для Michigan Data Hub"](#configure-michigan-data-hub-single-sign-on-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя в приложении "Единый вход для Michigan Data Hub"](#create-michigan-data-hub-single-sign-on-test-user)** требуется для того, чтобы в приложении "Единый вход для Michigan Data Hub" существовал пользователь B.Simon, связанный с представлением этого пользователя в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Единый вход для Michigan Data Hub** найдите раздел **Управление** и выберите элемент **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    В текстовом поле **URL-адрес входа** введите URL-адрес: `https://launchpad.midatahub.org`.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе объясняется, как разрешить пользователю B.Simon использовать единый вход Azure, предоставив доступ к приложению "Единый вход для Michigan Data Hub".

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Единый вход для Michigan Data Hub**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-michigan-data-hub-single-sign-on-sso"></a>Настройка единого входа для приложения "Единый вход для Michigan Data Hub"

Чтобы настроить единый вход на стороне приложения **Единый вход для Michigan Data Hub**, необходимо отправить **URL-адрес метаданных федерации приложения** [службе поддержки приложения "Единый вход для Michigan Data Hub"](mailto:support@midatahub.org). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

Чтобы добавить пользователей на платформу "Единый вход для Michigan Data Hub", обратитесь в [службу поддержки этой платформы](mailto:support@midatahub.org). Перед использованием единого входа необходимо создать и активировать пользователей.
