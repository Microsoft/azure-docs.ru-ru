# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-netvision-compas"></a>Руководство по интеграции единого входа Azure Active Directory с Netvision Compas

В этом руководстве описано, как интегрировать Netvision Compas с Azure Active Directory (Azure AD). Интеграция Netvision Compas с Azure AD обеспечивает следующие возможности:

* Контроль доступа к Netvision Compas с помощью Azure AD.
* Включение автоматического входа пользователей в Netvision Compas с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Netvision Compas с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Netvision Compas поддерживает единый вход, инициируемый **поставщиком услуг и поставщиком удостоверений**.
* После настройки единого входа в Netvision Compas вы сможете применить функцию управления сеансом, которая защищает от хищения конфиденциальных данных вашей организации и несанкционированного доступа к ним в режиме реального времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-aad).


## <a name="adding-netvision-compas-from-the-gallery"></a>Добавление Netvision Compas из коллекции

Чтобы настроить интеграцию Netvision Compas с Azure AD, необходимо добавить Netvision Compas из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите.
1. Выберите **Netvision Compas** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-single-sign-on-for-netvision-compas"></a>Настройка и проверка единого входа Azure AD для Netvision Compas

Настройте и проверьте единый вход Azure AD в Netvision Compas с использованием тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Netvision Compas.

Чтобы настроить и проверить единый вход Azure AD в Netvision Compas, выполните действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Netvision Compas](#configure-netvision-compas-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя Netvision Compas](#configure-netvision-compas-test-user)** требуется для того, чтобы в Netvision Compas существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Netvision Compas** найдите раздел **Управление** и щелкните **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. Если вы хотите настроить приложение в режиме, инициируемом **поставщиком удостоверений**, в разделе **Базовая конфигурация SAML** введите значения следующих полей.

    а. В текстовом поле **Идентификатор** введите URL-адрес в формате `https://<TENANT>.compas.cloud/Identity/Saml20`.

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://<TENANT>.compas.cloud/Identity/Auth/AssertionConsumerService`.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг**, щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес в формате `https://<TENANT>.compas.cloud/Identity/Auth/AssertionConsumerService`.

    > [!NOTE]
    > Эти значения приведены для примера. Замените их фактическими значениями идентификатора, URL-адреса ответа и URL-адреса входа. Обратитесь к [группе поддержки клиентов Netvision Compas](mailto:contact@net.vision), чтобы получить эти значения. Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите элемент **XML метаданных федерации** и нажмите кнопку **Скачать**, чтобы скачать файл метаданных и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/metadataxml.png)



### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B.Simon, предоставив этому пользователю доступ к Netvision Compas.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Netvision Compas**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-netvision-compas-sso"></a>Настройка единого входа для Netvision Compas

В этом разделе описано, как включить единый вход SAML в **Netvision Compas**.
1. Войдите в **Netvision Compas** с помощью учетной записи администратора и получите доступ к области администрирования.

    ![Область администрирования](media/netvision-compas-tutorial/admin.png)

1. Перейдите к области **Система** и выберите **Поставщики удостоверений**.

    ![Поставщики удостоверений администратора](media/netvision-compas-tutorial/admin-idps.png)

1. Выберите **Добавить**, чтобы зарегистрировать Azure AD в качестве нового поставщика удостоверений.

    ![Добавление поставщиков удостоверений](media/netvision-compas-tutorial/idps-add.png)

1. Выберите **SAML** для **Тип поставщика**.
1. Введите значимые значения для полей **Отображаемое им** и **Описание**.
1. Назначьте пользователей **Netvision Compas** в качестве поставщиков удостоверений, выбрав из списка **Доступные пользователи** и затем нажав кнопку **Добавить выбранные**. Пользователей также можно назначить в качестве поставщиков удостоверений, следуя процедуре подготовки.
1. Для параметра SAML **Метаданные** нажмите кнопку **Выбрать файл** и выберите файл метаданных, ранее сохраненный на компьютере.
1. Выберите команду **Сохранить**.

    ![Изменение поставщиков удостоверений](media/netvision-compas-tutorial/idp-edit.png)


### <a name="configure-netvision-compas-test-user"></a>Настройка тестового пользователя Netvision Compas

В этом разделе вы настроите существующего пользователя в **Netvision Compas**, чтобы использовать Azure AD для единого входа.
1. Выполните процедуру подготовки пользователей **Netvision Compas**, определенную в вашей компании, или измените существующую учетную запись пользователя.
1. При определении профиля пользователя убедитесь, что **адрес электронной почты пользователя (личный)** совпадает с именем пользователя Azure AD: username@companydomain.extension. Например, `B.Simon@contoso.com`.

    ![Изменение пользователя](media/netvision-compas-tutorial/user-config.png)

Перед использованием единого входа необходимо создать и активировать пользователей.

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD.

### <a name="using-the-access-panel-idp-initiated"></a>С помощью Панели доступа (поставщик удостоверений инициируется).

Щелкнув плитку Netvision Compas на Панели доступа, вы автоматически войдете в приложение Netvision Compas, для которого настроили единый вход. См. дополнительные сведения о [панели доступа](../user-help/my-apps-portal-end-user-access.md)

### <a name="directly-accessing-netvision-compas-sp-initiated"></a>Прямой доступ к Netvision Compas (инициируется SP).

1. Получите доступ к URL-адресу **Netvision Compas**. Например, `https://tenant.compas.cloud`.
1. Введите имя пользователя **Netvision Compas** и выберите **Далее**.

    ![Имя для входа](media/netvision-compas-tutorial/login-user.png)

1. **(необязательно)** Если пользователю назначено несколько поставщиков удостоверений в **Netvision Compas**, появится список доступных поставщиков удостоверений. Выберите поставщик удостоверений Azure AD, настроенный ранее в **Netvision Compas**.

    ![Выбор имени для входа](media/netvision-compas-tutorial/login-choose.png)

1. Вы будете перенаправлены в Azure AD для выполнения проверки подлинности. После успешной проверки подлинности вы должны автоматически войти в **Netvision Compas**, для которого настраивается единый вход.
