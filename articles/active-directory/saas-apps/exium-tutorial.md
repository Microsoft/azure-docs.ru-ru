# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-exium"></a>Учебник по интеграции единого входа Azure Active Directory с Exium

В этом учебнике описано, как интегрировать Exium с Azure Active Directory (Azure AD). Интеграция Exium с Azure AD обеспечивает следующие возможности:

* Управление доступом к Exium с помощью Azure AD.
* Автоматический вход пользователей в Exium с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Exium с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Exium поддерживает единый вход, инициированный **поставщиком услуг**.

## <a name="adding-exium-from-the-gallery"></a>Добавление Exium из коллекции

Чтобы настроить интеграцию Exium с Azure AD, необходимо добавить Exium из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Exium**.
1. Выберите **Exium** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-exium"></a>Настройка и проверка единого входа Azure AD для Exium

Настройте и проверьте единый вход Azure AD в Exium с помощью тестового пользователя **B.Simon**. Чтобы обеспечить работу единого входа, нужно установить связь между пользователем Azure AD и соответствующим пользователем в Exium.

Чтобы настроить и проверить единый вход Azure AD в Exium, выполните следующие действия:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Exium](#configure-exium-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя в Exium](#create-exium-test-user)** необходимо для того, чтобы в Exium существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **Exium** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** для изменения параметров.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    а. В текстовом поле **Идентификатор (сущности)** введите URL-адрес в следующем формате: `https://subapi.exium.net/saml/<WORKSPACE_ID>/metadata`.
    
    b.  В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://subapi.exium.net/saml/<WORKSPACE_ID>/acs`.

    c. В текстовом поле **URL-адрес для входа** введите URL-адрес: `https://service.exium.net/sign-in`

    > [!NOTE]
    > Эти значения приведены для примера. Измените их на фактические значения идентификатора и URL-адреса ответа. Чтобы получить эти значения, обратитесь к [группе поддержки клиентов Exium](mailto:support@exium.net). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B.Simon, предоставив этому пользователю доступ к Exium.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Exium**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-exium-sso"></a>Настройка единого входа в Exium

1. Войдите на корпоративный сайт с правами администратора.

1. В **консоли администрирования** выберите панель **Профиль организации**.

    ![Снимок экрана: консоль администрирования](./media/exium-tutorial/company-profile.png)

1. В разделе **Профиль** щелкните **Параметры единого входа** и **измените их**.

1. Выполните приведенные ниже действия в разделе **Параметры единого входа**.

    ![Снимок экрана: раздел "Параметры единого входа"](./media/exium-tutorial/update.png)

    а. В раскрывающемся списке выберите для параметра **Тип единого входа** значение **Azure AD**.

    b. Вставьте значение **URL-адреса метаданных федерации приложения** в поле **URL-адрес метаданных поставщика удостоверений SAML 2.0**.

    c. Скопируйте значение **SAML 2.0 SSO URL** (URL-адрес единого входа в SAML 2.0) и вставьте его в текстовое поле **URL-адрес ответа** в разделе **Базовая конфигурация SAML** на портале Azure.

    d. Скопируйте значение **идентификатора сущности поставщика услуг SAML 2.0** и вставьте его в текстовое поле **Идентификатор** в разделе **Базовая конфигурация SAML** на портале Azure.  

    д) Щелкните **Update** (Обновить).

### <a name="create-exium-test-user"></a>Создание тестового пользователя в Exium

1. Войдите на корпоративный сайт с правами администратора.

1. Выберите элементы **Управление пользователями > Пользователи** и нажмите кнопку **Добавить пользователя**.

    ![Снимок экрана: создания тестового пользователя](./media/exium-tutorial/add-user.png)

1. Заполните обязательные поля на следующей странице и щелкните **Сохранить**.

    ![Снимок экрана: создание полей тестового пользователя с помощью кнопки "Сохранить"](./media/exium-tutorial/add-user-2.png)

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов. 

* Выберите **Тестировать приложение** на портале Azure. Вы будете перенаправлены по URL-адресу для входа в Exium, где можно инициировать поток входа. 

* Перейдите по URL-адресу для входа в Exium и инициируйте поток входа.

* Вы можете использовать портал "Мои приложения" корпорации Майкрософт. Щелкнув на элемент Exium на портале "Мои приложения", вы перейдете по URL-адресу для входа в Exium. Дополнительные сведения о портале "Мои приложения" см. в [этой статье](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction).
