# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-contentful"></a>Руководство. Интеграция единого входа Azure Active Directory с Contentful

В этом учебнике описано, как интегрировать Contentful с Azure Active Directory (Azure AD). Интеграция Contentful с Azure AD обеспечивает следующие возможности.

* С помощью Azure AD вы можете контролировать доступ к Contentful.
* Вы можете включить автоматический вход пользователей в Contentful с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* подписка на Contentful с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Contentful поддерживает единый вход, инициированный **поставщиком услуг и поставщиком удостоверений**.
* Contentful поддерживает **JIT**-подготовку пользователей.
* После настройки Contentful можно применить функцию управления сеансом, которая защищает от несанкционированного перемещения конфиденциальных данных вашей организации в обоих направлениях в реальном времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

> [!NOTE]
> Идентификатор этого приложения является фиксированным строковым значением. В одном клиенте можно настроить только один экземпляр.

## <a name="adding-contentful-from-the-gallery"></a>Добавление Contentful из коллекции

Чтобы настроить интеграцию Contentful с Azure AD, необходимо добавить Contentful из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Contentful**.
1. Выберите **Contentful** из результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on-for-contentful"></a>Настройка и проверка единого входа Azure AD для Contentful

Настройте и проверьте единый вход Azure AD в Contentful с помощью тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Contentful.

Чтобы настроить и проверить единый вход Azure AD в Contentful, выполните действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    * **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    * **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Contentful](#configure-contentful-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    * **[Создание тестового пользователя Contentful](#create-contentful-test-user)** требуется для того, чтобы в Contentful существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Contentful** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. Если вы хотите настроить приложение в режиме, инициируемом **поставщиком удостоверений**, в разделе **Базовая конфигурация SAML** введите значения следующих полей.

    - В текстовом поле **URL-адрес ответа** скопируйте URL-адрес ACS (служба обработчика утверждений) на странице настройки единого входа в Contentful. Он должен выглядеть так: `https://be.contentful.com/sso/<organization_id>/consume`

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг**, щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    - В текстовом поле **URL-адрес входа** скопируйте тот же URL-адрес ACS (служба обработчика утверждений). Он должен выглядеть так: `https://be.contentful.com/sso/<organization_id>/login`

    > [!NOTE]
    > Эти значения приведены для примера. Замените эти значения фактическим URL-адресом ответа и URL-адресом входа, скопировав URL-адрес службы ACS (служба обработчика утверждений) на странице настройки единого входа в Contentful.

1. Приложение Contentful ожидает проверочные утверждения SAML в определенном формате, который требует добавить настраиваемые сопоставления атрибутов в конфигурацию атрибутов токена SAML. На следующем снимке экрана показан список атрибутов по умолчанию.

    ![Изображение](common/default-attributes.png)

1. В дополнение к описанному выше приложение Contentful ожидает несколько дополнительных атрибутов в ответе SAML, как показано ниже. Эти атрибуты также заранее заполнены, но вы можете изменить их в соответствии со своими требованиями.
    
    | Имя |  Исходный атрибут|
    | --------------- | --------- |
    | email | user.userprincipalname |

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificatebase64.png)

1. В разделе **Настройка Contentful** скопируйте URL-адрес входа, чтобы настроить единый вход Contentful.

    ![Копирование URL-адресов настройки](media/contentful-tutorial/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon, предоставив этому пользователю доступ к Contentful.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Contentful**.
1. На странице сводных сведений о приложении откройте раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а затем в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части страницы нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части страницы.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-contentful-sso"></a>Настройка единого входа в Contentful

Выполните следующие действия, чтобы настроить единый вход на стороне **Contentful**.

1. В разделе [Contentful](https://app.contentful.com) перейдите на страницу настройки единого входа в области **Параметры организации**.
1. Щелкните **Настроить единый вход**.
1. Скопируйте URL-адрес входа из раздела **Настройка Contentful** и вставьте его в Azure AD
1. Скопируйте и вставьте сертификат из файла сертификата Base64, скачанного из Azure AD.
1. Настройте имя единого входа для входа, инициированного поставщиком услуг.
1. Нажмите кнопку **Включить единый вход**.

Если это не поможет, обратитесь в [группу поддержки Contentful](mailto:support@contentful.com).

### <a name="create-contentful-test-user"></a>Создание тестового пользователя Contentful

В этом разделе описано, как в приложении Contentful создать пользователя с именем B.Simon. Приложение Contentful поддерживает JIT-подготовку пользователей, которая включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. Если пользователь еще не существует в Contentful, он создается после проверки подлинности.
