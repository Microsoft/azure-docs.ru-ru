# <a name="tutorial-integrate-way-we-do-with-azure-active-directory"></a>Руководство по Интеграция Way We Do с Azure Active Directory

В этом руководстве описано, как интегрировать Way We Do с Azure Active Directory (Azure AD). Интеграция Way We Do с Azure AD обеспечивает следующие возможности.

* С помощью Azure AD вы можете контролировать доступ к Way We Do.
* Вы можете включить автоматический вход пользователей в Way We Do с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, [здесь](https://azure.microsoft.com/pricing/free-trial/) вы можете получить бесплатную пробную версию сроком на один месяц.
* подписка Way We Do с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Приложение Way We Do поддерживает единый вход, инициированный **поставщиком услуг**.
* Приложение Way We Do поддерживает **JIT**-подготовку пользователей.

## <a name="adding-way-we-do-from-the-gallery"></a>Добавление Way We Do из коллекции

Чтобы настроить интеграцию Way We Do с Azure AD, необходимо добавить Way We Do из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Way We Do**.
1. Выберите **Way We Do** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-single-sign-on"></a>Настройка и проверка единого входа в Azure AD

Настройте и проверьте единый вход Azure AD в Way We Do с помощью тестового пользователя **B.Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Way We Do.

Чтобы настроить и проверить единый вход Azure AD в Way We Do, выполните действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
2. **[Настройка единого входа в Way We Do](#configure-way-we-do-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
3. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя Britta Simon.
4. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы разрешить пользователю Britta Simon использовать единый вход Azure AD.
5. **[Создание тестового пользователя Way We Do](#create-way-we-do-test-user)** требуется для того, чтобы в приложении Way We Do существовал пользователь Britta Simon, связанный с одноименным пользователем в Azure Active Directory.
6. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

### <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Way We Do** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения для следующих полей.

    а. В текстовом поле **URL-адрес для входа** введите URL-адрес в следующем формате: `https://<SUBDOMAIN>.waywedo.com/Authentication/ExternalSignIn`.

    b. В текстовом поле **Идентификатор (сущности)** введите URL-адрес в следующем формате: `https://<SUBDOMAIN>.waywedo.com`.

    > [!NOTE]
    > Эти значения приведены для примера. Необходимо обновить эти значения действующим URL-адресом для входа и идентификатором. Чтобы получить эти значения, обратитесь к [группе поддержки клиентов Way We Do](mailto:support@waywedo.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите элемент **Сертификат (необработанный)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

   ![Ссылка для скачивания сертификата](common/certificateraw.png)

1. Требуемые URL-адреса можно скопировать из раздела **Настройка Way We Do**.

   ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="configure-way-we-do-sso"></a>Настройка единого входа в Way We Do

1. Чтобы автоматизировать настройку в Way We Do, необходимо установить **расширение браузера My Apps Secure Sign-in Extension**, щелкнув **Установить расширение**.

    ![Расширение "Мои приложения"](common/install-myappssecure-extension.png)

1. Чтобы перейти к приложению Way We Do после добавления расширения в браузер, щелкните **Настройка Way We Do**. После этого укажите учетные данные администратора для входа в Way We Do. Расширение браузера автоматически настроит приложение и автоматизирует шаги 3–6.

    ![Настройка конфигурации](common/setup-sso.png)

1. Если необходимо вручную настроить Way We Do, откройте новое окно веб-браузера, зайдите на свой корпоративный сайт Way We Do с правами администратора и выполните следующие действия.

1. Нажмите щелкните **значок человека** в правом верхнем углу любой страницы в Way We Do и в раскрывающемся меню выберите **Account** (Учетная запись).

    ![Way We Do: учетная запись](./media/waywedo-tutorial/tutorial_waywedo_account.png)

1. Щелкните **значок меню**, чтобы открыть меню навигации, и выберите **Single Sign On** (Единый вход).

    ![Way We Do: единый вход](./media/waywedo-tutorial/tutorial_waywedo_single.png)

1. На странице **Single sign-on setup** (Настройка единого входа) выполните следующие действия.

    ![Way We Do: сохранение](./media/waywedo-tutorial/tutorial_waywedo_save.png)

    1. Щелкните переключатель **Turn on single sign-on** (Включить единый вход), установив для него значение **Yes** (Да).

    1. В текстовом поле **Single sign-on name** (Имя для единого входа) введите имя.

    1. В текстовое поле **Идентификатор сущности** вставьте значение **идентификатора Azure AD**, скопированное на портале Azure.

    1. В текстовое поле **URL-адрес единого входа SAML** вставьте **URL-адрес входа**, скопированный на портале Azure.

    1. Передайте сертификат, нажав кнопку **Select** (Выбрать) рядом с полем **Certificate** (Сертификат).

    1. **Необязательные параметры** -

        * "Enable Passwords" (Включить пароли). Если этот параметр отключен, для пользователей Way We Do действует обычный пароль и они могут использовать только единый вход.

        * "Enable Auto-provisioning" (Включить автоматическую подготовку). Когда этот параметр включен, адреса электронной почты, используемые для входа, автоматически сверяются со списком пользователей в Way We Do. Если адрес электронной почты не соответствует активному пользователю в Way We Do, для выполняющего вход пользователя автоматически добавляется новая учетная запись и ему предлагается ввести все недостающие сведения.

          > [!NOTE]
          > Пользователи, добавленные с помощью функции единого входа, являются обычными пользователями без каких-либо ролей в системе. Администратор может войти в систему и изменить их роль безопасности на роль редактора или администратора. Кроме того, он может назначить одну или несколько ролей для организационной диаграммы.

    1. Нажмите кнопку **Save** (Сохранить), чтобы сохранить настройки.

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход в Azure для пользователя B.Simon и как предоставить этому пользователю доступ к Way We Do.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. Из списка приложений выберите **Way We Do**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.

   ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Ссылка "Добавить пользователя"](common/add-assign-user.png)

1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

### <a name="create-way-we-do-test-user"></a>Создание тестового пользователя Way We Do

В этом разделе в приложении Way We Do создается пользователь с именем Britta Simon. Приложение Way We Do поддерживает JIT-подготовку пользователей, которая включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. Если пользователь еще не существует в Way We Do, он создается после проверки подлинности.

> [!Note]
> Чтобы создать пользователя вручную, обратитесь к [группе поддержки Way We Do](mailto:support@waywedo.com).

### <a name="test-sso"></a>Проверка единого входа

Щелкнув плитку "Way We Do" на Панели доступа, вы автоматически войдете в приложение Way We Do, для которого настроили единый вход. См. дополнительные сведения о [панели доступа](../user-help/my-apps-portal-end-user-access.md)
