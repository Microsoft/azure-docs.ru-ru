# <a name="tutorial-configure-wrike-for-automatic-user-provisioning"></a>Руководство по настройке Wrike для автоматической подготовки пользователей

В этом учебнике описаны шаги, которые нужно выполнить во Wrike и Azure Active Directory (Azure AD), чтобы настроить Azure AD для автоматической подготовки и отзыва пользователей и групп во Wrike.

> [!NOTE]
> В этом учебнике рассматривается соединитель, созданный на базе службы подготовки пользователей Azure AD. Подробные сведения о том, что делает эта служба, как она работает, а также часто задаваемые вопросы см. в статье [Что представляет собой автоматическая подготовка пользователей для приложений SaaS в Azure AD?](../app-provisioning/user-provisioning.md).
>
> Сейчас этот соединитель предоставляется в общедоступной предварительной версии. Дополнительные сведения об общих условиях использования продуктов в предварительной версии см. в документе [Дополнительные условия использования Предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

## <a name="prerequisites"></a>Предварительные требования

В сценарии, описанном в этом руководстве, предполагается, что у вас уже имеется:

* клиент Azure AD;
* [клиент Wrike](https://www.wrike.com/price/);
* учетная запись пользователя Wrike с разрешениями администратора.

## <a name="assign-users-to-wrike"></a>Назначение пользователей приложению Wrike
В Azure Active Directory для определения того, какие пользователи должны получать доступ к выбранным приложениям, используется концепция, называемая *назначением*. В контексте автоматической подготовки синхронизируются только те пользователи и группы, которые были назначены приложению в Azure AD.

Перед настройкой и включением автоматической подготовки пользователей решите, какие пользователи или группы в Azure AD должны иметь доступ к Wrike. Потом можно будет назначить этих пользователей или группы приложению Wrike, выполнив следующие инструкции:

* [Назначение корпоративному приложению пользователя или группы](../manage-apps/assign-user-or-group-access-portal.md)

## <a name="important-tips-for-assigning-users-to-wrike"></a>Важные рекомендации по назначению пользователей приложению Wrike

* Мы рекомендуем назначить одного пользователя Azure AD во Wrike, чтобы проверить настройки автоматической подготовки пользователей. Дополнительных пользователей или группы можно назначить позже.

* При назначении пользователя во Wrike в диалоговом окне назначения необходимо выбрать действительную роль для конкретного приложения (если доступно). Пользователи с ролью Доступ по умолчанию исключаются из подготовки.

## <a name="set-up-wrike-for-provisioning"></a>Настройка подготовки во Wrike

Перед настройкой автоматической подготовки пользователей с помощью Azure AD во Wrike необходимо включить подготовку SCIM (System for Cross-domain Identity Management).

1. Войдите в [консоль администрирования Wrike](https://www.Wrike.com/login/). Перейдите к параметру идентификатора клиента. Щелкните **Apps & Integrations** (Приложения и интеграции).

    ![Приложения и интеграции](media/Wrike-provisioning-tutorial/admin.png)

2.  Перейдите к элементу **Azure AD** и выберите его.

    ![Azure AD](media/Wrike-provisioning-tutorial/Capture01.png)

3.  Выберите SCIM. Скопируйте **базовый URL-адрес**.

    ![Базовый URL-адрес](media/Wrike-provisioning-tutorial/Wrike-tenanturl.png)

4. Выберите **API** > **Azure SCIM**.

    ![Azure SCIM](media/Wrike-provisioning-tutorial/Wrike-add-scim.png)

5.  Откроется всплывающее окно. Введите пароль, который вы создали ранее для учетной записи.

    ![Создание токена во Wrike](media/Wrike-provisioning-tutorial/password.png)

6.  Скопируйте **секретный токен** и вставьте его в Azure AD. Выберите **Save** (Сохранить), чтобы завершить настройку подготовки во Wrike.

    ![Постоянный маркер доступа](media/Wrike-provisioning-tutorial/Wrike-create-token.png)


## <a name="add-wrike-from-the-gallery"></a>Добавление Wrike из коллекции

Чтобы настроить Wrike для автоматической подготовки пользователей в Azure AD, добавьте Wrike из коллекции приложений Azure AD в список управляемых приложений SaaS.

Чтобы добавить приложение Wrike из коллекции Azure AD, выполните указанные ниже инструкции.

1. На [портале Azure](https://portal.azure.com) в области навигации слева выберите **Azure Active Directory**.

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, в области сверху нажмите кнопку **Новое приложение**.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **Wrike**, выберите **Wrike** на панели результатов, а затем — **Добавить**, чтобы добавить приложение.

    ![Wrike в списке результатов](common/search-new-app.png)


## <a name="configure-automatic-user-provisioning-to-wrike"></a>Настройка автоматической подготовки пользователей во Wrike 

В этом разделе описывается, как настроить службу подготовки Azure AD для создания, обновления и отключения пользователей или групп во Wrike на основе их назначений в Azure AD.

> [!TIP]
> Чтобы включить единый вход на основе SAML во Wrike, следуйте инструкциям из [этого учебника](wrike-tutorial.md). Единый вход можно настроить независимо от автоматической подготовки пользователей, хотя эти две возможности хорошо дополняют друг друга.

### <a name="configure-automatic-user-provisioning-for-wrike-in-azure-ad"></a>Настройка автоматической подготовки пользователей Wrike в Azure AD

1. Войдите на [портал Azure](https://portal.azure.com). Выберите **Корпоративные приложения** > **Все приложения**.

    ![Все приложения](common/enterprise-applications.png)

2. В списке приложений выберите **Wrike**.

    ![Ссылка на Wrike в списке "Приложения"](common/all-applications.png)

3. Выберите вкладку **Подготовка**.

    ![Вкладка "Подготовка"](common/provisioning.png)

4. Для параметра **Режим подготовки к работе** выберите значение **Automatic** (Автоматически).

    ![Для параметра "Режим подготовки к работе" задано значение "Авто"](common/provisioning-automatic.png)

5. В разделе "Учетные данные администратора" введите полученные ранее значения **базового URL-адреса** и **постоянного маркера доступа** в поля **URL-адрес клиента** и **Секретный токен** соответственно. Выберите **Проверить подключение**, чтобы убедиться, что Azure AD может подключиться к Wrike. Если установить подключение не удалось, убедитесь, что у учетной записи Wrike есть разрешения администратора, и повторите попытку.

    ![URL-адрес клиента и токен](common/provisioning-testconnection-tenanturltoken.png)

7. В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки. Установите флажок **Отправить уведомление по электронной почте при сбое**.

    ![Уведомление по электронной почте](common/provisioning-notification-email.png)

8. Нажмите кнопку **Сохранить**.

9. В разделе **Сопоставления** выберите **Synchronize Azure Active Directory Users to Wrike** (Синхронизировать пользователей Azure Active Directory с Wrike).

    ![Сопоставления пользователей Wrike](media/Wrike-provisioning-tutorial/Wrike-user-mappings.png)

10. В разделе **Сопоставления атрибутов** просмотрите пользовательские атрибуты, которые синхронизированы из Azure AD во Wrike. Атрибуты, выбранные как свойства **сопоставления**, используются для сопоставления учетных записей пользователей во Wrike для операций обновления. Чтобы зафиксировать изменения, щелкните **Сохранить**.

    ![Атрибуты пользователя Wrike](media/Wrike-provisioning-tutorial/Wrike-user-attributes.png)

11. Чтобы настроить фильтры области, ознакомьтесь с инструкциями в [этом учебнике](../app-provisioning/define-conditional-rules-for-provisioning-user-accounts.md).

12. Чтобы включить службу подготовки Azure AD для Wrike, в разделе **Параметры** измените значение параметра **Состояние подготовки** на **Включено**.

    ![Состояние подготовки "Включено"](common/provisioning-toggle-on.png)

13. Определите пользователей или группы для подготовки во Wrike, выбрав нужные значения в поле **Область** в разделе **Параметры**.

    ![Область действия подготовки](common/provisioning-scope.png)

14. Когда будете готовы выполнить подготовку, нажмите кнопку **Сохранить**.

    ![Сохранение конфигурации подготовки](common/provisioning-configuration-save.png)

После этого начнется начальная синхронизация пользователей или групп, определенных в поле **Область** раздела **Параметры**. Начальная синхронизация занимает больше времени, чем последующие операции синхронизации. Дополнительные сведения о том, сколько продлится подготовка пользователей или групп, см. в разделе [Сколько времени занимает подготовка пользователей](../app-provisioning/application-provisioning-when-will-provisioning-finish-specific-user.md#how-long-will-it-take-to-provision-users).

В разделе **Текущее состояние** можно отслеживать ход выполнения и переходить по ссылкам для просмотра отчетов о подготовке, в которых зафиксированы все действия, выполняемые службой подготовки Azure AD с приложением Wrike. Дополнительные сведения см. в статье [Проверка состояния подготовки пользователей](../app-provisioning/application-provisioning-when-will-provisioning-finish-specific-user.md). Сведения о чтении журналов подготовки Azure AD см. в руководстве [по отчетам об автоматической подготовке учетных записей пользователей](../app-provisioning/check-status-user-account-provisioning.md).
