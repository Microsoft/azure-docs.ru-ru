# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-trendminer"></a>Руководство по интеграции единого входа Azure Active Directory с TrendMiner

В этом учебнике описано, как интегрировать приложение TrendMiner с Azure Active Directory (Azure AD). Интеграция TrendMiner с Azure AD обеспечивает следующие возможности.

* Контроль доступа к TrendMiner с помощью Azure AD.
* Автоматический вход пользователей в TrendMiner с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка TrendMiner с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* TrendMiner поддерживает единый вход, инициированный **поставщиком услуг и поставщиком удостоверений**.

## <a name="adding-trendminer-from-the-gallery"></a>Добавление TrendMiner из коллекции

Чтобы настроить интеграцию TrendMiner с Azure AD, необходимо добавить TrendMiner из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **TrendMiner**.
1. Выберите **TrendMiner** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-trendminer"></a>Настройка и проверка единого входа Azure AD для TrendMiner

Настройте и проверьте единый вход Azure AD в TrendMiner с помощью тестового пользователя **B. Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в TrendMiner.

Чтобы настроить и проверить единый вход Azure AD в TrendMiner, выполните действия из следующих стандартных блоков:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в TrendMiner](#configure-trendminer-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя TrendMiner](#create-trendminer-test-user)** требуется для того, чтобы в TeamViewer существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **TrendMiner** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. Если вы хотите настроить приложение в режиме, инициируемом **поставщиком удостоверений**, в разделе **Базовая конфигурация SAML** введите значения следующих полей.

    а. В текстовом поле **Идентификатор** введите URL-адрес в формате `https://<CUSTOMER>.trendminer.cloud/security/saml/metadata`.

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://<CUSTOMER>.trendminer.cloud/security/saml/SSO`.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг**, щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес в формате `https://<CUSTOMER>.trendminer.cloud/`.

    > [!NOTE]
    > Эти значения приведены для примера. Замените их фактическими значениями идентификатора, URL-адреса ответа и URL-адреса входа. Чтобы получить эти значения, обратитесь в [группу поддержки клиентов TrendMiner](mailto:support@trendminer.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите элемент **XML метаданных федерации** и выберите **Скачать**, чтобы скачать сертификат и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/metadataxml.png)

1. Требуемые URL-адреса можно скопировать из раздела **Настройка TrendMiner**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)
### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B. Simon, предоставив этому пользователю доступ к TrendMiner.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **TrendMiner**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-trendminer-sso"></a>Настройка единого входа в TrendMiner

1. Откройте новое окно веб-браузера и войдите на корпоративный сайт TrendMiner с правами администратора.

1. В меню слева выберите **SECURITY > Identity Provider** (БЕЗОПАСНОСТЬ > Поставщик удостоверений).

1. На странице **Identity Provider** (Поставщик удостоверений) щелкните **SAML** и выберите **Next step** (Следующий шаг).

    ![Выбор параметра SAML](./media/trendminer-tutorial/saml.png)

1. Щелкните **Next step** (Следующий шаг) на странице **SAML Backup** (Резервная копия SAML).

    ![Выбор параметра SAML Backup (Резервная копия SAML)](./media/trendminer-tutorial/saml-backup.png)

1. Щелкните **Next step** (Следующий шаг) в разделе **Self-Signed certificates** (Самозаверяющие сертификаты).

    ![Страница Self-Signed certificates (Самозаверяющие сертификаты)](./media/trendminer-tutorial/self-signed-certificate.png)

1. Вы можете **пропустить** отправку ключа подписывания.

    ![Отправка ключа подписывания](./media/trendminer-tutorial/signing-key.png)

1. На экране **SAML Configuration** (Конфигурация SAML) в разделе **Entity base URL** (Базовый URL-адрес сущности) введите URL-адрес домена, например `https://trendminer.domain.com/`.

1. В разделе **Identity provider metadata** (Метаданные поставщика удостоверений) передайте **файл метаданных Azure**, скопированный на портале Azure, а затем щелкните **Next step** (Следующий шаг).

    ![Настройка SAML](./media/trendminer-tutorial/saml-configuration.png)

1. В разделе **SAML User mapping** (Сопоставление пользователей SAML) введите имена пользователей, которые будут использоваться для входа, и щелкните **Finish** (Готово).

    ![Сопоставление пользователей SAML](./media/trendminer-tutorial/user-mapping.png)

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов. 

#### <a name="sp-initiated"></a>Инициация поставщиком услуг:

* Выберите **Тестировать приложение** на портале Azure. Вы будете перенаправлены по URL-адресу для входа в TrendMiner, где можно инициировать поток входа.  

* Перейдите по URL-адресу для входа в TrendMiner и инициируйте поток входа.

#### <a name="idp-initiated"></a>Вход, инициированный поставщиком удостоверений

* На портале Azure выберите **Тестировать приложение**, и вы автоматически войдете в приложение TrendMiner, для которого настроен единый вход. 

Вы можете также использовать Панель доступа корпорации Майкрософт для тестирования приложения в любом режиме. Щелкнув плитку TrendMiner на Панели доступа, вы автоматически перейдете на страницу входа приложения для инициации потока входа (при настройке в режиме поставщика услуг) или автоматически войдете в приложение TrendMiner, для которого настроен единый вход (при настройке в режиме поставщика удостоверений). См. дополнительные сведения о [панели доступа](../user-help/my-apps-portal-end-user-access.md)
