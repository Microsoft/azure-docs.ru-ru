# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-raketa"></a>Руководство по интеграции единого входа Azure Active Directory с Raketa

В этом учебнике описано, как интегрировать Raketa с Azure Active Directory (Azure AD). Интеграция Raketa с Azure AD обеспечивает следующие возможности:

* Контроль доступа к Raketa с помощью Azure AD.
* Включение автоматического входа пользователей в Raketa с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

Чтобы узнать больше об интеграции приложений SaaS с Azure AD, прочитайте статью [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](../manage-apps/what-is-single-sign-on.md)

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Raketa с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Raketa поддерживает единый вход, инициированный **поставщиком услуг**.
* После настройки Raketa вы можете применить управление сеансами, которое защищает от хищения конфиденциальных данных вашей организации и несанкционированного доступа к ним в реальном времени. Управление сеансом является расширением функции условного доступа. [Узнайте, как применять управление сеансами с помощью Microsoft Cloud App Security](/cloud-app-security/proxy-deployment-any-app).

## <a name="adding-raketa-from-the-gallery"></a>Добавление Raketa из коллекции

Чтобы настроить интеграцию Raketa с Azure AD, необходимо добавить Raketa из коллекции в список управляемых приложений SaaS.

1. Войдите на [портал Azure](https://portal.azure.com) с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory** [1].

    ![rkt_1](./media/raketa-tutorial/azure-active-directory.png)

1. Перейдите в колонку **Корпоративные приложения** [2] и выберите **Все приложения** [3].

1. Чтобы добавить новое приложение, выберите **Новое приложение** [4]. 

    ![rkt_2](./media/raketa-tutorial/new-app.png)

1. В разделе **Добавление из коллекции** [5] в поле поиска [6] введите **Raketa**.

1. Выберите **Raketa** в области результатов [7], а затем нажмите кнопку **Добавить** [8]. 

    ![rkt_3](./media/raketa-tutorial/add-btn.png)


## <a name="configure-and-test-azure-ad-single-sign-on-for-raketa"></a>Настройка и проверка единого входа Azure AD для Raketa

Настройте и проверьте единый вход Azure AD в Raketa с использованием тестового пользователя **B. Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем AAD и соответствующим пользователем в Raketa.

Чтобы настроить и проверить единый вход Azure AD в Raketa, выполните действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Raketa](#configure-raketa-sso)** необходима, чтобы задать параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя приложения Raketa](#create-raketa-test-user)** требуется для того, чтобы в Raketa существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **Raketa** найдите раздел **Управление** и выберите **Единый вход** [9].

    ![rkt_4](./media/raketa-tutorial/manage-sso.png)

1. На странице **Выбрать метод единого входа** [9] выберите **SAML** [10].

    ![rkt_5](./media/raketa-tutorial/saml.png)

1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** [11] и изменить параметры.

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    1. В текстовые поля **Идентификатор (сущности)** [12] и **URL-адрес для входа** [14] введите URL-адрес: `https://raketa.travel/`.

    1. В текстовом поле **URL-адрес ответа** [13] введите URL-адрес в следующем формате: `https://raketa.travel/sso/acs?clientId=<CLIENT_ID>`.  

    ![rkt_6](./media/raketa-tutorial/enter-urls.png)

    > [!NOTE]
    > Значение URL-адреса ответа приведено для примера. Вместо него нужно указать фактический URL-адрес ответа. Чтобы получить это значение, обратитесь в [группу поддержки клиентов Raketa](mailto:help@raketa.travel). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать** [15], чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

1. Требуемые URL-адреса можно скопировать из раздела **Настройка Raketa**.

    1. URL-адрес для входа [16] — URL-адрес веб-страницы авторизации, который используется для перенаправления пользователей в систему проверки подлинности.

    1. Идентификатор Azure AD [17] — идентификатор Azure AD.

    1. URL-адрес выхода [18] — URL-адрес веб-страницы, который используется для перенаправления пользователей после выхода.

    ![rkt_7](./media/raketa-tutorial/copy-urls.png)


### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory** [1], **Пользователи** [19], а затем — **Все пользователи** [20].

1. В верхней части экрана выберите **Новый пользователь** [21].

    ![rkt_8](./media/raketa-tutorial/new-user.png)

1. В разделе **Свойства пользователя** выполните следующие действия.

   1. В поле **Имя пользователя** [22] введите username@companydomain.extension. Например, `B.Simon@contoso.com`.

   1. В поле **Имя** [23] введите `B.Simon`.

   1. Установите флажок **Показать пароль** [25] и запишите значение, которое отображается в поле **Пароль** [24].

   1. Нажмите кнопку **Создать** [26]. 

    ![rkt_9](./media/raketa-tutorial/create-user.png)


### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B. Simon, предоставив этому пользователю доступ к Raketa.

1. На портале Azure выберите **Корпоративные приложения** [2], а затем —**Все приложения** [3].

1. В списке приложений выберите **Raketa** [27].  

    ![rkt_10](./media/raketa-tutorial/add-raketa.png)

1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы** [28]. 

    ![rkt_11](./media/raketa-tutorial/users-groups.png)

1. Выберите **Добавить пользователя** [29], а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы** [30].

    ![rkt_12](./media/raketa-tutorial/add-user-raketa.png)

1. В диалоговом окне **Пользователи и группы** выберите **B. Simon** [31] в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать** [32].

1. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор роли** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.

1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить** [33]. 

    ![rkt_13](./media/raketa-tutorial/assign-user.png)

## <a name="configure-raketa-sso"></a>Настройка единого входа в Raketa

Чтобы настроить единый вход на стороне **Raketa**, нужно отправить скачанный **сертификат (Base64)** и соответствующие URL-адреса, скопированные на портале Azure, в [группу поддержки Raketa](mailto:help@raketa.travel). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

### <a name="create-raketa-test-user"></a>Создание тестового пользователя Raketa

В этом разделе вы узнаете, как создать пользователя B. Simon в приложении Raketa. Чтобы добавить пользователей на платформу Raketa, обратитесь к [группе поддержки Raketa](mailto:help@raketa.travel). Перед использованием единого входа необходимо создать и активировать пользователей.

## <a name="test-sso"></a>Проверка единого входа

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью панели доступа.

Щелкнув плитку Raketa на Панели доступа, вы автоматически войдете в приложение Raketa, для которого настроили единый вход. См. дополнительные сведения о [панели доступа](../user-help/my-apps-portal-end-user-access.md)
