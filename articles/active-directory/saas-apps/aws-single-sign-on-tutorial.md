# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-aws-single-sign-on"></a>Руководство по Интеграция единого входа Azure Active Directory с AWS Single Sign-on

В этом руководстве описано, как интегрировать Azure Active Directory (Azure AD) с AWS Single Sign-on. Интеграция AWS Single Sign-on с Azure AD обеспечивает следующие возможности:

* контроль доступа к AWS Single Sign-on в Azure AD;
* автоматический вход пользователей в AWS Single Sign-on с учетными записями Azure AD;
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка AWS Single Sign-on с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Приложение AWS Single Sign-on поддерживает единый вход, инициируемый **поставщиком службы и поставщиком удостоверений**.

* AWS Single Sign-on поддерживает [**автоматическую подготовку пользователей**](./aws-single-sign-on-provisioning-tutorial.md).

## <a name="adding-aws-single-sign-on-from-the-gallery"></a>Добавление AWS Single Sign-on из коллекции

Чтобы настроить интеграцию Azure AD с AWS Single Sign-on, вам нужно добавить AWS Single Sign-on из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **AWS Single Sign-on**.
1. Выберите **AWS Single Sign-on** в области результатов и добавьте приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-aws-single-sign-on"></a>Настройка и проверка единого входа Azure AD для AWS Single Sign-on

Настройте и проверьте единый вход Azure AD в AWS Single Sign-on с помощью тестового пользователя **B.Simon**. Чтобы обеспечить единый вход, нужно установить связь между пользователем в Azure AD и соответствующим пользователем в AWS Single Sign-on.

Чтобы настроить и проверить единый вход Azure AD в AWS Single Sign-on, выполните следующие действия:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в AWS Single Sign-on](#configure-aws-single-sign-on-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя в AWS Single Sign-on](#create-aws-single-sign-on-test-user)** требуется для того, чтобы в AWS Single Sign-on существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **AWS Single Sign-on** найдите раздел **Управление** и выберите элемент **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** для изменения параметров.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. Если у вас есть **файл метаданных поставщика службы**, выполните следующие действия в разделе **Базовая конфигурация SAML**:

    а. Щелкните **Отправить файл метаданных**.

    b. Щелкните **значок папки**, чтобы выбрать файл метаданных, скачанный, как описано в разделе **Настройка AWS Single Sign-on** (пункт 8), и нажмите кнопку **Добавить**.

    ![рисунок 2](common/browse-upload-metadata.png)

    c. После успешной передачи файла метаданных значения **Идентификатор** и **URL-адрес ответа** в разделе "Базовая конфигурация SAML" заполнятся автоматически.

    ![image3](common/idp-intiated.png)

    > [!Note]
    > Если поля **Идентификатор** и **URL-адрес ответа** автоматически не заполняются значениями, введите эти значения вручную в соответствии со своими требованиями.

1. Если у вас нет **файла метаданных, предоставленного поставщиком удостоверений**, в разделе **Базовая конфигурация SAML** выполните описанные ниже действия. Если вы хотите настроить приложение в режиме, **инициируемом поставщиком удостоверений**, введите значения для следующих полей:

    а. В текстовом поле **Идентификатор** введите URL-адрес в формате `https://<REGION>.signin.aws.amazon.com/platform/saml/<ID>`.

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://<REGION>.signin.aws.amazon.com/platform/saml/acs/<ID>`.

1. Чтобы настроить приложение для работы в режиме, инициируемом **поставщиком услуг**, щелкните **Задать дополнительные URL-адреса** и выполните следующие действия.

    В текстовом поле **URL-адрес входа** введите URL-адрес в формате `https://portal.sso.<REGION>.amazonaws.com/saml/assertion/<ID>`.

    > [!NOTE]
    > Эти значения приведены для примера. Замените их фактическими значениями идентификатора, URL-адреса ответа и URL-адреса входа. Чтобы получить их, обратитесь в [службу технической поддержки AWS Single Sign-on](mailto:aws-sso-partners@amazon.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. Приложение AWS Single Sign-on ожидает проверочные утверждения SAML в определенном формате. Поэтому вам нужно добавить настраиваемые сопоставления атрибутов в конфигурацию атрибутов токена SAML. На следующем снимке экрана показан список атрибутов по умолчанию.

    ![Изображение](common/edit-attribute.png)


    > [!NOTE]
    > Если в AWS SSO включен механизм ABAC, дополнительные атрибуты могут передаваться как теги сеанса непосредственно в учетные записи AWS.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните элемент **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificatebase64.png)

1. Требуемый URL-адрес можно скопировать из раздела **настройки AWS Single Sign-on**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе объясняется, как разрешить пользователю B.Simon использовать единый вход Azure, предоставив доступ к AWS Single Sign-on.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите вариант **AWS Single Sign-on**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-aws-single-sign-on-sso"></a>Настройка единого входа в AWS Single Sign-on

1. Чтобы автоматизировать настройку AWS Single Sign-on, установите **расширение браузера My Apps Secure Sign-in**, щелкнув **Установить расширение**.

    ![Расширение "Мои приложения"](common/install-myappssecure-extension.png)

2. Чтобы перейти к приложению AWS Single Sign-on после добавления расширения в браузер, щелкните **Настройка AWS Single Sign-on**. В открывшемся окне укажите учетные данные администратора для входа в AWS Single Sign-on. Расширение браузера автоматически настроит приложение и автоматизирует шаги 3–10.

    ![Настройка конфигурации](common/setup-sso.png)

3. Если вы хотите настроить AWS Single Sign-on вручную, в другом окне браузера войдите на свой корпоративный сайт AWS Single Sign-on с правами администратора.

1. Перейдите на страницу **Services -> Security, Identity, & Compliance -> AWS Single Sign-On** (Службы > Безопасность, удостоверения и соответствие > AWS Single Sign-on).
2. На панели навигации слева выберите элемент **Settings** (Параметры).
3. На странице **Settings** (Параметры) найдите пункт **Identity source** (Источник удостоверений) и щелкните **Change** (Изменить).

    ![Снимок экрана: служба смены источника удостоверений](./media/aws-single-sign-on-tutorial/settings.png)

4. На странице изменения источника удостоверений щелкните **External identity provider** (Внешний поставщик удостоверений).

    
    ![Снимок экрана: раздел выбора внешнего поставщика удостоверений](./media/aws-single-sign-on-tutorial/external-identity-provider.png)


1. Выполните следующие действия в разделе **Configure external identity provider** (Настройка внешнего поставщика удостоверений).

    ![Снимок экрана: раздел скачивания и отправки метаданных](./media/aws-single-sign-on-tutorial/upload-metadata.png)

    а. В разделе **Service provider metadata** (Поставщик метаданных службы) найдите **метаданные SAML для AWS SSO** и щелкните **Download metadata file** (Скачать файл метаданных), чтобы скачать файл метаданных, сохранить его на компьютере, а затем отправить его на портал Azure.

    b. Скопируйте значение **URL-адреса для входа в AWS SSO** и вставьте его в текстовое поле **URL-адрес для входа** в разделе **Базовая конфигурация SAML** на портале Azure.

    c. В разделе **Identity provider metadata** (Метаданные поставщика удостоверений) щелкните элемент **Browse** (Просмотр), чтобы отправить скачанный с портала Azure файл метаданных.

    d. Щелкните элемент **Next: Review** (Далее: проверка).

8. В текстовое поле введите **Accept** (Принять), чтобы изменить источник удостоверений.

    ![Снимок экрана: подтверждение конфигурации](./media/aws-single-sign-on-tutorial/accept.png)

9. Щелкните **Change identity source** (Изменить источник удостоверений).

### <a name="create-aws-single-sign-on-test-user"></a>Создание тестового пользователя в AWS Single Sign-on

1. Откройте **консоль AWS SSO**.

2. На панели навигации слева выберите элемент **Users** (Пользователи).

3. На странице Users (Пользователи) щелкните команду **Add user** (Добавить пользователя).

4. На странице Add user (Добавление пользователя) выполните следующие действия:

    а. В поле **Username** (Имя пользователя) введите B.Simon.

    b. В поле **Email address** (Адрес электронной почты) введите `username@companydomain.extension`. Например, `B.Simon@contoso.com`.

    c. В поле **Confirm email address** (Подтверждение адреса электронной почты) повторно введите адрес электронной почты, который вы использовали на предыдущем шаге.

    d. В поле First name (Имя) введите `Jane`.

    д) В поле Last name (Фамилия) введите `Doe`.

    е) В поле Display name (Отображаемое имя) введите `Jane Doe`.

    ж. Щелкните элемент **Next: Groups** (Далее: группы).

    > [!NOTE]
    > Убедитесь, что введенное для AWS SSO имя пользователя совпадает с именем входа для пользователя Azure AD. Это поможет избежать проблем с проверкой подлинности.

5. Выберите **Добавить пользователя**.
6. Теперь вам нужно назначить пользователя учетной записи AWS. Для этого на панели навигации слева в консоли AWS SSO выберите элемент **AWS accounts** (Учетные записи AWS).
7. На странице AWS Accounts (Учетные записи AWS) выберите вкладку AWS organization (Организация AWS). Затем установите флажок рядом с учетной записью AWS, которую хотите назначить этому пользователю. Затем выберите элемент **Assign users** (Назначить пользователей).
8. На странице Assign Users (Назначение пользователей) найдите пользователя B.Simon и установите флажок рядом с этим именем. Затем выберите элемент **Next: Permission sets** (Далее: наборы разрешений).
9. В разделе выбора наборов разрешений установите флажок рядом с набором разрешений, который хотите назначить пользователю B.Simon. Если у вас нет готовых наборов разрешений, щелкните элемент **Create new permission set** (Создать набор разрешений).

    > [!NOTE]
    > Наборы разрешений определяют уровень доступа к учетной записи AWS, который получают пользователи и группы. Дополнительные сведения о наборах разрешений см. на странице **наборов разрешений** для AWS SSO.
10. Нажмите кнопку **Готово**.

> [!NOTE]
> Кроме того, AWS Single Sign-on поддерживает автоматическую подготовку пользователей, дополнительные сведения о настройке которой можно найти [здесь](./aws-single-sign-on-provisioning-tutorial.md).

## <a name="test-sso"></a>Проверка единого входа 

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов. 

#### <a name="sp-initiated"></a>Инициация поставщиком услуг:

* Выберите **Тестировать приложение** на портале Azure. Вы будете перенаправлены по URL-адресу для входа в AWS Single Sign-on, где можно инициировать поток входа.  

* Перейдите по URL-адресу для входа в AWS Single Sign-on и инициируйте поток входа.

#### <a name="idp-initiated"></a>Вход, инициированный поставщиком удостоверений

* На портале Azure выберите элемент **Тестировать приложение**. Вы автоматически войдете в приложение AWS Single Sign-on, для которого настроили единый вход. 

Вы можете также использовать портал "Мои приложения" корпорации Майкрософт для тестирования приложения в любом режиме. Щелкните плитку AWS Single Sign-on в разделе "Мои приложения". Вы перейдете на страницу входа приложения для инициации потока входа (при настройке в режиме поставщика службы) или автоматически войдете в приложение AWS Single Sign-on, для которого настроен единый вход (при настройке в режиме поставщика удостоверений). Дополнительные сведения о портале "Мои приложения" см. в [этой статье](../user-help/my-apps-portal-end-user-access.md).
