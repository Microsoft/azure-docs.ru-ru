# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-adp-globalview-deprecated"></a>Учебник по интеграции единого входа Azure Active Directory с ADP Globalview (нерекомендуемое)

В этой инструкции описано, как интегрировать ADP Globalview (нерекомендуемое) с Azure Active Directory (Azure AD). Интеграция ADP Globalview (нерекомендуемое) с Azure AD обеспечивает следующие возможности:

* Контроль доступа к ADP Globalview (нерекомендуемое) с помощью Azure AD.
* Автоматический вход пользователей в приложение ADP Globalview (нерекомендуемое) с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка ADP Globalview (нерекомендуемое) с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Приложение ADP Globalview (нерекомендуемое) поддерживает единый вход, инициированный **поставщиком удостоверений**.

## <a name="adding-adp-globalview-deprecated-from-the-gallery"></a>Добавление ADP Globalview (нерекомендуемое) из коллекции

Чтобы настроить интеграцию ADP Globalview (нерекомендуемое) с Azure AD, необходимо добавить ADP Globalview (нерекомендуемое) из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **ADP Globalview (нерекомендуемое)** .
1. Выберите **ADP Globalview (нерекомендуемое)** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.

## <a name="configure-and-test-azure-ad-sso-for-adp-globalview-deprecated"></a>Настройка и проверка единого входа Azure AD для ADP Globalview (нерекомендуемое)

Настройте и проверьте единый вход Azure AD в ADP Globalview (нерекомендуемое) с помощью тестового пользователя **B. Simon**. Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в ADP Globalview (нерекомендуемое).

Чтобы настроить и проверить единый вход Azure AD в приложение ADP Globalview (нерекомендуемое), сделайте следующее:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в ADP Globalview (нерекомендуемое)](#configure-adp-globalview-deprecated-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя ADP Globalview (нерекомендуемое)](#create-adp-globalview-deprecated-test-user)** требуется для того, чтобы в приложении ADP Globalview (нерекомендуемое) существовал пользователь B. Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **ADP Globalview (нерекомендуемое)** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок карандаша, чтобы открыть диалоговое окно **Базовая конфигурация SAML** для изменения параметров.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Базовая конфигурация SAML** введите значения следующих полей.

    
    В текстовом поле **Идентификатор** введите URL-адрес в одном из следующих форматов:

    | Идентификатор |
    | ----------- |
    | `https://<subdomain>.globalview.adp.com/federate` |
    | `https://<subdomain>.globalview.adp.com/federate2` |
    |

    > [!NOTE]
    > Это значение приведено для справки. Вместо него нужно указать фактический идентификатор. Чтобы получить это значение, обратитесь в [группу поддержки клиентов ADP Globalview (нерекомендуемое)](https://www.adp.com/contact-us/overview.aspx). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificatebase64.png)

1. Требуемые URL-адреса можно скопировать из раздела **Настройка ADP Globalview (нерекомендуемое)** .

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как включить единый вход Azure для пользователя B. Simon, предоставив этому пользователю доступ к приложению ADP Globalview (нерекомендуемое).

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **ADP Globalview (нерекомендуемое)** .
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-adp-globalview-deprecated-sso"></a>Настройка единого входа для ADP Globalview (нерекомендуемое)

Чтобы настроить единый вход на стороне **ADP Globalview (нерекомендуемое)** , нужно отправить скачанный **сертификат (Base64)** и соответствующие URL-адреса, скопированные на портале Azure, в [группу поддержки ADP Globalview (нерекомендуемое)](https://www.adp.com/contact-us/overview.aspx). Специалисты службы поддержки настроят подключение единого входа SAML на обеих сторонах.

### <a name="create-adp-globalview-deprecated-test-user"></a>Создание тестового пользователя ADP Globalview (нерекомендуемое)

В этом разделе вы можете создать пользователя B. Simon в приложении ADP Globalview (нерекомендуемое). Обратитесь в [группу поддержки ADP Globalview (нерекомендуемое)](https://www.adp.com/contact-us/overview.aspx), чтобы добавить пользователей в учетную запись ADP Globalview (нерекомендуемое). Перед использованием единого входа необходимо создать и активировать пользователей.

## <a name="test-sso"></a>Проверка единого входа

В этом разделе описано, как проверить конфигурацию единого входа Azure AD с помощью указанных ниже способов.

* Выберите "Тестировать приложение" на портале Azure, и вы автоматически войдете в приложение ADP Globalview (нерекомендуемое), для которого настроен единый вход

* Вы можете использовать портал "Мои приложения" корпорации Майкрософт. Щелкнув плитку ADP Globalview (нерекомендуемое) на портале "Мои приложения", вы автоматически войдете в приложение ADP Globalview (нерекомендуемое), для которого настроили единый вход. Дополнительные сведения о портале "Мои приложения" см. в [этой статье](https://docs.microsoft.com/azure/active-directory/active-directory-saas-access-panel-introduction).
