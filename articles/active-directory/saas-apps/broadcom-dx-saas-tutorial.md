# <a name="tutorial-azure-active-directory-single-sign-on-sso-integration-with-broadcom-dx-saas"></a>Руководство. Интеграция единого входа Azure Active Directory с Broadcom DX SaaS

В этом руководстве описано, как интегрировать Broadcom DX SaaS с Azure Active Directory (Azure AD). Интеграция Broadcom DX SaaS с Azure AD обеспечивает следующие возможности:

* Контроль доступа к Broadcom DX SaaS с помощью Azure AD.
* Включение автоматического входа пользователей в Broadcom DX SaaS с помощью учетных записей Azure AD.
* Централизованное управление учетными записями через портал Azure.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе, потребуется следующее.

* Подписка Azure AD. Если у вас нет подписки, вы можете получить [бесплатную учетную запись](https://azure.microsoft.com/free/).
* Подписка Broadcom DX SaaS с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure AD в тестовой среде.

* Broadcom DX SaaS поддерживает единый вход, инициируемый **поставщиком удостоверений**.

* Broadcom DX SaaS поддерживает **JIT-подготовку** пользователей.

> [!NOTE]
> Идентификатор этого приложения — фиксированное строковое значение, поэтому в одном клиенте можно настроить только один экземпляр.

## <a name="add-broadcom-dx-saas-from-the-gallery"></a>Добавление Broadcom DX SaaS из коллекции

Чтобы настроить интеграцию Broadcom DX SaaS с Azure AD, вам нужно добавить Broadcom DX SaaS из коллекции в список управляемых приложений SaaS.

1. Войдите на портал Azure с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи.
1. В области навигации слева выберите службу **Azure Active Directory**.
1. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.
1. Чтобы добавить новое приложение, выберите **Новое приложение**.
1. В разделе **Добавление из коллекции** в поле поиска введите **Broadcom DX SaaS**.
1. Выберите **Broadcom DX SaaS** в области результатов и добавьте это приложение. Подождите несколько секунд, пока приложение не будет добавлено в ваш клиент.


## <a name="configure-and-test-azure-ad-sso-for-broadcom-dx-saas"></a>Настройка и проверка единого входа Azure AD для Broadcom DX SaaS

Настройте и проверьте единый вход Azure AD в Broadcom DX SaaS с помощью тестового пользователя **B.Simon**. Чтобы обеспечить работу единого входа, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Broadcom DX SaaS.

Чтобы настроить и проверить единый вход Azure AD в Broadcom DX SaaS, выполните следующие действия:

1. **[Настройка единого входа Azure AD](#configure-azure-ad-sso)** необходима, чтобы пользователи могли использовать эту функцию.
    1. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD с помощью пользователя B.Simon.
    1. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить пользователю B.Simon использовать единый вход Azure AD.
1. **[Настройка единого входа в Broadcom DX SaaS](#configure-broadcom-dx-saas-sso)** необходима, чтобы настроить параметры единого входа на стороне приложения.
    1. **[Создание тестового пользователя Broadcom DX SaaS](#create-broadcom-dx-saas-test-user)** требуется для того, чтобы в Broadcom DX SaaS существовал пользователь B.Simon, связанный с одноименным пользователем в Azure AD.
1. **[Проверка единого входа](#test-sso)** позволяет убедиться в правильности конфигурации.

## <a name="configure-azure-ad-sso"></a>Настройка единого входа Azure AD

Выполните следующие действия, чтобы включить единый вход Azure AD на портале Azure.

1. На портале Azure на странице интеграции с приложением **Broadcom DX SaaS** найдите раздел **Управление** и выберите **Единый вход**.
1. На странице **Выбрать метод единого входа** выберите **SAML**.
1. На странице **Настройка единого входа с помощью SAML** щелкните значок "Изменить" (значок пера), чтобы открыть диалоговое окно **Базовая конфигурация SAML** и изменить параметры.

   ![Изменение базовой конфигурации SAML](common/edit-urls.png)

1. На странице **Настройка единого входа с помощью SAML** введите значения для следующих полей:

    а. В текстовом поле **Идентификатор** введите URL-адрес в формате `DXI_<TENANT_NAME>`.

    b. В текстовом поле **URL-адрес ответа** введите URL-адрес в формате `https://axa.dxi-na1.saas.broadcom.com/ess/authn/<TENANT_NAME>`.

    > [!NOTE]
    > Эти значения приведены для примера. Измените их на фактические значения идентификатора и URL-адреса ответа. Чтобы получить эти значения, обратитесь в [службу поддержки Broadcom DX SaaS](mailto:dxi-na1@saas.broadcom.com). Можно также посмотреть шаблоны в разделе **Базовая конфигурация SAML** на портале Azure.

1. Приложение Broadcom DX SaaS ожидает утверждения SAML в определенном формате, поэтому следует добавить настраиваемые сопоставления атрибутов в вашу конфигурацию атрибутов токена SAML. На следующем снимке экрана показан список атрибутов по умолчанию.

    ![Изображение](common/default-attributes.png)

1. В дополнение к описанному выше приложение Broadcom DX SaaS ожидает несколько дополнительных атрибутов в ответе SAML, как показано ниже. Эти атрибуты также заранее заполнены, но вы можете изменить их в соответствии со своими требованиями.
    
    | Имя |  Исходный атрибут|
    | --------------- | --------- |
    | Группа | user.groups |

1. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** найдите пункт **Сертификат (Base64)** и щелкните **Скачать**, чтобы скачать сертификат. Сохраните этот сертификат на компьютере.

    ![Ссылка для скачивания сертификата](common/certificatebase64.png)

1. Скопируйте требуемые URL-адреса из раздела **Настройка Broadcom DX SaaS**.

    ![Копирование URL-адресов настройки](common/copy-configuration-urls.png)

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как на портале Azure создать тестового пользователя с именем B.Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.
1. В верхней части экрана выберите **Новый пользователь**.
1. В разделе **Свойства пользователя** выполните следующие действия.
   1. В поле **Имя** введите `B.Simon`.  
   1. В поле **Имя пользователя** введите username@companydomain.extension. Например, `B.Simon@contoso.com`.
   1. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.
   1. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как предоставить пользователю B.Simon доступ к Broadcom DX SaaS, чтобы он мог использовать единый вход Azure.

1. На портале Azure выберите **Корпоративные приложения**, а затем — **Все приложения**.
1. В списке приложений выберите **Broadcom DX SaaS**.
1. На странице "Обзор" приложения найдите раздел **Управление** и выберите **Пользователи и группы**.
1. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.
1. В диалоговом окне **Пользователи и группы** выберите **B.Simon** в списке пользователей, а затем в нижней части экрана нажмите кнопку **Выбрать**.
1. Если пользователям необходимо назначить роль, вы можете выбрать ее из раскрывающегося списка **Выберите роль**. Если для этого приложения не настроена ни одна роль, будет выбрана роль "Доступ по умолчанию".
1. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

## <a name="configure-broadcom-dx-saas-sso"></a>Настройка единого входа Broadcom DX SaaS

1. Выполните вход на веб-сайт компании Broadcom DX SaaS в качестве администратора.

2. Перейдите в раздел **Settings** (Параметры) и выберите вкладку **Users** (Пользователи).

    ![Пользователи](./media/broadcom-dx-saas-tutorial/settings.png "Пользователи")

3. В разделе **User management** (Управление пользователями) щелкните многоточие и выберите **SAML**.

    ![Управление пользователями](./media/broadcom-dx-saas-tutorial/local.png "управление пользователями;")

4. В разделе **Identify SAML account** (Определение учетной записи SAML) выполните следующие действия.
   
    ![Учетная запись](./media/broadcom-dx-saas-tutorial/broadcom-1.png "Учетная запись") 

    а. В текстовое поле **Issuer** (Издатель) вставьте значение **идентификатора Azure AD**, скопированное на портале Azure.

    b. В текстовое поле **Identity Provider (IDP) Login URL** (URL-адрес входа поставщика удостоверений) вставьте **URL-адрес входа**, скопированный на портале Azure.

    c. В текстовое поле **Identity Provider (IDP) Logout URL** (URL-адрес выхода поставщика удостоверений) вставьте значение **URL-адреса выхода**, скопированное на портале Azure.

    d. Откройте в блокноте скачанный на портале Azure файл **сертификата (Base64)** и вставьте его содержимое в текстовое поле **Identity Provider certificate** (Сертификат поставщика удостоверений).

    д) Щелкните **NEXT** (Далее).

5. В разделе **Map attributes** (Сопоставление атрибутов) заполните необходимые атрибуты SAML на следующей странице и щелкните **NEXT** (Далее).

    ![Атрибуты](./media/broadcom-dx-saas-tutorial/broadcom-2.png "Атрибуты") 


6. В разделе **Identify user group** (Идентификация группы пользователей) введите идентификатор объекта этой группы и щелкните **NEXT** (Далее).

    ![Добавление группы](./media/broadcom-dx-saas-tutorial/broadcom-3.png "Добавление группы") 

7. В разделе **Populate SAML Account** (Заполнение учетной записи SAML) проверьте заполненные значения и щелкните **SAVE** (Сохранить).

    ![Учетная запись SAML](./media/broadcom-dx-saas-tutorial/broadcom-4.png "Учетная запись SAML")
