# <a name="features-and-licenses-for-azure-ad-multi-factor-authentication"></a>Функции и лицензии для многофакторной идентификации Azure AD

Для защиты учетных записей пользователей в своей организации используйте многофакторную проверку подлинности. Эта функция особенно важна для учетных записей, которые имеют привилегированный доступ к ресурсам. Основные функции многофакторной проверки подлинности доступны для Microsoft 365 и Azure Active Directory (Azure AD) без дополнительных затрат. Если вы хотите обновить функции для администраторов или расширить многофакторную проверку подлинности для остальных пользователей, вы можете приобрести многофакторную идентификацию Azure AD несколькими способами.

> [!IMPORTANT]
> В этой статье описаны различные способы лицензирования и использования многофакторной идентификации Azure AD. Конкретные сведения о ценах и выставлении счетов см. на [странице с ценами на многофакторную идентификацию Azure AD](https://azure.microsoft.com/pricing/details/multi-factor-authentication/).

## <a name="available-versions-of-azure-ad-multi-factor-authentication"></a>Доступные версии многофакторной идентификации Azure AD

Многофакторную идентификацию Azure AD можно использовать и лицензировать с помощью нескольких различных способов в зависимости от потребностей вашей организации. Возможно, вы уже имеете право использовать многофакторную идентификацию Azure AD в зависимости от имеющейся лицензии Azure AD, EMS или Microsoft 365. В следующей таблице описаны различные способы получения многофакторной идентификации Azure AD и некоторые функции и варианты использования для каждого из них.

| Если вы являетесь пользователем | Возможности и варианты использования |
| --- | --- |
| Microsoft 365 бизнес Premium и EMS или Microsoft 365 E3 и в. | EMS E3, Microsoft 365 E3 и Microsoft 365 бизнес Premium включают Azure AD Premium P1. EMS E5 или Microsoft 365 E5, включая Azure AD Premium P2. Для обеспечения многофакторной проверки подлинности пользователей вы можете использовать функции условного доступа, которые описаны в следующих разделах. |
| Azure AD Premium P1 | Используйте [Условный доступ Azure AD](../conditional-access/howto-conditional-access-policy-all-users-mfa.md), чтобы запрашивать у пользователей многофакторную проверку подлинности в определенных сценариях или событиях в соответствии с бизнес-требованиями. |
| Azure AD Premium P2 | Обеспечивает наивысший уровень безопасности и улучшает взаимодействие с пользователем. Добавляет [Условный доступ на основе рисков](../conditional-access/howto-conditional-access-policy-risk.md) к функциям Azure AD Premium P1, которые адаптируются к шаблонам пользователей и сокращают запросы многофакторной проверки подлинности. |

Управление многофакторной идентификацией Azure AD осуществляется на портале Microsoft 365. Для повышения удобства работы пользователей выполните обновление до Azure AD Premium P1 или P2 и используйте условный доступ. Дополнительные сведения см. в статье [Защита ресурсов Microsoft 365 с помощью многофакторной проверки подлинности](/microsoft-365/admin/security-and-compliance/set-up-multi-factor-authentication). |

У вас нет детализированного контроля включенных пользователей или сценариев, но это обеспечивает дополнительный уровень безопасности.<br /> Даже если параметры безопасности по умолчанию не используются для включения многофакторной проверки подлинности для всех пользователей или пользователей, которым назначена роль *глобального администратора Azure AD*, можно настроить использование многофакторной проверки подлинности. Эта функция уровня "Бесплатный" обеспечивает защиту критических учетных записей администратора с помощью многофакторной проверки подлинности. |

## <a name="purchase-and-enable-azure-ad-multi-factor-authentication"></a>Приобретение и включение многофакторной идентификации Azure AD

Чтобы использовать многофакторную идентификацию Azure AD, зарегистрируйте или приобретите подходящего уровня Azure AD. Azure AD поставляется в четырех выпусках — бесплатном, Microsoft 365 приложениях, Premium P1 и Premium P2.

Выпуск "Бесплатный" входит в состав подписки Azure. Сведения об использовании параметров безопасности по умолчанию или защите учетных записей с помощью роли *глобального администратора Azure AD* см. в [разделе ниже](#azure-ad-free-tier).

Выпуски Azure AD Premium доступны через представителя корпорации Майкрософт, в [программе Open Volume License](https://www.microsoft.com/licensing/licensing-programs/open-license.aspx) и [программе поставщиков облачных решений](https://go.microsoft.com/fwlink/?LinkId=614968&clcid=0x409). Подписчики Azure и Microsoft 365 могут приобрести Active Directory Premium P1 и P2 через Интернет. Для этого необходимо выполнить [вход в систему](https://portal.office.com/Commerce/Catalog.aspx).

После приобретения требуемого уровня Azure AD выполните [планирование и развертывание многофакторной идентификации Azure AD](howto-mfa-getstarted.md).

### <a name="azure-ad-free-tier"></a>уровень Azure AD Free

Все пользователи в клиенте Azure AD Free могут использовать многофакторную идентификацию Azure AD с помощью параметров безопасности по умолчанию. Приложение для проверки подлинности мобильных устройств — единственный метод, который можно использовать для многофакторной идентификации Azure AD при использовании Azure AD Free умолчаний безопасности.

* [Дополнительные сведения о параметрах безопасности по умолчанию Azure AD](../fundamentals/concept-fundamentals-security-defaults.md)
* [Включение параметров безопасности по умолчанию для пользователей в Azure AD Free](../fundamentals/concept-fundamentals-security-defaults.md#enabling-security-defaults)

Если вы не хотите включать многофакторную идентификацию Azure AD для всех пользователей, можно вместо этого выбрать защиту учетных записей пользователей с помощью роли *глобального администратора Azure AD* . Этот подход предоставляет дополнительные запросы проверки подлинности для критически важных учетных записей администратора. Включить многофакторную идентификацию Azure AD можно в одном из следующих способов, в зависимости от типа используемой учетной записи.

* Если вы используете учетную запись Майкрософт, [выполните регистрацию для многофакторной проверки подлинности](https://support.microsoft.com/help/12408/microsoft-account-about-two-step-verification).
* Если вы не используете учетную запись Майкрософт, [включите многофакторную проверку подлинности для пользователя или группы в Azure AD](howto-mfa-userstates.md).
