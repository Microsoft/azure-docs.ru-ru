# <a name="assign-azure-ad-roles-to-users"></a>Назначение ролей Azure AD пользователям

Теперь вы можете просматривать всех членов ролей администраторов в центре администрирования Azure AD и управлять ими. Если вы часто управляете назначениями ролей, возможно, вам пригодится эта функция. В этой статье описывается назначение ролей Azure AD с помощью центра администрирования Azure AD.

## <a name="assign-a-role"></a>Назначение роли

1. Войдите в [центр администрирования Azure AD](https://aad.portal.azure.com) с разрешениями администратора глобального администратора или привилегированной роли.

1. Выберите **Azure Active Directory**.

1. Выберите **роли и администраторы** , чтобы просмотреть список всех доступных ролей.

    ![Снимок экрана со страницей "роли и Администраторы"](./media/manage-roles-portal/roles-and-administrators.png)

1. Выберите роль, чтобы просмотреть ее назначения.

    Чтобы помочь вам найти нужные роли, Azure AD может отобразить подмножества ролей на основе категорий ролей. Просмотрите фильтр **типа** , чтобы отобразить только роли в выбранном типе.

1. Выберите **добавить назначения** , а затем выберите пользователей, которых нужно назначить этой роли.

    Если вы видите что-то другое на следующем рисунке, прочитайте примечание в [Управление привилегированными пользователями (PIM)](#privileged-identity-management-pim) , чтобы проверить, используется ли PIM.

    ![список разрешений для роли администратора](./media/manage-roles-portal/add-assignments.png)

1. Нажмите кнопку **Добавить** , чтобы назначить роль.

## <a name="privileged-identity-management-pim"></a>Управление привилегированными пользователями (PIM)

Вы можете выбрать **Управление в PIM** для дополнительных возможностей управления с помощью [Azure AD privileged Identity Management (PIM)](../privileged-identity-management/pim-configure.md). Администраторы привилегированных ролей могут менять "постоянные" назначения (пользователь всегда выступает в этой роли) на "разрешенные" (роль назначается в отдельных случаях). Если у вас нет управление привилегированными пользователями, вы по-прежнему можете выбрать **Управление в PIM** , чтобы зарегистрироваться для получения пробной версии. Для управления привилегированными пользователями (PIM) требуется [план лицензирования Azure AD Premium P2](../privileged-identity-management/subscription-requirements.md).

![Снимок экрана, на котором показана страница "Администраторы пользователей" с выбранным действием "Управление в PIM"](./media/manage-roles-portal/member-list-pim.png)

Если вы являетесь глобальным администратором или администратором привилегированных ролей, вы можете добавлять или удалять участников, отфильтровывать список или выбирать участников, чтобы просмотреть их активные назначенные роли.

> [!Note]
> Если у вас есть лицензия Azure AD Premium P2 и вы уже используете управление привилегированными пользователями, все задачи управления ролями выполняются в управлении удостоверениями прав, а не в Azure AD.
>
> ![Роли Azure AD, управляемые в PIM для пользователей, которые уже используют PIM и имеют лицензию Premium P2](./media/manage-roles-portal/pim-manages-roles-for-p2.png)
