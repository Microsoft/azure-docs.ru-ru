# <a name="conditional-access-user-risk-based-conditional-access"></a>Условный доступ: условный доступ на основе рисков для пользователей

Корпорация Майкрософт работает с исследователими, применением закона, различными группами безопасности в корпорации Майкрософт и другими надежными источниками для поиска потерянных пар имен пользователей и паролей. Организации с лицензиями Azure AD Premium P2 могут создавать политики условного доступа, включающие [Защита идентификации Azure AD обнаружения рисков пользователей](../identity-protection/concept-identity-protection-risks.md#user-risk). 

Эта политика может быть назначена в двух местах. Организациям следует выбрать один из следующих вариантов, чтобы включить политику условного доступа на основе рисков для пользователей, которая требует безопасного изменения пароля.

## <a name="enable-with-conditional-access-policy"></a>Включить с политикой условного доступа

1. Войдите на **портал Azure** с учетными данными глобального администратора, администратора безопасности или администратора условного доступа.
1. Выберите **Azure Active Directory** > **Безопасность** > **Условный доступ**.
1. Выберите **Новая политика**.
1. Присвойте политике имя. Мы рекомендуем организациям присваивать политикам понятные имена. Для получения дополнительных сведений [Задайте стандарты именования для политик](./plan-conditional-access.md#set-naming-standards-for-your-policies).
1. В разделе **Назначения** выберите **Пользователи и группы**.
   1. В разделе **Включить** выберите **Все пользователи**.
   1. В разделе **Исключить** выберите **Пользователи и группы**, а затем выберите учетные записи для аварийного доступа или для обхода стандартной системы контроля доступа в вашей организации. 
   1. Нажмите кнопку **Готово**.
1. В разделе **облачные приложения или действия**  >  укажите **все облачные приложения**.
1. В разделе **условия**  >  **риск для пользователя** задайте для параметра **настроить** значение **Да**. В разделе **Настройка уровней риска для пользователей, необходимых для применения политики** выберите **высокий**, а затем нажмите кнопку **Готово**.
1. В разделе **Управление доступом**  >  **предоставление** выберите **предоставить доступ**, **требовать изменение пароля** и нажмите **кнопку Выбрать**.
1. Подтвердите параметры и установите для **параметра включить политику** значение **вкл**.
1. Нажмите **Создать**, чтобы создать и включить политику.

## <a name="enable-through-identity-protection"></a>Включить с помощью защиты идентификации

1. Войдите на **портал Azure**.
1. Выберите **все службы**, а затем перейдите к **Защита идентификации Azure AD**.
1. Выберите **Политика риска пользователя**.
1. В разделе **назначения** выберите **Пользователи**.
   1. В разделе **Включить** выберите **Все пользователи**.
   1. В разделе **исключить** выберите **выберите Исключенные пользователи**, выберите учетные записи аварийного доступа или непрозрачного стекла в вашей организации и нажмите **кнопку Выбрать**.
   1. Нажмите кнопку **Готово**.
1. В разделе **условия** выберите **риск для пользователя**, а затем щелкните **высокий**.
   1. Щелкните **выбрать**, затем **Готово**.
1. В разделе **Управление**  >  **доступом** выберите **Разрешить доступ**, а затем выберите **Требовать смену пароля**.
   1. Щелкните **Выбрать**.
1. Установите для **параметра включить политику** значение **вкл**.
1. Щелкните **Сохранить**.
