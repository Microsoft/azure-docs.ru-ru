# <a name="adsync-service-account"></a>Учетная запись службы ADSync
Azure AD Connect устанавливает локальную службу, которая управляет синхронизацией между Active Directory и Azure Active Directory.  Служба синхронизации Microsoft Azure AD синхронизации (ADSync) выполняется на сервере в локальной среде.  Учетные данные для службы задаются по умолчанию в экспресс установки, но их можно настроить в соответствии с требованиями безопасности Организации.  Эти учетные данные не используются для подключения к локальным лесам или Azure Active Directory.

Выбор учетной записи службы ADSync — это важное решение планирования, которое необходимо выполнить перед установкой Azure AD Connect.  Любая попытка изменить учетные данные после установки приведет к сбою запуска службы, потере доступа к базе данных синхронизации и сбою проверки подлинности в подключенных каталогах (Azure и AD DS).  Синхронизация не будет выполняться, пока не будут восстановлены исходные учетные данные.

Служба синхронизации может работать от имени разных учетных записей. Она может выполняться под учетной записью виртуальной службы (VSA), управляемой учетной записью службы (gMSA/sMSA) или обычной учетной записью пользователя. Поддерживаемые параметры были изменены с выпуском 2017 апреля и 2021 марта Azure AD Connect при установке новой версии. При обновлении с более ранней версии Azure AD Connect эти дополнительные параметры недоступны.

>[!IMPORTANT]
> При использовании Connect сборки за март 2017 года или более ранней не следует сбрасывать пароль учетной записи службы, так как после этого Windows удалит ключи шифрования из соображений безопасности. Не удается изменить учетную запись на любую другую учетную запись без повторной установки Azure AD Connect. При обновлении до сборки с 2017 апреля или более поздней версии поддерживается изменение пароля учетной записи службы, но нельзя изменить используемую учетную запись. 

> [!IMPORTANT]
> Учетную запись службы можно задать только при первой установке. Изменение учетной записи службы после завершения установки не поддерживается. Если вам нужно изменить пароль учетной записи службы, это поддерживается, и инструкции можно найти [здесь](how-to-connect-sync-change-serviceacct-pass.md).

## <a name="virtual-service-account"></a>учетной записи виртуальной службы 

Учетная запись виртуальной службы — это особый тип управляемой локальной учетной записи, которая не имеет пароля и автоматически управляется Windows. 

 ![Учетная запись виртуальной службы](media/concept-adsync-service-account/account-1.png)

Учетная запись виртуальной службы предназначена для использования в сценариях, где модуль синхронизации и SQL находятся на одном сервере. Если используется удаленный SQL, вместо него рекомендуется использовать групповую управляемую учетную запись службы. 

Учетную запись виртуальной службы нельзя использовать на контроллере домена из-за проблем с [API защиты данных Windows (DPAPI)](https://msdn.microsoft.com/library/ms995355.aspx) . 

## <a name="managed-service-account"></a>Управляемая учетная запись службы 

При использовании удаленного SQL Server рекомендуется использовать групповую управляемую учетную запись службы. Дополнительные сведения о подготовке Active Directory для групповой управляемой учетной записи службы см. в разделе [Общие сведения о групповой управляемой](https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/hh831782(v=ws.11))учетной записи службы. 

Чтобы использовать этот параметр, на странице [Установка необходимых компонентов](how-to-connect-install-custom.md#install-required-components) выберите **Использовать существующую учетную запись службы**, а затем выберите **Управляемая учетная запись службы**. 

 ![Управляемая учетная запись службы](media/concept-adsync-service-account/account-2.png)

Также поддерживается использование автономной управляемой учетной записи службы. Однако они могут использоваться только на локальном компьютере, и их использование через учетную запись виртуальной службы по умолчанию не имеет смысла. 

### <a name="auto-generated-standalone-managed-service-account"></a>Автоматически созданная автономная управляемая учетная запись службы 

При установке Azure AD Connect на контроллере домена мастер установки создает автономную управляемую учетную запись службы (если учетная запись не указана для использования в настраиваемых параметрах). Учетная запись предваряется **ADSyncMSA_** и используется для запуска самой службы синхронизации. 

Эта учетная запись является управляемой учетной записью домена, которая не имеет пароля и автоматически управляется Windows. 

Эта учетная запись предназначена для использования в сценариях, где модуль синхронизации и SQL находятся на контроллере домена. 

## <a name="user-account"></a>Учетная запись пользователя 

Локальная учетная запись службы создается мастером установки (если только вы не укажете нужную учетную запись в пользовательских параметрах). Эта учетная запись имеет префикс AAD_ и используется для запуска от имени фактической службы синхронизации. Если Azure AD Connect устанавливается на контроллере домена, учетная запись создается в этом домене. Учетная запись службы AAD_ должна быть размещена в домене, если: 
- используется удаленный сервер, на котором работает SQL Server 
- используется прокси-сервер, требующий проверки подлинности. 

 ![учетная запись пользователя](media/concept-adsync-service-account/account-3.png)

При создании этой учетной записи используется длинный сложный пароль, который имеет неограниченный срок действия. 

Эта учетная запись используется для безопасного хранения паролей других учетных записей. Эти пароли хранятся в зашифрованном виде в базе данных. Закрытые ключи защищены службами шифрования с закрытым ключом с использованием API защиты данных Windows (DPAPI). 

При использовании полной версии SQL Server учетной записью службы является DBO созданной базы данных для модуля синхронизации. Служба не будет работать должным образом с любым другим разрешением. Кроме того, создается имя входа SQL. 

Учетной записи также предоставляется разрешение на доступ к файлам, разделам реестра и другим объектам, связанным с модулем синхронизации.
