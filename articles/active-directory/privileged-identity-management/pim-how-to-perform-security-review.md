# <a name="review-access-to-azure-ad-roles-in-privileged-identity-management"></a>Проверка доступа к ролям Azure AD в управление привилегированными пользователями

Управление привилегированными пользователями (PIM) упрощает управление привилегированным доступом к ресурсам в Azure Active Directory (AD) и других веб-службы Майкрософт, таких как Microsoft 365 или Microsoft Intune, предприятиям. Выполните действия, описанные в этой статье, чтобы самостоятельно проверить назначенные роли.

Если вам назначена роль администратора, администратор привилегированных ролей вашей организации может попросить вас регулярно подтверждать, что эта роль по-прежнему требуется вам для работы. Вы можете получить сообщение электронной почты, содержащее ссылку, или перейти непосредственно к [портал Azure](https://portal.azure.com) и начать.

Если вы администратор привилегированных ролей или глобальный администратор и вас интересуют функции проверки доступа, дополнительные сведения см. в статье [Как запустить проверку доступа](pim-how-to-start-security-review.md).

## <a name="add-a-pim-dashboard-tile"></a>Добавление плитки панели мониторинга PIM

Если служба управление привилегированными пользователями не закреплена на панели мониторинга в портал Azure, выполните следующие действия, чтобы приступить к работе.

1. Войдите на [портал Azure](https://portal.azure.com/).
2. Выберите свое имя пользователя в правом верхнем углу портал Azure и выберите организацию Azure AD, в которой вы будете работать.
3. Выберите **Все службы** и введите **Azure AD Privileged Identity Management** в текстовом поле "Фильтр".
4. Установите флажок **Закрепить на панели мониторинга** и нажмите кнопку **Создать**. Откроется приложение "Управление привилегированными пользователями".

## <a name="approve-or-deny-access"></a>Утверждение или отклонение доступа

При утверждении или отклонении доступа вы просто сообщаете проверяющему, используете ли вы еще эту роль. Чтобы оставаться в этой роли, выберите **Утвердить**, а когда доступ вам больше не требуется — **Запретить**. Состояние роли изменится лишь тогда, когда проверяющий применит результаты.
Выполните следующие действия, чтобы найти проверку доступа и завершить ее.

1. В службе управление привилегированными пользователями выберите **проверить привилегированный доступ**. При наличии ожидающих проверок доступа они отображаются на странице проверки **доступа** Azure AD.
2. Выберите проверку, которую необходимо завершить.
3. Если эта проверка создана не вами, вы должны быть единственным пользователем, отображаемым при проверке. Установите флажок рядом со своим именем.
4. Выберите **Утвердить** или **Запретить**. Возможно, в текстовом поле **Указание причины** потребуется ввести причину своего решения.  
5. Закройте колонку **Проверка ролей Azure AD** .
