# <a name="azure-ad-audit-activity-reference"></a>Справочные материалы по действиям аудита в Azure Active Directory

Функция отчетов в Azure Active Directory (Azure AD) позволяет получать всю необходимую информацию о работе среды.

Архитектура создания отчетов в Azure AD состоит из следующих компонентов.

- **Отчеты о действиях** 
    - [Входы](concept-sign-ins.md) — информация об использовании управляемых приложений и входах пользователей.
    - [Журналы аудита](concept-audit-logs.md) — обеспечивает трассировку с помощью журналов для всех изменений, выполненных различными функциями в Azure AD. 
    
- **Отчеты системы безопасности** 
    - [Рискованные входы](../identity-protection/overview-identity-protection.md). Рискованный вход означает, что в систему пытался войти пользователь, который не является законным владельцем учетной записи. 
    - [Пользователи с пометкой о совершении рискованных действий](../identity-protection/overview-identity-protection.md). Такая пометка означает возможность компрометации учетной записи пользователя. 

В этой статье перечислены действия аудита, которые могут записываться в журналах аудита.
