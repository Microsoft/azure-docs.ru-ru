# <a name="reference-for-as2-message-settings-in-azure-logic-apps-with-enterprise-integration-pack"></a>Справочник по параметрам сообщений AS2 в Azure Logic Apps с Пакет интеграции Enterprise

Эта ссылка описывает свойства, которые можно задать для указания того, как соглашение AS2 обрабатывает сообщения, отправляемые и получаемые между торговыми партнерами. Настройте эти свойства на основе вашего соглашения с партнером, который обменивается сообщениями с вами.

<a name="AS2-incoming-messages"></a>

## <a name="as2-receive-settings"></a>Параметры получения AS2

![Выберите "Параметры получения".](./media/logic-apps-enterprise-integration-as2-message-settings/receive-settings.png)

| Свойство | Обязательно | Описание |
|----------|----------|-------------|
| **Override message properties** | Нет | Переопределяет свойства входящих сообщений с помощью параметров свойств. |
| **Message should be signed** | Нет | Указывает, должны ли все входящие сообщения быть подписаны цифровой подписью. Если требуется подписывание, в списке **сертификат** выберите существующий открытый сертификат гостевого партнера для проверки подписи сообщений. Если у вас нет сертификата, см. Дополнительные сведения о [добавлении сертификатов](../logic-apps/logic-apps-enterprise-integration-certificates.md). |
| **Message should be encrypted** | Нет | Указывает, должны ли быть зашифрованы все входящие сообщения. Незашифрованные сообщения будут отклонены. Если требуется шифрование, в списке **сертификат** выберите существующий частный сертификат главного партнера для расшифровки входящих сообщений. Если у вас нет сертификата, см. Дополнительные сведения о [добавлении сертификатов](../logic-apps/logic-apps-enterprise-integration-certificates.md). |
| **Message should be compressed** | Нет | Указывает, должны ли быть сжаты все входящие сообщения. В противном случае сообщения будут отклонены. |
| **Запретить дублирование ИДЕНТИФИКАТОРов сообщений** | Нет | Указывает, следует ли разрешить сообщения с повторяющимися идентификаторами. Если вы запретили дублирование идентификаторов, выберите число дней между проверками. Можно также выбрать, следует ли приостанавливать дубликаты. |
| **"MDN Text" (Текст MDN)** | Нет | Указывает уведомление о размещении сообщений по умолчанию (MDN), которое требуется отправить отправителю сообщения. |
| **"Send MDN" (Отправлять MDN)** | Нет | Указывает, следует ли отсылать синхронные MDN для полученных сообщений.  |
| **"Send signed MDN" (Отправлять MDN с подписью)** | Нет | Указывает, следует ли отсылать подписанные MDN для полученных сообщений. Если требуется подписывание, в списке **алгоритм MIC** выберите алгоритм, который будет использоваться для подписи сообщений. |
| **"Send asynchronous MDN" (Отправлять асинхронные MDN)** | Нет | Указывает, следует ли отсылать MDN асинхронно. Если вы выбрали асинхронный MDN, в поле **URL-адрес** укажите URL-адрес для отправки MDN. |
||||

<a name="AS2-outgoing-messages"></a>

## <a name="as2-send-settings"></a>Параметры отправки AS2

![Выберите "Параметры отправки".](./media/logic-apps-enterprise-integration-as2-message-settings/send-settings.png)

| Свойство | Обязательно | Описание |
|----------|----------|-------------|
| **"Enable message signing" (Включить подписание сообщений)** | Нет | Указывает, должны ли исходящие сообщения быть подписаны цифровой подписью. Если требуется подписывание, выберите следующие значения: <p>— Из списка **алгоритм подписывания** выберите алгоритм, который будет использоваться для подписи сообщений. <br>— В списке **сертификат** выберите существующий частный сертификат главного партнера для подписи сообщений. Если у вас нет сертификата, см. Дополнительные сведения о [добавлении сертификатов](../logic-apps/logic-apps-enterprise-integration-certificates.md). |
| **"Enable message encryption" (Включить шифрование сообщений)** | Нет | Указывает, должны ли быть зашифрованы все исходящие сообщения. Если требуется шифрование, выберите следующие значения: <p>— В списке **алгоритм шифрования** выберите алгоритм открытого сертификата гостевого партнера, который будет использоваться для шифрования сообщений. <br>— В списке **сертификат** выберите существующий частный сертификат гостевого партнера для шифрования исходящих сообщений. Если у вас нет сертификата, см. Дополнительные сведения о [добавлении сертификатов](../logic-apps/logic-apps-enterprise-integration-certificates.md). |
| **Включить сжатие сообщений** | Нет | Указывает, должны ли быть сжаты все исходящие сообщения. |
| **"Unfold HTTP headers" (Развернуть заголовки HTTP)** | Нет | Помещает заголовок HTTP в `content-type` одну строку. |
| **Передать имя файла в заголовке MIME** | Нет | Указывает, следует ли включать имя файла в заголовок MIME. |
| **"Request MDN" (Запрашивать MDN)** | Нет | Указывает, следует ли получать уведомления о размещении сообщений (MDN) для всех исходящих сообщений. |
| **"Request signed MDN" (Запрашивать подписанные MDN)** | Нет | Указывает, следует ли принимать подписанные MDN для всех исходящих сообщений. Если требуется подписывание, в списке **алгоритм MIC** выберите алгоритм, который будет использоваться для подписи сообщений. |
| **"Request asynchronous MDN" (Запрашивать асинхронные MDN)** | Нет | Указывает, следует ли получить MDN асинхронно. Если вы выбрали асинхронный MDN, в поле **URL-адрес** укажите URL-адрес для отправки MDN. |
| **"Enable NRR" (Включить NRR)** | Нет | Указывает, требуется ли получение уведомления о неподдельности (NRR). Этот атрибут связи предоставляет свидетельство о том, что данные были получены как адресованные. |
| **Параметр "Формат алгоритма SHA2"** | Нет | Указывает формат алгоритма MIC, используемый для входа в заголовки исходящих сообщений AS2 или MDN |
||||
