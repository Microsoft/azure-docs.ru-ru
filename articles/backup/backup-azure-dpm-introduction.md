# <a name="prepare-to-back-up-workloads-to-azure-with-system-center-dpm"></a>Подготовка к резервному копированию рабочих нагрузок в Azure с помощью System Center DPM

В этой статье объясняется, как с помощью службы Azure Backup подготовиться к резервному копированию в Azure с помощью System Center Data Protection Manager (DPM).

Эта статья содержит следующие сведения:

- обзор развертываний DPM с Azure Backup;
- предварительные условия и ограничения по использованию Azure Backup с DPM;
- действия по подготовке служб Azure, такие как настройка хранилища резервных копий для Служб восстановления и (при необходимости) изменение типа хранилища Azure для хранилища;
- процедура подготовки сервера DPM, которая включает скачивание учетных данных для хранилища, установку агента Azure Backup и регистрацию сервера DPM в хранилище;
- действия по устранению распространенных проблем.

## <a name="why-back-up-dpm-to-azure"></a>Для чего нужно резервное копирование серверов DPM в Azure?

[System Center DPM](/system-center/dpm/dpm-overview) производит резервное копирование данных файлов и приложений. DPM взаимодействует со службой архивации Azure по описанной ниже схеме.

- **DPM, работающий на физическом сервере или локальной виртуальной машине** . в дополнение к резервному копированию на диск и на ленту можно выполнять резервное копирование данных в хранилище службы архивации Azure.
- **DPM, выполняющийся на виртуальной машине Azure** — начиная с System Center 2012 R2 с обновлением 3 или более поздней версии, можно развернуть DPM на виртуальной машине Azure. Вы можете создавать резервные копии данных на дисках Azure, подсоединенных к виртуальной машине, или в резервное хранилище с помощью Azure Backup.

Благодаря резервному копированию серверов DPM в Azure компания получает следующие преимущества:

- Для локальных экземпляров DPM Azure Backup служит альтернативой долгосрочному хранению на магнитной ленте.
- Если DPM выполняется на виртуальной машине Azure, Azure Backup позволяет снизить нагрузку на диск Azure. Сохранение старых данных в резервном хранилище позволит увеличить масштаб для бизнес-целей, освобождая диск для новых данных.

## <a name="modify-storage-settings"></a>Изменение параметров хранилища

Вы можете выбрать геоизбыточное хранилище или локально избыточное хранилище.

- По умолчанию это геоизбыточное хранилище.
- Если это хранилище является основной целью резервного копирования, оставьте параметр установленным на геоизбыточное хранилище. Если вам нужно более дешевое и не такое надежное решение, настройте локально избыточное хранилище, выполнив следующую процедуру.
- Сведения о [службе хранилища Azure](../storage/common/storage-redundancy.md)и [геоизбыточном](../storage/common/storage-redundancy.md#geo-redundant-storage), [локально избыточном](../storage/common/storage-redundancy.md#locally-redundant-storage) и хранилище, [избыточном в зоне](../storage/common/storage-redundancy.md#zone-redundant-storage) .
- Измените параметры хранения перед начальным резервным копированием. Если вы уже выполняете резервное копирование элемента, остановите этот процесс до изменения параметров хранилища.

Чтобы изменить параметр репликации хранилища:

1. Откройте панель мониторинга хранилища.

2. В окне **Управление** выберите **инфраструктура резервного копирования**.

3. В меню **Конфигурация резервного копирования** выберите вариант хранилища.

    ![Список хранилищ службы архивации](./media/backup-azure-dpm-introduction/choose-storage-configuration-rs-vault.png)

## <a name="download-vault-credentials"></a>Скачивание учетных данных хранилища

Для регистрации сервера DPM в хранилище вам потребуются учетные данные хранилища.

- Файл учетных данных хранилища представляет собой сертификат, который создается порталом для каждого хранилища архивации.
- Затем портал отправляет открытый ключ в службу контроля доступа (ACS).
- Во время рабочего процесса регистрации компьютера закрытый ключ сертификата будет доступен пользователю, который выполняет проверку подлинности компьютера.
- Основываясь на проверке подлинности, служба Azure Backup отправляет данные в идентифицированное хранилище.

### <a name="best-practices-for-vault-credentials"></a>Советы и рекомендации по учетным данным хранилища

Чтобы получить учетные данные, скачайте файл учетных данных хранилища по безопасному каналу из портал Azure:

- Учетные данные хранилища используются только в рабочем процессе регистрации.
- За сохранность и безопасность файла учетных данных хранилища отвечаете вы сами.
  - При потере контроля над учетными данными хранилища они могут использоваться для регистрации других компьютеров в хранилище.
  - Однако данные резервного копирования шифруются с помощью парольной фразы, которая принадлежит вам, поэтому существующие резервные копии данных не могут быть скомпрометированы.
- Убедитесь, что файл сохранен в расположении, к которому можно получить доступ с сервера DPM. Если он хранится в общей папке или SMB, проверьте наличие разрешений на доступ.
- Учетные данные хранилища действительны в течение 48 часов. Вы можете скачивать новые учетные данные хранилища столько раз, сколько потребуется. Однако только последний файл учетных данных хранилища можно использовать в рабочем процессе регистрации.
- Служба Azure Backup не знает о существовании закрытого ключа сертификата, а сам закрытый ключ недоступен ни на портале, ни в службе.

Чтобы скачать файл учетных данных хранилища на локальный компьютер, сделайте следующее.

1. Войдите на [портал Azure](https://portal.azure.com/).
2. Откройте хранилище, в котором нужно зарегистрировать сервер DPM.
3. В окне **Параметры** выберите **свойства**.

    ![Открытие меню хранилищ](./media/backup-azure-dpm-introduction/vault-settings-dpm.png)

4. В окне **Свойства**  >  **учетные данные архивации** выберите **скачать**. Портал создаст учетные данные хранилища, используя сочетание имени хранилища и текущей даты, и предоставит их вам для скачивания.

    ![Загрузка учетных данных](./media/backup-azure-dpm-introduction/vault-credentials.png)

5. Выберите **сохранить** , чтобы скачать учетные данные хранилища в папку, или **Сохранить как** и укажите расположение. Создание файла займет около одной минуты.

## <a name="install-the-backup-agent"></a>Установка агента службы Backup

На каждом компьютере, для которого Azure Backup создает резервные копии, должен быть установлен агент службы Backup, также известный как агент Служб восстановления Microsoft Azure (MARS). Чтобы установить агент на сервере DPM, сделайте следующее.

1. Откройте хранилище, в котором нужно зарегистрировать сервер DPM.
2. В окне **Параметры** выберите **свойства**.

    ![Открыть параметры хранилища](./media/backup-azure-dpm-introduction/vault-settings-dpm.png)
3. На странице **Свойства** скачайте агент Azure Backup.

    ![Скачать](./media/backup-azure-dpm-introduction/azure-backup-agent.png)

4. Когда скачивание завершится, запустите файл MARSAgentInstaller.exe для установки агента на компьютере DPM.
5. Выберите папку для установки агента и папку для хранения кэша. В расположении для кэша должно быть свободное место в объеме не менее 5 % от объема данных для резервного копирования.
6. При использовании прокси-сервера для подключения к Интернету на экране **Конфигурация прокси-сервера** введите данные прокси-сервера. Если используется проверенный прокси-сервер, введите имя пользователя и пароль.
7. И наконец, агент Azure Backup устанавливает .NET Framework 4.5 и Windows PowerShell (если эти компоненты еще не установлены).
8. После завершения установки агента **закройте** окно.

    ![Закрыть](../../includes/media/backup-install-agent/dpm_FinishInstallation.png)

## <a name="register-the-dpm-server-in-the-vault"></a>Регистрация сервера DPM в хранилище

1. В консоли администрирования DPM > **Управление** выберите в **сети**. Выберите **Зарегистрировать**. Откроется мастер регистрации сервера.
2. В разделе **Конфигурация прокси-сервера** настройте обязательные параметры прокси-сервера.

    ![настройки прокси-сервера;](../../includes/media/backup-install-agent/DPM_SetupOnlineBackup_Proxy.png)
3. В разделе **Резервное хранилище** найдите и выберите файл учетных данных хранилища, который вы скачали ранее.

    ![Учетные данные хранилища](../../includes/media/backup-install-agent/DPM_SetupOnlineBackup_Credentials.jpg)

4. В разделе **Параметр регулирования** вы можете включить регулирование пропускной способности для процессов резервного копирования. Здесь можно установить ограничения скорости для рабочих часов и дней.

    ![Параметр регулирования](../../includes/media/backup-install-agent/DPM_SetupOnlineBackup_Throttling.png)

5. В разделе **Recovery Folder Setting** (Настройка папки восстановления) укажите расположение, которое будет использоваться во время восстановления данных.

    - Azure Backup использует это расположение для временного хранения восстанавливаемых данных.
    - Завершив восстановление данных, Azure Backup очистит все данные из этой области.
    - Расположение должно содержать достаточно места для хранения элементов, которые должны быть восстановлены параллельно.

    ![Recovery Folder Setting](../../includes/media/backup-install-agent/DPM_SetupOnlineBackup_RecoveryFolder.png)

6. В **параметре шифрования** Создайте или укажите парольную фразу.

    - Эта парольная фраза используется для шифрования процесса резервного копирования в облако.
    - Ее длина должна составлять не менее 16 символов.
    - Сохраните файл в безопасном расположении, так как он потребуется для восстановления данных.

    ![Encryption](../../includes/media/backup-install-agent/DPM_SetupOnlineBackup_Encryption.png)

    > [!WARNING]
    > У вас есть собственная парольная фраза для шифрования, и корпорация Майкрософт не имеет возможности видеть ее.
    > Если парольная фраза утеряна или забыта, корпорация Майкрософт не сможет помочь в восстановлении данных резервных копий.

7. Выберите **зарегистрировать** , чтобы зарегистрировать сервер DPM в хранилище.

После успешной регистрации сервера в хранилище теперь можно приступить к резервному копированию в Microsoft Azure. Для резервного копирования рабочих нагрузок в Azure необходимо настроить группу защиты в консоли DPM. [Узнайте, как](/system-center/dpm/create-dpm-protection-groups) развертывать группы защиты.

## <a name="troubleshoot-vault-credentials"></a>Устранение неполадок с учетными данными хранилища

### <a name="expiration-error"></a>Ошибка окончания срока действия

Файл учетных данных хранилища действителен только в течение 48 часов (после скачивания с портала). Если на этом экране возникли ошибки (например, "срок действия файла учетных данных хранилища истек"), войдите в портал Azure и скачайте файл учетных данных хранилища еще раз.

### <a name="access-error"></a>Ошибка доступа

Убедитесь, что файл учетных данных хранилища доступен в расположении, доступном для приложения установки. При возникновении ошибок, связанных с доступом, скопируйте файл учетных данных хранилища во временное расположение на этом компьютере и повторите операцию.

### <a name="invalid-credentials-error"></a>Ошибка недопустимых учетных данных

Если возникла ошибка в учетных данных хранилища (например, "указаны недопустимые учетные данные хранилища"), файл либо поврежден, либо не имеет последних учетных данных, связанных со службой восстановления.

- Скачайте новый файл учетных данных хранилища на портале, а затем повторите операцию.
- Эта ошибка обычно возникает при выборе параметра **скачать учетные данные хранилища** в портал Azure дважды в режиме быстрого выполнения. В этом случае действительным является только второй файл учетных данных хранилища.
