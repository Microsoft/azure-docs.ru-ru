# <a name="deploy-secure-applications-on-azure"></a>Развертывание безопасных приложений в Azure
В этой статье представлены действия по обеспечению безопасности и элементы управления, которые следует учитывать при развертывании приложений для облака. Рассматриваются контрольные вопросы и основные понятия, которые следует учитывать на этапах выпуска и реагирования [жизненного цикла разработки безопасности Microsoft (SDL)](/previous-versions/windows/desktop/cc307891(v=msdn.10)) . Цель — помочь вам определить действия и службы Azure, которые можно использовать для развертывания более безопасного приложения.

В этой статье описаны следующие этапы SDL:

- Release
- Ответ

## <a name="release"></a>Release
Основное внимание на этапе выпуска — подготовить проект для общедоступной версии.
Сюда входят способы планирования эффективного выполнения задач обслуживания после выпуска и устранения уязвимостей, которые могут возникнуть позже.

### <a name="check-your-applications-performance-before-you-launch"></a>Проверка производительности приложения перед запуском

Проверьте производительность приложения перед его запуском или развертыванием обновлений в рабочей среде. Запуск облачных [нагрузочных тестов](https://www.visualstudio.com/docs/test/performance-testing/getting-started/getting-started-with-performance-testing) с помощью Visual Studio для поиска проблем с производительностью в приложении, улучшения качества развертывания, обеспечения того, что приложение всегда работает или доступно, а приложение может управлять трафиком для вашего запуска.

### <a name="install-a-web-application-firewall"></a>Установка брандмауэра веб-приложения

Веб-приложения все чаще подвергаются вредоносным атакам, использующим общеизвестные уязвимости. К общим атакам относятся атаки путем внедрения кода SQL и атаки с использованием межузловых сценариев. Предотвращение этих атак в коде приложения может оказаться непростой задачей. Для этого может потребоваться жесткое обслуживание, установка исправлений и мониторинг на многих уровнях топологии приложения. Централизованная WAF помогает упростить управление безопасностью. Решение WAF может также реагировать на угрозу безопасности путем исправления известной уязвимости в центральном расположении и безопасности каждого отдельного веб-приложения.

[Шлюз приложений Azure WAF](../../web-application-firewall/ag/ag-overview.md) обеспечивает централизованную защиту веб-приложений от распространенных эксплойтов и уязвимостей. WAF основан на правилах из [набора правил OWASP Core наборов](https://www.owasp.org/index.php/Category:OWASP_ModSecurity_Core_Rule_Set_Project) 3,0 или 2.2.9.

### <a name="create-an-incident-response-plan"></a>Составление плана реагирования на инциденты

Подготовка плана реагирования на инциденты крайне важна для того, чтобы помочь в устранении новых угроз, которые могут возникать с течением времени. Подготовка плана реагирования на инциденты включает в себя определение соответствующих контактных лиц для системы безопасности и создание планов обслуживания для кода, наследуемого от других групп в Организации и для лицензионного кода стороннего производителя.

### <a name="conduct-a-final-security-review"></a>Окончательная проверка безопасности

Намеренная проверка всех выполненных действий безопасности помогает обеспечить готовность к выпуску программного обеспечения или приложению. Окончательная проверка безопасности (ФСР) обычно включает в себя анализ моделей угроз, выходных данных инструментов и производительности для шлюза качества и полос ошибок, определенных на этапе требований.

### <a name="certify-release-and-archive"></a>Сертификация выпусков и архивов

Сертификация программного обеспечения перед выпуском гарантирует соблюдение требований безопасности и конфиденциальности. Архивация всех необходимых данных необходима для выполнения задач обслуживания после выпуска. Архивирование также помогает снизить долгосрочные затраты, связанные с устойчивым проектированием программного обеспечения.

## <a name="response"></a>Ответ
Этапы этапа пост-Release в группе разработчиков, которые позволяют реагировать на любые отчеты о новых угрозах и уязвимостях программного обеспечения.

### <a name="execute-the-incident-response-plan"></a>Выполнение плана реагирования на инциденты

Возможность реализации плана реагирования на инциденты, реализованная на этапе выпуска, необходима для защиты клиентов от уязвимостей в области безопасности программного обеспечения или конфиденциальности.

### <a name="monitor-application-performance"></a>Мониторинг производительности приложений

Постоянный мониторинг приложения после его развертывания потенциально помогает выявить проблемы с производительностью, а также уязвимости системы безопасности.
Службы Azure, которые помогают отслеживать приложения:

  - Azure Application Insights
  - Центр безопасности Azure

#### <a name="application-insights"></a>Application Insights

[Application Insights](../../azure-monitor/app/app-insights-overview.md) — это расширяемая служба управления производительностью приложений (APM) для веб-разработчиков на нескольких платформах. Используйте ее для мониторинга вашего работающего веб-приложения. Application Insights автоматически обнаруживает аномалии производительности. Она включает мощные средства анализа, помогающие диагностировать проблемы и понять, что пользователи фактически делают с вашим приложением. Эта служба помогает постоянно улучшать производительность и удобство использования.

#### <a name="azure-security-center"></a>Центр безопасности Azure

[Центр безопасности Azure](../../security-center/security-center-introduction.md) помогает предотвращать и обнаруживать угрозы, а также реагировать на них, повышая уровень видимости и контроля над безопасностью ресурсов Azure, включая веб-приложения. Центр безопасности Azure помогает обнаруживать угрозы, которые в противном случае могут быть незамеченными. Он работает с различными решениями безопасности.

Центр безопасности ценовой категории "Бесплатный" предлагает ограниченные возможности безопасности только для ваших ресурсов Azure. [Уровень "Стандартный" центра безопасности](../../security-center/security-center-get-started.md) расширяет эти возможности на локальные ресурсы и другие облака.
Центр безопасности Standard помогает:

  - Поиск и устранение уязвимостей системы безопасности.
  - Примените элементы управления доступом и приложениями для блокирования вредоносных действий.
  - Обнаружение угроз с помощью аналитики и аналитики.
  - Быстро реагировать на атаки.
